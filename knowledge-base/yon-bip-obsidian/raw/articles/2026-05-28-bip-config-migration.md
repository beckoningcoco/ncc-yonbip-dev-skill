# 配置迁移

> 来源：YonBIP 社区文档中心 - c2.yonyoucloud.com
> 抓取时间：2026-05-28
> 原始URL：https://c2.yonyoucloud.com/iuap-hc-client/ucf-wh/client/index.html#/detail/CZZN-PZQY12

## 概述

在BIP系统进行实施交付时，通过沙箱环境/租户 → 生产环境/租户的验证与上线过程。配置迁移工具用于环境之间、租户之间的配置数据和基础数据的复制、导入。

**背景**：实施交付中环境间配置复制和验证工作占实施成本20%以上，小型项目达40%。

**关键词**：源环境、目标环境、沙箱环境、生产环境、传输包

## 名词解释

- **沙箱环境（UAT）**：商开环境开通，用于实施配置和验证测试
- **生产环境（PROD）**：正式生产租户，所有改动需通过配置迁移工具进行

## 操作流程

### 传输包构建
新增传输包 → 配置数据加入清单 → 加入传输包 → 传输包构建 → 下载传输包

### 传输包部署
导入传输包 → 环境校验 → 数据校验 → 传输包部署 → 数据检查

### 迁移顺序（必须严格）
**对象建模 → 应用 → 枚举 → 实体 → 页面**
自建应用首次必须先迁移"业务对象"

### 两种部署模式
- **指定目标部署**：传输包绑定目标环境/租户，部署时校验一致性
- **任意目标部署**：可在任意环境部署（不含当前租户）

## 环境校验内容
- 环境校验：确保目标环境匹配
- 租户校验：确保目标租户匹配
- 微服务校验：版本一致性（弱校验模式可跳过）
- 库表一致性校验：数据库、数据表结构一致（弱校验：目标不缺少字段即可通过）

## 数据校验
- 确保迁移配置数据的所有依赖项在目标环境存在
- 主键对照：来源系统和当前系统配置数据的映射关系
- 数据对照确认：迁移数据覆盖目标已有数据时需人工确认

## 目标环境管理
- 环境编码：租户所属数据中心域名（在API文档请求地址中查看）
- 租户编码：租户ID（数字化建模 → 我的企业）
- 不允许重复的环境编码+租户编码

## 私有云 COPY 数据库注意事项

copy数据库前必须清理配置迁移相关表，否则主键对照残留导致后续迁移出错。

**R5版本**：delete from iuap_apcom_migration.dms_category_data 等17张表
**YonBIP 5.0/R6版本**：在R5基础上增加 dms_category_dependent、dms_category_queue、dms_job_calls_relation 共20张表

清理条件：`where ytenant_id='当前租户'`

## 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| UI模板迁移校验失败"目标环境不存在该元数据" | 依赖的特征未迁移，或客开扩展元数据属性未安装 | 迁特征或安装扩展 |
| 特征迁移后UI模板校验仍失败 | 特征分配到的实体所在应用未安装到目标环境 | 在源环境UI设计器删除该特征字段后重新迁移 |
| 传输包构建失败"存在不是启用保存并发布状态的流程模型" | 工作流未发布 | 工作流设计器→保存并发布 |
| 传输包构建失败"参照默认值档案数据不存在" | UI设计器选了具体的档案数据，目标环境无对应数据 | 先通过档案迁移同步该数据 |
| 开多语后翻译丢失 | 多语资源不会随配置迁移带走 | 需单独迁移"翻译工作台"（按租户全量迁移） |

## 特征迁移注意事项
- 两个环境相同编码特征但数据类型不同 → 无法迁移
- 自定义档案引用URI不同 → 无法迁移
- 创建人/修改人工号、手机号、邮箱在两个环境无匹配 → 无法迁移
- 枚举类型不同 → 无法迁移
