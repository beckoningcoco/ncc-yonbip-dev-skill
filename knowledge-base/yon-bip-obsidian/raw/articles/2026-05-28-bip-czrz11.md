业务场景

系统中有多种日志需要记录: 用户操作日志、业务日志。操作日志记录用户点击的菜单或按钮操作；业务日志单独记录重要业务操作，如订单提交、审批等。日志服务要求对业务系统无侵入且不影响性能，日志需统一存储，方便后续统计分析。

关键词

日志服务、操作日志、 业务日志、统一存储

自定义日志开发

日志SDK在脚手架中已默认依赖，如果工程中没有默认依赖，可手动添加依赖，如下（版本号根据平台二方包管理的版本定义，YPD脚手架工程无需指定，可通过引擎脚手架中查看该版本号）：

<dependency>
<groupId>com.yonyou.iuap</groupId>
<artifactId>iuap-ap-log-sdk</artifactId>
</dependency>

业务日志SDK接入

后端工程业务日志记录，使用iuap平台的IBusiLogService 接口方法来统一存储日志数据，注意：该方式使用的是rpc通信，ISV开发不能使用该方式，可使用下面章节OpenAPI接入的方式，代码示例如下：

package com.yonyou.ucf.mdf.bill.plugin;

import org.springframework.beans.factory.annotation.Autowired;

import com.yonyou.iuap.context.ContextConstans;
import com.yonyou.iuap.context.InvocationInfoProxy;
import com.yonyou.iuap.log.cons.BusinessLogMode;
import com.yonyou.iuap.log.cons.DeviceTypes;
import com.yonyou.iuap.log.cons.OperCodeTypes;
import com.yonyou.iuap.log.model.BusinessLogObjectBuilder;
import com.yonyou.iuap.log.model.BusinessLogObjectDTO;
import com.yonyou.iuap.log.model.OperationLogVO;
import com.yonyou.iuap.log.rpc.IBusiLogService;
import com.yonyou.iuap.log.rpc.IOperationLogService;
import com.yonyou.ypd.bill.annotation.BillPlugin;
import com.yonyou.ypd.bill.context.YpdBillContext;
import com.yonyou.ypd.bill.plugin.AbstractBillPlugin;

@BillPlugin(busiObj = "gzyProductPurchaseRequisition")
public class GzyProductPurchaseRequisitionBillPlugin extends AbstractBillPlugin {

@Autowired
private IBusiLogService busiLogService;

private void saveBusiLog(YpdBillContext billContext) {
/**
* 使用单据上下文构建业务日志对象实体
* newObject 业务数据 字段应包括id/code/name或通过元数据打标签的字段
* uri 实体元数据URI
* operCodeTypes 操作类型枚举（新增、更新、删除等）
* serviceCode 工作台服务编码
* busiObjTypeCode 自定义的code，区分业务即可，可以是BillNum
* busiObjTypeName 业务类型名称 例如 组织新增 物料新增 用户新增等 支持多语词条
* defaultOperationName 默认操作名称
* defaultBusiObjTypeName 默认单据名称
* logMode 业务对象日志类型接入（自定义日志、ui元数据、业务对象）
*/
BusinessLogObjectDTO businessLogObject = new BusinessLogObjectBuilder()
.newObject(billContext.getBillDO())
.uri(billContext.getBillDO().getMetaFullName())
.operCodeTypes(OperCodeTypes.open)
.serviceCode(billContext.getBaseBillContext().getServiceCode())
.busiObjTypeCode(billContext.getBillDO().getCBillNo())
.busiObjTypeName(billContext.getBaseBillContext().getBillName())
.defaultOperationName(billContext.getBaseBillContext().getActionInfo().getName())
.defaultBusiObjTypeName(billContext.getBaseBillContext().getBusiObj())
.logMode(BusinessLogMode.BUSINESS_OBJECT)
.build();

busiLogService.saveBusinessLog(businessLogObject);
}

/**
* 保存前业务扩展
*
* @param billContext
* @throws Exception
*/
@Override
public void beforeSave(YpdBillContext billContext) throws Exception {
super.beforeSave(billContext);

//保存业务日志
saveBusiLog(billContext);

}
}

操作日志SDK接入

后端工程操作日志记录，使用iuap平台的IOperationLogService接口方法来统一存储日志数据，注意：该方式使用的是rpc通信，ISV开发不能使用该方式，可使用下面章节OpenAPI接入的方式，代码示例如下：

package com.yonyou.ucf.mdf.bill.plugin;

import org.springframework.beans.factory.annotation.Autowired;

import com.yonyou.iuap.context.ContextConstans;
import com.yonyou.iuap.context.InvocationInfoProxy;
import com.yonyou.iuap.log.cons.BusinessLogMode;
import com.yonyou.iuap.log.cons.DeviceTypes;
import com.yonyou.iuap.log.cons.OperCodeTypes;
import com.yonyou.iuap.log.model.BusinessLogObjectBuilder;
import com.yonyou.iuap.log.model.BusinessLogObjectDTO;
import com.yonyou.iuap.log.model.OperationLogVO;
import com.yonyou.iuap.log.rpc.IBusiLogService;
import com.yonyou.iuap.log.rpc.IOperationLogService;
import com.yonyou.ypd.bill.annotation.BillPlugin;
import com.yonyou.ypd.bill.context.YpdBillContext;
import com.yonyou.ypd.bill.plugin.AbstractBillPlugin;

@BillPlugin(busiObj = "gzyProductPurchaseRequisition")
public class GzyProductPurchaseRequisitionBillPlugin extends AbstractBillPlugin {

@Autowired
private IOperationLogService operationLogService;

private void saveOperationLog(YpdBillContext billContext) {
/**
* 使用单据上下文构建操作日志对象实体
* serviceCode 服务Code（工作台维护）
* device 登录设备 (1/2/3/4) web/android/client/iphone
* serviceName 没serviceCode可自定义服务名称
* applicationName 没serviceCode可自定义应用名称
* ip 登录的IP地址
* button 操作按钮，传com.yonyou.iuap.log.cons.OperCodeTypes 枚举对应的code，枚举不满足可不传
* operCode 操作编码，自定义操作类型可传入多语编码，OperCodeTypes 枚举满足需求可不传
* defaultButton 短路多语领域操作按钮名称
* operUser 友互通用户ID
* tenantId 友互通租户ID
*/
OperationLogVO operationLog = new OperationLogVO();
operationLog.setServiceCode(billContext.getBaseBillContext().getServiceCode());
operationLog.setDevice(DeviceTypes.web.getIndex());
operationLog.setServiceName(billContext.getBaseBillContext().getBillName());
operationLog.setApplicationName(billContext.getBaseBillContext().getAppName());
operationLog.setIp(InvocationInfoProxy.getValByKey(ContextConstans.X_FORWARDED_CLIENTIP));
operationLog.setButton(OperCodeTypes.save.getDefaultOperateName());
operationLog.setOperCode(OperCodeTypes.save.getCode());
operationLog.setDefaultButton(OperCodeTypes.save.getMultilingual());
operationLog.setOperUser(InvocationInfoProxy.getUserid());
operationLog.setTenantId(InvocationInfoProxy.getTenantid());

operationLogService.saveOperation(operationLog);
}

/**
* 保存前业务扩展
*
* @param billContext
* @throws Exception
*/
@Override
public void beforeSave(YpdBillContext billContext) throws Exception {
super.beforeSave(billContext);

//保存操作日志
saveOperationLog(billContext);

}
}

常见问题
后端在线函数，怎样打印日志？

答：后端脚本可以使用logBusinessObject方法记录业务日志。

日志数据统一存储到哪里了？

答：日志存储在ES里。

想用操作日志制作报表，操作日志是存在ES里，怎么实现？

答：不能直接操作ES，R5及以后版本提供基本操作日志统计报表。需要更复杂报表，可以使用操作日志API自定义读取数据并开发报表。

新功能开发的单据，怎样接入业务日志？

答：新功能开发的单据对接业务日志，需要在元数据中勾选业务日志标签。

日志API在开放平台的内容和项目上的不一致，该怎么办？

答：以项目环境上的API说明为准，开放平台的内容为公有云环境的。
