业务场景

应用程序中经常会用到跑定时任务的需求，比如定时垃圾回收。有关任务调度需求，有时候很复杂，如每隔多长时间重复执行，一个任务在不同时间段执行等。

关键词

调度任务、调度类型、专业版、定时任务、租户级

专业名词术语

调度任务服务：提供基于预设固定时间点、预设时间间隔或给定执行次数自动执行任务的调度框架，主要包括调度控制器、调度触发器和调度任务。

调度类型：为开发态的调度类型配置。

调度任务：支持任务的添加、删除、停用、启用等功能，且支持外部调用的 REST 服务。调度任务还提供完整的任务配置界面，包括任务调度、数据分析等功能。通过后台调度任务功能可以启动和终止作业调度、配置调度策略、查看执行历史和执行日志，进行统一的后台管理。

Scheduler：Quartz 框架的定时任务调度器，负责定时任务的调度与分发。

系统级任务：指全局性的、跨租户的定时任务，通常是运营管理类的任务，如权限同步任务等。配置此类任务的是用友云的运营人员，在开发态配置页面进行配置。

租户级任务：指租户内部的定时任务，通常由开发运营人员预置任务类型和默认定时任务配置，但不启用，只有在租户开通初始化时才启用。租户管理员可在租户态配置页面进行配置，同时也可以直接在租户态配置页面创建租户级任务。

调度任务实践核心内容
总体架构设计
设计图

高可用的分布式任务调度系统；

（1） 支持系统级/租户级调度任务的分级管理；

（2） 支持场景内提交异步任务；

（3） 提供异步任务的统一监控；

（4） 支持租户配置个性化的时间规则，参数，消息配置等；

（5） 支持任务限流；

（6） 任务全异步执行。

关键技术

任务分发过程：

启动调度任务服务，通过任务工厂创建计划实例；
业务用户发起增删改查任务时，从已有的实例选择执行；
对于业务数据进行持久化操作；
开启分发线程，循环分发到工作线程；
工作线程执行任务，发送任务到 Kafka,客户端调用 Rest 接口处理业务；
consumer 执行 Rest 调用时，设定超时设置，目前设置为 2 秒，超过此限制后， 任务调度系统会记录此任务失败，需要推动业务优化其任务接口，推荐调度任 务接口异步化；
分表，每个 Scheduler 只负责整个调度任务中的一部分，确保每个 Scheduler 每次分发任务总量在 4 万以内，这样每个 Scheduler 需要操作的 quart 数据表 的数量下降后，quartz 能正常分发。
功能介绍
功能概述

分布式任务调度系统；支持系统级/租户级调度任务的分级管理；支持场景 内提交异步任务；提供异步任务的统一监控；支持租户配置个性化的时间规则，参数， 消息配置等；支持任务限流；任务全异步执行

创建调度类型

注意：调度类型无法在公有云维护，公有云只能选择【后端脚本】类型。

打开调度类型节点：

在自己的应用节点下点击“新建类型”按钮创建一个任务类型：

调度类型的编码、名称不能与当前租户下其他类型的编码、名称重复。
接口字段需要完整路径，只支持 REST 接口的post请求; 强烈建议业务接口按照异步方式实现（同步方式会有60S超时限制），调度任务已提供回调接口。
同时 SDK 支持了异步执行器，具体使用方式见代码示例。
启用如果不勾选的话，在任务配置时无法选取到该类型。
可为调度类型定义参数，目前支持的类型有：字符串，数字，日期，参照。
当选择用户可见时，该参数会出现在调度任务界面，用户可动态修改；如果 不勾用户可见，则不会显示给用户；或者打开一下连接也能创建。
接口建议写为动态参数 ${domain.url}。
调度任务

打开调度任务菜单，会显示调度任务的任务总数、已启用任务数量、任务执行并发量统计、任务管理、任务查询和历史任务。

任务管理

点击【任务管理】，显示任务管理页面：

1、页面左侧为配置了调度类型的应用；

2、页面右侧为分组下的任务列表界面，包括任务名称、任务创建时间、任务状态、 任务执行的成功失败次数，点击行，可进行任务的停用、删除、进入详情；

3、点击【新建任务】按钮创建新的任务：

首先选择调度类型（公有云只能选择后端脚本），可以设置是否启用，填写任务的其它信息。

条件设置

如果调度类型之前配置了参数，那么调度任务的条件设置页签可以带出这些参数并对参数进行设值：

定时规则设置

在定时规则页签中，可以设置任务执行情况，包含时区、时间范围、时间配置、执行频率和时间间隔等。

消息设置

在消息配置页签中，可以设置接收人的信息以及发送时机。

注意：调度任务只支持三种消息发送时机，成功失败都发送、失败发送、都不发送，并且这里的消息接收人是直接选用户、角色或者组织，调度任务将按照固定格式给指定接收人发送任务执行成功或者失败的消息。

代码示例

当在自建调度类型配置接口时，只支持 REST 接口，REST接口分为异步和同步，示例分别如下。

注意：REST 接口不适用于公有云调度任务实现。

异步代码示例

调度任务平台提供的异步回调接口说明：

类型	参数
url	/warning/warning/async/updateTaskLog
格式	JSON
http 请求方式	POST
编码类型	UTF-8

请求参数说明：

参数字段	必选	类型	说明
id	TRUE	String	日志 ID，请求时 header 中的 logId
status	TRUE	Integer	任务执行结果 1：成功；0： 失败；
content	TRUE	String	发送信息内容
title	TRUE	String	发送信息标题
msg	TRUE	String	失败信息

代码示例如下：

private ExecutorService ymsExecutor = YmsExecutors.getYmsExecutor();
@RequestMapping(value = "/xxxxx", method = RequestMethod.POST)
@ResponseBody
public Object dispatch( HttpServletRequest request ,@RequestBody(required = false) Map<String, Object> paramMap) {
    String logId =  Optional.ofNullable(request.getHeader("logId")).orElse("");
    String tenantId = Optional.ofNullable(request.getHeader("tenantId")).orElse("");
    Map<String, Object> retMap = Maps.newHashMap();
    
        ymsExecutor.execute(()->{
        //异步执行业务逻辑 执行成功status传1 失败传0；
        int status = 业务逻辑service.exec(xxx,xxx,xxx);
        //回调也可放到业务逻辑最下面执行
        callbackPlatform(logId, status);
        })
    retMap.put("asynchronized", "true");
    return retMap;
}
private void callbackPlatform(String logId, int status) {
    // 回调调度任务
    JSONObject param = new JSONObject();
    param.put("status", status);
    param.put("id", logId);
    param.put("msg", "xxxxxx执行成功");
    param.put("content","执行成功" );
    try {
        String url = PropertyUtil.getPropertyByKey("domain.iuap-apcom-coderule") + "/warning/warning/async/updateTaskLog";
        AuthHttpClientUtils.execPost(url, null, null, param.toString());
    } catch (Exception e) {
        log.error("回调平台，更改任务状态异常 " + e.getMessage());
    }
}

后端函数实现

调度类型预置了后端脚本类型，在公有云创建调度任务时，调度类型只能选择【后端脚本】，如果有后端脚手架，可在后端脚本中通过postman函数调用REST接口,从后端脚本调用接口时，只能同步调用。

代码示例如下：

let AbstractTrigger = require('AbstractTrigger');
class MyTrigger extends AbstractTrigger {
    execute(context, param) {
         //后端REST接口地址
        let url = “https://xxxx.com/shippingPacking/expressageTrajectory?”;

        let header = {
            Referer: url,
        };
        let body = {};
        let response = ObjectStore.postman('post', url, header, body);

        return { data: response, url: url };
    }
}
exports({ entryPoint: MyTrigger });

常见问题
我注册的调度任务，都储存在了哪里？

答：iuap_yms_data下的任务类型：dispatch_taskway;调度任务：dispatch_task

调度任务最小可以设置多少分钟？

答：5分钟

我的版本比较老，如何进行调度任务注册

答： BIP2111及2203版本，任务类型注册地址：私有化域名/iuap-apcom-scheduler/warning/#/ruleslistdev/

问调度任务的在任务用户表在哪里？

答：在dispatch_taskuser下

调度任务立即执行不执行？

答：如果调度任务立即执行和定时执行都不执行，请确认对应的系统级的调度任务是否处于停用状态，如果停用，则租户级的调度任务都不会执行；如果领域想用的话，需要抽取启用系统级的调度任务并且抽取脚本同步到别的环境
