公有云ISV生态产品制品
最后更新时间：2025-05-27
前言
本文档用途、面向对象

本文主要面向需要开发ISV生态产品所有相关人，本文档带领大家学习一下生态产品制品上线流程，从前期准备工作、发布0租户、抽取脚本、脚本上线等这几个部分为大家进行详细讲解。后续文档会持续进行更新。

面向对象：ISV总负责人、ISV产品负责人、ISV开发负责人、ISV验证负责人、ISV交付负责人

准备工作
在开发租户完成应用开发
配置迁移开发成果到管理租户
已申请ppm码
管理租户开通“产品包管理”菜单
管理租户开通“规格计量管理”菜单

注意事项：

1.尽量避免重复添加按钮，删除按钮发布菜单，容易导致角色管理里边出现重复的按钮

2.校验自建单据引用原厂档案，不需要校验原厂档案删除，该功能由原厂实现

3.原厂扩展迁移管理租户需要再把所有页面扩展出来

基本概念

开发沙箱：用于进行低代码开发的租户，可以进行业务对象建模、页面管理、流程管理、应用发布等功能。

管理沙箱（Release）：用于进行应用开发完成后，可实现将应用发布到商开0租户，进行规格配置，流水线管理等能力。

商开0租户：开发完成后的应用需要发布到商开环境的0租户作为模板，产品开通时需要从0租户开通到指定租户。

YB/YS验收沙箱租户：用于开通验证的商开租户，在这个租户购买产品，验证产品的可用性。

生产0租户：商开环境验证好的0租户数据需要同步到生产上，用于生产环境的开通和购买。每个数据中心都存在一个生产0租户，用于当前数据中心的产品购买。

生产上线验证租户：用于在生产环境验证0租户的产品是否可以正常购买。和商开0租户类似。

生态产品迭代及修复bug流程
生态产品上线及迭代步骤说明

伙伴开发人员在开发沙箱迭代开发应用，开发完成后迁移到测试沙箱由开发人员或测试人员进行验证，验证失败后重新修改BUG，然后开发人员或测试人员继续在测试沙箱验证。
验证通过后从测试环境迁移到管理沙箱，由产品负责人配置规格和计量，进行制品制作，采用一键发布的方式，然后发布到商开0租户。
由用友人员在YB/YS验收沙箱租户进行开通购买，由伙伴测试人员进行全面验证，无问题后下载制品，然后由生态技术人员将脚本放入到脚本流水线，填写上线申请单，将脚本提交到生产环境0租户，专业版应用构建需要提前准备生产环境流水线。
在生产上线验证租户开通购买生产0租户的产品到当前租户，然后验证功能，出现问题要立即修复解决，重新从原流程重新修改解决，将所有问题解决完，总体流程也完成。
生态产品BUG修复紧急问题上线流程说明

在管理沙箱（Release）中，伙伴开发人员修改定位线上的BUG，当修复完成后，直接在租户内伙伴测试人员进行验证。
验证成功后，走制品发布流程，伙伴人员进行一键发布0租户，下载制品，然后由用友生态技术负责将制品放到脚本流水线，然后提上线单，审批通过后发布到生产修复。
管理沙箱做的功能，同步修改到开发沙箱。用于继续迭代。
后续的验证和上线流程同上。
实践步骤-发布0租户

本章节由生态ISV开发独立配置完成。

创建项目

生态开发将开发成果迁移到管理租户，打开“产品包管理”菜单，新建项目

填写项目信息，编码和名称，建议包含引擎编码避免重复

发布应用
自建应用发布0租户

在管理租户点击已创建的产品包

导入应用，选择需要打包的自建应用。

点击一键发布，等待提示发布成功即发布到0租户了

原厂扩展发布0租户

生态开发在管理租户打开产品包，导入原厂扩展应用。** 原厂扩展和自建应用建议都放在一个产品包中**。

选择应用后点击一键发布，等待页面提示“发布成功”即可。

扩展配置
私有云【独立用户许可】规格导出

规则编码配置：0042_iuap_workbench_system-sys_user_license
参数配置（注意：ppm0000145098替换为规格计量管理（ISV）中的相应产品主数据码/产品编码 ）：{"code":"ppm0000145098"}，如果包含多个产品编码，参数配置为：{"code":"ppm0000145098,ppm0000145097"}


私有云 ISV REST API 路由脚本配置

规则编码配置：iuap-api-gateway_isvapirouter
参数配置：{"apiName":"ISV REST API 路由"}

步骤-商开环境开通验证

该章节由集团生态测试负责开通，ISV提供验收资料提交验收联系集团生态产品测试即可。

环境开通后，ISV验证负责人对产品功能进行测试。

实践步骤-抽取脚本

本章节暂时由集团生态开发对接人进行操作。

管理租户产品包管理菜单，下载需要上线的产品包

下载的包内包含自建单据和原厂扩展的脚本包

解压应用包

自建应用有MONGO脚本和SQL脚本

原厂扩展只有MONGO脚本

将脚本复制到脚本流水线代码

scripts\db\patch\rdb\0001_{产品编码}\0010_iuap_common\DML目录下

MONGO脚本

解压压缩包后

resources\db\patch\mongodb\V6_R5_2312\0001_iuap_dms\0010_iuap_common\DML

目录为MONGO脚本

将MONGO脚本复制到脚本流水线代码

scripts\db\patch\mongodb\0001_{产品编码}\0010_iuap_common\DML 目录下

实践步骤-生产上线

本章节一、二、四、五步由集团生态开发对接人进行操作，第三步由生态ISV开发进行操作。

预发生成镜像

集团租户打开脚本流水线，选择预发环境执行流水线生成镜像，生成镜像成功后，提上线单推送镜像到生产环境。

生产执行脚本

镜像同步到生产环境后，由集团生态技术对接人使用运维大盘进行脚本确认。

执行生产服务流水线

由生态ISV开发自己去升级前后端服务流水线。

租户升级

联系对应生态产品对接人对已开通租户进行租户升级

清理缓存

自建缓存：清理redis缓存

枚举缓存：首次上线，或枚举变化时需要联系支持同事清理公共枚举缓存

领域缓存：使用原厂扩展，且原厂扩展有迭代更新时，需要联系支持同事清理领域缓存

问题：一键发布报错应用名称（中文、英文、繁体）都不可以为空

答案：在应用构建的应用编辑按钮，重新保存下应用

问题：一键发布报错“应用编码重复”或“服务编码重复”

答案：检查是否存在测试菜单，如果存在，删除测试菜单后重新发布。如果还报重复，需要联系生态技术对接同事检查是否是存在已发布且冲突的数据。

问题：开通后枚举不全

答案：检查系统级数据是否缺少枚举，缺少枚举需要在应用构建重新保存一下缺少的枚举。

问题：原厂扩展开通后枚举不显示

答案：需要清理公共枚举缓存，自建单据显示，单原厂扩展的单据枚举不显示需要联系生态技术对接人清理领域缓存。

问题：租户级存在UI模板导致系统级不生效

答案：租户级的UI模板点击升级按钮同步系统级模板，但需要注意此动作会初始化模板，导致自定义的一些配置初始化。

问题：开通后原厂扩展没有生效

答案：检查0租户发布时，原厂扩展的页面有没有扩展出来，没扩展出来需要重新扩展出来，发布0租户。

问题：生态应用测试时候发现计量不生效，如果查看下应用及服务的计量表示标识是否已经开启？

答案：首先在规格计量管理，检查下对应规格是否已经启用计量标识;然后在运营租户“服务管理”检查服务管理的计量标识是否已经开启，如未开启可以在“生态伙伴服务管理”进行启用；

问题：旗舰版私有云订阅和旗舰版私有云许可的规格计量配置如何区分？

答案：旗舰版私有云订阅，第一步计量类型选择特性；旗舰版私有云订阅，第一步计量类型选择独立用户许可。在配置规格信息，统一都选择YonBIP旗舰版私有云。

问题：产品迭代新开发的菜单，如何配置规格计量？

答案：找到规格计量管理，编辑，找到关联服务步骤，补充勾选关联服务，保存后重新提交审核即可。

问题：规格计量配置关联服务的时候，看不到服务，是什么原因？

答案：确认产品包管理是否已经发布制品，如果还未发布，则无法关联服务。

问题：客户环境开通生态应用后，在哪里可以查看生态应用的服务明细和许可占用信息？

答案：在应用许可看板，找到应用所属领域，可以看到生态应用的服务和许可占用情况；
