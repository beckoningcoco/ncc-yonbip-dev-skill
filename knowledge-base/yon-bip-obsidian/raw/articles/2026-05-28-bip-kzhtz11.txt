业务场景

供应商认证会存在很多发工作通知的场景。如：

1 、采购商侧现场认证任务时，现场认证任务保存后，可点击【发起打分】按钮， 根据打分指标分配的情况按照设置的打分人生成个人的现场认证评分任务单据，形成工 作通知到相应的打分人员，然后打分人员会进行评分操作

2 、对采购商资质认证任务在单据发布状态=待发布时，点击【发布】按钮，则将该 资质认证任务单据同步到对应的供应商租户，形成供应商的工作通知。

基于以上业务场景，会存在如下两种发送场景

1 、采购商给采购商发送通知（同租户）；

2 、采购商给供应商发送通知（跨租户）；

问题：使用OpenApi发送工作通知，目前第二种场景，采购商给供应商发送通知后，

供应商查看时，会自动切换到采购商租户，导致查看结果不对。原因是OpenApi不支持跨租户发送通知。

跨租户发送工作通知

由于OpenApi不支持跨租户的接口调用，需要使用其他方式来实现跨租户的使用场

景。有如下两种方式：

1、api函数

2、后端脚手架使用sdk

推荐使用第二种，方便代码的统一管理。

消息发送接口

接口： IMsgSendService

接口类型： rpc

说明：发送普通消息给友户通用户接口， 调用该接口可以发送普通消息，支持的通

道有短信、邮件、工作通知、待办、预警通知。（注：租户分库需要走越权机制）

方 法 名 ： sendPlainMessageToUsers(String sysId, String tenantId,

List receivers, List channels,String messageType, String

title, String content, Map<String, Object> extParams)

参数说明

参数名	意义	是否必须
sy sId	系统 Id，公有云为 diwork	是
tenantId	租户 Id	是
receivers	接收人友户通 ID 列表	是
channels	发送通道	是
messageType	消息类型，支持 notice(通 知)和 creatToDo(待办)和 prewarn(预警)	否，默认为 notice
title	消息标题，如果是短信，这里是短信签名；如果是邮件，这里是邮件主题；如果是友空间，这里是标题	是
content	消息内容	是
extParams	扩展参数，发送消息时传入各个通道的扩展参数	否

参数示例：

发送短信消息
{
    "tenantId": "xcy78984",
    "sysId": "diwork",
    "receiver":["1b1123f0-5d62-41fe-b619-15922397f917"],
    "channels": ["sms"],
    "subject": "用友网络",
    "content": "您好，测试短信。 "
}


2.发送友空间工作通知，需要在 extParams 中传入工作通知需要的参数， key 为 uspaceExt，

value 是个 Map 格式，按需传入，包括：

•tabId:工作通知展示的页签 ID（不传使用默认值）；

•url:工作通知跳转 URL；

•webUrl:web 端跳转 URL；

•miniProgramUrl:小程序跳转 URL。

{
    "tenantId": "xcy78984",
    "sysId": "diwork",
    "receiver": [ "1b1123f0-5d62-41fe-b619-15922397f917"],
    "channels": [ "uspace"],
    "subject": "通知测试",
    "content": "您好，工作通知测试。 ",
    "extParams": {
        "uspaceExt": {
            "tabId": "5d2ec7a89d12c02c65e0f557",
            "url": "www.yonyoucloud.com",
            "webUrl": "www.yonyoucloud.com",
            "miniProgramUrl": "www.yonyoucloud.com"
        }
    }
}


3.发送友空间待办消息，需要在 extParams 中传入待办需要的参数，createTodoExt，value 是

个 Map 格式，按需传入，包括：

•appId:应用 Id，不传使用默认值

•businessKey:待办 key，用来标记待办使用，不传使用 UUID

•typeName:移动端页签名称

•mobileUrl:移动端跳转 URL；

•webUrl:web 端跳转 URL；

•miniProgramUrl:小程序跳转 URL

{
    "tenantId": "xcy78984",
    "sysId": "diwork",
    "receiver": ["1838579e-6b2a-40e0-8c36-9d64817ba53c"],
    "channels": ["uspace"],
    "messageType": "createToDo",
    "subject": "待办测试",
    "content": "您好，待办测试。 ",
    "extParams": {
        "createTodoExt": {
            "appId": "5d2ec7a89d12c02c65e0f557",
            "businessKey": "123",
            "typeName": "待办",
            "mobileUrl": "www.yonyoucloud.com",
            "webUrl": "www.yonyoucloud.com",
            "miniProgramUrl": "www.yonyoucloud.com"
        }
    }
}

1.调用 OpenAPI 出现异常 java.net.SocketTimeoutException: Read timed out

SDK 中设置的超时时间默认值为 5000ms，调用方使用时根据实际调用量按需调整：

HttpOptions httpOptions = new HttpOptions() 
.setConnectionRequestTimeout(30 * 1000) //从连接池获取连接的超时时间 单位ms 
.setConnectionTimeout(30 * 1000) //建立连接的超时时间 单位ms 
.setSocketTimeout(180 * 1000); //等待响应的超时时间 单位ms 
OpenServiceBuilder openServiceBuilder = new OpenServiceBuilder().setHttpOptions(httpOptions)

API未被授权：APPKEY[xxx]未获得要调用的API[/yonbip/uspace/rest/openapi/idempotent/work/notify/push]的授权

需要在API授权节点对指定的APPKEY执行API授权

{"code":"10018","message":"获取token失败，应用不存在或appKey已停用"}
