业务场景

在本文档中，将重点讲解如何通过脚手架前端工程代码编写，完成业务功能的扩展。

在“商品采购需求单”功能中添加业务处理，具体业务场景为：

新增数据时，录入方式不可编辑并赋值为“手工录入”；
编辑数据时，采购日期<=付款日期。

MDF、前端开发、事件、发布订阅模式、MVVM

扩展开发核心内容
前端扩展开发说明

线上某些页面的业务逻辑非常复杂时，可以通过在自建引擎中的前端脚手架中，在创建单据页面对应的脚本文件里继续写业务逻辑。

脚手架的相关内容，请参考：

YonBuilder私有云开发-前端脚手架介绍

(出处: 开发者社区)

扩展脚本命名

扩展文件名字是由应用编码（配置cSubId，如：cpurart）和当前的单据号billNo（配置cBillNo，如：kkfnProductPurchaseRequisition）以及固定扩展字符（VM.Extend.js）组成，中间用下划线（_）进行连接，例如：cpurart _kkfnProductPurchaseRequisition_VM.Extend.js 。

!

扩展文件放在src/business/｛应用编码｝ 中。business文件夹根据应用划分，因此在business下建立自己应用编码的文件夹，并将所有扩展脚本放置里面。这里就是cpurart文件夹。

规则： 应用编码/应用编码_页面billNo_VM.Extend.js

或者是查看meta接口返回结果的extendFileName 字段值

扩展脚本的使用

node层在生成元数据模板后，会在前端加载扩展脚本，并执行init方法，加载业务的扩展逻辑。

方法说明：

define：加载依赖的公共扩展脚本

doAction：框架中加载扩展脚本的入口

init：初始化整个扩展脚本

/**
 * 所属节点：商品采购需求单-卡片页面
 * 创建人：ckkfn
 * 创建时间：2025年2月22日
 * 最后修改人：ckkfn
 */
cb.define(process.env.__DOMAINKEY__, [], function () {
  const cpurart_kkfnProductPurchaseRequisition_VM = {
      doAction: function (name, viewModel) {
          if (this[name])
              this[name](viewModel);
      },
      init: function (viewModel) {
        
      }
  }
  try {
      module.exports = cpurart_kkfnProductPurchaseRequisition_VM;
  } catch (error) {

  }
  return cpurart_kkfnProductPurchaseRequisition_VM;
});


在扩展脚本中，可以获取页面控件层面的viewModel，调用相关viewModel的API或 监听viewModel对应事件编写逻辑，实现具体的特定的交互。可以通过相关api获取当前页面UI组件所对应的 Model对象.

通常扩展业务逻辑一般都写在init函数内，当单据渲染某些生命周期触发业务逻辑时可参考文档：

【[专业开发][前端框架] MDF中的ViewModel单据模型】 (出处: 开发者社区)

新增时，确认状态不可编辑并赋默认值为待确认。
// 新增时，确认状态不可编辑并赋默认值为待确认
viewModel.on('modeChange', (mode) => {
if (mode.toLocaleLowerCase() == 'add') {
viewModel.get('inputType').setValue('1');
        viewModel.get('inputType').setDisabled(true);
    }
});

模态枚举

页面渲染的mode值，描述页面的模态信息。

mode值	描述
browse	预览态，不可编辑信息，只能查看信息
edit	编辑态，可编辑信息，对已有数据进行更新操作
add	新增态，新增数据信息
编辑数据时，要求采购日期<=付款日期。
// 采购日期<=付款日期
viewModel.get('purDate').on('afterValueChange', (data) => {
    let payDate = viewModel.get('payDate').getValue();
    if (data.value && payDate && new Date(data.value) > new Date(payDate)) {
       cb.utils.alert('采购日期不能晚于付款日期！', 'error');
       viewModel.get('purDate').setValue('');
       return false;
}
});
viewModel.get('payDate').on('afterValueChange', (data) => {
    let purDate = viewModel.get('purDate').getValue();
if (data.value && purDate && new Date(data.value) < new Date(purDate)) {
cb.utils.alert('采购日期不能晚于付款日期！', 'error');
        viewModel.get('payDate').setValue('');
        return false;
    }
});
