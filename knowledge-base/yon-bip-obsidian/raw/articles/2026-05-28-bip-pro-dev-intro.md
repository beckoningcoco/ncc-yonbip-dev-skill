业务场景

为了解决传统应用升级缓慢、架构臃肿、不能快速迭代、故障不能快速定位、问题无法快速解决等问题，云原生这一概念横空出世。云原生可以改进应用开发的效率，改变企业的组织结构，甚至会在文化层面上直接影响一个公司的决策。 YonBuilder专业开发工具就是一个帮助企业用户摆脱传统应用开发模式，重塑数字化体制的一整套云原生开发解决方案。 当企业的数字化转型过程中，有了微服务自治、业务数据私有化、Devops、敏捷转型的需求时，YonBuilder专业开发工具将为这些企业提供帮助。

关键词

专业版、应用引擎、应用构建、应用构建升级版、对象建模、页面建模、DevOps、流水线、微服务

名词解释

工作台：即YonBIP数字化工作台服务，是YonBIP的统一工作入口，目前已经集成YonBIP各领域云产品，包括财务云、营销云、制造云、供应链云、协同云、人力云、云平台等；YonBIP数字化工作台提供统一的应用入口、全局菜单、个人中心、角色化工作台、智能搜索、统一代办、应用管理等基础能力。

应用构建-基础版：指在工作台入口内为企业租户提供的基础版应用构建产品，提供无代码声明式开发和在线函数式低代码开发的租户快速实施能力。

应用引擎：应用构建通过应用引擎为开发者提供本地代码扩展的高级定制能力，提供脚手架代码工程下载、本地开发调试、持续集成、快速发布等开发全生命周期的应用运行时支撑。当企业租户/isv需要源码开发或者独立部署时，即可将线上应用迁移托管到自己创建的应用引擎内。

技术平台：面向开发者提供开发运维的能力，提供应用构建高端定制服务，提供云服务研发、运维、运营全生命周期的平台能力。

应用构建-专业版：指在工作台入口内为原厂/isv/专属化客户提供的专业版应用构建产品，专业版产品包含基础版全部能力，且支持对多企业租户同时在线实施开发，支持本地代码扩展和独立部署的全生命周期管理。

应用构建升级版：应用构建升级版支持全新自建应用和原厂领域扩展应用，是应用构建的新版本，支持一个应用同时含有零代码、低代码、专业开发三种开发模式的资源，同时支持零代码应用到低代码、专业开发的平滑升级；支持多个角色协同开发，实现业务在不同阶段的诉求，让应用扩展不再受限，让开发没有天花板。

单据设计态：应用构建页面建模单据设计器中对单据页面进行字段布局，样式等设计开发的阶段。

单据运行态：单据页面设计开发完成，实际预览页面效果的状态。

YPD框架：全称为Yonyou Platform Development Framework，一套面向强类型实体，以插件扩展开发模式为主的后端框架。

YDS工具：全称为YonBuilder Developer Studio，提供全面的一站式开发解决方案，涵盖了从项目创建、工程协同调试、部署，到制品生成和补丁制作等关键环节，确保开发服务流程的高效与连贯性。

元数据：描述数据的数据，对数据及信息资源的描述性信息。

UI元数据：描述前端UI的数据，对前端页面UI数据级信息资源的描述性信息。

开发租户：是指用应用构建创建应用的租户。

测试租户：在这特指测试或使用系统应用的租户。

租户应用：开发租户使用应用构建创建的应用属于租户应用，该应用只能在开发租户中使用。

系统应用：系统应用，又称原厂预制应用。只要有相应使用权限，就可以使用的应用。

产品特性

专业开发过程与技术平台的容器、微服务等有机融合，为专业开发人员提供开发运维一体化的全生命周期管理能力，让对性能和安全敏感的企业应用有了保障。

自动注册流程中心，编码规则等基础支撑服务。满足YonBuilder标准版能力的同时，支持自定义和自行管理持久化数据库。同时为专业开发者提供了基础的脚手架代码工程，支持本地开发调试。

技术架构
功能架构

应用构建专业引擎服务对专业版引擎进行统一维护管理，主要包括成员管理，托管应用，实现自动注册微服务，无缝对接基础支撑服务，提供生成脚手架代码的能力。支持简便快捷方式创建流水线，为专业版应用运行时提供能力。

脚手架

通过代码工程模板，使用freemarker模板引擎，结合引擎信息以及一些环境信息生成对应的脚手架代码工程。脚手架工程代码会在服务内做一级缓存，提高多次下载/推送的效率。代码模板通过远端git仓库获取实时最新的版本。针对一些私有云部署的环境，服务中会预置一份固化版本的代码模板。

目前后端脚手架分为基于YPD框架脚手架和基于MDD框架的两种脚手架，前端脚手架是基于MDF框架和YNF框架提供的脚手架。

功能介绍

YonBuilder专业开发工具是针对专业开发人员，方便开发人员。通过自建引擎为开发服务向流程中心，编码规则等基础支撑服务无缝注册对接；通过引擎管理进行引擎信息、开发协作人员、托管应用的自主管理；通过代码模板，根据微服务编码自动生成服务对应脚手架代码，方便开发团队进行本地开发和代码管理；支持便捷快速创建前后端分离的流水线服务，通过使用开发者自己的资源服务，轻松使用技术平台能力，实现CICD。同时，不同环境不同租户下数据共享与复用，提供了专业数据迁移工具。YonBuilder专业开发工具为开发人员提供开发运维一体化的管理能力。针对领域开发，原厂开发，YonBuilder提供了租户级应用发布系统级的标准工具。

应用开发

创建专业版应用，首先进行应用引擎创建，应用创建，然后进行对象建模、页面建模、流程&自动化配置、集成配置，过程中通过Git下载脚手架代码进行本地前后端开发调试，完成业务需求的开发。

应用发布

由开发租户在应用构建中创建出的租户级应用，可以通过系统应用发布功能，发布成系统应用(即原厂预制应用)。

通过开发迁移工具功能形成制品，导出在目标环境安装，进而完成开发成果的迁移。

开发准备
技术栈

前端：nodejs、React

后端：Java、Springboot

中间件：关系型数据库（mysql、oracle、达梦、Oscar、postgresql） + YonMatrix + redis + kafka + nginx + es + minio/sftp

云原生底座：k8s 、 Docker 、 DevOps

开发团队组成

前端开发工程师：根据项目业务需求评估是否需要自定义页面、自定义组件，如果需要则项目组中要具备专业前端开发人员，熟练使用REACT；如果平台提供的组件和页面布局能满足产品开发需求，团队中需包含具备一定JS代码能力的工程师。

后端开发工程师：掌握JAVA语言，熟悉Springboot框架开发技术，熟悉关系型数据库的开发，熟练使用常见中间件，对云原生开发体系有一定的理解。

开发流程

整体流程和软件研发流程是一致的，包括需求分析、功能设计、开发、测试、上线验收。开发人员根据业务功能需求进行开发方案的设计，根据开发方案进行开发，开发完成进行开发成果的迁移，安装到测试环境进行测试或验收，测试通过或用户验收通过后安装到生产环境进行企业级的应用。

客户化开发与ISV产品开发在本地部署环境下开发技术是一致的，流程上有些区别，开发规范上也有些区别，例如引擎编码，具体可参考开发规范；客户化项目交付过程可参考高端项目交付流程，ISV产品项目过程可参考ISV原生开发流程指引。

ISV产品开发过程需要注意以下事项：进行生态产品的开发需取得产品开发认证证书，学习认证可参考开发者认证章节；开发过程会使用到沙箱，此阶段会提供3个租户分别做为：开发沙箱、测试沙箱、管理沙箱，开发沙箱：用于生态产品的开发工作（参与人员：伙伴开发人员）；测试沙箱：通过制品包或配置迁移将开发沙箱的开发成果迁移至测试沙箱，进行产品验证（参与人员：伙伴开发人员、伙伴测试人员、用友生态测试人）；管理沙箱：测试沙箱验证通过后，同样的方式将开发成果迁移至管理沙箱，在此沙箱进行制品，发布零租户（参与人员：伙伴开发人员、伙伴测试人员、用友生态测试人员）。

公有云可以做专业开发吗？

答：可以的，在商开环境进行专业版应用的开发，商开环境属于公有云。

应用引擎是做什么的？

答：应用引擎提供了后端和前端服务，驱动应用功能的展现。

专业开发需要具备哪些技术能力？

答：专业开发会涉及前端与后端的开发，因此需要具备js和Java开发能力，具体可参考开发团队组成的内容。

ISV产品开发者必须要认证通过吗？

答：必须通过，具备一定的开发能力才能进行产品开发。

业务需求实现怎么判断是否使用专业开发？

答：专业开发是为了满足企业对性能、安全的更高要求，同时专业开发可以使用前端、后端工程更好地实现复杂的业务场景，因此，如果有这些需求，就可以使用专业开发。
