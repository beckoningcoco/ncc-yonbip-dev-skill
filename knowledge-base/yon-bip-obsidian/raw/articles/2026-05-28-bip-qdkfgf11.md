前端性能规范
目的

提升YonBIP产品整体性能，节省资源流量，优化页面访问效率。

规范说明
【强制】页面下行流量：首次加载不超过2M，非首次不超过1M；
【推荐】页面打开效率：首次打开不超过2s，非首次不超过1s；
【强制】单个前端文件资源大小（gzip、压缩后）不超过500K；
【强制】公有云静态资源必须走CDN加速；
【推荐】整站系统资源不能超过40M，尽可能的首页把所有资源下载完成；
【强制】不影响首屏交互的接口，需要进行异步加载；
【强制】渲染顺序上先加载样式文件、后加载js文件，避免页面错乱；
【强制】白屏时间10ms以内；
【推荐】服务节点在BIP数字化工作台中以非Iframe嵌入方式接入。

HTML 编程规范
代码格式

(1)【强制】缩进使用 soft tab（4个空格）

正例：

<ul>
<li>first</li>
<li>second</li>
</ul>

<select>
<option value="1" selected>1</option>


【推荐】每行不得超过120个字符。

(2)【强制】嵌套的节点应该缩进；

(3)【强制】在属性上，使用双引号，不要使用单引号；

(4)【强制】属性名全小写，用中划线做分隔符；

(5)【强制】不要在自动闭合标签结尾处使用斜线（HTML5 规范指出他们是可选的）；

(6)【强制】不要忽略可选的关闭标签

正例：

<!DOCTYPE html>
<html>
<head>
<title>Page title</title>
</head>
<body>
<img src="images/company_logo.png" alt="Company">

<h1 class="hello-world">Hello, world!</h1>
</body>
</html>

<select>
<option value="1" selected>1</option>
</select>


(7) 【强制】boolean 属性指不需要声明取值的属性，XHTML 需要每个属性声明取值，但是 HTML5 并不需要

正例：

<input type="text" disabled>

<input type="checkbox" value="1" checked>

命名风格

(1)【强制】class必须单词全字母小写，单词间以-分隔。

(2)【推荐】class必须代表相应模块或部件的内容或功能，不得以样式信息进行命名。

正例：

<div class="sidebar"></div>


反例:

<div class="left"></div>


(3)【强制】元素 id 必须保证页面唯一。

(4)【强制】同一页面，应避免使用相同的 name 与 id。

HTML5 doctype

(1) 【推荐】在页面开头使用这个简单地 doctype 来启用标准模式，使其在每个浏览器中尽可能一致的展现；

(2) 【推荐】虽然 doctype 不区分大小写，但是按照惯例，doctype 大写

正例：

<!DOCTYPE html>
<html>
...
</html>

lang 属性

【强制】根据 HTML5 规范：应在 html 标签上加上 lang 属性。语言列表如下： zh-cn、zh-hk、zh-tw、en-us等。

正例：

<!DOCTYPE html>

<html lang="en-us">

...

</html>

字符编码

【强制】通过声明一个明确的字符编码，让浏览器轻松、快速的确定适合网页内容的渲染方式，通常指定为 'UTF-8'。

资源引入顺序

【强制】根据HTML5规范, 通常在引入CSS和JS时不需要指明 type，因为 text/css 和 text/javascript 分别是他们的默认值。

正例：

<style> ... </style>

<script src="code_guide.js"></script>

<script> ... </script>

IE兼容模式

【推荐】用标签可以指定页面应该用什么版本的 IE 来渲染，不同doctype在不同

浏览器下会触发不同的渲染模式。

<!DOCTYPE html>
<html>
<head>
<meta http-equiv="X-UA-Compatible" content="IE=Edge">
</head>
...
</html>

属性顺序

【推荐】属性应该按照特定的顺序出现以保证易读性：

class
id
name
data-*
src, for, type, href, value , max-length, max, min, pattern
placeholder, title, alt
aria-*, role
required, readonly, disabled

class是为高可复用组件设计的，所以应处在第一位；

id更加具体且应该尽量少使用，所以将它放在第二位。

异常捕获和上报

通过YMC 自动采集上报，需引入YMC UBA前端采集SDK，工作台环境默认已引入。

自动采集错误类型包含：

Error 全局抛出的Error类异常，通过监控error事件捕获收集，并匹配对应错误类型及错误码，同时支持自定义类错误通过可选的errorCode属性指定其错误码；
console.error打印的错误，通过重写包装console.error方式捕获采集，错误码可通过第一个可选参数指定，规则/^E\d{5}$/；

示例：

console.error('E00007', '上下文缺失错误'); // 指定错误码方式1
console.error(Error.E00007, '测试错误信息'); // Error.E00007为对象{code: 'E00007', text: '上下文缺失错误'} // 指定错误码方式2
console.error('测试信息'); // 未指定错误码
