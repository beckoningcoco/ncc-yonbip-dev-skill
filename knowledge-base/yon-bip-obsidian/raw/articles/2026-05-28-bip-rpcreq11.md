最后更新时间：2025-05-20
概述
业务场景

YMS-JDBC 是一个轻量级数据持久化组件，基于 JDBC 进行封装，提供面向对象的 ORM 映射能力，简化数据库操作代码，支持原生 SQL 与动态条件构造，适用于需要灵活控制 SQL 但希望减少重复代码的场景。

作为YPD,MDD框架的底层持久化框架，同时也支持非YPD,MDD框架下的持久化功能。

关键词

持久化组件、注解驱动、元数据映射、类型处理器、多数据源

注解设计
实体

实体注解（@YMSEntity）：用于描述java实体与数据库表的映射关系。

属性	类型	描述	默认值
name	String	实体名(namespace+domain+name 要保证唯一性)	空
domain	String	域	空
namespace	String	所属命名空间	空
列

列注解（@YMSColumn）：用于描述java实体的属性与数据库字段的对应关系，以及对数据的行为、数据长度、数据精度等进行约束。

属性	类型	描述	默认值
name	String	名称	空
unique	Boolean	是否唯一	false
nullable	Boolean	是否可以为null	true
insertable	Boolean	是否支持插入	true
updatable	Boolean	是否支持更新	true
columnDefinition	String	列定义	空
table	String	对应表	空
length	int	长度限制	255
precision	int	精度（字段长度）	0
scale	int	范围（小数位数）	0
主实体

主实体（@YMSRoot）：平行表或主子孙实体中用于标识该实体是主实体

主键

主键（@YMSId）：用于标识该字段是主键

主键生成策略：

主键生成策略（@YMSGeneratedValue）：用于描述主键的生成策略

属性	类型	描述	默认值
strategy	Stragegy(枚举)	策略	YMS OID
domain	String	域
customClass	String	定制类全限定名	空
表

表注解（@YMSTable）：用于描述java实体与数据库表的对应关系。

属性	类型	描述	默认值
tableName	String	表名称	空
一对一

一对一（@OneToOne）：用于描述Java实体与Java实体的一对一关系。

属性	类型	描述	默认值
targetEntity	Class	目标实体
columnName	String	关联外键，不填默认使用@YMSId主键关联OneToOne表	""
cascade	CascadeType[]	级联持久化控制枚举，与JPA类似	CascadeType.ALL
fetch	FetchType	级联查询控制枚举，与JPA类似	FetchType.EAGER
一对多

一对多（@OneToMany）：用于描述Java实体与Java实体的一对多关系。

属性	类型	描述	默认值
targetEntity	Class	目标实体
associationProperty	String	一对多关联属性(如pid)
cascade	CascadeType[]	级联持久化控制枚举，与JPA类似	CascadeType.ALL
fetch	FetchType	级联查询控制枚举，与JPA类似	FetchType.NOT
DTO别名

DTO别名（@YMSAlias）： 用于映射DTO属性和查询SQL中的别名

属性	类型	描述	默认值
name	String	名称	空

查询结果集处理器
解析器类名	说明
BeanProcessor	值对象处理器，返回一个javaBean,每个javaBean对应结果中一行数据
BeanListProcessor	值对象集合处理器，返回一个ArrayList集合，集合中的每一个元素是一个javaBean,每个javaBean对应结果集合中一行数据
ColumnProcessor	列值处理器，返回一个Java对象，结果集中只有一行数据，该对象对应与结果集中某一列的值，该处理器通过结果集列的序号或名称来确定列
ColumnListProcessor	列值处理器，返回一个ArrayList对象，结果集中有多行数据，该对象对应与结果集中某一列的值，该处理器通过结果集列的序号或名称来确定列
MapProcessor	HashMap处理器，返回一个HashMap, 结果集中只有一行数据，其中结果集合中每一列的列名和列值对应HashMap的一个关键字和相应的值
MapListProcessor	HashMap集合处理器，返回一个ArrayList集合，集合中的每一个元素是一个HashMap，每个HashMap对应结果集中的一行数据,其中结果集合中每一列的列名和列值对应HashMap的一个关键字和相应的值
ArrayProcessor	数组处理器，返回一个对象数组，结果集中只有一行数据，其中结果集中每一列对应数组的一个元素
ArrayListProcessor	数组集合处理器，返回一个ArrayList集合，集合中的每一个元素是一个数组，每个数组对应结果集中的一行数据，其中结果集中每一列对应数组的一个元素
DTOProcessor	DTO处理器，返回一个DTO, 结果集中只有一行数据，其中结果集合中每一列的列名和列值对应DTO的一个关键字和相应的值，DTO支持带注解与不带注解两种方式
DTOListProcessor	DTO集合处理器，返回一个ArrayList集合，集合中的每一个元素是一个DTO，每个DTO对应结果集中的一行数据,其中结果集合中每一列的列名和列值对应DTO的一个关键字和相应的值，DTO支持带注解与不带注解两种方式
AutoBeanProcessor	值对象处理器，返回一个javaBean,每个javaBean对应结果中一行数据，支持自动映射
AutoBeanListProcessor	值对象集合处理器，返回一个ArrayList集合，集合中的每一个元素是一个javaBean,每个javaBean对应结果集合中一行数据，支持自动映射

常见问题
1. 数据源
1.1. 数据源没有配置jdbcUrl

检查数据源配置文件yms_dynds.json是否存在
如果数据源配置文件不存在，查看yms_config.json内容是否存在异常
如果yms_config.json也不存在，查看ym制台地址是否正确，spring.application.name、moduleName、yms控制台的领域编码是否一致
检查数据源bean名称是否规范：beanName是逻辑数据源下划线后的值。
1.2. 当前查询条件查询的结果超过100001，数据量过多，建议缩小范围查询，提升查询效率，例如缩小时间范围到一月或一周
优先优化自己代码，缩小查询范围
增加配置 uap.resultset.maxRowsValue=-1
1.3. 本地启动缺少数据库驱动
添加对应驱动的依赖
1.4. 租户 [租户ID] 已经创建,但尚未激活,请联系商务代表! 错误码:200002[没有在YMS控制台数据源连接池配置租户映射，请联系运维配置]
检查租户是否绑定到对应的连接池上，连接池是否绑定到逻辑数据源上，发布配置重启
1.5. 创建数据源失败
检查是否缺少驱动
检查数据库是否可以连接
1.6. 启用了租户路由，但是获取数据源连接时上下文没有租户id，检查此处调用是否有上下文, 没有上下文需要检查调用的上游有没有设置上下文

检查该逻辑数据源是否需要开启租户路由，如果不需要，在yms控制台修改配置，发布，重启服务。

检查此处调用是否有上下文, 没有上下文需要检查调用的上游有没有设置上下文。

检查是否在新线程内操作，检查是否在定时任务中操作。修改方法：使用yms线程池或者使用联邦查询。

1.7. com.yonyou.iuap.yms.datasource.ds.YMSDataSource.init()方法出现空指针

查看YMSDataSource的dsProvider属性是否为空

是否能获取到ymsProvider这个bean

查看是否有其他报错，导致spring容器启动失败

1.8. 数据加密失败

检查yms控制台是否开启加密 uap.encrypt.switch=1 需要配置在对应的微服务上。

检查yms控制台配置文件是否发布，发布后重启服务。

ognl -x 3 '@com.yonyou.cloud.tualatin.monitor.YMSSlowSqlMonitorUtils@encryptSwitch' 通过arthas命令查看是否开启加密开关 返回值大于0就可以。如果是-1 需要检查yms控制台配置 发布yms控制台配置并重启服务。

1.9. 获取默认连接池为空

先确定是哪个实例出现的异常，然后通过arthas查看实例是哪个维度的 vmtool -x 3 --action getInstances --className com.yonyou.iuap.yms.config.service.YmsConfigService --express 'instances[0].getCurrentDimension()'

然后询问运维这个租户组是哪个维度，如果租户组的维度和实例的维度不一致，说明请求打错实例了，需要联系赵永军老师或张威老师排查。

如果租户组维度和实例维度一致需要接着排查。

先通过arthas命令查看逻辑数据源绑定了哪些连接池 vmtool -x 3 --action getInstances --className com.yonyou.iuap.yms.datasource.ds.provider.impl.YMSGlobalDataSourceProviderV2 --express 'instances[0].groupConfigMap.get("逻辑数据源编码").getDsList()'

再通过arthas命令查看连接池绑定的租户组 vmtool -x 3 --action getInstances --className com.yonyou.iuap.yms.datasource.ds.provider.impl.YMSGlobalDataSourceProviderV2 --express 'instances[0].dsDefineGroupMap.get("连接池编码").getTenantGroupSet()'

检查逻辑数据源绑定的连接池是否有绑定异常信息中的租户组，如果没有绑定需要检查yms控制台配置，询问运维这个租户组对应的连接池并把这个连接池绑定到逻辑数据源上，发布重启，如果这个租户组没有绑定连接池，询问运维这个租户组需要绑定到哪个连接池，绑定后再把这个连接池绑定到逻辑数据源上 发布重启。
