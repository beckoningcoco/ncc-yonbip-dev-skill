业务场景

业务流，是多个单据类型串接起来的一个业务流程，每个单据类型在流程中有对应的上游、下游单据。在上游-->下游的单据流转过程中，可定义流转规则，包括单据的拆分、字段映射、回写等，在项目交付过程中，如果存在原厂单据到自建单据的流转，

就可能涉及到原厂单据推自建单据的开发。

本文档结合几百个业务流ISP问题单，从中筛选出项目开发共性问题，发现项目开发过程中存在如下几个比较典型的问题：

业务流推单不触发，原厂单据需要在对应的规则链中预制推单规则，部分领域未预制
业务流回写不自动调用，需要开发添加对应的脚本。
业务流设计选不到拉单画面，需要开发拉单页面。
业务流设计中，添加业务流规则，原厂单据未自动生成按钮，项目开发人员不知需要满足一定的条件才会自动生成按钮

结合业务流常用开发场景，总结出的开发案例，目的是帮助项目开发人员在开发过程中提前识别开发过程中可能遇到的问题，提高项目开发效率。

专业名词术语

上/下游单据：在一个业务流程中，若A单据会直接生成B单据，我们把A单据叫做B单据的上游单据。把B单据叫做A单据的下游单据

原厂单据：领域MDD单据，本案例暂不考虑部分领域YPD框架

自建单据：项目开发人员采用专业版YPD引擎开发的单据，本案例暂不考虑公共引擎自建单据和MDD引擎自建的单据

业务流开发规范
开发规范

单据编码需与业务对象保持一致

业务流设计中配置的是业务对象与业务对象之间的转换关系，不是指定具体的单据，业务流在处理时候，会将业务对象的编码转换成单据编码获取单据相关的信息，如果单据编码与业务对象编码不一致，会导致业务流在处理的过程中找不到单据信息。

例如，转单画面无法跳转到对应的转单界面，权限校验无法获取正确的servisecode进行权限校验，默认的全局联查无法跳转到对应的单据。

业务流开发常见问题
业务流自动推单找不到数据库表

原厂单据自动推单生成自建的单据，推送报找不到数据库表，日志中显示的insert语句只有表名，没有加上库名，该问题是因为自建引擎没有在业务流中注册，导致生成下游单据是直接在当前服务执行insert，没有到下游单据的自建服务中执行保存，而当前服务是操作的当前服务对应的schema，导致找不到表报错。

单据转换规则中选不到原厂扩展字段

业务流设计中，在单据转换规则中，选择不到原厂扩展的平行表字段或分配的特征字段。

解决办法：

在应用平台中选择业务对象节点，找到对应的业务对象，编辑该业务对象，找到对应的扩展字段，选择场景，勾选转换规则（有些系统叫业务流），保存并发布。

添加自定义规则，原厂单据中未自动生成推单按钮

先在原厂单据模版中启用租户级模版，包括列表页和卡片页面，然后在业务流设计中重新停用启用业务流。

原厂页面添加手动规则按钮，页面预览时按钮不显示

前端请求中没有加载developplatform的extend.min.js

业务流的按钮是在这个扩展里面，如果没有加载资源是没法渲染的

在前端MDF服务中配置文件加上如下内容依赖，然后重启MDF服务

{ "yonbip-cpu-cooperation": { "dependencies": [ "developplatform" ] } }

原厂自动推单没有触发业务流推单

原厂的推单是通过在对应规则链中执行业务流的规则来实现推单，通过查询对应的规则链中是否有推单规则

查看原厂单据对应的动作下是否有推单规则

select * from billruleregister t where t.action='对应单据动作' and billnum in ('common','对应单据编码') and t.ytenant_id='对应的租户' and t.ruleId='bizFlowBatchPushRule'
