UI模板
最后更新时间：2024-05-14
本文摘要

UI 模板，是原厂单据在运行态的显示扩展，UI 模板对单据 UI 元数据进行扩展配置， 通过 UI 元数据配置和高级设计器的扩展配置，依赖 MDD 和 MDF 对元数据在运行态的公式能力、参照带入能力、交互规则能力、组件显示属性自定义以及页面布局能力的支撑， 结合 UI 模板多维度的管理分配以及业务规则预置功能，满足原厂单据在不同业务场景的个性化支持。

本文主要介绍了UI 模板典型使用场景，可概括为以下核心要点：

1、不同角色不同模板：实现租户级不同用户打开业务单据显示字段数量不同、名称不同、布局不同等；

2、不同组织不同模板：实现租户级单据不同业务组织单据显示字段数量不同、名称不同、布局不同等；

3、运用公式满足个性化需求：通过模板配置公式能力，满足客户提出的业务计算、档案取数的个性化逻辑等；

开发者通过对本文的学习，可了解 UI 模板对外支持的功能以及如何接入管理菜单，

第一章 基本概念
背景

UI 模板管理的功能，为租户对领域出厂的 UI 模板做个性化定制的功能，相关支持的功能如下几个部分

配置功能支持：简单的配置功能，可以定制租户级字段名称、显示隐藏、是否必输、宽度、tips 等功能定制
设计查询字段功能：新增查询字段，可以在详情态展示相关的字段，或者新增公式字段，用于特定展示公式计算数据
多模板功能支持：对于租户下不同的组织、不同的交易类型、不同的用户、不同的角色可以使用各自定义的模板
高级设计器功能支持：对于特殊的 UI 定制场景，UI 模板服务还提供了低代码的高级设计器对模板进行定制修改，包括 pc 端和移动端的

UI 模板服务的核心：为租户个性化各个领域的 UI 模板提供功能支撑。

第三章 功能介绍
功能概述

UI 模板是原厂单据在运行态的显示扩展，UI 模板对单据 UI 元数据进行扩展配置， 通过 UI 元数据配置和高级设计器的扩展配置，依赖 MDD 和 MDF 对元数据在运行态的公式能力、参照带入能力、交互规则能力、组件显示属性自定义以及页面布局能力的支撑，结合 UI 模板多维度的管理分配以及业务规则预置功能，满足原厂单据在不同业务场景的个性化支持。

UI 模板管理入口

通过工作台菜单 UI 模板进入 UI 模板管理节点，左侧树为领域-应用-业务对象。

UI 模板字段配置功能

UI 模板的配置功能是模板配置的快接入口，在配置中可以对 UI 模板元数据进行字段重命名、新增显示字段、设置必填、设置默认值、设置显示顺序的基础功能。

UI 模板设计功能

管理用户可通过设计进入低代码单据设计器，对 UI 模板进行配置，可配置内容包括：

新增显示字段；

配置字段显示属性； 设置公式与函数； 配置参照带入；

修改单据布局与显示顺序； 配置页面规则；

交易类型模板支持

支持管理用户对 UI 模板按照交易类型维度进行模板管理和配置。支持在交易类型菜单设置或原厂预置的交易类型节点显示，用户可切换当前单据的交易类型，配置相应交易类型模板。

组织级模板支持

支持管理用户对 UI 模板按照组织维度进行模板管理和配置，不同用户管理的组织权限不同，看到的组织数量不同，UI 模板按照组织分配的功能将组织作为全域筛选条件。

用户可切换有权限的组织，配置相应组织级模板。  

3.7** **分配用户和角色功能支持

支持卡片类型原厂单据为了满足在同一租户下不同角色、不同人员使用不同的 UI

模板的业务场景，需要通过系统模板复制出多个模板后逐个对 UI 模板进行分配，达到不同用户打开的 UI 模板不同的目的。

组织级模板默认获取领域单据上的主组织属性作为组织级模板的组织加载标识，在列表跳转卡片的时候，add、detail 接口中，都会携带 masterOrgKeyField 属性来标记领域单据中哪个属性是主组织属性。

开通组织级模板的单据加载时，会将原来列表跳转卡片的 tpllist 接口将切换成assignedtpllist

扩展开发

后端服务扩展：

领域可手动改写 masterOrgKeyField 返回的属性用于标识组织级模板使用的属性字段
领域可手动改写 assignedtpllist 方法返回值，用于过滤前端可选择的组织级模板数据
