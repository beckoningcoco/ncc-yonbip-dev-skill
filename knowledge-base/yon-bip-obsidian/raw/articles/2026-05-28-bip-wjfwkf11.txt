文件服务开发
最后更新时间：2024-10-23
1 简介

平台提供附件能力，满足文件上传、下载及相关扩展开发能力。 附件管理有两种方式：

1、MDF附件控件对接协同套件。
2、MDD基于MDF框架和文件服务实现的附件管理逻辑。
2 快速上手

在YonBuilder应用构建中可通过对象建模使单据实现附件上传下载的能力。

2.1 使用流程

在数字化工作台YonBuilder应用构建中对应的单据上添加附件类型的字段即可，单据开发请参考 示例 ，添加附件类型的字段如下图：

2）开发中定义附件类型的字段，通过页面建模实现附件组件的布局，在运行态附件组件调用文件服务进行交互，如下图：

3 前端附件扩展

MDF加载协同资源，部分MDF组件只使用协同服务，通过window.__MDF_FILEURL__获取服务地址，然后拼接正确接口路径，主要用到下面接口地址：
window.__MDF_FILEURL__ = https://<工作台域名>/iuap-apcom-file

3.1 附件组件

在工作台打开或者node渲染时都会先去加载协同的资源文件，加载完成后资源文件会在window上挂载所有操作组件和渲染组件的方法window.YYCooperationBridge ：

渲染附件组件方法 window.YYCooperationBridge.YYRenderFile
渲染协同套件方法 window.YYCooperationBridge.YYRenderCooperation
渲染评论组件方法 window.YYCooperationBridge.YYRenderComment

3.2 附件事件

附件事件根据终端的不同分为：PC端和移动端。

3.2.1 PC端
初始化或者更新 beforeCreateAttachmentData 附件组件初始化或者更新时候需要修改传入objectId的事件
/**
* data: 当前附件数据
* config: 当前组件配置信息
*/
viewmodel.on('beforeCreateAttachmentData', ({ data, config }) => {
  data.objectId = 'xxxxx'; // 修改objectId
})

beforeRenderFileComponent 组件加载之前处理authId， authI不需要可以如下面处理
this.viewModel.on('beforeRenderFileComponent', (obj) => {obj.authId = ''});

获取协作套件实例ecsuiteDidMount
viewmodel.on('ecsuiteDidMount', function ({ ecsuiteComponent }) {
    ecsuiteComponent.then(function(ins) {
        // ins附件实例

MDD接入方式
MDD依赖，要求版本高于3.0.8.4-SNAPSHOT
<!-- #######MDD版本引入建议采用iuap-pom引入方式 -->
<!-- MDD版本为5.3.0-SNAPSHOT -->

    <groupId>com.yonyou.iuap</groupId>
    <artifactId>yonyou-mdd-starter-parent</artifactId>
    <version>5.3.0-SNAPSHOT</version>
</parent>
 <!-- MDD版本为5.3.0-RELEASE -->

    <groupId>com.yonyou.iuap</groupId>
    <artifactId>yonyou-mdd-starter-parent</artifactId>
    <version>5.3.0-RELEASE
</version>
</parent>

附件管理入口FileUpload
客户化开发中可根据业务场景进行附件查询、上传等操作。MDD文件管理的入口Controller，默认注入BeanName为fileUploadController，请求统一路径为/pub/fileupload，主要功能接口如下：
1.获取附件管理服务地址/pub/fileupload/getFileServerUrl
2.附件上传/pub/fileupload/upload
3.附件暂存到本地服务器/pub/fileupload/upload2Local
4.附件下载，后端转发模式/pub/fileupload/privateObject/**
5.附件下载，后端流输出模式/pub/fileupload/privateStream/**
5 常见问题

5.1 附件删除不了，报错“不支持文件删除”

分析：单据的verifyState状态不是0或4只允许上传人才能删除，如果是上传人可能是平台问题。
解决思路：判断不是verifyState状态及操作人的问题，需要检查环境及时打补丁。

5.2 MDD附件下载如何加密下载地址

分析：MDD附件下载加密依赖MDD框架验签机制，基于yht_access_token的验签，领域或者专属化如有其它验签方式请自行处理验签。
解决思路：
1.spring项目增加如下两个配置项：
#getFileServerUrl请求是否返回加密地址
mdd.enable.file.server=true 
#配置后端请求的域名，不要以特殊字符结尾
mdd.file.domain.url=http://xxxx.xxx.com 
#如果附件路径需要加密请将此开关打开
mdd.enable.file.secret=true 


2.请确认项目注入了FileUploade这个Controller，如果领域自行实现上产下载逻辑，请忽略此步骤 代码如下：

@RequestMapping("/getFileServerUrl")
   public void getFileServerUrl(@RequestParam(value = "type", required = false) String type, HttpServletRequest request, HttpServletResponse response) throws Exception {
       String enablePrivateUrl = AppContext.getEnvConfig(MddConstants.MDD_ENABLE_FILE_SERVER, "false");
       if (BooleanUtils.b(enablePrivateUrl) && "mdf".equalsIgnoreCase(type)) {
           String domainURL = AppContext.getEnvConfig(MddConstants.MDD_FILE_DOMAIN_URL);
           if (StringUtils.isNotBlank(domainURL)) {
               if (domainURL.endsWith("/")) {
                   domainURL = domainURL.substring(0, domainURL.lastIndexOf("/"));
               }
               domainURL = domainURL + "/pub/fileupload/privateStream";
               renderJson(response, ResultMessage.data(domainURL));
           } else {
               renderJson(response, ResultMessage.data(uploader.getFileServer()));
           }
       } else {
           renderJson(response, ResultMessage.data(uploader.getFileServer()));
       }
   }


3.如果领域复写了FileUpload的getFileServerUrl方法，请领域自行比对代码处理，如无特殊处理请忽略此步骤。
4.请确认前端是否开启配置，确认方式为查看前端getFileServerUrl中是否有type参数，正确实例请求uri如下：https://xxxx.yyuap.com/uniform/pub/fileupload/getFileServerUrl?&type=mdf
5.配置确认完毕后，验证文件上传功能和下载功能是否正常即可。
