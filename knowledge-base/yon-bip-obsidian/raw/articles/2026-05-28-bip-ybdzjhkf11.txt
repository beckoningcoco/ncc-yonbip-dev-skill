YonBuilder组件化开发
最后更新时间：2024-03-05
基本概念
背景

通常开发者可以通过低代码设计器中已经提供了一些默认模板进行单据页面的开发，这些单据都是由默认的内置组件拼接而成，也可以通过使用内置组件进行拖拽开发。但是这远远不能满足于复杂的业务需求，目前复杂的业务界面可以通过专业版来完成，通过默认提供的专业版脚手架对源码进行二开处理，方可满足复杂的业务组件需求，但是这种开发模式复用性较低。

组件化开发，通过统一的组件化开发规范开发出业务组件，既可以在源码中自己使用，也可以上传到组件扩展中心，组件扩展中心对资源统一的管理调度，可以将组件发布到各个租户设计器，可以直接通过设计器拖拽使用，同时资源中心也开放所有前端框架使用。

各领域及其开发者共同开发出的业务组件与，大家共同使用、共同发展、相互促进 ，实现真正的共创共赢。

名词解释

代码设计器：“0”代码拖拽式快速定制,支持根据数据创建应用的模型驱动设计、构建高度定制化应用的画布式设计、基于模板的快速构建。

组件扩展中心：将各领域下的组件统一注册/收集起来管理；将组件及扩展组件数据共享起来，供运行态、设计态、展示态、开发平台等使用； 对组件进行分级运营，为后续组件大数据分析、组件生态购买等提供支撑。

**专业版脚手架：**专业版除了涵盖标准版的全部设计和脚本能力之外，还可以支持开发者自建应用引擎来实现本地源码开发、独立运维部署等企业云服务高级定制能力。

内置组件： 基于ArcUI或TinperNext为低代码设计器提供的一套业务组件，供给设计器进行拖拽开发。其中包含布局容器、数据容器、基础控件、表单控件、功能组件、业务组件。

**Arc-UI：**简单、轻量的B端移动应用组件库；服务用友BIP生态，构建数智化移动应用新体验。

**TinperNext：**是一套基于 React.js 的UI组件库，它从丰富的企业级中后台应用场景中实战沉淀而来，为复杂应用的快速开发提供一致性 UI 解决方案，也是用友BIP产品一致使用的UI设计规范实现。

yonyou-cli： 组件化开发的命令行工具；开箱即用，你只需要一个 @yonyou/yon-ui-cli 启动开发即可，无需安装 react、preact、webpack、react-router、babel、jest 等等。

技术架构
总体架构设计

通过统一的协议多技术栈的组件、渲染的支持

架构设计

通过协议来开发。不管是内置组件，还是业务组件，都是与资源无关的。你定义一套资源描述，而具体实现可以是 react 的各种资源库，而对于搭建页面，你可以在设计态用资源库组件，渲染可以是 react任意引擎，也可以是MDF或其它前端框架。

特别说明：目前iuap 多以react作为主要框架，后续的实例介绍主要以react为主，该组件化整体设计是支持vue等其它框架组件接入，后续有更多vue或其它组件要接入组件化会补充相关文档。

开发原则
业务内聚

组件之间拼接组装原则，将业务功能内聚到组件中，减少拼接的配置项。

纯组件+协议

通过配置文件加上原生的React组件进行开发即可。

组件目录

默认所有组件平铺在 components 目录下。

每添加一个组件，就需要在components目录中创建一个子目录，名称和组件名称保持一致。同时在组件目录中创建相应的 index.js，以及配套的样式文件等。

最后，需要在顶层目录的 index.js 中导出这个组件。

编译目录

即：执行yonui build

dist/ 整个组件库打包压缩后的代码。
lib/ 各个组件源码经过babel转译后的资源，可在源码中引入并使用。
demo/ 组件的demo文件打包后的代码，按组件为单元划分，用于注册中心中的组件预览页面。

设计器配置content属性，并映射到props

1、在组件使用增加content属性，定义之后，在设计器右侧会开放出属性供用户编辑

2、在manifest.js文件定义props属性，定义之后，在运行时会将用户在设计器中定义的content属性添加到组件的props里

import { FieldTypes, EditTypes, ComponentManifest, UIObject, TypeProps } from 'yonui-extension'
const manifest: ComponentManifest = {
  name: 'Mybutton',//组件名称
  label: 'Mybutton',//组件中文名称
  type: TypeProps.BasicControls,//FormControls:表单控件,BasicControls:基础控件,LayoutContainer:布局容器,DataContainer:数据容器

  props: [
    {
      name: 'content',           //属性名称

      defaultValue: '',          //属性默认值
      showDesign: false,         //属性是否在设计器右侧展示

        props: {},               //属性的参数

      }
    }
  ],
  children: []
}

  content?: string
  onClick?: any
  disabled?: boolean

export default class Mybutton extends Component<ButtonProps, any> {
  onClick = () => {
    const { onClick, disabled } = this.props
    if (disabled) {
      alert('禁用')
    } else {
      onClick && onClick()
    } 
  }

  render() {
  	const { content } = this.props
    return <button onClick={this.onClick}>{content}</button>
  }
}

组件通信
事件发布与订阅（PC和移动都适用）

我们在window上挂载一个cb.events对象实现事件的订阅与发布

组件A可以订阅组件B的某事件，当组件B的某事件被触发时，通过发布订阅机制，组件A会接收到组件B触发的事件，然后执行某些操作

订阅

组件A中可以使用window.cb.events.on(eventName: string, callback: function)方式来订阅一个事件，eventName为订阅的事件的名称，callback为订阅的事件被触发时执行的方法，一个组件可以订阅多个事件

// 订阅事件clickA

const callback = (e) => {

// 回调

console.log(e.info)

}

// 订阅

window.cb.events.on('clickA', callback)

// 取消订阅

window.cb.events.un('clickA', callback)

发布

组件B可以使用window.cb.events.execute(eventName: string, args: any)方式来发布一个事件，eventName为发布的事件的名称，args为参数

// 发布事件clickA 参数 {info: 'click-a'}window.cb.events.execute('clickA', {info: 'click-a'})// 订阅了该事件的组件，会接收到{info: 'click-a'}，执行对应回调函数
组件之间通信
getState

如果想要获取某个组件的某个属性，可以使用viewModel.get()方法获取组件model，之后调用model的getState方法。

var comp = viewModel.get('组件编码')// 获取想要控制的组件的model模型，不存在的话返回undefined// 判断模型是否存在，存在的话再调用setStateif (comp) {comp.getState('title') // 获取组件的title属性}
setState

如果想要修改某个组件的某个属性，可以使用viewModel.get()方法获取组件model，之后调用model的setState方法。setState调用之后，对应的组件会刷新。

var comp = viewModel.get('组件编码')// 获取想要控制的组件的model模型，不存在的话返回undefined// 判断模型是否存在，存在的话再调用setStateif (comp) {comp.setState('title', '新标题')// 设置组件的title属性}

PC端扩展事件

用户在组件中心配置xxxaction事件，定义之后，在设计器右侧会开放出事件供用户编辑

在组件中定义以下两个方法用来触发事件，直接复制即可

/**
   * 获取属性
   * @param {*} attrKey 属性名
   */
   getAttr = (attrKey) =>{
    const { meta = {} } = this.props;
    let attr = meta[attrKey] || this.props[attrKey]
    if(!attr){
      let cExtProps = meta.cExtProps || this.props.cExtProps
      if(cExtProps){
        let cExtPropsObj = JSON.parse(cExtProps)
        attr = cExtPropsObj[attrKey]
      }
    }
    return attr
  }

  /**
   * 执行命令
   * @param {*} actionKey  命令编码
   */
  execute = (actionKey, params) =>{
    // 依赖设计器需要设置actionKey对应的属性记录mdf框架的命令名称
    const { viewModel, pageModel, model} = this.props;
    let nowModel = viewModel || pageModel
    if(!nowModel){
      nowModel = model && model.__data &&  model.__data.cache &&  model.__data.cache.parent
    }
    return nowModel && nowModel.execute(this.getAttr(actionKey), params)
  }



之后在想要触发事件的位置定义如下方法触发定义的xxxaction即可

execute('xxxaction')

多语
实现一个多语组件

多语的实现基于react-intl，引入 IntlProvider包裹需要实现多语的组件，被包裹的组件中即可使用react-intl提供的接口以及传入 IntlProvider的配置信息，以FormattedMessage为例，示例代码如下

component.jsx

import React, { Component } from 'react'
import {IntlProvider, FormattedMessage} from 'react-intl'
import { chooseLocale, chooseMessage } from '../_utils'
export default class Testinput extends Component {
  render () {
    return (
    <IntlProvider
      locale={chooseLocale()} // 浏览器语言信息
      messages={chooseMessage()} // 配置的多语文本
    >
      <FormattedMessage
        id='superHello'
        defaultMessage='defaultMSG'
        values={{someone: 'yonbuilder'}}
      />
    </IntlProvider>
    )
  }
}



react-intl提供的功能以组件的形式使用，FormattedMessage可以根据传入intlProvider的配置显示不同的语种, 在多语组件中显示文本的地方都可以用该组件替换，默认将多语文本以span标签包裹返回，也可以通过传入tagName属性改变包裹的标签，上述例子中，返回结果为你好，yonbuilder !

utils/index.js

import Messages from '../../locales'
// 根据浏览器预览设置信息判断显示哪个语种，新增语种需要在此处新增判断
export const chooseMessage = () => {
  switch(navigator.language.split('_')[0]){
      case 'en':
          return Messages['en_US'];
          break;
      case 'zh':
          return Messages['zh_CN'];
          break;
      default:
          return Messages['zh_CN'];
          break;
  }
}
// 返回浏览器语言设置信息
export const chooseLocale = () => {
  return window?.navigator?.language || 'zh'
}

ybcomponent/manifest.js为组件描述文件，控制组件在设计器和运行时的属性，可在发布到设计器时再进行编写

import { FieldTypes, UIObject, TypeProps } from 'yonui-extension'

  name: 'Ybcomponent',
  label: '测试组件',
  uiObject: UIObject.Controls,

      name: 'testProp',
      type: FieldTypes.string,
      showDesign: true
    }
  ],
  events: [
      {
          name: 'onClick',
          des: '点击'
      }
  ],
  transformers: [
      'inherit',
      ['bindEvent', {events: ['onClick']}]
  ]
}
