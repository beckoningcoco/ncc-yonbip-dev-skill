## 1 UFIND介绍

## 1.1 什么是UFIND

UFIND属于业务规则中计算规则的一种，用于预算系统取业务系统中单据类型对应单据的执行数。UFIND相关节点包括：任务管理、应用模型、套表管理、业务规则、日常执行等。

![](https://community.yonyou.com/datadict/bipbook/assets/1-1.BxBx7NXL.png)

## 1.2 为什么要用UFIND

预算的分析和执行监控平台：

提供了执行监控功能，能够从业务数据中获取执行数，对业务系统进行预算控制和预警；支持预算多维分析，可以按不同的维度进行数据分析；预置了差异分析、趋势分析和预算分析等分析方法。

![](https://community.yonyou.com/datadict/bipbook/assets/1-2.DbhlbsTc.png)

## 2.3 IBusiSysReg 接口关于界面的方法

1.业务系统：

getBusiSysID()：返回业务系统编码

getBusiSysDesc()：返回业务系统名称

2.单据类型：

getBillType()：该接口中有两个 getBillType() 方法，实现无参的返回值为 ArrayList<ControlBillType>getBillType()。返回值为 ArrayList<ControlBillType>，其中每一个 ControlBillType 代表一种单据类型，其包含的主要内容有：主组织主键集合（一般只包含一个主键）、单据类型主键（非数据库字段，可自定义）、单据类型编码、单据类型名称、是否预占数（UFIND为false）、是否执行数（UFIND为true）、是否适用取数、是否适用控制。

3.属性、取值内容：

getControlableObjects(String billtype)：该接口中有两个 getControlableObjects() 方法，实现有参的返回值为 `ArrayList<ControlObjectType> getControlableObjects(String billtype)` 方法，参数值为单据类型编码，需要根据不同的单据类型返回不同的属性，属性和取值内容存在联动关系，属性发生改变后，取值内容下拉框的填充内容随之改变。

返回值为 ` ArrayList<ControlObjectType>` ，其中每一个 ControlObjectType 代表一种属性，其包含的主要内容有：属性主键值（非数据库字段）、属性编码、属性名称、ControlObjectType 数组形式的取值内容。

## 2.4 IBusiSysReg接口关于界面的方法

1.开始日期、结束日期：

需要实现 nc.itf.tb.control.IDateType 接口，并重写其 getDataType()、getDateType(String paramString) 和 getDataTypeDesc() 方法，返回日期类型编码和日期类型描述字符串数组。

2.业务系统取数实现类：

getExecDataProvider()：此方法返回实现了 nc.itf.tb.control.IBusiSysExecDataProvider 接口的业务系统的取数实现类，供预算取数时调用。

## 3 UFIND后台计算实现

## 3.1 参数介绍

getExecDataGroupBatch:

groupDocType：分组参数的基本档案类型，即界面上所选的基本档案的值最多的基本档案类型注册在ntb\_id\_bdcontrast 中的 att\_fld 字段。

groupParaVOs：以 NtbParamVO 为 key， List<String> 作为 value 的键值对，groupParaVOs 的数量为界面上所选择条件（除最大维度外）的笛卡尔积（条件中包含 UFIND 中所选择的基础档案值和 FIND 中的维度选值），键值对的值为最大维度的主键值集合。

childGroupDocs：以 String 为 key，Map<String, String\[\]> 为 value 的键值对，如果最大维度勾选了包含下级，那么预算会解析出最大维度的下级数据传给业务系统，key 为上级主键，value 为上级对应的下级主键的数组。

方法的返回值为 NtbParamVO 和 Map<String, UFDouble\[\]> 组成的键值对，NtbParamVO 为传入参数的groupParaVOs 中的 key 值，Map 中的 key 值为最大维度的主键值，value 值为分组 sql 查询到的每一组数据构成的 UFDouble 数组。

## 3.2 成本凭证单UFIND取数

1. 成本中心会计包含两种单据类型，开始取数时要将两种单据类型的参数区分开，分别调用不同的接口进行取数，对应方法为 nc.vo.cca.bugetinterface.BugetServiceProvider.getExecDataGroupBatch() 方法；
2. 调用成本凭证单的接口进行取数，对应方法为 nc.vo.cca.bugetinterface.IGroupVoucherDataGetter.getGroupVoucherExcData() 方法，方法接收三个参数 mostDim(最大维度编码)、groupMap(单据类型为成本凭证单的预算接口参数)、voucherChildMaps(最大维度包含下级时上级主键和下级主键数组的键值对)；
3. 进行取数前sql参数的构造，方便起见，成本凭证单创建了一个查询实体类 nc.vo.cca.bugetinterface.CCAQueryVO，用来组织sql的查询条件。这里主要是需要理解 NtbParamVO 中 m\_typedim、m\_busiAttrs、m\_pkdim、m\_includelower、lowerArrays 等参数的含义，分组字段的确定以及辅助核算档案和其它档案的区分。对应方法为nc.bs.cca.bugetinterface.GroupVoucherDataGetter.buildNtb2ccaMap() 方法；
4. 通过构造好的参数拼接sql语句进行查询，需要重点关注的点有：
	1)分组字段拼接到 sql 中需要确定其表名，以“表名.分组字段名”作为分组条件;
	2)调用接口获取辅助核算 sql 条件时以辅助核算分组和不以辅助核算分组两种情况的区分;
	3)对应方法为nc.bs.cca.bugetinterface.GroupVoucherDataGetter.getAmountArrays() 方法。
5. 根据界面上属性（数量，金额等）、取值内容（借方发生额，贷方发生额，净发生额等）计算查询结果并返回给预算，需要重点关注的点有：
	1)分组维度的值为空时需要添加默认值 NtbParamVO.EMPTY\_GROUPDOC;

2)保证返回结果中 key 和 value 的关系和预算传过来的参数中 key 和 value 的关系保持一致;

3)对应方法为：nc.bs.cca.bugetinterface.GroupVoucherDataGetter.matchData() 方法。

## 3.3 UFIND联查

在“日常执行”节点进行规则计算后，选中计算后的某个单元格，点击联查执行按钮，选择相应规则后弹出一个模态框，列表态展示主体、单据类型、执行数等基本信息，此处功能预算已实现，不需要业务系统单独实现

![](https://community.yonyou.com/datadict/bipbook/assets/3-1.BBVgjclR.png)

业务系统需要实现的是点击上图中的联查按钮，根据预算提供的参数，跳转到单据类型对应的节点，并查询数据展示在联查界面。

在业务系统注册类中，需要实现 nc.itf.tb.control.IBusiSysNCcloudReg 接口，并实现其中注册联查信息的方法：

1.应用类型注册：getAppType：返回1为小应用，返回2为其它类型应用，一般返回1；

2.应用编码注册：getAppcode：根据单据类型返回不同的应用编码，应用编码即应用注册节点对应的应用编码；

3.应用地址注册：getLinkUrl：根据单据类型返回不同的应用地址，应用地址即应用注册中的页面地址；

4.页面编码注册：getPagecode：根据单据类型返回不同的页面编码，页面编码即应用注册中的页面编码。
