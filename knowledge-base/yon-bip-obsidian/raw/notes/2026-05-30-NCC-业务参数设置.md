# 业务参数设置 | YonBIP高级版

## 业务参数设置

---

业务参数是指高级版当中的配置类参数，关键业务逻辑可以通过在业务参数设置当中增加参数项，从而控制不同的业务流程走向，影响系统表现。针对于关键性的可让用户调整的系统行为，可以在业务参数设置当中增加参数项，从而用户可以通过修改参数的方式调控业务行为。

## 1 业务参数结构

业务参数是配置项。高级版的业务参数支持多种不同的配置方式，例如某个业务参数可以是布尔类型的，也可以是整数类型的，也可以是字符串；当然业务参数也可以是自定义的格式，就是本节课第三部分所要讲述的内容。业务参数主要包含两部分的结构，这两部分的结构相辅相成。第一个部分的结构就是业务参数模板，模板的含义就是描述一下参数范作用、取值范围，定义校验方式、取值类型、取值方式等。

![](https://community.yonyou.com/datadict/bipbook/assets/img-0.DrX-8HD_.png)

第二部分就是业务参数的存储。根据业务参数模板定义的取值范围等，用户在取值范围内选择了对应的值以后就会将对应数据落库，此即是业务参数的存储。存储节点除了存储具体的值以外，还区分除了全局、集团、组织级的业务参数。前端界面展示的时候即从业务参数存储当中获取对应的值。

![](https://community.yonyou.com/datadict/bipbook/assets/img-1.EgY1f40G.png)

## 2 业务参数开发

参数模板系统表当中包含了如本页所展示的这些字段，其中必输类型的字段当然是必不可少的了；其中参数编码、参数名称涉及到业务参数的显示，所以在做业务参数的时候需要起号名字方便理解。其中值类型和取值范围也需要注意一下。最方便的处理方式是从系统当中找一个已经预置好的参数，然后复制该参数模板的纪录并修改关键字段。

![](https://community.yonyou.com/datadict/bipbook/assets/img-2.3BksrJeC.png)

业务参数模板重要属性的说明：

**stateflag - 渲染方式**

- 当 state = 0 ：渲染 input 框；
- 当 state = 1 ：渲染 selected 框，valuelist 字段必须是以“,”分隔的形式；
- 当 state = 2 ：渲染 bool 组件 valuelist 字段必须是 是/否，Y/N 形式；
- 当 state = 3 ：渲染 check 组件 valuelist 字段必须是以“,”分隔的形式；
- 当 state = 5 ：渲染日期组件；
- 当 sysflag = N 时 state 才会有效，sysflag=Y 为自定义参照，state 无效。

参数设置是业务参数存储的地方，我们在新开发一个业务参数的时候，需要在该表当中增加一条记录，其中关联模板字段将该业务参数同参数模板关联了起来。参数值字段用于存储业务参数的实际值。

![](https://community.yonyou.com/datadict/bipbook/assets/img-3.CSCrHNVb.png)

开发一个标准的业务参数的过程就是在这两张表当中各增加一条记录。增加记录完毕以后，需要注意抽取脚本的问题，在相关的配置文件当中增加抽取脚本的条目，抽取对应的脚本后放到Git工程当中。

## 3 自定义参数开发

下图展示的是个性化参数的样例，示例当中的参数在点击参数值后会弹出一个对话框，对话框当中的内容是定制化的，我们可以通过编写前端代码的方式实现一个自定义的组件用于展示业务参数。编写的方式就是自定义个一个React组件，在render函数当中编排对应的组件。

![](https://community.yonyou.com/datadict/bipbook/assets/img-7.DDEnIpL0.png)

编写自定义前端组件需要满足以下规范：

- 请严格遵守React数据传递机制；
- 组件props当中包含以下两个重要参数：
```json
json    data: 打开编辑界面时，会以参数名为 data 传入，正在编辑业务参数的数据以及对应的业务参数模板数据，数据格式为
    {
      sysinittempvo   当前数据对应的模板数据
      m_sysinitvo     当前参数值 VO
      sysinitvo       返回参数值 VO，属性结构与 m_sysinitvo 结构相同
    }

   valueChange：改变值并确认后的回调函数，在编辑完成后需要调用valueChange(data)
```

此处提供一个简单的示例程序，在该程序就是第三部分开头演示自定义参数的前端组件，通过点击确定按钮将将参数更新为当前时间。

![](https://community.yonyou.com/datadict/bipbook/assets/img-8.nAComlAm.png)

当我们编写完成了前端组件以后，还需要将对应的组件注册到业务参数设置当中去，如果我们没有注册的话平台就不知道加载哪个前端组件，也就不能正确显示内容。注册的方式是修改参数模板的数据库表，将我们前端组件的路径放到其中，修改 refpath 值为组件路径（在 pub\_sysinittemp 表中）。 如下图所示。
