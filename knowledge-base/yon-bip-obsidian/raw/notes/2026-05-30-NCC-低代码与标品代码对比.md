# 低代码产品与标准产品代码对比 | YonBIP高级版

## 低代码产品与标准产品代码对比

## 1 React介绍

### 1.1 React是什么

- React是什么：React 是一个帮助构建页面 UI 的库，相较于 MVC 框架，只能算是其中的 View 层。能够帮助我们把页面分割成各个独立的小模块，每个小模块就是一个组件，这些组件相互组合、嵌套，就构成了复杂庞大的页面。
- React核心概念：JSX、props 和 state、组件、生命周期（API）、数据绑定（单向数据流）
- 为什么要先介绍下React：不管是 YonBIP 高级版的 platform 还是 tinper-bee 均使用的 React 库，所以我们开发的前端技术基础就是 React

### 1.2 React重要性

![](https://community.yonyou.com/datadict/bipbook/assets/1.pDaaAq3W.png)

此图展示了React的生命周期，任何页面的从创建、运行到关闭都会遵循React 的生命周期，所以掌握React生命周期对我们至关重要。在React的生命周期中，总共分为三个阶段：

1. 一是挂载阶段，进入页面时，React将先通过 construct 完成 props 和 state 的初始化，然后依次执行 componentWillMount 函数、render 函数渲染页面、componentDidMount 函数；
2. 二是渲染阶段，页面初始化并渲染完成后将进入到运行中状态，当 props 或者 state 发生改变时，将触发相应的生命周期函数并重新执行 render 函数渲染页面，然后回到运行中状态；
3. 三是卸载阶段，关闭页面时触发，将执行 componentWillUnMount 函数后，完成页面卸载。

### 1.3 React必要性

![](https://community.yonyou.com/datadict/bipbook/assets/2.Cj3BchEx.png)

此图表达的是React 动作派发的一个流程图，当页面处在运行中状态时，用户的操作将触发动作派发，执行对应的业务功能，业务功能主动触发setState方法使 state 发生改变，从而执行渲染阶段的生命周期。

## 2 目录结构介绍

### 2.1 代码目录的介绍

![](https://community.yonyou.com/datadict/bipbook/assets/3.C5KGV3bK.png)

左边是低代码工具生成的代码，右边是标准产品的代码，对比下两边的代码目录有何不同：

- 低代码工具代码：主要包含lcdpsuppier、lcdpsuppierl、lcdpsuppierp三个目录，其中，lcdpsuppier表示正常应用目录，lcdpsupplierl 是联查、追溯小应用的目录，lcdpsupplierp 是审批中心查看应用卡片信息的目录，再详细看下lcdpsuppier目录中的文件，config.json文件是配置依赖的，index.js文件就是我们的入口文件，主要的代码都在这里实现，Utils.js文件是工具类文件。
- 标准产品的代码：这是收入合同的目录结构，标准产品的目录结构比较多，其中main目录主要是路由功能，分别路由到列表或者卡片页，card目录表示卡片页相关代码目录，list表示列表页相关代码目录，constant目录表示常量目录，card目录下，btnClicks目录表示按钮点击功能实现目录，eventHandler目录表示编辑前后事件的目录，init目录是页面初始化相关的目录，utils是工具类相关的目录，list目录结构与card相似

### 2.2 低代码开发平台为什么不是在线可配置的

1. 历史问题：从 NC 6 系列的 uap-studio 到 NC Cloud 的 iuap-studio 再到 YonBuilder for NC Cloud 等工具，无一例外全是按 元数据 + 配置项 走代码生成的路线。
2. 安全性：在网页上直接配置代码或者写代码具有一些安全风险
3. 代码蕴含赋能思想：从代码层面教给使用者编写前后端代码的思路以及思维方式（更贴近 React 蕴含的核心思想），在使用过程中赋能使用者技术能力，逐步使其具备自主完成项目交付的能力和后期的自主创新力。
4. 可能的变革：随着使用者技术能力的不断精进、底层框架的不断完善，低代码开发平台后续可能变革成在线配置、在线编写代码的方式。

## 3 代码对比介绍

### 3.1 代码风格介绍

![](https://community.yonyou.com/datadict/bipbook/assets/4.C5kuBq7z.png)

此图是低代码与标准产品的对比图，看一下两者有何区别：

低代码开发平台代码：

- 低代码开发平台采用组件安装卸载的方式完成页面间的切换，在render函数中可以看到功能的应用，
- 低代码采用契合视图模型驱动视图（MVVM）的指导思想，即模型驱动视图，代码中createState（）方法便体现了该思想。
- 方法职责基本单一即定义的每个方法只包含自己的功能，不会去处理其他功能，state构建应用VM即通过createState方法构建了页面的视图模型。
- 一个应用一个js文件、一个配置类、一个工具类，这个在前面小节介绍目录结构的时候也可以提体
- 出发动作名为on+具体动作，表示的是出发动作的方法名的命名规则。
- 方法参数基本是对象和回调。

标准产品代码：

- 通过标准SPA方式完成应用，单页应用SPA是一种网络应用程序或网站的模型，它通过动态重写当前页面来与用户交互，这种方法避免了页面之间切换打断用户体验在单页应用中，所有必要的代码（HTML、JavaScript和CSS）都通过单个页面的加载而检索，或者根据需要（通常是为响应用户操作）动态装载适当的资源并添加到页面，页面在任何时间点都不会重新加载，也不会将控制转移到其他页面。
- 每个单据基本由路由、list、card、others构成，这个在前面小节介绍目录结构的时候也可以体现
- 按事件类型拆分，采用this指针的形式，this指针放在后面小节介绍下。
- 复杂的业务功能是说每个功能的实现都单独放在一个js文件中，可能我们要跟踪一个功能的实现，需要跳转很多文件才可以找到。
- 采用平台提供的缓存机制代替state部分功能的实现。
- 各个page单独请求模板

低代码工具生成的代码中，通过调用 `Utils.LoadNCCResource` 方法完成了模板、按钮等的初始化功能，而在标准产品的代码中，模板、按钮等的初始化是放在initTemplate.js文件中进行的，两者实现的功能基本是相同的。

## 5 总结

通过前面的介绍，可以看到低代码开发平台与标准产品的代码在目录结构、 `state` 使用、 `this` 指针、 `initTemplate` 、 `Before` 、 `After` 事件、 `Button` 事件等方面存在差异化，但是在写法上基本是相通的，从低代码开发平台产出的代码变成标准产品代码的风格仅需要稍作调整。许多非常复杂的业务功能，单纯依赖低代码开发平台至少目前还不具备这个能力（市面上任何低代码开发平台应该都不具备这个能力），更多的还需要使用者的能动性。

低代码开发平台在高效提升项目交付效率的同时更多蕴含了赋能的思想，在遵循 React 核心思想、平台开发规范的前提下，赋能使用者自主完成更多更复杂的业务功能。更有机会聚拢、发展、壮大生态伙伴。

伴随 YonBIP 高级版版本的迭代，低代码开发平台的不断完善，还会融入更多具备赋能效果的代码、功能。进一步达成轻交付、提能力、促生态的发展思路。
