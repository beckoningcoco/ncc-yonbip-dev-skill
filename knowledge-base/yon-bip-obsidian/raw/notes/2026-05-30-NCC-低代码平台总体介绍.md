    从Wiki的这段对低代码的定义中，我们可以提炼几个关键信息![image-20220823142358926](https://community.yonyou.com/datadict/bipbook/assets/1.8Ykop6cf.png)

     低代码开发平台（LCDP）提供了用于通过图形用户界面创建应用软件的**开发环境**。低编码平台可能会**产生完全可操作的应用程序**，或者需要针对特定情况**进行额外编码**。低代码开发平台可以减少传统花费的时间，从而加快业务应用程序的交付。一个共同的好处是，更多的人可以为应用程序的开发做出贡献。

从低代码核心价值和目标总结

-   低代码开发平台能够实现业务应用的快速交付。

-   低代码开发平台能够降低业务应用的开发成本。

## 1 低代码和无代码 [​](https://community.yonyou.com/datadict/bipbook/chapter1701/chapter1701.html#_1-%E4%BD%8E%E4%BB%A3%E7%A0%81%E5%92%8C%E6%97%A0%E4%BB%A3%E7%A0%81)

    低代码和无代码概念接近，技术相似、产品形态相似，但是低代码和无代码存在本质上是完全不相同的，带来的效果和隐性影响也不同，从专业性和长远的角度来看，低代码更具备韧性和持久力，无代码更能代码短期体验效果。

**低代码和无代码的本质差异是使用者角色的差异**

     无代码更适合与业务人员，可以快速构建应用，并适应企业的各种需求变化，但是，业务人员一般完全不具备设计能力，缺少软件工程缺乏扎实的了解，和从整体视角什么系统设计，没有足够的经验来就任何计算机系统的工作方式做出合理的决定。不会像程序员那样具有相同的直觉或对上下文的理解，因此，存在着不断出现的风险。例如，低代码应用程序可能不支持某些未由程序员预见的要求例如检索，计算和存储一些关键的基于事件的历史数据，这些数据如果当前未捕获，将永远丢失。再者程序的性能上的优化，需要更具备专业性的知识和经验。更深层次的隐性问题是，持续扩展的能力都会有很大下降，当不断的需求变更和程序功能升级时，会带来很大的维护成本，直至不可维护。

     虽然这种情况确实使业务团队有责任适当地确定需求范围，但这是软件工程师众所周知的事实，范围界定的错误一直发生，并且通常在开发和开发过程中由程序员即时修正。维护周期，程序员可以清楚地看到规格满足实际的程度。没有经验丰富的专业人员来解决这些微妙但往往是破坏性的错误。

     低代码平台主要针对技术开发人员，使他们能够在几天甚至几个小时内快速构建应用前端。这使得他们能够更快地进入软件开发中最有趣的部分:定制。低代码平台适合创建更复杂的应用程序和流程，这些应用程序和流程需要与其他应用程序数据库或系统集成。

**应用规模和类型有着根本性的差异。**

     与低代码平台相比，无代码平台更加简单。无代码平台可以使用户实现可视化的、拖拽式方法创建基本的功能性应用，但却无法在平台上改造或是扩展遗留系统。除此之外，无代码平台的集成能力有限。因此，这种创建模式最适合用于在特定范围内有特定需求的团队。

     无代码平台的简单性和易用性也是它的缺点。由于其大部分框架是由开发人员决定的，因此它的自定义范围有限，甚至无法自定义。这就为安全和合规问题留下了潜在的漏洞。

     在将应用集成到整个企业架构方面，无代码平台的功能十分有限，甚至并不具备这一功能。如果开发人员在创建时不加以监督和考虑，那么无代码应用最终还会引发影子 IT 的盛行。

     可扩展性上无代码平台倾向于运营效率方面的用例，它们不具备专注用户体验的功能，也无法连接到遗留系统，各厂商也不支持为第三方解决方案或自主系统创建的自定义集成。

规模化软件开发的复杂度非常高。与业务要解决的特定问题域强相关，；这部分复杂度不是任何开发方法或工具能解决的。这种复杂度一般与开发阶段的技术细节强相关，恰好就是低代码所擅长且适合解决的。

     为开发者尽可能屏蔽底层技术细节、减少不必要的技术复杂度，并支撑其更好地应对业务复杂度（满足灵活通用的业务场景需求），这是低代码开发平台要尽到的核心职责。\*\*

### 2.1 低代码功能整体介绍 [​](https://community.yonyou.com/datadict/bipbook/chapter1701/chapter1701.html#_2-1-%E4%BD%8E%E4%BB%A3%E7%A0%81%E5%8A%9F%E8%83%BD%E6%95%B4%E4%BD%93%E4%BB%8B%E7%BB%8D)

![](https://community.yonyou.com/datadict/bipbook/assets/8.EkG2CcQY.jpg)

​ 低代码平台主要由以下基本构件组成，包括核心框架，应用节点模型, 页面表现扩展抽象，功能特征插件抽象，元数据结构扩展，数据脚本引擎，模板引擎。

**低代码核心 -** 框架组织了生成代码基本流程，声明了扩展点和扩展接口，定义了数据脚本结构。

**应用节点模型 -** 是把节点设计的所有要素设计为模型，包括页面，UI组件，controller模型，service模型， service Implement模型，配置模型等。

\*\*页面表现扩展 - \*\*定义了页面的形式的接口，封装页面操作的基本功能，提供了各种注册UI组件的功能。

\*\*功能特性插件 - \*\*定义节点具备具体功能的接口，提供脚本，代码注册了业务对象注册等功能。

\*\*元数据结构扩展 - \*\*在高级版元数据接口的基础上，扩展了更多元数据的功能，为应用模型提供支持。

\*\*数据引擎 - \*\*定义了数据模型接口，同统一javabean和一般Java对象。是这些对象都可以生成数据脚本和执行。

​ 通过**页面表现扩展**和**功能特性插件**两个主要扩展，使低代码可以在两个维度上独立演化，相互独立同时能够相互增强，任何**页面表现扩展**都可以和一个或者多个**功能特性插件**进行组装，满足全场景全功能的应用代码生成。 ![](https://community.yonyou.com/datadict/bipbook/assets/9.CEF1Wf-B.jpg)

三种输入方式：

```
1、代码定制新增：选择元数据
2、代码定制修改：已经代码定制过，修改了元数据，再次通过代码定制生成
3、应用定制新增：通过无代码（应用定制）生成的功能，需要个性化处理，可通过该输入方式处理
```

基础配置：

```
主要配置要生成的应用注册在应用注册节点的什么目录下，应用菜单注册在菜单注册节点的什么目录下。以及对应的组织类型和功能点类型，这里要注意组织类型和功能点类型，一定要选择正确。
```

页面配置：

```
主要是配置当前要生成的应用的应用类型（分成四种：主子、树表、单表、树卡），以及所支持的插件功能（包括：附件管理、打印输出、导入导出、单据特性、档案特性、编码规则、审批流、业务流、拉单、openapi、外部交换平台、自定义项等功能）
```

其他配置：

```
主要是配置当前要生成的应用是否需要创建对应的数据库表、反向生成元数据文件、前后端导出的路径（前后端工程根目录）等
```

其他的一些配置是配合页面配置中的插件功能的详细配置。

### 3.3 代码定制输出 [​](https://community.yonyou.com/datadict/bipbook/chapter1701/chapter1701.html#_3-3-%E4%BB%A3%E7%A0%81%E5%AE%9A%E5%88%B6%E8%BE%93%E5%87%BA)

经过代码定制可以输出 应用源代码、应用的标准结构的脚本以及allSql、相关的xml（如导入导出的xml）等。拿到源代码后，即可根据项目实际场景对应用进行个性化开发。其中在源代码中预置了很多示例性的代码，开发者可通过阅读源码获取不同的功能示例、技术手段，来帮助自己完成一些个性化的功能。

### 3.5 代码定制版本差异 [​](https://community.yonyou.com/datadict/bipbook/chapter1701/chapter1701.html#_3-5-%E4%BB%A3%E7%A0%81%E5%AE%9A%E5%88%B6%E7%89%88%E6%9C%AC%E5%B7%AE%E5%BC%82)

**所有版本都满足向后兼容(2005版,2015版由于平台变化不兼容)**

**2005版** 具备开发主子，树卡，树表，编辑表页面形态，支持编码规则，导出导出，打印，外部交换平台，审批流，推拉单，附件，业务流，Openapi特性功能。

**2105版** 2005版功能一致，兼容满足2015版平台规范的功能。

**2207版（里程碑版本)** 2005版功能一致，增加了元数据设计。 增加主子孙页面形态功能。

**2212版（里程碑版本)** 增加节点配置项(器)功能。 低代码功能，从设计，开发， 配置。基本满足面向开发的全流程开发工具。

\*\*2312版 \*\* 增加元数据设计器分层模式。 增加自由布局(树卡，树表，行编辑表，主子表)。 增加移动应用低代码。 增加参照生成低代码。

### 3.5 代码定制注意事项 [​](https://community.yonyou.com/datadict/bipbook/chapter1701/chapter1701.html#_3-5-%E4%BB%A3%E7%A0%81%E5%AE%9A%E5%88%B6%E6%B3%A8%E6%84%8F%E4%BA%8B%E9%A1%B9)

a、申请到代码定制安装盘之后，安装之后一定要升库。

b、代码定制并不知道项目上使用的菜单是否是复制的，故而会在系统预置的菜单中注册新的应用菜单，需要在复制出来的菜单中点升级菜单

c、代码定制过程中如果组织类型和功能点类型选择有问题，可能会导致在分配应用职责时选不到对应的应用，所以需要注意。
