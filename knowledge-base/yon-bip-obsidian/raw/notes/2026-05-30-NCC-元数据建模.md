# 元数据建模 | YonBIP高级版

## 元数据建模

---

目前，元数据建模工具集成在标准产品中，我们可以通过打开【应用工厂】 -> 【元数据设计】节点进行建模。

## 1 简介

元数据设计器中，一个重要概念是视图面板（View），元数据设计器由多个视图面板（Views）组成。提供了不同的功能和特性。

![MDP](https://community.yonyou.com/datadict/bipbook/assets/1.CZu2EmT9.png)

### 1.1元数据设计器

上图中，元数据设计器包含以下主要面板。即 特征面板，属性面板，工具面板，画板面板，模型面板，图元面板。其中，模型面板中还包含，实体属性面板，接口映射面板，参照面板。

#### 1.1.1 画布面板

画布面板通过图形方式来表现一个模型的内部所有对象的关系结构，实现的接口，及对象拥有的属性。与UML类似，他是模型对象关系图，仅用来对象关系。画布右侧的工具面板，提供各种创建 选择，创建，引入，连线工具。

#### 1.1.2 特征面板

特征面板包含多个特征按钮，其作用为，快速补充当前画布面板的模型，按功能实现高级版定义的功能接口同时添加对应的属性，和接映射。

#### 1.1.3 属性面板

属性面板是查看画布面板选中对象的属性，即，属性模板是画布面板选中激活对象时，对对象的更详细显示。所以，选中不同的对象(对象包括，实体，枚举，接口，关联，聚合，依赖)，属性面板的内容也不相同。当画布面板没有选中时，则默认显示当前模型组件对象的属性。

下面，我们按照选中不同的对象，分别说明属性面板的相关属性

**情况1：** 选中画布空白区域（组件激活时) (在元数据设计器中，所有对象都归属组件)

![properties](https://community.yonyou.com/datadict/bipbook/assets/2.Dv9ESGaN.png)

- 属性说明

| 名称 | 说明 |
| --- | --- |
| 版本号 | 记录组件当前版本，防止旧版本覆盖新版本。版本号是自动生成的，每点击一次保存，就会增加1 |
| 代码风格 | 主要针对vo结构，有NC传统样式、标准样式和自定义样式三种，传统样式生成的vo和5系列一致不包含getMetaData方法，自定义样式生成的vo包含getMetaData方法，使用于6版本元数据操作方式 |
| 多语言资源模块 | 对应以前多语言里的文件夹名，现在按照组件划分，包括resmodel和resid |
| 是否预加载 | 默认false。如果选择true，而且所用的数据源也配置为预加载元数据（在nc\_home\\ierp\\bin\\preload-datasource.properties中配置），那么该组件就会在中间件启动时被预先加载 |
| 主实体 | 一个组件里如果有实体，则必须设置一个主实体 |
| 名称 | 该组件的名称。必须全局唯一 |
| 名称空间 | 一般情况与所属模块一致 |

**情况2：** 选中画布中的实体对象

画布中的实体存在两类，实体和引用实体， 不归属当前组件的实体为引用实体，即引用的其他组件实体，引用实体的属性面板不可编辑(禁用)。

![properties](https://community.yonyou.com/datadict/bipbook/assets/3.Bgjhchzk.png)

- 属性说明

| 名称 | 说明 |
| --- | --- |
| 主属性 | 实体必须设置主属性(对应数据库表的主键) |
| 访问器 | 当组件代码风格选择传统样式时，针对主子表或多子表中主表对应的实体，访问器要设置为AggVO，其他一般选择NCVO |
| 类名称 | 生成VO时完整类路径 |
| 动态信息修改类 | 用于动态修改实体，比如运行时增加或者设置某些属性 |
| 可见性 | public，protected，default，private |
| 扩展标签 | 可以给实体加任意的标签，查询时可根据标签过滤，如：BDMODE-管控模式、 BUSILOG-业务日志、URC-唯一性、BILLCODE-支持编码规则、 TREE-树形结构 |
| 缺省表名 | 生成数据库时对应的表名 |
| 业务接口实现类 | 如果实体实现了业务接口，这里可以注册实现类 |

## 2 建立元数据

2.1 通过(动态建模平台)->(应用工厂)->(应用工厂高阶定制) ，找到元数据设计节点

![](https://community.yonyou.com/datadict/bipbook/assets/6.D1UpK4MQ.png)

2.2 在【元数据设计】节点中，点击新增，开始创建元数据。

![](https://community.yonyou.com/datadict/bipbook/assets/8.Bs5B0lYP.png)

**注意** ：组件描述的是一个相对独立和完整的业务块，主要是从业务层次上进行划分，比如“销售订单”可以作为一个组件，“客商档案”也可以作为一个组件，划分组件时，注意组件间的重用，且注意不要形成相互依赖。组件属于模块，一个模块可以包含多个组件

2.3 在工具箱中，点击“实体”，将其拖到左侧空白区域，这样一个空实体建立完成。注意默认第一个实体为主实体。

![](https://community.yonyou.com/datadict/bipbook/assets/10.BvDw5Ew6.png)

2.4 设置模型视图属性

![](https://community.yonyou.com/datadict/bipbook/assets/11.BkHyFRs7.png) ![](https://community.yonyou.com/datadict/bipbook/assets/12.CQ9H-YMv.png)

```
**关键属性**:
  - 类型样式:主要分为Single,REF,Array,List
     - Single:单一样式，最终类型就是原始数据类型
     - REF:参照，只用于实体，值对象。
     - Array:数组样式，最终的类型为数组 集合类型
     - List:列表样式，最终的类型为 \`\`\` List\<数据类型> \`\`\`
  - 类型：自行选择，但需注意，主键必须为UFID
  - 字段名称:对应数据库中列名
  - 访问策略：对于设置AggVO样式的实体，检查子表属性（1-n聚合关系），访问策略要设置为BodyOfAggVOAccesssor
  - 计算属性：由其他属性计算得到，本身不需要保存到数据库，不会生成数据库列
  - 扩展标签：可以自由给属性加上标注如： BDMODE-管控模式、 BUSILOG-业务日志、 URC-唯一性、BILLCODE-支持编码规则、 TREE-树形结构等。
```

2.8 创建枚举 创建一个枚举类型时，要注意以下几点: 1. 枚举类型必须设置返回值类型，当前支持Integer和String两种 2. 如果是自定义枚举，需要注册自定义枚举值实现类 3. 枚举类型不对应java类，枚举类型的fullClassName为其返回类型，即int或者String

枚举有3种情况:

    1.开发时就可以固定下来的，可以在模型上直接建立

    2.开发时不确定，那么可以在模型上将不确定的属性设为自定义项类型，运行时在元数据管理工具中可以新增枚举类型，将自定义项类型的属性设为具体的类型

    3.运行时也不固定，枚举值可能要从数据库中动态取得，此时需要开发人员实现枚举扩展类IEnumType

![](https://community.yonyou.com/datadict/bipbook/assets/23.Dw4UI2TO.png)

## 3 元数据的分层

**元数据常用层级定义**

\*\*领域级：\*\*集团开发标准产品层级。

\*\*行业级：\*\*行业开发标准层级，继承领域级元数据，可修改和扩展行业级元数据。对继承的元数据不可修改。

\*\*伙伴级：\*\*伙伴开发标准层级，继承行业级元数据，可修改和扩展伙伴级元数据。对继承的元数据不可修改。

\*\*客户级：\*\*客户开发标准层级，继承伙伴级元数据，可修改和扩展客户级元数据。对继承的元数据不可修改。

\*\*层级关系：\*\*领域级 -> 行业级 -> 伙伴级 -> 客户级

伙伴级可以跳过行业级，直接继承领域级的元数据

客户级可以跳过伙伴级或行业级，直接继承行业级或领域级的元数据

**分层目标：**

上层继承下层所有属性实体，属性，关系，各个层级只能修改自己的层级的模型。

## 4 升级元数据

## 4.1 正常升级

    管理员登录NC系统，【业务系统管理】->【模型定制管理】->【元数据管理】打开元数据节点，维护元数据时采用升级元数据的方式，而不是发布元数据,点击【升级】，在窗口中找到对应的元数据文件，之后点击【确定】 如下图所示：

![](https://community.yonyou.com/datadict/bipbook/assets/22.D8DbBv0O.png)

## 4.2 不确定是否有打过补丁或客开

1. 先从数据库查询出需要升级元数据当前的版本号：select t.version from md\_component t where t.id=‘’,此处id可以在MDP透视图点击画布中间空白部分在属性面板中可获取到
2. 把修改后的元数据bmf文件使用nodepad++或EditPlus或其他编辑器打开，找到第一行中配置的version，随后执行update md\_component set version=(此处值要比bmf中找到的要小) where id=‘’;
3. 然后按照正常实际元数据的方式进行升级即可

![](https://community.yonyou.com/datadict/bipbook/assets/25.RGr3neyM.png)
