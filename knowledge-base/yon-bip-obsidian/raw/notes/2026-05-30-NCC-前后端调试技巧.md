# 前后端调试技巧 | YonBIP高级版

## 前后端调试技巧

## 1 Eclipse调试技巧

Eclipse常用的调试窗口

| 窗口 | 说明 |
| --- | --- |
| Debug窗口 | 主要显示当前线程方法调用栈，以及代码行数（有调试信息的代码）。还可以让当前方法回滚。 |
| 断点Breakpoints窗口 | 断点列表窗口，可以方便增加断点，设置断点条件，删除断点等 |
| 变量Variables窗口 | 显示当前方法的本地变量，非static方法，包含this应用，可以修改变量值 |
| 代码编辑窗口 | 查看需要调试的代码 |
| 输出Console窗口 | 日志等输出内容，调试时，可以将关注的组件日志级别设置低一点，以便获得更多输出信息 |
| 表达式expression窗口 | 查看需要观察的数据的表达式以查看表达式的值 |

如下图是进入调试时eclipse的界面

![](https://community.yonyou.com/datadict/bipbook/assets/002.DC2Nih2g.png)

1.将所有断点设置为被跳过，设置了Skip All Breakpoints之后，所有断点上都会有一个斜线，表示断点将被跳过，线程不会在该断点处被挂起。

2.表示当前线程继续运行直到下一个断点，快捷键为F8。

3.表示打断整个程序运行，执行后程序退出运行。

4.相当于单步执行，如果有方法调用，则会进入方法中执行，快捷键为F5。

5.表示运行下一行代码，如果有方法调用不会进入方法中，快捷键为F6。

6.表示退出当前方法，返回到调用层，快捷键为F7。

7.Drop to Frame：这个命令可以让程序回到当前方法开头的第一行代码重新开始执行，可以重新执行这个java堆栈帧，也可以在堆栈树上选择一个指定的堆栈帧，然后点击 Drop to Frame，这样就可以重新进入指定的堆栈帧，局部变量会被重置。使用Drop to Frame时候需要注意：

（1）不能drop到已经执行过的方法栈中的方法中。

（2）drop到stack frame中时，不会改变全局数据原有的值，比如，一个包含元素的vector并不会被清空。

8.Step Filters：这个功能比较简单，就是当我们在debug的时候想要忽略一些我们不关注的类时，可以开启Step Filters进行过滤，程序会一直执行直到遇到未经过滤的位置或断点。设置Step Filters步骤如下：

步骤 1：打开 Windows -> Preferences -> Java -> Debug -> Step Filtering

步骤 2：选择‘Use Step Filters’

步骤 3：在屏幕上选中所需的选项，也可以添加自己代码库中的部分代码。 步骤 4：点击‘Apply’

## 2 IDEA调试技巧

![](https://community.yonyou.com/datadict/bipbook/assets/009.B5wxdrrH.png)

1. 显示执行点：快捷键 Alt + F10，显示当前执行断点
2. 步进：快捷键 F8，单步执行当前代码行，并带到下一行，即使有方法调用，也会跳过方法的实现，直接转到调用方方法的下一行。
3. 步入：快捷键 F7，进入当前方法以显示其中发生的情况。当不确定该方法是否返回正确的结果时，请使用此选项。
4. 步出：快捷键 Shift+F8，跳到下一个断点或者跳出方法。
5. 运行到光标处：快捷键 Alt+F9，继续执行，直到到达当前光标所在的行。
6. 评估表达式：快捷键 Alt+F8，可以在调试和运行期间对表达式进行求值，获取表达式运行结果。
7. 跟踪当前流链：Java 8 Streams有时可能难以调试，可以通过跟踪流链功能查看每个操作如何转换值，该功能不适用于库或反编译代码。
8. 重启服务：快捷键 Ctrl+F5，顾名思义重启当前调试的会话服务。
9. 编辑运行服务：点击后弹出对话框，可以修改当前服务名称、模块、虚拟机参数等信息。一般需要重启方能生效。
10. 恢复程序：快捷键 F9，继续直至遇到下个断点。
11. 暂停程序：暂停调试会话程序。
12. 终止调试器会话：快捷键 Ctrl+F2，停止服务运行。
13. 查看断点：快捷键 Ctrl+Shift+F8，可以查看、删除断点，也可以禁用断点或添加新的断点。
14. 忽略断点：点击后禁用所有断点，程序执行不再进入断点。再次点击后恢复断点。
15. 获取线程转储：获得当前的线程堆栈。

## 3 远程调试及SPR报告

### 3.1 远程调试配置

某些情况下可能需要对非本地环境代码进行调试查找问题原因，比如测试环境或用户环境调试，这时就需要用到远程调试。远程调试需要远程环境提供调试端口。

添加调试端口方法：运行 home\\bin\\sysConfig.bat 脚本，出现 yonyou UAP 配置工具后，在服务器信息一项点击“读取”按钮后在虚拟机参数最后添加

\-agentlib:jdwp=transport=dt\_socket,server=y,suspend=n,address=ip地址:8099

\-agentlib:jdwp 表明使用 JDWP 协议

transport：有两个内置类型，dt\_socket（使用 socket 接口）和 dt\_shmem（使用共享内存）。共享内存类型仅在本机调试时使用

server：y 表明此虚拟机在调试中扮演服务端角色

suspend：是否在客户端连接前挂起主进程。为不影响服务正常使用，通常需要设置为 n；当需要调试启动过程时，可设置为 y

address：应用程序监听绑定的ip和调试端口号

![](https://community.yonyou.com/datadict/bipbook/assets/016.BFhurqNC.png)

**Eclipse添加远程调试**

在 Eclipse 工具栏点击 debug 下拉箭头找到 Debug Configurations 打开界面如右图所示，可以在 Remote Java Application 添加一个远程调试应用。

Project 选择要远程调试的项目。Host 填写服务器IP地址，port 为远程调试端口即上个环节所配置调试端口。点击 Debug 后如果没有报错即连接上远程调试端口。

调试过程与本地调试基本一致，使用完毕后需要在工具栏点击 图标断开连接释放远程调试端口。 ![](https://community.yonyou.com/datadict/bipbook/assets/017.Bo-J4zpY.png)

### 3.2 SPR报告

SPR报告是 YonBIP高级版系统在一段时间内进行业务操作时系统运行情况的总结报告。这个报告涵盖了服务器端环境信息、概要、耗时分布图、远程调用、SQL耗时TOP10、SQL结果集TOP10、SQL信息、报错信息、Weblog 等内容。这个报告有利于诊断业务操作的性能及报错问题。

YonBIP高级版录 SPR报告：在界面点用户头像->录制SPR 或用快捷键 ctrl+~ 开始录制后界面出现图标 ，再次使用快捷键 ctrl+~ 或点击图标结束录制，系统会自动在新页签打开 SPR 报告。 ![](https://community.yonyou.com/datadict/bipbook/assets/020.DXnggItR.png)

SPR 主要由说明信息、服务器端环境信息、耗时分布图、远程调用、SQL耗时TOP10、SQL结果集TOP10、SQL信息、报错信息、Weblog等构成。

服务器端环境信息：展示服务器操作系统、JDK版本、虚拟机内存使用情况等信息。

概要：统计操作总共调用的远程调用次数、总耗时、服务器端耗时、中间件耗时、sql耗时、sql数量、读结果集耗时、结果集、是否有连接泄露等信息。

远程调用：统计出每次远程调用耗时、sql耗时、sql数量、结果集等信息。当数据超过指标时会标红警示。 ![](https://community.yonyou.com/datadict/bipbook/assets/021.B2C4yPDS.png)

\*\*耗时分布图：\*\*用饼状图展现数据库耗时、网络耗时，以及后端web耗时。可以更直观地展示不同耗时段分布的比例关系，视觉上更加直观。 ![](https://community.yonyou.com/datadict/bipbook/assets/022.Q4WwhaHg.png)

SQL耗时TOP10：统计业务操作TOP10耗时的SQL。

SQL结果集TOP10：统计业务操作TOP10结果集的SQL。

SQL信息：统计出所有执行调用的SQL信息。

![](https://community.yonyou.com/datadict/bipbook/assets/023.bOpYLdPN.png)

报错信息：统计出所有远程调用的报错信息。建议通过远程调用的链接到对应的报错信息。

Weblog信息：统计出web应用action调用类以及远程接口调用信息。

![](https://community.yonyou.com/datadict/bipbook/assets/024.Cdf3O1ya.png)

## 4 前端调试技巧

浏览器都提供了强大的开发者工具，用来排查页面样式、js代码、性能等问题。在浏览器通过页面右键->检查或者 F12 调出开发者调试窗口。调试窗口包括页面元素、控制台、源代码、网络、性能、应用等多个页签。

![](https://community.yonyou.com/datadict/bipbook/assets/025.BcQNM3u0.png)

\*\*元素：\*\*在元素面板中，可以查看和编辑元素的 HTML 结构、CSS样式和相关属性。可以通过元素选择工具定位 html 代码，也可以通过 html 代码定位页面元素位置。

![](https://community.yonyou.com/datadict/bipbook/assets/026.B8gJitY9.png)

\*\*CSS样式窗口：\*\*在样式窗口，可以查看元素样式属性，例如大小、颜色、字体、边框等。如果属性有冲突被其他样式覆盖会用带删除线的字体。也可以直接修改样式属性值预览界面效果，快速调试CSS样式。 ![](https://community.yonyou.com/datadict/bipbook/assets/027.C1ErXHZH.png)

\*\*Console控制台：\*\*Console 控制台是一个强大的调试工具，可以帮助开发人员输出日志信息、调试代码以及进行交互式的开发。

![](https://community.yonyou.com/datadict/bipbook/assets/028.Bk2zweIV.png)

\*\*Console控制台：\*\*在代码开发时可以通过内置 console 对象向控制台输出一些调试信息，以下是一些常见的 Console 输出方法： console.log()：使用 console.log() 方法可以在控制台输出一条普通的日志信息。例如：console.log("Hello，YonBIP高级版!"); console.error()：使用 console.error() 方法可以输出错误信息，并在控制台中以红色显示。例如：console.error("程序出现错误，请检查代码逻辑!"); console.warn()：使用 console.warn() 方法可以输出警告信息，并在控制台中以黄色显示。例如：console.warn("检测到输入数据不合法!"); ![](https://community.yonyou.com/datadict/bipbook/assets/029.BvD2fsEw.png)

Console 控制台还可以帮助开发人员调试代码、定位错误和查找问题。以下是一些常用的调试方法：

1. console.assert()：使用 console.assert() 方法可以进行条件检查，并在条件为 false 时输出错误信息。例如：console.assert(2 + 2 === 5, "Math doesn't work!");
2. console.dir()：使用 console.dir() 方法可以输出一个对象的详细信息，包括其属性和方法。例如：console.dir(obj);
3. console.dirxml()：以 XML 或 HTML 元素作为参数，以相应的结构化方式显示在控制台中。例如：console.dirxml(document);
4. console.trace()：使用 console.trace() 方法可以输出当前代码的调用堆栈，帮助定位到错误发生的位置。例如：console.trace("Error occurred");
5. console.memory：console.memory 是 console 对象的一个属性，而不是一个方法。它可以用来查看当前内存的使用情况，如果使用过多的 console.log() 会占用较多的内存，导致浏览器出现卡顿情况。

**源代码/来源** ：源代码页签可以找到页面执行的 js 代码。YonBIP高级版普通节点所引用代码位置一般为mainframe->name->src->源代码及路径。如果知道 js 文件名称，可以通过 Ctrl+O 快速查找打开文件。 ![](https://community.yonyou.com/datadict/bipbook/assets/030.BAg9g9a-.png)

找到需要调试的 js 代码后可以在行号处鼠标左键点击添加断点。如果需要添加条件断点，在行号右键->添加条件断点->编辑条件，在符合条件时会进入断点。 前端还支持日志点，行号右键->添加日志点->编辑输出内容，在代码运行经过日志点时会向控制台输出内容但不会在断点停留。 右侧“断点”视图可以查看所有添加的断点。同时也可以通过勾选停启用异常断点。 ![](https://community.yonyou.com/datadict/bipbook/assets/031.BA2v0eY0.png)

继续执行：继续执行直至遇到下一个断点，快捷键 F8 跳过下一个函数：执行下一步，如果有函数调用不会进入函数内部。快捷键 F10 进入下一个函数调用：如果有函数会进入函数内部执行，没有继续向下执行。快捷键 F11 跳出当前函数：执行完当前函数返回调用处，快捷键 Shift+F11 单步调试：单步执行，遇到函数进入函数内部执行，基本和 F11 类似但是在异步函数略有差异。快捷键 F9 停用断点：停用所有断点，不再进入断点。快捷键 Ctrl+F8 ![](https://community.yonyou.com/datadict/bipbook/assets/032.HmJptfFP.png)

堆栈：与后端代码调试类似，前端调试时也可以查看调用堆栈，展示前端代码调用关系。 如果需要代码重新执行时可以选中某一帧右键重启帧，这样可以避免所有代码重新走一遍。 也可以右键复制堆栈轨迹，前端代码调用关系会复制到剪切板。 ![](https://community.yonyou.com/datadict/bipbook/assets/033.CMlXElYb.png)

\*\*网络：\*\*可以实时监控浏览器与服务器之间的网络请求。通过查看请求的结果和相应的信息，可以更好地了解网页的加载和数据传输情况。当点击某个特定的网络请求时，在网络页签面板上将显示有关该请求的详细信息，这包括请求和响应的头部信息、传输的数据、请求的时间线等。这些详细信息可以帮助分析和排查网络请求问题。 如果有需要也可以选择某个请求右键重放请求。 ![](https://community.yonyou.com/datadict/bipbook/assets/034.Ojp20HVL.png)

上图几个按钮主要作用如下：

1. 开启/停止录制网络请求：如果关闭有网络请求时不会被录制，如果开启时才会记录请求信息。
2. 清除网络日志：清空录制的网络请求信息。
3. 过滤：打开请求过滤面板，可以通过过滤面板对请求进行过滤。
4. 搜索：打开搜索面板，是对文件内容的搜索。
5. 保留日志：勾选后网页重新加载/导航时不清除已经录制的网络请求内容。
6. 停用缓存：勾选后不使用浏览器缓存的文件，对请求文件会重新下载。
7. 节流模式：可以控制网络的快慢，默认提供快速 3G、慢速 3G、离线（Offline）也可以添加自定义设置。
8. 更多网络状况：打开网络状况控制面板，可以设置网络节流、用户代理等更多网络相关配置。
9. 导入、导出 HAR 文件：可以把当前录制的网络请求保存成.har 文件，或者导入其他.har 文件进行查看分析。

点击某一个请求可以在网络页签查看请求具体内容，比如 http 请求标头、向后台发送的数据、相应返回的数据、启动器、时间、请求携带的 cookie 等等。其中 YonBIP高级版请求一般都做了压缩加密处理，如果需要查看明文内容可以修改配置文件 HOME\\hotwebs\\nccloud\\WEB-INF\\config\\miscellaneous.xml。 比较常用的是查看启动器，即发送请求的具体文件，可以通过调用关系快速定位到具体代码。 ![](https://community.yonyou.com/datadict/bipbook/assets/035.AbsftTVH.png)

\*\*性能：\*\*性能页签用于分析和优化网页的性能，点击录制在页面进行操作后停止录制会生成性能报告。在性能报告中可以看到各种事件的持续时间，例如加载资源、执行脚本、渲染页面等。通过分析这些事件，找到性能瓶颈所在，确定需要优化的部分。 性能报告可以导出成 JSON 文件，也可以导入其他的性能报告继续分析。 ![](https://community.yonyou.com/datadict/bipbook/assets/036.BhF3xVBY.png)

\*\*应用：\*\*可以找到平台和领域缓存的数据，例如菜单、主题皮肤、单据模板以及部分系统参数。也可以找到系统缓存的图片、代码、html 等文件，点击某个文件，还可以在右侧的预览窗格中查看该文件的内容。 ![](https://community.yonyou.com/datadict/bipbook/assets/037.oYjm7GeM.png)
