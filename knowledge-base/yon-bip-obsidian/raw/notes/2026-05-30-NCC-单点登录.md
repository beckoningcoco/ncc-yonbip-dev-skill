# 单点登录 | YonBIP高级版

## 单点登录

---

## 1单点登陆概述

单点登陆（英文全称Single Sign On，简称为 SSO）它的目的在于解决多系统之间切换时频繁认证问题，是企业中常见的多个系统间认证的整合方案。举个例子我们的系统需要在登陆页输入用户名和密码进行登陆认证后才能进入系统做业务操作，而企业内部可能并不仅限于一套业务系统可能还有邮箱系统、绩效考核系统、财务报销系统、知识库管理系统等等。如果每个系统登陆都是使用独立的用户名密码，用户不仅要管理多个账号密码而且要挨个登陆，这种设计体验效果显然不好。如果能有某个的系统对用户进行登陆统一认证，基于该系统认证信息，用户再进入其他系统时不再需要额外的身份验证，并直接附带上用户信息进行业务操作，减少频繁的认证过程，显然这才是高效、快捷、安全的办公设计。YonBIP高级版提供了第三方系统单点登陆的解决方案，本章节会对如何进行单点登陆开发进行详细介绍。

## 2第三方单点登陆

## 2.1单点登陆流程

第三方系统单点登陆到“YonBIP高级版”服务可以分为下面六个步骤：

1. 客户端向第三方服务器发起单点登陆“YonBIP高级版”系统请求。
2. 第三方服务向“YonBIP高级版”系统服务器注册校验用户登陆和上下文信息。
3. 如果校验成功“YonBIP高级版”系统服务器响应给客户服务器token信息。
4. 客户服务器将token响应给客户端。
5. 客户端收到token后拼接请求“YonBIP高级版”系统节点的url信息。
6. “YonBIP高级版”系统收到请求并验证token是否合法，如果合法，则重定向到客户端请求的节点页面

## 2.2单点登陆实现

（1）注册第三方系统：需要在YonBIP高级版系统注册表sm\_oauth\_security注册第三方系统。

表中各字段含义如下：

- CLIENT\_ID ：第三方系统编码，必需项
- CLIENT\_NAME： 第三方系统名称，非必需 。
- CLIENT\_SECURITY： 第三方系统和YonBIP高级版共同维护的秘钥对
- PK\_OAUTH\_SECURITY： 表的主键
- CLIENT\_URL： 第三方系统的url (供系统集成访问第三方系统使用，本环节不需要)
- CLIENT\_AUTHCLASS： 登录第三方系统授权类 (供系统集成访问第三方系统使用，本环节不需要)
- PK\_GROUP：非本环节使用，设置为空即可。

（2）第三方系统调用接口获取token：通过秘钥对获取登录token，第三方服务调用接口 `http://ip:port/service/genThirdPartyAccessToken` ，ip、port为YonBIP高级版服务的ip地址和端口号。同时需要传递如下访问参数：

- type 获取方式，此方式为固定为 type\_security且必需项
- dsname 数据源名称，本地开发模式为design，必需项
- usercode NCC用户编码，必需项
- langcode 多语语种，非必需项，为空默认中文语种
- busicentercode 账套编码，本地开发模式为develop，为空登录时会循环在数据源中查找用户，非必需项
- client\_id 第三方系统id,即第一步在sm\_oauth\_security注册的系统编码，必需项
- security第三方由共同维护的秘钥对根据给定的算法自己生成的秘钥值，必需项。security是通过 **SHA256** 加签，其中是根据usercode，ts，client\_security和当前系统时间动态拼接的字符串生成。这块编码方式可以参考我们后面的示例。

（3）配置白名单：基于安全方面的考虑，如果获取token时只允许某些服务器允许调用上面步骤2里的接口，可以选择配置白名单IP地址。在home\\ierp\\sf\\nccssoConfig.xml里面可以配置token的有效时间和合法的ip地址。配置文件内容示例：

```xml
xml<?xml version="1.0" encoding="UTF-8"?>
<NCCSSOConfigVO>
    <authenticator id="default" classname="ncc.sso.bs.DefaultNCCSSOAuthenticator">
        <regTimeOut>200</regTimeOut>
        <listParam key="IPAddress">
            <string>127.0.0.1</string>
            <string>10.11.115.*</string>
        </listParam>
    </authenticator>
</NCCSSOConfigVO>
```

配置文件中classname是对应的获取token时的校验类，regTimeOut对应token的有效时间，这里单位为秒，如果不配置regTimeOut则默认有效时间60秒。listParam里面为ip地址的白名单，这里白名单可以使用\*号进行匹配。单点登陆时白名单匹配失败会报401错误“单点登录失败，非法客户端 IP”。如果不希望配置白名单，对客户端的IP地址不做校验则应删除listParam标签。

（4）拼接单点登陆YonBIP高级版系统URL：拼接路径中需要accesstoken值即为第二步成功返回的token，超时或使用过一次则失效，token的有效时间即为前面步骤3里xml配置的regTimeOut。此外还需要单点成功后跳转的路径，看下面的路径示例：

`http://ip:port/nccloud/resources/uap/rbac/thirdpartylogin/main/index.html?accesstoken=xxx&redirect_uri=http://ip:port/nccloud`

这个路径中ip、port是YonBIP服务的ip地址和服务端口， `http://ip:port/uap/rbac/thirdpartylogin/main/index.html` 为固定地址，accesstoken=xxx是获取的临时token。拼接路径参数redirect\_uri值为 ` http://ip:port/nccloud` 对应单点登陆后跳转的路径，这里是跳转至首页，如果需要跳转到审批中心或者其他节点可以参考我们后面的介绍。拼接完成访问URL时，服务器收到该请求，会验证token和用户登录权限，验证通过则重定向到redirect\_uri。

（5）单点地址校验：一般来说单点登陆是由第三方页面发起请求，YonBIP高级版为了安全考虑对第三方发起者进行了校验，校验类为nccloud.framework.core.filter.CorsFilter。比如发起请求的第三方页面地址为 `https://nccdev.yonyou.com/index` ，则我们需要在home\\hotwebs\\nccloud\\WEB-INF\\corsfilter.properties 配置文件中加入如下配置：

```
properties....//省略部分内容
referer=https://nccdev.yonyou.com
```

如果有多个系统单点登陆到YonBIP高级版的需求，则多个地址之间应该使用英文逗号进行分割，例如除了上面的例子外还有第三方系统：“ [http://192.168.0.1:8080/user/index.jsp](http://192.168.0.1:8080/user/index.jsp) ”、" [http://test.yonyou.com:8900/](http://test.yonyou.com:8900/) "页面也需要单点登陆到YonBIP系统中来，则corsfilter.properties 配置文件应该写成下面方式：

```
properties....//省略部分内容
referer=https://nccdev.yonyou.com,http://192.168.0.1:8080,http://test.yonyou.com:8900
```

如果不配置或者配置错误时会报403错误“未获取授权，无法查看此网页”，如下面的例子：

## 2.3 示例代码

根据前面的介绍如果要实现单点登陆，我们需要先往数据库SM\_OAUTH\_SECURITY表中添加第三方数据，所以先执行如下sql：

```sql
sqlINSERT INTO SM_OAUTH_SECURITY (CLIENT_AUTHCLASS, CLIENT_ID, CLIENT_NAME, CLIENT_SECURITY, CLIENT_URL, DR, PK_GROUP, PK_OAUTH_SECURITY, TS) VALUES (null, 'TestThird', '测试单点', 'dee656e3c1', null, 0, null, '1001J210000000005NM9', '2021-12-15 19:24:03');
```

下面这是一个简单的应用程序，这里只是为说明问题方便各位读者理解应该如何获取token及如何拼接路径。需要注意的是示例中用户编码应该根据自己环境选择一个能正常登陆的用户，可以是普通用户或者集团管理员、系统管理员，错误的用户编码会导致单点登陆失败。如果是本地开发环境账套编码为develop，数据源编码为design；如果是测试服务或者正式环境上面需要修改为SM\_BUSICENTER数据库表里对应账套和数据源编码。这里我们示例代码针对的是本地开发服务：

```java
javapackage nc.demo.third;

import java.io.DataOutputStream;
import java.io.InputStream;
import java.io.OutputStream;
import java.net.HttpURLConnection;
import java.net.URL;
import java.net.URLConnection;

import org.apache.commons.codec.binary.Base64;

import nccloud.security.impl.SignatureTookKit;

public class PostThirdPartyAccessTokenTest {
    public static void main(String[] args) throws Exception  {
        String dsname = "design";//数据源的编码，本地开发为design
        String usercode = "dcs";//用户的编码
        String client_id = "TestThird";//第三方系统id
        String client_security = "dee656e3c1";//第三方秘钥
        String busicentercode = "develop";//账套编码，本地开发时为develop

        String url="http://127.0.0.1:8080/service/genThirdPartyAccessToken";
        String security = genKey(usercode,usercode + client_security + (System.currentTimeMillis() + "").substring(0, 6));
        String write="type=type_security&dsname="+dsname+"&usercode="+usercode+"&client_id="+client_id+"&security="+security+"&busicentercode="+busicentercode;
        write=write.replaceAll("\\+","%2B");//避免出现特殊符号问题。

        //获取token
        String token = getToken(url,write);
        if("".equals(token)) {
            System.out.println("获取token失败");
            return;
        }
        //单点成功后的跳转路径，这里为首页工作桌面
        String redirect_uri = "http://127.0.0.1:8080/nccloud";
        //单点登陆时的全路径，可以直接粘贴到浏览器进行访问。
        String fullUrl="http://127.0.0.1:8080/nccloud/resources/uap/rbac/thirdpartylogin/main/index.html?accesstoken="+token+"&redirect_uri="+redirect_uri;

        System.out.println("获取的token为："+token);
        System.out.println("单点登陆全路径，复制到浏览器地址栏即可："+fullUrl);
    }

    /**调用接口获取token
     *
     * @param url 要调用的接口
     * @param write 需要写出的数据
     * @return token值
     * */
    private static String getToken(String url,String write) {
        OutputStream outStream=null;
        DataOutputStream dataOutput=null;
        InputStream inStream  = null;
        String ret="";
        try {
            URL preUrl = new URL(url);
            URLConnection con = preUrl.openConnection();

            //设置为true，后面可以调用getOutputStream并传输数据
            con.setDoOutput(true);
            //不使用http缓存
            con.setUseCaches(false);
            //设置http请求头
            con.setRequestProperty("Content-Type", "application/x-www-form-urlencoded");
            con.setRequestProperty("Content-Length", "10000" );

            HttpURLConnection httpCon = (HttpURLConnection) con;
            //这里使用post方式调用接口
            httpCon.setRequestMethod("POST");
            //获取输出流
            outStream = httpCon.getOutputStream();
            dataOutput = new DataOutputStream(outStream);
            //接口需要的参数作为字节序列写入输出流
            dataOutput.writeBytes(write);
            dataOutput.flush();
            //获取输入流
            inStream  = httpCon.getInputStream();
            //读取服务返回的数据
            int ch;
            while ((ch = inStream.read()) != -1) {
                ret += String.valueOf((char) ch);
            }
        } catch (Exception e) {
            System.out.println(e);
        } finally {
            //关闭服务输入流和输出流
            if (dataOutput != null) {
                try {
                    dataOutput.close();
                } catch (Exception e2) {
                }
            }
            if (outStream != null) {
                try {
                    outStream.close();
                } catch (Exception e2) {
                }
            }
            if (inStream != null) {
                try {
                    inStream.close();
                } catch (Exception e2) {
                }
            }
        }
        //返回获取到的token
        return ret;
    }

    private static String genKey(String userid,String key) throws Exception{
        return new Base64().encodeToString(SignatureTookKit.digestSign(userid.getBytes(), key.getBytes()));
    }
}
```

## 2.4 单点登陆到普通节点

（1）需要先确定要跳转到节点的url。这里以人员节点为例，打开【人员】节点后，可以看到浏览器地址栏有一段 `page=10140PSN-10140PSN_main`

按F12 打开浏览器控制台找到【应用或Application】->【会话存储空间或session storage】->右边密钥找到NCCAPPURL，下面可能会有多个key和url，我们只需要找到需要页面的10140PSN-10140PSN\_main即可。

![](https://community.yonyou.com/datadict/bipbook/assets/3%E4%BA%BA%E5%91%98%E8%8A%82%E7%82%B9%E7%95%8C%E9%9D%A2.DHyB_wab.jpg)

这里我们复制出来的值为：

`ifr=%252Fuapbd%252Fpsninfo%252Fpsndoc%252Fmain%252Findex.html&c=10140PSN&p=10140PSN_main&ar=0001Z010000000004F8W&n=%25E4%25BA%25BA%25E5%2591%2598&b1=%25E5%258A%25A8%25E6%2580%2581%25E5%25BB%25BA%25E6%25A8%25A1%25E5%25B9%25B3%25E5%258F%25B0&b2=%25E5%259F%25BA%25E7%25A1%2580%25E6%2595%25B0%25E6%258D%25AE&b3=%25E4%25BA%25BA%25E5%2591%2598%25E4%25BF%25A1%25E6%2581%25AF`

相信读者看到这段字符串的时候并不明白它代表的是什么意思、包含哪些信息，如果找一下URL解码工具对这段字符串进行两次解码（注意是两次解码）可以得到下面的字符串：

`ifr=/uapbd/psninfo/psndoc/main/index.html&c=10140PSN&p=10140PSN_main&ar=0001Z010000000004F8W&n=人员&b1=动态建模平台&b2=基础数据&b3=人员信息`

这个字符串相信各位读者应该能看明白了。这里ifr=/uapbd/psninfo/psndoc/main/index.html是人员节点的页面路径，其他的c：应用编码，p：页面编码，ar：节点小应用主键（sm\_appregister），n：人员，b1：动态建模平台，b2：基础数据，b3：人员信息。这些都是平台在确定小应用、打开小应用时所必要的信息。

（2）代码拼接url：与2.3小节里面基本类似，我们这里需要做的是把获取到的token与需要跳转的redirect\_uri进行拼接。这里redirect\_uri为打开节点后浏览器地址栏里的地址把ifr?后面的 `page=10140PSN-10140PSN_main` 去掉并替换成我们复制出来的字符串。参考示例：

```java
java//单点后跳转的url
String redirect_uri="http://127.0.0.1:8080/nccloud/resources/workbench/public/common/main/index.html#/ifr?ifr=%252Fuapbd%252Fpsninfo%252Fpsndoc%252Fmain%252Findex.html&c=10140PSN&p=10140PSN_main&ar=0001Z010000000004F8W&n=%25E4%25BA%25BA%25E5%2591%2598&b1=%25E5%258A%25A8%25E6%2580%2581%25E5%25BB%25BA%25E6%25A8%25A1%25E5%25B9%25B3%25E5%258F%25B0&b2=%25E5%259F%25BA%25E7%25A1%2580%25E6%2595%25B0%25E6%258D%25AE&b3=%25E4%25BA%25BA%25E5%2591%2598%25E4%25BF%25A1%25E6%2581%25AF";
//拼接全部url，accesstoken为单点登陆获取的token
String url="http://127.0.0.1:8080/nccloud/resources/uap/rbac/thirdpartylogin/main/index.html?accesstoken="+accesstoken+"&redirect_uri="+redirect_uri;
```

## 2.5 单点登陆到审批详情

（1）与前面2.4类似，在浏览器地址栏先确定最后面的page=xxxx，然后浏览器打开开发者工具找到【应用或Application】->【会话存储空间或session storage】->右边密钥找到NCCAPPURL。可能会有多个我们应该找到key和浏览器地址栏匹配的字符串。可参考下图方式：

![](https://community.yonyou.com/datadict/bipbook/assets/4%E5%AE%A1%E6%89%B9%E4%B8%AD%E5%BF%83%E7%95%8C%E9%9D%A2.kXVdNsbL.png)

取到的字符串示例： `ifr=%252Fnccloud%252Fresources%252Fuap%252Fmsgcenter%252Fmessage%252FapproveDetail%252Findex.html%2523%252F%253Fpk_message%253D1001Z31000000008Q01T%2526pageMsgType%253Dworklist%2526appcode%253D10160501%2526pagecode%253D10160501APPROVE%2526c%253D10140CUSTPFA%2526p%253D10140CUSTPF_custpfcard%2526n%253D%25E5%25AE%25A2%25E6%2588%25B7%25E7%2594%25B3%25E8%25AF%25B7%25E5%258D%2595%25E5%25AE%25A1%25E6%2589%25B9%2526checknote%253D&p=10160501APPROVE&n=%25E5%25AE%25A1%25E6%2589%25B9%25E4%25B8%25AD%25E5%25BF%2583&b1=%25E5%258A%25A8%25E6%2580%2581%25E5%25BB%25BA%25E6%25A8%25A1%25E5%25B9%25B3%25E5%258F%25B0&b2=%25E6%25B5%2581%25E7%25A8%258B%25E7%25AE%25A1%25E7%2590%2586&b3=%25E5%25AE%25A1%25E6%2589%25B9%25E7%25AE%25A1%25E7%2590%2586&c=10160501&ar=0001Z0100000000042SD`

与上一小节类似，想要了解字符串里面的信息需要对字符串进行两次URL解码，这时我们会得到下面信息：

`ifr=/nccloud/resources/uap/msgcenter/message/approveDetail/index.html#/?pk_message=1001Z31000000008Q01T&pageMsgType=worklist&appcode=10160501&pagecode=10160501APPROVE&c=10140CUSTPFA&p=10140CUSTPF_custpfcard&n=客户申请单审批&checknote=&p=10160501APPROVE&n=审批中心&b1=动态建模平台&b2=流程管理&b3=审批管理&c=10160501&ar=0001Z0100000000042SD`

在这个字符串中 `ifr=/nccloud/resources/uap/msgcenter/message/approveDetail/index.html#/` 是审批中心的页面路径，单点登陆到审批中心时这个不需要再调整。剩下的字符串中还有几个重要的参数（pk\_message、pageMsgType、appcode、pagecode、ar、c、p），具体含义和获取方式如下：

- pk\_message：消息中心的主键值（sm\_msg\_approve），具体获取方式可直接调用接口：nccloud.message.itf.IApproveMessageQueryService中 queryNCMessages(JSONObject paramJSONObject) 这个接口，具体的参数意义可找到对应的实现类（nccloud.message.bs.MessageQueryServiceImpl）进行查看
- pageMsgType：对应消息中心的消息类型（sm\_msg\_approve中message\_type字段），获取方式同pk\_message这两个字段调用一次接口都能直接获取
- appcode：审批中心应用编码，此处从上图中复制出来的Url已自带，不必修改
- pagecode：审批中心页面编码，此处从上图中复制出来的Url已自带，不必修改
- ar：审批中心小应用主键（sm\_appregister），不必修改
- c：对应当前单据的审批联查页面应用编码，此处从上图中复制出来的Url已自带，根据单据类型调整
- p：对应当前单据的审批联查页面编码，此处从上图中复制出来的Url已自带，根据单据类型调整
- n、b1、b2、b3：面包屑导航参数。

（2）代码拼接url：与前面单点到普通节点类似，这里我们拼接时的redirect\_uri为浏览器地址栏里的路径把ifr?后面的 `page=2022341003110` 替换成我们上面复制出来的路径，参考下面的例子：

## 3 友户通单点登陆NCC

## 3.1 基本流程

1、客户端向第三方服务端发起登陆YonBIP高级版请求。

2、第三方服务端将请求友户通获accessToken

3、友户通收到客户端请求并将验证信息发送给YonBIP高级版服务器。

4、YonBIP高级版接收到友户通的请求，校验信息是否合法，如果合法，响应给友户通access\_token。

5、友户通将接收到的access\_token信息响应给服务器。

6、服务器将access\_tokenx响应给客户端。

7、客户端收到access\_token后拼接请求ncc节点的url信息。

8、NCC收到请求并验证access\_token是否合法，如果合法，则重定向到客户端请求的节点页面。

## 3.2 具体配置流程

（1）友户通设置：友户通租户与NCC集团绑定，友户通账号与NCC用户绑定功能。

（2）YonBIP高级版配置：

- 在友户通企业服务中心下载秘钥文件，放在nchome的/ierp/yht目录下。

![](https://community.yonyou.com/datadict/bipbook/assets/5%E5%8F%8B%E4%BA%92%E9%80%9A%E4%B8%8B%E8%BD%BD%E5%AF%86%E9%92%A5.3UoKNiOh.png)

- 点击绑定友户通按钮进行绑定 ![](https://community.yonyou.com/datadict/bipbook/assets/6%E7%BB%91%E5%AE%9A%E5%8F%8B%E4%BA%92%E9%80%9A.COlmg97L.png)
- 登录YonBIP高级版普通用户，会提示用户与友户通账号绑定。该步骤只需按照页面指导操作即可完成用户绑定，用户绑定完成后会在YonBIP高级版和友户通后台保存对方的用户信息，以便登陆需要。

（3）在第三方系统中，提供登录YonBIP高级版系统链接，该链接请求后台，从友户通获取登录YonBIP高级版的临时token即accesstoken，获取方式如下

```java
javaUserCenter.getNCTokenByAccessToken(String ncId, String ncDs, String accessToken);
```

jar包依赖配置：

```xml
xml<groupId>com.yonyou.yht</groupId>

```

（4）获取到access\_token后拼接完整url，格式固定如下： ` http://ip:port/uap/rbac/thirdpartylogin/main/index.html?accesstoken=xxx &redirect_uri=http://ip:port/nccloud` ip和port是YonBIP高级版的服务地址的ip和端口号。 ` http://ip:port/uap/rbac/thirdpartylogin/main/index.html` 为固定地址 accesstoken=xxx为(4)中获取的临时token。 ` redirect_uri=http://ip:port/nccloud` 为成功登录后要重定向的地址，如后面为空，默认跳到工作桌面。如果是列表或卡片节点，需改为节点的url的地址。如果是审批详情页面，需要客开，因为审批详情页面的url不同其他页面，需要向NCC服务器请求获取。拼接完成访问url时，NCC服务器收到该请求，会验证token和用户登录ncc权限，验证通过则重定向到redirect\_uri。
