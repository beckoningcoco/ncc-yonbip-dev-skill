# 多语言 | YonBIP高级版

## 多语言

---

多语处理包括四个方面，即元数据、Java代码、前端模板、前端代码多语。

多语资源id命名规则为：（分类(1位)+功能点+编码），其中分类：0代表代码，1代表系统预置脚本，2代表元数据。

## 1 元数据多语

YonBIP高级版系列产品支持两种方式导出元数据多语：

- 使用YonBuilder Premium开发工具， [下载地址](https://nccdev.yonyou.com/studio)
- 通过低代码平台的“元数据设计”节点进行导出

一、 使用YonBuilder Premium开发工具

在开发工具中打开元数据文件，在画布空白出点击右键，选择“导出多语”。

1、不同版本的开发工具这个右键菜单名称可能不同，有的叫“多语资源处理”。
![](https://community.yonyou.com/datadict/bipbook/assets/1-1.CLlr25iG.png)

2、根据业务需要勾选要导出多语的字段
![](https://community.yonyou.com/datadict/bipbook/assets/1-2.DpJrUD-4.png)

3、选择多语文件导出位置，填写要生成多语资源的前缀。注意：文件名最好是元数据名称+6res，依赖尽量不要勾选。
![](https://community.yonyou.com/datadict/bipbook/assets/1-3.D7vEcB_i.png)

4、最后点击Next，然后检查下生成的多语资源id，点击Finish
![](https://community.yonyou.com/datadict/bipbook/assets/1-4.LGL1X22v.png)

5、检查元数据中的多语资源id是否已全部赋值，确认后需要重新发布元数据。
![](https://community.yonyou.com/datadict/bipbook/assets/1-5.sZ_IyWze.png)

二、通过低代码平台的“元数据设计”节点进行导出

1、在”元数据设计“节点打开元数据，点击右上角的”添加多语“按钮
![](https://community.yonyou.com/datadict/bipbook/assets/1-6.CaAGlo-M.png)

2、添加多语前缀
![](https://community.yonyou.com/datadict/bipbook/assets/1-7.QLDzVLtg.png)

3、点击确定导出多语文件后，查看”实体属性“页签的”多语资源ID“字段已更新
![](https://community.yonyou.com/datadict/bipbook/assets/1-8.BUfjBIa4.png)

## 2 Java内多语

Java内的多语抽取主要是通过【代码多语工具包】来做的。如果没有此工具，公司内开发人员请找对应的技术负责人获取，伙伴请找对应分公司技术负责人获取。具体操作步骤如下：

1、把【代码多语工具包】下的restools.jar添加到工程所使用的NCHOME的lib下
2、把【代码多语工具包】下的lang的common文件夹目录拷贝到组件的\\lang下
3、在pub\_billtemplet表中查询节点所对应的nodecode，此处以blacklist为例
4、在eclipse中以Application方式运行类ml.tools.CollectionTempletResUI
5、配置数据库连接和要保存多语资源文件的目录(”blacklist“是第3步中黑名单管理的nodecode值)

注：在抽取重量端模板多语资源时，工具只支持SqlServer数据库
![](https://community.yonyou.com/datadict/bipbook/assets/2-1.DDJKtqdS.png)

6、点下一步，工具根据blacklist查询出单据模板（非元数据）的相关资源，如果资源在公有资源中存在，则取得公有资源的ID，否则生成私有的ID，点击更新数据库资源ID号把生成的私有ID号存入数据库
![](https://community.yonyou.com/datadict/bipbook/assets/2-2.BED4JCHj.png)

7、点下一步进入进入收集节点java代码资源和处理替换Java代码资源的界面
![](https://community.yonyou.com/datadict/bipbook/assets/2-3.CJ9Mw301.png)

8、点代码资源收集按钮，在弹出框录入要进行代码文件收集的文件包范围
![](https://community.yonyou.com/datadict/bipbook/assets/2-4.DFVHzA9U.png)

9、点击确定按钮，工具收集到的有多语需要处理的java文件
![](https://community.yonyou.com/datadict/bipbook/assets/2-5.nbKNo5HY.png)

10、根据各java文件下所罗列的资源，删除掉不需要的资源，或者无法直接替换的资源，然后点击生成资源ID按钮把这些资源的ID存入数据库中

注意：a) 中文字符串有特定业务含义的，比如if(“中文字符串”.equals(str)) …，需要更改java代码不用中文字符串作条件或业务逻辑的判断依据；b) 中文字符串存在换行的，需要考虑更改java代码，比如 setText(“多语处理”+”字符串”); 改为:setText(“多语处理字符串换行了”)；c) 其他一些不作为界面展示item的相关中文字符串，不需要多语。
![](https://community.yonyou.com/datadict/bipbook/assets/2-6.BGrV2AV8.png)

11、点击替换java文件按钮对搜索出的资源进行多语替换。

## 3 前端模板多语

抽取模板多语前需要先备份数据（平台那边现在模板多语按钮只能点击一次， 如果第一次操作失败，就需要把对应的数据还原回去，然后再次操作）

涉及的表如下（pub\_page\_templet，pub\_area，pub\_query\_property，pub\_form\_property四张表）

## 3.1 备份脚本

查询查询模板区域属性注册

```sql
sqlselect *
from pub_query_property
where areaid in
      (select pk_area
       from pub_area
                left join pub_page_templet
                          on pub_area.templetid = pub_page_templet.pk_page_templet
       where pub_page_templet.appcode in ('60071010'));
```

查询页面模板区域属性注册

```sql
sqlselect *
from pub_form_property
where areaid in
      (select pk_area
       from pub_area
                left join pub_page_templet
                          on pub_area.templetid = pub_page_templet.pk_page_templet
       where pub_page_templet.appcode in ('60071010'));
```

查询页面模板区域注册

```sql
sqlselect *
from pub_area
where templetid in (select pk_page_templet
                    from pub_page_templet
                    where appcode in ('60071010'));
```

查询页面模板注册

```sql
sqlselect *
from pub_page_templet
where appcode in ('60071010');
```

## 3.2 抽取模板多语

在应用注册节点选中要导出多语的节点，点击“导出该应用下所有页面模板多语”。一个应用点一次就可以，不能重复点（重复点击，会导致生成的多语文件乱码）。需要提前建好多语文件，文件名为“应用编码.properties”
![](https://community.yonyou.com/datadict/bipbook/assets/3-1.DKr2xqb6.png)

修改导出的文件的编码格式（编码格式为UCS-2 Big Endian）
![](https://community.yonyou.com/datadict/bipbook/assets/3-2.D75rRofi.png)

文件夹命名规则：中文zh-CN文件夹更改为 simpchn文件夹、英文文en-US文件夹更改为english文件夹、繁体中文文件夹更改为tradchn文件夹。下面以中文为例。把此文件放入工程目录下的resources/lang/simpchn目录下，建一个 文件夹，命名为应用编码（文件夹命名必须全部都是小写）+后缀（后缀是为了不和NC的多余文件冲突）
![](https://community.yonyou.com/datadict/bipbook/assets/3-3.BfNSRIXM.png)

## 3.3 开发环境生效

在nchome\\ resources\\lang\\simpch下建一个 文件夹，命名为应用编码，放入多语文件，重启中间件，多语文件即可生效。

## 4 前端代码多语抽取

## 4.1 前端代码多语抽取

前端代码中多语的抽取所需要的工具，公司内开发人员请找对应的技术负责人获取，伙伴请找对应分公司技术负责人获取。

抽取多语前先备份一份原文件，工具回写并不能保证百分百正确。该工具识别到中文时，会在当前检测到的中文所在位置，以该位置前后方向的第一个单引号或双引号为分割标识来转译多语。前端多语文件格式为：UTF-8。

具体操作步骤：打开工具，配置好对应内容，先点击抽取，然后点击会写即可。其中“key规则”填入应用编码。

## 4.2 前端多语适配

平台提供的方法是将查询到的多语放在state中维护
![](https://community.yonyou.com/datadict/bipbook/assets/5-1.BOjzdtFd.png)

一般的多语语句替换
![](https://community.yonyou.com/datadict/bipbook/assets/5-2.BOTOf5bb.png)

多语语句需要拼接时的适配
![](https://community.yonyou.com/datadict/bipbook/assets/5-3.ANEjPZ0P.png)

在constructor中涉及到的多语，可以先设置为多语code，当使用时，用相应的方法来读取
![](https://community.yonyou.com/datadict/bipbook/assets/5-4.CKqEqA0h.png)

在相应的组件中涉及到多语，可以在调用时通过传参的形式传递到组件中来获取
![](https://community.yonyou.com/datadict/bipbook/assets/5-5.BgIAS0U3.png)

回写规则

```js
js{this.props.MutiInit.getIntl(2052)&& this.props.MutiInit.getIntl(2052).get('20521030-0003')}
```

多语中如果有变量值，多语文件 json中

```js
js{"20521030-0021": "{name}" }
```

对应的js code中

```js
js{multiLang && multiLang.get('20521030-0021',{name:'第一帧'})}
```

页面替换 为 name 值。

使用这个方法需注意：产生多语资源不能确保在使用多语资源之前就加载完成，可能会报错。
