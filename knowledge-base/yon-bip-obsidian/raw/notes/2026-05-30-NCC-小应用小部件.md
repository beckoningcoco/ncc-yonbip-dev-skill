# 小应用小部件 | YonBIP高级版

## 小应用小部件

---

## 1.小部件介绍

当我们登陆系统后在桌面右上角点击头像进入桌面设置（2111及以后版本在工作桌面右上角）可以拖拽小应用或者小部件到工作桌面上来。保存设置后可以看到下图效果，对于小应用来说更像是一种快捷方式，我们点击小应用图标可以快速打开应用节点，小部件与小应用不同之处在于我们不需要打开节点也可以直接查看数据。小部件可以每隔一段时间刷新数据，当需要查看详情时点击小部件也可以打开对于节点。根据前面的章节内容我们已经知道该如何开发一个普通应用节点，那么如何开发一个小部件可以参考本章节内容。

![](https://community.yonyou.com/datadict/bipbook/assets/1%E5%B0%8F%E5%BA%94%E7%94%A8%E4%B8%8E%E5%B0%8F%E9%83%A8%E4%BB%B6.BPHjbwZt.png)

\*\*定义：\*\*有具体的操作（如待审批信息10条）、需要图形化展示（如投资收益率展现折线图）。登陆后加载工作桌面中的小部件会先检查功能权限，并占用license，如果该小部件没有license，会有明确提示。

\*\*分配：\*\*系统发版时，会预置上本版的所有小部件，系统管理员在初次实施时，会新建角色、新建职责，然后把小部件分配到不同的职责上，将职责挂在不同的角色上。

![](https://community.yonyou.com/datadict/bipbook/assets/%E7%94%A8%E6%88%B7%E6%9D%83%E9%99%90%E6%A8%A1%E5%9E%8B.BkyuiLKy.png)

\*\*复制：\*\*小部件不支持复制。

## 2.小部件开发过程

（1）应用注册节点注册小部件

应用类型注册为小部件，小部件的路径对应前端js路径。

![](https://community.yonyou.com/datadict/bipbook/assets/2%E5%BA%94%E7%94%A8%E6%B3%A8%E5%86%8C.Z6oUyBjL.png)

（2）编写前端js

如下是一个小部件的index.js代码示例：

```javascript
JavaScriptimport React, { Component } from 'react';
import ReactDom from 'react-dom';
import { ajax, pageTo,  viewModel } from 'nc-lightapp-front';

const names = {"approve": '审批中心', "todo": '业务中心', "prealert": '预警中心', "notice": '通知中心'}
const openParams = {
    approve: {appcode: "10160501", pagecode: "10160501APPROVE"},
    todo: {appcode: "10160502", pagecode: "10160502TODO"},
    prealert: {appcode: "10160503", pagecode: "10160503PREALERT"},
    notice: {appcode: "10160504", pagecode: "10160504NOTICE"}
};

class MessageCenterSummary extends Component {
    constructor(props) {
        super(props);
        this.state = {
            json: {},
            inlt: null,
            unreadCounts: {approve: 0, todo: 0, prealert: 0, notice: 0},
            refreshing: false,
        }
        this.container = React.createRef();
    };
    componentWillMount() {
        //如果需要获取多语等可以放到这里来
    }

    componentDidMount(){
        //加载初始数据
        this.initData();
    }

    /**
    * render里面渲染刷新的图标及点击事件，如果点刷新时重新加载数据。
    */
    refreshSummary = e => {
        if(this.state.refreshing) return;
        this.setState({ refreshing: true });
        this.initData(e);
    }

    initData = (e) => {
        let _this = this;
        ajax({
            url: '/nccloud/riart/message/summary.do',
            data: {},
            method: 'POST',
            loading: false,
            success: (res) => {
                let retVal = { counts: res.data };

                _this.setState({
                    unreadCounts: retVal.counts,
                });
                setTimeout(() => {
                    this.setState({ refreshing: false})
                }, 1000);
            }
        });
        e && e.stopPropagation();
    }

    /**
    * div标签里面添加onClick事件，当用户点击时打开对应的节点
    */
    openLink(msgType){
        pageTo.openTo("/uap/msgcenter/message/main/index.html?msgType=" + msgType, openParams[msgType], null, { uniqueTab: true });
    }

    render() {
        let { unreadCounts, refreshing } = this.state;
        let arr = ["approve", "todo", "prealert", "notice"].map(item =>{
            return <div onClick={() => {this.openLink(item);}}>
                        <h4 className="count font-26">
                            {unreadCounts[item] > 99 ? '99+' : unreadCounts[item]}
                        </h4>
                        <p className="font-12 msg-center-12 label"><span>{names[item]}</span></p>
                        { item == "approve" && <div className="approve_icon_bg"/>}
                        { item == "todo" && <React.Fragment><div className="todo_icon_bg_left"/><div className="todo_icon_bg_right"/></React.Fragment>}
                        { item == "prealert" && <div className="prealert_icon_bg"/>}
                        { item == "notice" && <div className="notice_icon_bg"/>}
                        <div className="icon_1"/>
                    </div>
        });
        return (<div id="messageCenter_messageCenter_summary" class="platform-app">
            <div className="message-center-container">
                <div className="msg-title nc-theme-Widgets-font-c" style={ {fontSize: 14} }>消息中心</div>
                <div className="center-mar pointer">
                    <div className="block-wrapper">
                        {arr}
                    </div>
                </div>
                <div className="refresh-bottom">
                    { refreshing ? <div className="ball-clip-rotate"><div/></div> : <div className="refresh-icon messageCenter pointer" onClick={this.refreshSummary}></div>}
                </div>
            </div>
        </div>);
    }
}

// ReactDOM.render(<MessageCenterSummary />, document.querySelector('#app'));
//小部件最后要用“export default 组件名”
export default MessageCenterSummary;
```

小部件的index.js末尾需要export该组件 `export default MessageCenterSummary;`开发阶段可以使用 `ReactDOM.render` 先调试，最后发布出去之后一定要改成 `export default 组件名`

在小部件入口index.js平级增加一个 `config.json` 文件，目的是打包之后的js不带hash值，不加这个 `config.json` 文件的话会找不到部件文件路径，内容如下所示：

```json
json{
    "template": "./template/index.html",
    "hash": false
}
```

将部件打包放到home\\hotwebs\\nccloud\\resources下对应模块里

## 3小部件自动刷新方案

（1）在应用注册找到小部件的应用，在参数注册添加一个参数refresh\_time，参数值写该小部件定时刷新的间隔，单位是秒

![](https://community.yonyou.com/datadict/bipbook/assets/5%E5%88%B7%E6%96%B0%E5%8F%82%E6%95%B0.B95aw-8b.png)

（2）从platform引入widgetAutoRefresh

```javascript
import { widgetAutoRefresh } from 'nc-lightapp-front';
```

widgetAutoRefresh是一个函数，接收两个参数：

| 参数名 | 类型 | 描述 |
| --- | --- | --- |
| appcode | String \\Function | 小部件的appcode，用于查询参数注册中的刷新间隔 |
| refreshCallback | Function | 小部件自动刷新执行的逻辑 |

（3）引入widgetAutoRefresh后有两种写法：

**第一种，class组件可以使用装饰器写法**

```javascript
javascript//_this是包裹组件的引用，可以通过_this拿到props和内部方法
@widgetAutoRefresh('10160511',_this => {
    //这里写小部件自动刷新时执行的逻辑
    _this.initData();
})
class WidgetDemo extends Component {...}
```

**第二种，函数组件可以使用普通写法**

```javascript
javascript//不使用自动刷新写法
function WidgetDemo() {...}

export default WidgetDemo;

//自动刷新写法：
const autoRefreshCallback = function(_this) => {
    //这里写小部件自动刷新时执行的逻辑
    _this.initData();
}

WidgetDemo = widgetAutoRefresh('10160511', autoRefreshCallback)(WidgetDemo)

export default WidgetDemo;
```

注意

如果小部件使用了createPage包装组件，widgetAutoRefresh必须在写在createPage之前

```javascript
javascript//一定要写在createPage之前，不然会找不到_this
WidgetDemo = widgetAutoRefresh('10160511', autoRefreshCallback)(WidgetDemo);

WidgetDemo = createPage({})(WidgetDemo);

export default WidgetDemo;
```

如果小部件引用的是公共组件widgetAutoRefresh第一个参数可以传function，必须要返回appcode

```javascript
javascript@widgetAutoRefresh(_this => {
    //_this是包裹组件的引用，可以访问props
    return _this.props.appcode; //必须要返回appcode
},
    _this => _this.initData()
)
```
