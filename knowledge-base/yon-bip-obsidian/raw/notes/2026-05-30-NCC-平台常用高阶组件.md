## 1 基础按钮组件

1、基础组件按钮的引入

```javascript
javascriptimport {base} from 'nc-lightapp-front';
const {NCButton} = base;
```

2、基础组件的使用

```javascript
javascript<NCButton
```

基础组件按钮的使用很简单，这里不再做基础组件的讲解，一般我们做NC Cloud开发都是应用级别的开发，按钮创建不可能一个一个的创建，需要一个api把我们按钮注册的按钮数据按规则统一创建，而底层给我们封装了一个api:

## 3 按钮生成

在代码中props对象里有一个button对象，从button对象中可以解构出createButtonApp方法，该方法用于创建按钮，并且注册按钮点击事件监听

```javascript
javascriptlet {button} = this.props;
let {createButtonApp} = button;
return <div className="btn-group">
    {createButtonApp({
         area :  '按钮区域',//按钮组件区域编码,
         buttonLimit : 3,//限制个数（好像没用）
         onButtonClick : (props,btncode)=>{},//按钮点击事件监听
    })}
</div>
```

通过 按钮注册 和 按钮生成两个步骤就可以创建出对应的按钮，createButtonApp这里需要注意：

1、area指向的是按钮注册界面上按钮信息的“按钮区域”列的值，想要挂在同一个地方的按钮，需要有相同的按钮区域值

2、onButtonClick是创建出来的按钮的点击事件监听

为什么在按钮注册页面注册了按钮信息，这里就可以使用按钮区域去创建出按钮，这其中还缺少一步按钮的代码层面的注册,在每个页面的js代码中都有一个逻辑：

请求页面资源，使用createUIDom:

```javascript
javascriptthis.props.createUIDom({//请求页面资源
    pagecode,
    appcode
},(data)=>{//请求完成后的回调方法
    let {template,button,context} = data;//data中含有我们在应用注册里页面信息上注册的所有按钮
    //此时需要调用setButtons方法，把按钮数据注册给底层
    this.props.button.setButtons(button,()=>{//通过这一步，其实就是告诉底层：我把按钮给你了，到时候调用你的方法的时候给我创建我想要的按钮出来啊
        //other business code
    });
});
```

```javascript
javascriptlet {
```

## 2 createButtonApp

这个方法用于创建 **列表界面右上角的按钮** 、 **卡片界面右上角的按钮** 以及 **卡片界面子表格右上角肩部的按钮** 。

使用方式：

```javascript
javascript<div className="header-button-area">
    {/*头部按钮*/}
    {this.props.button.createButtonApp({...btn})}
</div>
```

参数btn的结构:

```javascript
javascriptbtn : {
    area :  'common',//按钮组件区域编码,即在按钮注册那里注册按钮时填写的按钮区域编码，渲染时会把按钮区域编码为common的按钮，按照注册顺序渲染到界面上
    buttonLimit : 3,//限制个数（好像没用）不写也没问题
    onButtonClick : (props,btncode)=>{this.onButtonClick({props,btncode})},//按钮点击事件
},
```

参数btn中对于onButtonClick的实现:

```javascript
javascript//★★★注册时，对原始onButtonClick的松散参数做了一个小封装，变成了对象参数★★★!!!
onButtonClick = ({props,button})=>{
    switch(btncode){
         case 'Add'//点击新增按钮时触发该分支逻辑
            this.onAdd();//自己实现新增的逻辑
            break;
         default:break;
    }
}
```

## 3 createOprationButton

该方法作用是创建操作列按钮，其使用方式如下：

```javascript
javascript//meta对象的来源：this.props.meta.getMeta()
//tableId是需要添加操作列的表格的表格区域编码（应用注册-模板注册 里面 添加一个表格区域的区域编码）
meta[tableId].items.push({
    attrcode: 'opr',
    itemtype:'customer',
    label: '操作',
    width: 150,
    className : 'table-opr',
    fixed: 'right',
    visible: true,
    render:(text,record, index) => {
        //过滤出当前表格的操作列按钮
        let button = this.props.button.getButtons();
        //获得按钮区域编码为 tablId+"Opr" 值 的所有按钮的按钮编码（★★★这里注意区分 表格区域编码、按钮区域编码、按钮编码★★★）
        let btncodes = ((button || []).filter(btn => { return btn.area == tableId+"Opr" }) || []).map(btn => {
            return btn.key;
        });
        //创建操作列按钮
        return this.props.button.createOprationButton(btncodes, {
            area:tableId+"Opr",
            buttonLimit: 3,
            onButtonClick: (props, key) => {
                /**
                 * key: 按钮编码
                 * index：当前要删除的数据行序号
                 * areacode：当前操作的卡片表格的区域编码
                 * 这里的参数可以根据自己的业务向参数对象中添加
                 */
                this.onButtonClick({btncode:key,index,areacode:tableId,record})
            }
        });
    }
});
```

以上写法常见于低代码平台生成的前端代码的initMeta方法中，如果是标准产品的代码常见与initTemplate.js或者index.js的componentDidMount等方法中。

## 5 setUploadConfig

该方法比较特殊，由于导入是excel导入数据，它专门负责设置导入按钮的配置，其写法如下：

```javascript
javascript//设置导入按钮属性
this.props.button.setUploadConfig('Import', excelImportconfig(
    this.props,
    moduleName,//模块名称（元数据上的所属模块）
    billType,
    true,
    '',
    {pagecode,appcode},//pagecode：页面编码；appcode:应用编码
    this.doRefresh //回调函数执行刷新（这个doRefresh对于excelImportconfig来说是一个回调方法，是根据自己的节点自定义的）
));
```

上述代码中 方法setUploadConfig的参数：

1、"Import" 是按钮注册注册的导入按钮编码

2、excelImportconfig的来源：

2005版本之前的引入方式： `import { excelImportconfig } from "nc-lightapp-front"` 。

2005及以后版本引入方式： `import excelImportconfig from 'uap/common/components/excelImportconfig'` 。

## 4 高阶组件（表单、表格）个性化渲染

在项目交付中，经常使用的高阶组件是表格和表单

表格的创建：

```javascript
this.props.table.createSimpleTable(tableAreaCode,{})

this.props.cardTable.createCardTable(tableAreaCode,{})

this.props.editTable.createEditTable(tableAreaCode,{})
```

表单的创建：

```javascript
this.props.form.createForm(formAreaCode,{})
```

在实际应用中经常因为个性化的展示，需要改变表格或者表单的默认渲染方式。

众所周知，不管是表单和表格，其实底层对于每一表单项或者表格列，都有默认的render方法，如果需要个性化的展示，则需要我们对表单项或者表格列进行个性化的render处理，也就是给需要个性化展示的表单项或者表格列实现一个自己的render方法。

```javascript
         * tree:树组件配置
         * card:卡片组件配置
         * defLeftWid:默认左侧宽度
         * leftMinWid:左侧最小宽度
         */
        let {tree,card,defLeftWid,leftMinWid} = dragComp;
        /**
         * 解构card
         * form:表单组件配置
         * formList:卡片表格区域编码
         */
        let {form,formList} = card;
        //解构form area: 表单区域编码
        let {area} = form;
        /**
         * 解构tree
         * treeId: 树组件编码
         * disabledSearch:树组件查询区禁用
         */
        let {treeId,disabledSearch} = tree;
        //禁用树
        this.disableTree(treeId,disabledSearch);
        //选中当前节点
        this.selectTreeNode(node.refpk);
        //清空卡片信息
        this.clearCard(
        ).then(()=>{
            let billCard = this.buildReqParam();
            //加载卡片信息
            return this.loadCard({
                url:URLS.addUrl,
                data : {
                    parentId : node.refpk,//父节点pk
                    billCard,
                    userJson : {
                        //parentKey:父属性的field，parentPk:父节点pk
                        data : {parentKey : FIELDS.PARENT_ID,parentPk : node.refpk}
                    }
                }
            });
        }).then((res)=>{
            //res : 加载的卡片数据
            //填充卡片数据
            return this.fillCard({data : res.data}).then(()=>{return Promise.resolve(res);});
        }).then((res)=>{
            //res : 编码规则返回结果
            //设置表单状态
            this.setFormStatus();
            //设置表格状态
            this.setTableStatus();
            //更新按钮状态
            this.updateBtnStatus();
            //存在编码规则，设置编码字段的编辑性
            let {externalData : {billCodeContext}} = res;
            billCodeContext && this.props.form.setFormItemsDisabled(
                area,
                {[FIELDS.CODE] :  !(billCodeContext.isPrecode && billCodeContext.isEditable)}
            );
            //***********************************
            this.selfRenderForm();
            //***********************************
        });
    });
}
```

高阶表单组件对FormItem的个性化渲染分成两种 浏览态和编辑态各有个性化渲染的方式。其实这里我建议过平台个性化渲染跟组件的状态没有关系，但是目前鉴于种种情况改不了，所以只有按照现有的方式来。

## 4 同步树与异步树

绝大多数场景使用同步树是可以的，只有很少一部分场景可能存在异步树，即树数据量本身非常庞大，一次性加载数据非常耗时，这时候就需要使用异步树。

而异步树在使用时有个问题无法解决：如果需要关键字过滤树节点需要怎么做，是要就目前已经加载的树数据进行过滤，还是要查所有数据进行过滤？

如果查所有数据，那又回到了同步树的情况，加载可能非常耗时；如果就目前已经加载的数据进行过滤，很有可能用户想要的数据拿不到。而数据量非常大

的时候用户很有可能非常想要对树的关键字过滤功能，所以这就产生了矛盾。因此异步树就留给客开自行解决。
