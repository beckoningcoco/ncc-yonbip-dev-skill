# 开发环境搭建 | YonBIP高级版

## 开发环境搭建

---

## 1 一键搭建环境

## 1.1 工具申请地址

工具申请地址： [https://](https://nccdev.yonyou.com/article/detail/1275) [nccdev.yonyou.com/article/detail/1275](https://nccdev.yonyou.com/article/detail/1275)

## 1.2 目录结构

  D盘下新建“DevHome”目录，把下载的压缩包拷贝到该目录并解压，形成如下目录结构。

如果不使用 D:\\DevHome 这样的目录，后续的一些环境变量、nchome 路径、项目路径等需要调整成本地的目录。

![](https://community.yonyou.com/datadict/bipbook/assets/11.2-01.CqJsGC3v.jpg)

![](https://community.yonyou.com/datadict/bipbook/assets/11.2-02.C4LwPm8C.jpg)

## 1.3 环境配置

拷贝 ncchome 到 ncc-home 目录下，ncc-home 下不要再有二级目录。

保证 prop.xml 文件的路径是这样的：D:\\DevHome\\ncc-home\\ierp\\bin\\prop.xml

保证 sysconfig 中配置的服务器端口和前端 D:\\DevHome\\ncc-front\\config.json 中的端口设置一致，

并在 config.json 中的 buildEntryPath 中添加需要调试的 js 文件路径

![](https://community.yonyou.com/datadict/bipbook/assets/11.3-01.ByKjfhNK.jpg)

![](https://community.yonyou.com/datadict/bipbook/assets/11.3-02.Brp7MCcC.jpg)

## 1.4 启动命令

   方式一：

第一步：双击“1.启动 Idea.bat”启动idea

第二步：双击“2.启动 VSCode.bat”启动VS Code

第三步：双击“3.启动 前端脚手架.bat”启动前端脚手架

方式二：

双击“0.一键三联.bat”可以同时启动以上3个工具

![](https://community.yonyou.com/datadict/bipbook/assets/11.4-01.D3UDuRjy.jpg)

## 2 IDEA环境搭建

## 2.1 前期准备

初次打开 idea 开发工具时，需要打开一个目录作为工作空间

![](https://community.yonyou.com/datadict/bipbook/assets/12.1-01.BWNDU57U.jpg)

然后打开项目结构，配置相关信息

![](https://community.yonyou.com/datadict/bipbook/assets/12.1-02.B6lOQx6D.jpg)

其中 SDK 配置的是 jdk

windows 需要选择 nchome 中的 ufjdk 目录作为开发中使用的 jdk；

MacOs 选择 zulu 的 openjdk；

Language level 选择 8；

Compiler output 是选择整个工作空间的编译输出目录，使用默认即可

![](https://community.yonyou.com/datadict/bipbook/assets/12.1-03.R7O5vDa6.jpg)

## 2.2 安装插件

添加自定义插件存储库，参考文章： [https://nccdev.yonyou.com/article/detail/1220](https://nccdev.yonyou.com/article/detail/1220)

打开设置窗口，添加自定义插件存储库路径： [https://nccdev.yonyou.com/ide/idea/latest/updatePlugin.xml](https://nccdev.yonyou.com/ide/idea/latest/updatePlugin.xml)

![](https://community.yonyou.com/datadict/bipbook/assets/12.2-01.BVKQq-hO.jpg)

![](https://community.yonyou.com/datadict/bipbook/assets/12.2-02.CxN4IFDg.jpg)

在 Marketplace 中搜索 **YonBuilder** **Premium** **开发者工具** 点击安装

![](https://community.yonyou.com/datadict/bipbook/assets/12.2-03.DcCHBpRU.jpg)

在 Installed 中找到 **YonBuilder** Premium开发者工具点击更新

![](https://community.yonyou.com/datadict/bipbook/assets/12.2-04.Dg6kCk5Q.jpg)

## 2.3 配置HOME

点击Home配置(Alt+H)按钮，位置在工具栏右侧第一个。

![](https://community.yonyou.com/datadict/bipbook/assets/12.3-01.CE7JiwZH.jpg)

检查数据源配置，要注意：design数据源是我们启动项目时使用的数据源；基准库是导出预制脚本时用到的数据源；本插件支持的数据库类型与配置的home支 持的数据库类型一致。

![](https://community.yonyou.com/datadict/bipbook/assets/12.3-02.DHdw3G5Z.jpg)

可以通过此界面的【数据源迁移】按钮将旧home的数据源配置信息复制到当前选择的home中，复制内容包括：sysconfig中的数据源配置信息和数据库类型。

![](https://community.yonyou.com/datadict/bipbook/assets/12.3-03.CN6JMDez.jpg)

也可通过此界面的【增加驱动类型】按钮添加高级版支持的其他数据库类型，包括：瀚高HighGo、神通数据库(原OSCAR数据库）、人大金仓KingBase、

PolarDB、DB2、腾讯云数据库TDSQL

![](https://community.yonyou.com/datadict/bipbook/assets/12.3-04.BiWdb7MG.jpg)

## 2.4 代码

提前下载安装好本地git程序，然后选择File->New->Project from Version Control…

![](https://community.yonyou.com/datadict/bipbook/assets/12.4-01.Ct052hx3.jpg)

维护好URL，然后注意目录路径，前面对应的是之前打开的工作空间路径，后面跟模块名

![](https://community.yonyou.com/datadict/bipbook/assets/12.4-02.CYMyQIaP.jpg)

Clone完成后，弹出的窗口选择\*\*【Cancel】!!!\*\* 然后就能看到刚才克隆的代码在工作空间下了，图标是一个文件夹的样子

![](https://community.yonyou.com/datadict/bipbook/assets/12.4-03.hSHOOTfc.jpg)

![](https://community.yonyou.com/datadict/bipbook/assets/12.4-04.DvxnGwAB.jpg)

选中刚克隆下的文件夹右键，选择 YonBIP高级版项目工具->转化成YonBIP高级版项目，输入模块名称，然后等待加载完成。完成后重启idea。在此操作期间的 报错忽略即可。

![](https://community.yonyou.com/datadict/bipbook/assets/12.4-05.ZYL_SpJX.jpg)

![](https://community.yonyou.com/datadict/bipbook/assets/12.4-06.CacgAH_U.jpg)

![](https://community.yonyou.com/datadict/bipbook/assets/12.4-07.B0BgfRZ8.jpg)

## 2.5 启动服务

选中高级版模块右键，选择 YonBIP高级版项目工具->启动YonBIP高级版中间件，等待模块编译完成启动服务

![](https://community.yonyou.com/datadict/bipbook/assets/12.5-01.CLyzTUYq.jpg)

## 3 前端环境搭建

## 3.1 脚手架

引入前端脚手架： [https://git.yyrd.com/nc-pub/ncpub-multipage-demo](https://git.yyrd.com/nc-pub/ncpub-multipage-demo)

![](https://community.yonyou.com/datadict/bipbook/assets/13.1-01.D6W9K66k.jpg)

![](https://community.yonyou.com/datadict/bipbook/assets/13.1-02.CtTqwmSt.jpg)

## 3.2 脚手架依赖

切换 npm 淘宝镜像（淘宝镜像下载相关依赖的速度更快）在 windows 系统下 win+R 输入 cmd 进入命令行终端，输入以下命令：

npm config set registry [https://registry.npm.taobao.org](https://registry.npm.taobao.org/)

![](https://community.yonyou.com/datadict/bipbook/assets/13.2-01.DMASF4Va.jpg)

下载前端项目依赖进入前端脚手架目录，在命令行终端输入 npm install 命令，下载前端项目依赖，第一次下载的时间可能较长，请耐心等待。下载完成后，前 端脚手架工程中会出现 node\_modules 文件夹。

![](https://community.yonyou.com/datadict/bipbook/assets/13.2-02.BRZgKktG.jpg)

## 3.3 开发环境代码依赖

更新前端开发环境将 NChome 下列文件夹复制到脚手架 src 目录下：（第4步配置的 uapbd 文件如果不生效，以NChome 下的 uapbd 文件夹为准）

`${nchome}\hotwebs\nccloud\resources` 下的 platform、lappreportrt、uap、uapbd

![](https://community.yonyou.com/datadict/bipbook/assets/13.3-01.pcs25TdQ.jpg)

## 3.4 config.json 配置文件

修改 config.json 配置文件buildEntryPath：配置启动时需要编译的路径，以./src 开头。proxy：配置代理 IP 和端口，与 Eclipse 开发配置的客户端连接配置信 息保持一致。

![](https://community.yonyou.com/datadict/bipbook/assets/13.4-01.C5OTlmpc.jpg)
