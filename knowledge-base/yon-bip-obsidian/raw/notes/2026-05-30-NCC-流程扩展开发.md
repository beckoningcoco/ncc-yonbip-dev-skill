# 流程扩展开发 | YonBIP高级版

## 流程扩展开发 [​](https://community.yonyou.com/datadict/bipbook/chapter3106/chapter3106.html#%E6%B5%81%E7%A8%8B%E6%89%A9%E5%B1%95%E5%BC%80%E5%8F%91)

## 1 扩展开发目的 [​](https://community.yonyou.com/datadict/bipbook/chapter3106/chapter3106.html#_1-%E6%89%A9%E5%B1%95%E5%BC%80%E5%8F%91%E7%9B%AE%E7%9A%84)

为满足行业化、本地化甚至是企业化个性应用，YonBIP高级版提供了一套开放的产业链分层研发框架，目标是通过二次开发与标准产品的分层隔离，达到降低二次开发难度、减少甚至避免修改标准产品源代码的主要目的。基于分层研发框架进行二次开发，可以解耦开发成果与标准产品的依赖关系，提升行业、项目等二次开发成果的复用性，同时，也会降低项目版本升级的难度和风险。

在不修改原始代码的情况下，我们可以通过流程扩展开发，实现单据某些扩展功能，来完成客户的个性化需求。

### 2.1 单据类型扩展开发 [​](https://community.yonyou.com/datadict/bipbook/chapter3106/chapter3106.html#_2-1-%E5%8D%95%E6%8D%AE%E7%B1%BB%E5%9E%8B%E6%89%A9%E5%B1%95%E5%BC%80%E5%8F%91)

目前主要通过单据类型管理中配置单据类型子表的方式来实现流程的扩展开发。对应数据库表【bd\_billtype2】

![image1.png](https://community.yonyou.com/datadict/bipbook/assets/image1.C3754Np1.png)

例如：我们想在单据脚本执行前后执行一些业务处理，此时可以通过注册扩展类型选择“单据动作脚本类执行监听类”，填入实现类路径代码，实现类需要实现对应的接口【nc.bs.pub.pf.IBillActionClassRunListener】 。

![image2.png](https://community.yonyou.com/datadict/bipbook/assets/image2.BAM23P5L.png)

代码实现对应的接口【nc.bs.pub.pf.IBillActionClassRunListener】 ，实现其中的脚本执行前事件或执行后事件即可。

java

```
public class GLBillActionClassRunHandler implements IBillActionClassRunListener {
    @Override
    public void beforeActionExecute(PfParameterVO pfParameterVO) throws BusinessException {
        //TODO 业务流程处理
    }
    @Override
    public void afterActionExecute(PfParameterVO pfParameterVO) throws BusinessException {
        //TODO 业务流程处理
    }
}
```

### 2.3 举例：交易类型扩展 [​](https://community.yonyou.com/datadict/bipbook/chapter3106/chapter3106.html#_2-3-%E4%B8%BE%E4%BE%8B-%E4%BA%A4%E6%98%93%E7%B1%BB%E5%9E%8B%E6%89%A9%E5%B1%95)

下面再以交易类型扩展开发为例，讲一下单据类型扩展开发的一些内容。图中展示的是收入合同的交易类型增加保存即提交和直接确认收入两个选项。

![image3.png](https://community.yonyou.com/datadict/bipbook/assets/image3.BIb8P4M_.png)

首先是元数据设计，添加对应内容。

![image4.png](https://community.yonyou.com/datadict/bipbook/assets/image4.CZSd3gzv.png)

收入合同的单据类型要勾选“是否可扩展交易类型” ，在下面单据类型子表要添加三个扩展类型： 【交易类型扩展属性编辑器URL】； 【交易类型扩展属性VO转换类】； 【交易类型业务扩展类】。

![image5.png](https://community.yonyou.com/datadict/bipbook/assets/image5.Cb3D7XQI.png)

前端页面按照正常的应用页面开发完成后，有以下几个地方需要修改： 需要在componentDidMount方法中获取相应数据，在setExtendPaneStat方法中获取当前页面状态，需要自己写一个showData方法来控制扩展页面状态及数据显示。 具体内容参见：交易类型扩展的课程内容

![image6.png](https://community.yonyou.com/datadict/bipbook/assets/image6.CfPLCPiI.png)

![image7.png](https://community.yonyou.com/datadict/bipbook/assets/image7.peCA3JMO.png)

交易类型扩展属性VO转换类：需要写在client端。 类中请添加方法：public Object onvertToExtendVO(String str, BilltypeVO transTypeVO)。该方法请将扩展属性的str转换为所需的VO类。

![image8.png](https://community.yonyou.com/datadict/bipbook/assets/image8.ASUfBsvU.png)

后端也要增加交易类型业务扩展类。实现ncccloud.pubitf.riart.transtype.InccTranstypeBiz接口，实现对扩展属性的修改删除更新的方法。至此收入合同的交易类型扩展就完成了。
