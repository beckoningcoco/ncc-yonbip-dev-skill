# 低代码 - 移动应用开发 | YonBIP高级版

## 低代码 - 移动应用开发

---

## 1 向导生成代码

**元数据设计**

做移动应用的元数据，跟 PC 端节点的元数据一样，没有任何区别。

低代码生成移动节点支持： **一主、一主一子、一主多子** 元数据。这里用一主二子的元数据做演示。

![](https://community.yonyou.com/datadict/bipbook/assets/001.DtmcAoaP.png)

**向导生成代码**

1. 用开发者身份用户登录系统。
2. 打开移动代码定制节点（动态建模平台/应用工厂/高阶定制/移动代码定制）

![](https://community.yonyou.com/datadict/bipbook/assets/002.JmGeiykw.png)

基础配置部分，跟低代码生成 PC 端节点的配置类似。应用分类选择移动应用分类，填写应用编码时，需要注意 **不能跟系统已有的应用编码重复。**

![](https://community.yonyou.com/datadict/bipbook/assets/003.BVEboItf.png)

页面及插件配置这里，移动代码定制目前只支持 **审批流插件** 和 **附件管理插件。**

![](https://community.yonyou.com/datadict/bipbook/assets/004.-MikHHZo.png)

选择审批流插件时填写必要的单据类型编码外，还需要填写 PC 端审批应用配置。生成移动节点时会自动生成对应的PC端审批应用，移动端提交的单据，可以到 PC 端节点去审批。

![](https://community.yonyou.com/datadict/bipbook/assets/005.5KmDICF_.png)

其余的生成代码配置，跟低代码生成 PC 端节点的配置一致的。如果是开发环境，可以直接填写前端代码和后端代码的下载路径，分别填写前端脚手架路径和后端工程路径。

![](https://community.yonyou.com/datadict/bipbook/assets/006.BbqnuHSl.png)

配置信息填写完后，点击下载源码按钮，开始下载代码。这里以下载代码压缩包 Zip 文件为例。

![](https://community.yonyou.com/datadict/bipbook/assets/007.DevHft3j.png)

## 2 本地环境调试

**代码路径**

下载的代码压缩包中，有两个文件夹。文件夹名称为 java 的，是后端代码。把它放到后端工程目录中。如下图所示，把 mobile\_test 文件夹放到后端工程目录中。文件夹名称为 ui 的，是前端代码，把它放到前端脚手架 src 目录下。如果是开发环境下，生成代码时直接填写前后端代码下载路径，代码会下载到开发工具目录中，无需手动移动代码。

![](https://community.yonyou.com/datadict/bipbook/assets/008.DW8vMDSy.png)

**启动项目**

后端代码添加到编译路径，启动项目。

![](https://community.yonyou.com/datadict/bipbook/assets/009.uWxAL6RM.png)

前端 config.json 中添加新节点代码编译路径，用 **npm run dev-m** 命令启动前端。

![](https://community.yonyou.com/datadict/bipbook/assets/010.JUv0E_yq.png)

**节点调试**

1. 前端启动后浏览器自动打开登录页面，输入账号密码登录。
2. 浏览器打开新的标签页，访问： `http://localhost:前端端口/mobile_platform/mobile_runtime/light-front/main/index.html?appcode=移动应用编码`
3. 直接在浏览器中调试即可。

![](https://community.yonyou.com/datadict/bipbook/assets/011.C1jy61jL.png)

**代码结构**

- 后端代码：结构跟 PC 端低代码生成的后端代码结构一致。
- 前端代码：结构跟 PC 端低代码生成的前端代码结构不同。

![](https://community.yonyou.com/datadict/bipbook/assets/012.DOWkuvws.png)

- public 目录：放节点公共代码。默认生成的只有一个 constant.js 文件，放节点中用到的常量。
- list 目录：列表页面的代码。
- card 目录：卡片页面的代码。

list 和 card目录中的代码结构是一致的：

- index.js：节点入口文件。
- config.json：配置文件，内容是固定的。
- init 目录：模板初始化相关代码。
- btnClick 目录：按钮事件。每个按钮事件都是单独的一个文件。

## 3 导出节点补丁

**后端补丁**

后端代码出补丁，没有特殊配置，直接出就行。

**前端补丁**

生成节点时勾选了审批流插件的情况下，前端代码会有两个目录，如下图所示。mobile\_ 开头的是移动端的代码，非 mobile\_ 开头的是 PC 端审批应用的代码。 **这两个文件夹需要分开出补丁，然后合并到一个补丁里。**

PC 端用 **npm run patch** 或 **npm run test** 命令出补丁；移动端用 **npm run test-m** 命令出补丁。

![](https://community.yonyou.com/datadict/bipbook/assets/013.DzeqWXMT.png)

**注意事项**

最后是注意事项。给已有的 PC 端节点，添加移动应用时，需要注意是否满足以下场景。本版本的移动代码定制， **不支持给标准节点补做移动应用** 。这一点一定要注意，别把标准产品节点给覆盖了。如果是低代码生成的节点，而且生成代码时，代码风格选择的是客开风格，可以用移动代码定制做移动应用，但是有客开的内容和脚本可能会被覆盖。所以，建议PC端和移动端同时生成，然后再做后面的客开内容。

| PC 端节点开发方式 | 支持增加移动应用 | 注意事项 |
| --- | --- | --- |
| 低代码生成的(客开风格) | **支持** | 己有客开代码时先备份好，避免覆盖。审批流相关的脚本会被覆盖。 |
| 低代码生成的(领域风格) | 不支持 |  |
| 产品标准节点 | 不支持 |  |
