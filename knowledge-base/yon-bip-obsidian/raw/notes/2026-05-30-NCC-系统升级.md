# 系统升级 | YonBIP高级版

## 系统升级

## 1安装盘配置文件说明

YonBIP高级版安装盘分为新安装盘和补丁盘两种，新装盘会在 home 下生成 ncscript 文件夹，而补丁盘则会在 home 下生成 update 文件夹，新盘和补丁盘结构大同小异。

### 1.1安装盘结构

安装盘包含一个到多个产品，每个产品表现形式上为一个目录， 目录的名称必须为产品的模块标识名称，每个产品下面又会包含一个到多个模块。

如下是安装盘目录结构：

![](https://community.yonyou.com/datadict/bipbook/assets/001.BOXtX5P1.png)

一个产品内包含一个到多个模块，产品的目录结构如图所示。

![](https://community.yonyou.com/datadict/bipbook/assets/002.BB2RS5dE.png)

对于每个产品来说，都有一个setup.ini的配置文件，其配置文件中的内容及格式如下所示：

```
ini#编码
code=10
#名称
name=智能客户端应用平台
#版本号
version=NC Cloud 2021.1120211122093306
#安装盘类型,取值为 new(新盘) 或 patch(补丁盘)
disk.type=new
#该模块是否必须安装 ,取值为true 或 false
must.selected=false
#依赖的前一个版本号,可以使用通配符*或?
preversion=
#依赖的其他产品,表示为[code,name, version1, version2,...]&[code,name, version1, version2,...]
required.related.module=
adjust.class0=
#数据升级的代码调整类
dataupdateclass1=
dataupdateclass2=
dataupdateclass3=
```

对于每个模块内一般会有若干个jar包，properties文件及配置文件setup.ini，如下图所示：

![](https://community.yonyou.com/datadict/bipbook/assets/004.DnjRq7ep.png)

## 2安装升级相关说明

### 2.1安装过程介绍

YonBIP高级版产品安装分为以下几个步骤：

1、运行客户化安装盘的安装程序，把代码及数据库脚本安装到指定目录作为系统运行环境。

2、以超级管理员的身份进入系统管理界面，创建系统，运行数据库安装脚本及初始化数据。

3、保存客户化版本信息和账套信息。

第一步安装代码实质就是代码解压过程，此过程无特殊之处。下面将对创建和升级系统，即数据库安装升级进行详细描述。

在YonBIP高级版产品，用户需要以超级管理员（root或super用户）的身份登陆系统，登录地址为

`http://IP:端口/nccloud/resources/uap/rbac/login/admin/index.html` ，在系统管理节点可以增加系统，也可以根据需要设置主语种、语种顺序。 ![](https://community.yonyou.com/datadict/bipbook/assets/006.WmTWz1Y2.png)

新增（升级）系统时，会读取ncscript（update）中脚本，构造产品树，如下所示： ![](https://community.yonyou.com/datadict/bipbook/assets/007.DEIYxONx.png)

如下图是补丁盘的升级界面，安装程序会比较 setup.ini 中 version 的属性值和数据库中记录的版本新旧，如果 setup.ini 中版本较新则该模块可以升级，其中 sm\_product\_version 记录的是安装盘版本，sm\_patch\_version 记录的是补丁盘版本信息。升级时会按照树结构以及 setup.ini 中的依赖关系来决定脚本的执行顺序。会先执行在树上靠前的模块脚本，如果该模块对其他模块有依赖，先执行被依赖的模块。 ![](https://community.yonyou.com/datadict/bipbook/assets/008.B4pOC2i5.png)

### 2.2建表

关于升级时建表：不管是新安装还是升级，数据库结构都可能会发生变化，但是不同的数据库类型的关键字是不同的，所以执行 create 或者 alter 前是需要经过数据库类型和关键字的映射关系转换的，而这个映射关系是在HOME\\ncscript\\产品\\模块\\研发层次\\dbcreate\\Dtype 下的各种数据类型 xml，如图所示。 ![](https://community.yonyou.com/datadict/bipbook/assets/009.Bp4NHy4B.png)

转换过的建库脚本和数据库中的数据结构进行比对生成差异文件 ncscript\\产品\\模块\\研发层级\\diff.xml，在这个过程中，对于数据库中不存在的表，将在 diff.xml 中插入相应的 create 语句；已经存在但结构发生变化了的表，生成对应的 alter 语句。如下图 ![](https://community.yonyou.com/datadict/bipbook/assets/010.CJ6I4o1c.png)

### 2.3插入预置脚本

关于升级时预置脚本：在安装盘里，初始化脚本都是 insert 语句，这样在执行升级操作的时候就会产生唯一键冲突的情况，高级版对此种情况有如下几种策略： 1、将 insert 语句转化为 update 语句，对数据进行更新 2、将 insert 语句转化为 update 语句，只对部分字段更新 3、忽略该语句，该表的预置数据将不会被执行 4、抛出违反唯一性约束异常

语句使用何种策略由业务人员根据业务需要制定，如果没有指定，默认按照抛异常的策略执行，策略注册到 **HOME\\ierp\\install\\sqlTransConf** 目录下，该目录下直接放 xml 文件，不要再放文件夹。 sqlTransConf\*.xml 文件中，table 标签表示对一个表策略的描述，name 属性为表名称，pk 为主键字段，operate 表示策略，值如下：

- 若无值使用策略1；
- 若无值且存在 ignorefield 标签，则使用策略2，ignorefield 中 name 属性表示不会被更新的字段名；
- 值为 ignore 表示使用策略3
- 值为 notranslate 表示使用策略4

其中sqlTransConf\*.xml示例文件如下所示：

```xml
xml<?xml version="1.0" encoding="UTF-8"?>
<sqltransconf>
    <tables>
        <!-- operate无值时使用策略1 -->
        <table name="bd_userdefruleref" pk="pk_userdefruleref" />

        <!-- operate无值且存在ignorefield标签，则使用策略2 -->
        <table name="bd_userdefitem" pk="pk_userdefitem">
            <ignorefield name="dataoriginflag" />
            <ignorefield name="classid"/>
        </table>

        <!-- operate值为ignore，使用策略3 -->
        <table name="bd_accperiodscheme" pk="pk_accperiodscheme" operate="ignore" />

        <!-- operate值为notranslate表示使用策略4 -->
        <table name="bd_exratescheme" pk="pk_exratescheme" operate="notranslate" />
    </tables>
</sqltransconf>
```

### 2.4代码调整

在很多时候，安装升级过程中只是执行脚本并不能完全解决问题，需要进行一些代码调整，在高级版中对以下几种情况做了相应处理：

1. 新装时代码调整：

需实现 nc.bs.sm.accountmanage.INewInstallAdjust 、nc.bs.sm.accountmanage.INewInstallAdjust2 接口，将升级类注册到setup.ini的adjust.class属性值。

nc.bs.sm.accountmanage.INewInstallAdjust ：

adjust(String newVersion) 在新安装时，所有其他操作完成后调用

nc.bs.sm.accountmanage.INewInstallAdjust2：

doBeforeUpdateDB(String newVersion) 在数据库表结构安装之前调用；

doBeforeUpdateData(String newVersion) 在数据库表结构安装之后，预置数据之前调用；

1. 升级时代码调整：

需实现 nc.bs.sm.accountmanage.IUpdateAccount 接口，该接口提供了3个方法，可以在不同时机进行代码调整。

doBeforeUpdateDB(String oldVersion,String newVersion) 在数据库表结构升级之前调用；

doBeforeUpdateData(String oldVersion,String newVersion) 在数据库表结构升级之后，数据初始化之前调用；

doAfterUpdateData(String oldVersion,String newVersion) 在做升级的最后阶段调用。

INewInstallAdjust 实现类注册到 setup.ini 的 adjust.class 中。对 nc.bs.sm.accountmanage.IUpdateAccount 接口的实现类可以注册到两个地方： 注册到 setup.ini 的 dataupdateclass 属性值，那么该实现类只会在升级本模块时调用 注册到 HOME\\ierp\\install\\au.xml 中，那么在升级每一个模块的时候都会调用。 setup.ini 中注册新安装和升级时代码调整 class 样例： ![](https://community.yonyou.com/datadict/bipbook/assets/012.BbbxpMkN.png)
