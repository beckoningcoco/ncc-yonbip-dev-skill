# 编码规则 | YonBIP高级版

## 编码规则

## 1 编码规则介绍

在YonBIP高级版中常常会有需要给新生成的单据或者档案填写一个编码的情况，通常这些编码需要具有易于识别且符合一定规范的特点。例如使用字母、数字或特定符号组合，使用前缀、日期、序列号等元素。有些可能还需要保证唯一，如果手动去填写容易出错，因此就需要一种由系统自动生成编码的机制。

在YonBIP高级版系统中，提供了编码规则。用户可以按照需要，为单据类型定义编码规则，然后在生成单据号时，按照定义的编码规则来自动生成编码。 单据编码通常由固定的字符串、分隔符、单据相关的数据项的值、时间和流水号构成，在编码规则中用四种编码元素来描述上述的编码项：

1. **固定值：** 表示固定字符串和分隔符；
2. \*\*编码实体：\*\*表示与单据相关的数据项的值（客商、人员等），而这些值通常不是简单的使用其编码名称等，需要通过编码映射功能，将这些值映射为最终展现到编码段中的值。
3. **时间项：** 表示时间属性，包含系统时间（取值为服务器时间）和业务时间（取值为单据上的相关时间项，如制单日期等）；
4. **流水号：** 顺序记录单据编号的定长号。

## 2 编码注册

编码注册涉及到的节点包括：编码对象注册、编码映射、编码规则定义。

- 编码对象注册：可以设定编码的基本信息，例如：编码对象编码、编码对象名称、编码的长度、关联元数据等。
- 编码映射：可以实现单据编码中参与编码的实体值转换为编码中出现的编码值的映射转换关系。
- 编码规则定义：可以设定基本的编码规则，包括“编码名称”、“编码方式”、“编码是否可编辑”、“是否产生随机码”、“编码由哪些元素组成”等。

![](https://community.yonyou.com/datadict/bipbook/assets/001.KHt-4So5.png)

1. \*\*编码对象编码：\*\*该对象的唯一标识，在代码中作为参数来识别编码对象；
2. \*\*编码对象范围：\*\*定义了该编码对象上定义的编码规则使用的范围，共有三个参数值 ①集团：编码规则只能定义到集团级； ②全局：只能定义全局级的编码规则； ③皆可：该对象即可定义全局级编码规则，也可定义集团级编码规则；
3. \*\*编码方式：\*\*用来限制为该编码对象定义编码规则时编码方式的选择范围，包括 前编码，后编码和都支持（关于编码方式的含义可以参考后面的描述）；
4. \*\*编码长度：\*\*用来限制为该编码对象定义编码规则时最大的编码长度；
5. **关联组织类型：** 编码规则定义后可对组织进行分配，该参数指定了可供分配组织的组织类型；
6. \*\*编码对象元数据：\*\*指定编码对象关联的元数据；
7. \*\*候选属性：\*\*编码规则中可使用的属性，包括实体和枚举；

编码映射功能用于为每一个可用于编码规则的业务实体的值进行重新映射。在生成编码规则时，按照映射后的值，作为编码的组成部分。编码映射支持“按照规则设置”或者“离散选择设置”，同时可以通过该节点查询实体在编码规则中的引用情况。 左侧树上的节点，就是可供编码规则定义使用的业务实体。在编码对象和编码实体关联表中注册了编码对象需要使用的编码实体，为编码实体定义映射规则有几个关键属性：

- 映射值长度：指编码实体在编码规则中所占的位数；定义编码映射值使用一段时间之后，如发现映射值长度不够，可以修改映射值长度；
- 补位方式：包括右补位、左补位、不补位，默认为右补位。
	该处选择右补位、左补位时，指的是当根据默认规则或者离散值得到的映射值小于“映射值长度”设置的长度时，采用的补位方式；
	该处选择不补位时，如果某一个映射值的长度达不到指定长度时，就采取不补位的形式；
- 补位符号：用什么占位符来为映射值补位，默认为“@”；

“编码规则定义”节点定义编码的最基本的功能规则，一个完整的编码规则包含了基本信息和编码元素各部分。 ![](https://community.yonyou.com/datadict/bipbook/assets/003.Bj2LfNVB.png)

编码规则的基本信息包含如下几项：

- **规则编码、规则名称** ：左侧树上显示的是规则的名称；
- **编码方式** ：前编码方式即在用户新增单据时即生成编码，此时新增单据很多相关项还没有赋值，所以前编码方式不能支持在编码中出现业务实体和业务时间；后编码方式是在用户新增单据后，保存时生成单据编码，没有什么特殊限制；
- **编码是否可编辑** ：对于前编码方式可用，控制单据编码的可编辑性；
- **是否断码补码** ：如果删除一个单据，或者申请一个单据出错，那么它们所使用的单据号是否要再被使用。编码的可编辑性和断码补码是互斥的。
- **按组织归零** ：流水号是否随着组织的改变而重新计数。
- **是否集团默认规则** ：集团默认使用的规则，如果组织没有分配编码规则那么就使用这个默认规则，一个集团只能有一个默认规则。

## 3 应用场景及接口说明

**1、后编码新增保存：**

获取编码规则上下文； 如果编码规则上下文不为空且为后编码，则调用接口获取后编码，并设置到 VO 对象中。 保存 VO 数据。

![](https://community.yonyou.com/datadict/bipbook/assets/004.D6MpH1ua.jpg)

**2、后编码修改：**

获取编码规则上下文； 根据编码规则上下文获取编码可编辑性，并根据可编辑性设置界面组件是否可修改。 ![](https://community.yonyou.com/datadict/bipbook/assets/005.CxG-fkIy.jpg)

**3、前编码新增：** 获取编码规则上下文； 从编码规则上下文中获取是否为前编码，如果是前编码则取号，并根据编码规则是否可编辑设置界面上编码字段的值及可修改属性； ![](https://community.yonyou.com/datadict/bipbook/assets/006.BMt_qnGh.jpg)

**4、前编码新增保存：**

保存 VO 数据；根据编码规则上下文如果编码方式为前编码，则调用接口进行前编码的提交，即 commitPreBillCode(String nbcrcode, String pk\_group, String pk\_org, String billcode); 如果提交前编码过程中发生重号异常，则调用弃号相关接口，即 AbandonBillCode\_RequiresNew (nbcrcode, pk\_group, orgPre, vo.getCode()); ![](https://community.yonyou.com/datadict/bipbook/assets/007.kLTpW-Pz.jpg)

**5、前编码新增取消：** 如果编码方式为前编码，新增取消时回退已获取的编码值； 前编码退号：rollbackPreBillCode(String nbcrcode, String pk\_group, String pk\_org, String billcode)

**6、前编码修改：** 步骤1、获取编码规则上下文； 步骤2、根据编码规则上下文设置界面上编码字段的可编辑性；

本小节关于前编码、后编码等相关接口介绍都是 `nc.pub.billcode.itf.IBillcodeManage` 编码管理服务接口的介绍。 批量获取，单据基于元数据模型：getBatchBillCodes\_RequiresNew(String nbcrcode, String pk\_group, String pk\_org, Object billvo,int num); 批量获取，单据基于非元数据模型：getBatchBillCodesNoMeta\_RequiresNew(String nbcrcode, String pk\_group, String pk\_org,EntityValueVO entivalue, int num)

非批量，单据基于元数据模型 getBillCode\_RequiresNew(String nbcrcode, String pk\_group, String pk\_org, Object billvo); 非批量，单据基于非元数据模型：getBillCodeNoMeta\_RequiresNew(String nbcrcode, String pk\_group, String pk\_org,EntityValueVO entivalue)

获取前编码接口： getPreBillCode\_RequiresNew(String nbcrcode, String pk\_group, String pk\_org)

保存提交前编码： commitPreBillCode(String nbcrcode, String pk\_group, String pk\_org, String billcode)

取消回滚前编码： rollbackPreBillCode(String billTypeCode, String pk\_group, String pk\_org, String billcode)

获取编码规则上下文，可以用来判断前后编码、是否可编辑等： getBillCodeContext(String nbcrcode, String pk\_group, String pk\_org)

删除时回退编码，基于元数据模型： returnBillCodeOnDelete(String nbcrcode, String pk\_group, String pk\_org, String billCode, Object billvo)

删除时回退编码，基于非元数据模型： returnBillCodeOnDeleteNoMeta(String nbcrcode, String pk\_group, String pk\_org, String billCode, EntityValueVO entivalue)

编码重复时，弃用编码接口： AbandonBillCode\_RequiresNew(String nbcrcode, String pk\_group, String pk\_org, String billcode)

对于编码规则可以使用工具类 nc.impl.pubapp.bill.billcode.BillCodeUtils

void checkUnique ( IBill \[\] bills) ： 检查单据号是否重复

void createBillCode(IBill\[\] bills)：获得单据的后编码单据号

void createPreBillCode(IBill\[\] bills)：获得单据的前编码单据号

void returnBillCode(IBill\[\] bills)：回退单据号

void rollbackPreBillCode(IBill\[\] bills)：如果编码方式为前编码，则单据取消时调用该方法回退已获取的单据号

void upadteBillCode(IBill\[\] bills, IBill\[\] originbills)：修改单据时，若相应的字段发生变化，则需要重新获取单据号
