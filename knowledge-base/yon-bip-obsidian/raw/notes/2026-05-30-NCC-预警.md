# 预警 | YonBIP高级版

## 预警

---

## 1.预警介绍

通过预警平台统一管理按一定条件触发的业务预警。业务预警功能就是通过当前系统运行的指标、数据与各类预设定条件预算的比较，当达到条件后自动向相关人来发出警告性的提示信息，用于提示企业的管理者针对将来或已经发生的情况采取相应的措施，及时进行处理。

预警服务的整体示意图如下图所示：

![](https://community.yonyou.com/datadict/bipbook/assets/1.BttlXVHE.png)

由上图可知，预警平台主要由预警类型、预警条目以及预警消息三个组成部分。

\*\*预警类型：\*\*预警的一个插件类型。它的目的是对某个业务或操作的抽象，其可以定义一系列的阈值

\*\*预警条目：\*\*是具体的预警任务，是预警平台调度执行的单位。一个预警类型可以根据不同的业务情况定义多个预警条目。支持用户基于元数据根据业务需要定义预警条件和内容

于是想要使用预警平台的预警功能，开发步骤如下：

1. 打开 动态建模平台 -> 开发配置 -> 预警类型注册 节点，进行预警类型注册
2. 打开 动态建模平台 -> 客户化配置 -> 预警平台 -> 预警条目配置 节点，添加对应的预警条目
3. 打开 动态建模平台 -> 开发配置 -> 消息管理 -> 消息模板类型注册 节点，注册对应的消息类型
4. 打开 动态建模平台 -> 客户化配置 -> 统一沟通平台 -> 消息模板-全局 节点，新增消息模板
5. 预警服务执行预警条目产生预警消息
6. 预警消息通过消息、邮件、短信等方式发送，通知相关人员

## 2.预警类型注册

预警有自定义预警和预置预警类型，预置预警需要开发业务插件，一般是标准产品或客开人员预置进的预警类型，预警插件开发可以参考本章的插件开发指南。自定义预警一般由实施人员根据需要再去定义一些预警，不需要开发预警插件，但是需要先设计预警模型，有了预警模型我们才能在自定义预警选择设计的预警模型。本小节我们将先介绍预警模型定义，然后介绍自定义预警类型，最后介绍预置预警类型。

## 2.1预警模型定义

![](https://community.yonyou.com/datadict/bipbook/assets/3.D8DjEqD3.png)

预警模型定义节点主要是用来定义产生预警的预警对象和产生预警的条件信息。预警模型定义一般定义一条业务预警首先需要确定这几项内容：产生预警的预警对象、产生预警的条件、产生预警后需要通知哪些人、发送预警消息时应用的消息模板等信息。预警模型定义左侧树可以添加预警模型分类和设置分类的上下级关系，预警模型分类是根据业务需要把预警模型进行分类管理，方便以后应用。如果没有分类可以将鼠标移动到左侧“预警模型分类”上点加号按钮，添加一个分类。

![](https://community.yonyou.com/datadict/bipbook/assets/5.BnKijuX4.png)

名称、编码可以根据需要填写即可，所属模块可以根据该预警模型所适用的模块来划分，也可以根据该预警模型中引用的业务实体来划分。上级分类用来定义分类的上下级关系。

新增预警模型时，有一项是“类型”，下拉选择简单模式或高级模式，这是根据预警模型的复杂程度不同：

- 简单模式，该种模式应用比较简单，如果组成该预警的业务实体只有一个，则可以应用该简单模式来定义；
- 高级模式，如果触发预警的条件需要多个业务实体，且需要这几个实体做关联等比较复杂的情况下可以应用高级模型来定义。

### 2.1.1简单模式设计

第一步：选择产生预警的预警对象实体编码：选择触发预警时所需要用到的预警对象。

![](https://community.yonyou.com/datadict/bipbook/assets/6.DJY632JU.png)

第二步：选择构成预警条目的阈值条件；

![](https://community.yonyou.com/datadict/bipbook/assets/7.XBUlPIqJ.png)

属性说明：

- 从表新增：从第一步中选择的业务实体中的属性作为阈值条件；
- 自定义属性：性利用函数编辑器，通过函数计算来定义阈值条件；
- 修改：修改选中的阈值条件，可以修改阈值的名称、描述、值类型；
- 删除：删除选中的阈值条件；
- 编码：阈值条件的编码，如果是自定义阈值条件在点击自定义属性后可以设置编码；
- 名称：阈值条件的显示名称；
- 描述：对阈值条件的解释说明；
- 值类型：如果该条件是选择的实体属性，值类型会自动带出；如果是应用函数编辑器自定义的条件，用来约束设置预警时输入的阈值范围。

第三步：设置一些过滤数据的固定条件，有些条件是作为固定的筛选数据的条件，不会根据情况的不同而变化，这时候就可以把这些条件设置为筛选条件，不需要定义预警条目时再去定义值。

![](https://community.yonyou.com/datadict/bipbook/assets/8.DOYhUolU.png)

典型模式：主要是简单的利用实体中的字段来设置筛选条件

项目说明：

- 逻辑符：用来定义条件与条件之间的关系，如果需要同时满足的条件，则需要设置为“并且”，如果条件不需要同时都满足则需要设置为“或者”；
- 字段：参与比较的字段，可以通过新增按钮新增实体属性或者自定义属性；
- 操作符：表示字段与值之间的关系，包含等于、大于、小于、大于等于、小于等于、不等、like、in；
- 值类型：用来定义该字段的类型，包含常量、字段、表达式、参数、宏变量；
- 值：需要比较的值，根据前面值类型进行选择或者手动输入。

![](https://community.yonyou.com/datadict/bipbook/assets/9.Bj1hCTaF.png)

高级模式：需要利用字段表达式计算作为筛选条件

项目说明：

- 逻辑符：用来定义条件与条件之间的关系，与典型模式逻辑符功能一样
- 筛选条件表达式：利用实体中的字段，通过函数编辑器定义字段与字段之间的计算公式。

第四步：选择字段对实体对象进行排序。可以设置多个字段，排序顺序可以是升序或者是降序。

![](https://community.yonyou.com/datadict/bipbook/assets/10.BjIcvh_G.png)

### 2.1.2高级设计模式

第一步：选择表，产生预警的关联实体。可以利用数据字典选择库里的数据表，也可以选择查询对象，从而实现查询条件的组合。

- 元数据：选择系统中存在的单据或档案；
- 元定义：可以选择数据字典、也可以选择语义模型中定义的语义模型来作为查询的对象。
- 语义脚本：支持自由 SQL 的方式，由于复杂 SQL 在解析、适配、执行等方面的不稳定性较大，因此不推荐普通用户使用。语义脚本包括 SQL 录入和字段显示名设置两个环节。
- 数据加工：指某个（或某几个）结果集在经过一系列加工算法的变换后得到另一个结果集，而后者通常是 业务上要求的展现结果。相关的算法又称为加工滤镜，滤镜通常支持输入多个结果集和输出一个结果集， 并可以嵌套使用。

第二步：连接条件，建立表与表之间的连接条件，支持典型和高级两种模式。

第三步：选择字段，在该处选择的字段会出现在关联的消息模板类型中，可以应用这些字段来配置消息的 内容和标题。

第四步：选择阈值条件；

第五步：可以设置筛选条件

## 2.3预置预警类型

定义一个预警类型需要提供：名称、所属模块、业务插件、描述、阈值名称、编辑类型、参照名称(如果编辑类型为参照)、默认值。预警类型注册右侧界面显示预警类型详细信息，主要包含“常规属性”和“阈值条件”两个页签。它目的是对某个业务或操作的抽象，其可以定义一系列阈值。(这里也只是定义，真正的值还是由条目来设置的)。

![](https://community.yonyou.com/datadict/bipbook/assets/2.CBiviH49.png)

**常规属性项目说明：**

- 编码：输入预警类型的编码。必填项。
- 名称：输入预警类型的名称。必填项。
- 所属模块：是区分各产品组模块的标识，即通常说的模块名。这里的模块是指开发模块，对应的是中间件/modules目录下的子目录。按规定模块名都是小写的。必填项。
- 消息模板类型：使用消息模板时需填。一个预警类型对应一个消息模板类型。
- 所属行业：该预警类型适用于哪个行业，参照行业目录。
- 业务插件：输入业务插件的类名，要求实现了 nc.bs.pub.pa.IPreAlertPlugin 接口的实现类的完整类名。此项不能为空。必填项。
- 最小间隔：如果预警定时周期执行时的最小时间间隔，填正整数。
- 时间单位：最小间隔的时间单位，下拉选择小时或分钟。
- 描述：指对预警类型的文字描述。
- 组织类型：用于限定阈值的参照取值范围，这样在该类型预警条目新增时，通过选择相应的组织，来缩小参照类型阈值的取值。如：A预警类型在注册时，指定了其组织类型为公司，则在注册A类型的预警条目a时，其组织单元参照的取值为公司参照，当指定组织单元数据，如公司1时，其相关的预警取值，均为公司1下的相关数据。
- 组织可多选：该设置决定预警条目配置节点组织单元是否允许多选
- 组织是否可空：该设置决定预警条目配置节点组织单元是否允许为空
- 组织类型：用于限定阈值的参照取值范围，这样在该类型预警条目新增时，通过选择相应的组织，来缩小参照类型阈值的取值。如：A预警类型在注册时，指定了其组织类型为公司，则在注册A类型的预警条目a时，其组织单元参照的取值为公司参照，当指定组织单元数据，如公司1时，其相关的预警取值，均为公司1下的相关数据。

**阈值条件项目说明：**

- 阈值名称：也就是条件的名称。
- 阈值描述：对阈值名称的说明。
- 编辑类型：此处定义输入阈值的样式，系统提供五个选择：字符型、逻辑型、整型、Double型和参照基础档案类型。如果将某个阈值的编辑类型定义为逻辑型，那么在进行预警条目设置时，此阈值的值以下拉框的形式出现，有是和否两个选择；如果选择编辑类型为基础档案，还要在后面的参照名称栏中选择参照哪个基础档案，这样当用户输入该阈值时会弹出相应的基础档案参照。
- 是否非空：该阈值是否允许为空。
- 参照名称：如果编辑类型选择为参照基础档案，那么参数名称变为可选项，提供选择的项有人员档案、部门档案、客户档案等等。
- 是否单选：此选项针对参照而言，表示该参照是否支持多选。
- 默认值：该阈值的默认值。

## 3预警条目注册

## 3.1预警条目简介

预警条目一般由实施人员或用户根据具体的业务环境和需要来定义。通过选择预警类型，并设置该预警类型中需要用户设置的阈值变量，以及定义预警方式来实现具体的预警任务。预警方式包括产生方式和发送方式。产生方式控制预警的发生时间，有即时和定时两种。发送方式是定义预警以何种方式发送给用户。

预警条目是具体的预警任务，是预警平台调度执行的单位。一个预警类型可以根据不同的业务情况定义多个预警条目。

预警条目保存预警信息的产生条件及发送方式设置，基于预警类型进行编辑。预警平台的后台服务线程定时读取预警条目信息，根据条目中设置的条件，调用相应的预警类型与业务信息相比较，当符合预警条件时，就会产生预警信息，并根据相应配置进行信息发送。

条目与类型的关系其实就是一个具体与抽象的关系。打开 动态建模平台->客户化配置->预警平台->预置条目配置-集团 节点，单击增加，可以增加一个预警条目。选中一条已注册的预警条目，单击“删除”、“修改”，可以对已有预警条目进行删除或修改的操作。

![](https://community.yonyou.com/datadict/bipbook/assets/12.zKFGLGky.png)

预警条目配置主要功能：

- 新增：弹出预警条目设置的界面，共分为四个页签，对相应页签的信息进行设置，所有的页签定义好了以后，点确定按钮，回到主界面，新增的预警条件将会显示在主界面的列表中。
- 修改：修改当前选中的预警条目的配置数据
- 复制：复制当前预警条目数据
- 删除：删除当前预警条目数据
- 联查日志：在主界面中选中一个预警条目，弹出“预警日志”的窗口，可以查询相应的预警日志。
- 激活：使该预警条目生效，预警系统会产生预警信息。
- 休眠：休眠表示不产生预警信息。（休眠的预警在业务预警监控节点不显示）

## 3.2新增/编辑预警条目

（1）预警定义页签 ![](https://community.yonyou.com/datadict/bipbook/assets/13.D_P0JOkk.png)

- 预警名称：即该条目的名称，一个显示的标记。同公司同类型不允许条目名相同。
- 预警消息文件名：用于存储预警提示信息的文件，文件名称只能包含英文字母、数字或下划线，否则有可能无法通过浏览器正常显示预警信息。
- 条目状态：是否启用当前定义的预警条目，激活表示预警系统产生预警信息，休眠表示不产生预警信息。默认为激活态。
- 预警信息：可以选择已配置的消息模板，用于对消息文件内容的定义，如何配置消息模板可以参考业务消息章节中的介绍。

（2）阈值设置页签

![](https://community.yonyou.com/datadict/bipbook/assets/14.IvO18jcT.png)

- 类型：即预警类型注册的预警类型。其以下拉框的形式显示，此处对预警类型的选择将决定此预警条目将调用的业务插件。阈值设置中显示的预警类型、业务插件、描述、条件，将根据选择的“预警类型”自动带入。
- 组织单元：限定阈值参照的范围。组织单元参照的数组，由注册的预警类型决定。
- 阈值列表：在此处编辑预警条目的阈值。这里的阈值是从类型定义中带过来的，这里要做的只是设置操作符和阈值设置。

（3）触发配置页签

![](https://community.yonyou.com/datadict/bipbook/assets/15.BwuPNmMA.png)

触发策略：分为立即执行与定时执行，两个只能选择其中一个。如果选择立即执行，则系统会根据定义的触发方式来触发业务操作，并依据条件满足与否，来产生预警提示信息。若使用立即执行产生方式，预警方式需要 **至少** 选择系统登录、触发点提示、应用按钮中的一种，用来触发预警条目进行预警业务校验。

- 系统登录：用户登陆时触发
- 触发点提示：设置的节点被打开时触发
- 应用按钮：按钮点击时触发

如果选择定时产生，则预警平台会在设定的时间配置到来时进行预警检查，并产生预警信息。两者产生的预警消息的如何接收都由消息接收者配置面板来配置。

点击定时按钮时会弹出“编辑发生时间”的对话框，如下所示：

![](https://community.yonyou.com/datadict/bipbook/assets/17.BFMqv8Oy.png)

- 间隔单位：包含天、周，月三个时间单位，以及关于时间单位的间隔数。当为周或月时，还能够选择对应的间隔频率。周可以选择周一到周日，月除了可以选择1号到31号，还能选择“最后一天”。
- 触发时间：因为不管频率制定的如何，具体到还是某一天中。这里就是具体设置在预警需要触发时是触发一次还是周期触发，如果是发生一次则需要设置触发的时间，如果是周期发生需要设置“周期发生”的时间间隔以及周期起止时间。
- 有效期：这是优先级最高的设置，即前面设置的间隔单位、触发时间点必须要在此有效期内。系统会在设定的时间点进行业务检查，触发并合适地产生预警信息。

（4）消息接收配置页签

![](https://community.yonyou.com/datadict/bipbook/assets/16.CaYYAmQb.png)

消息接收者配置：预警信息产生时，预警平台通知用户的方式，系统提供了 3 种消息接收方式：消息中心、电子邮件、手机短信，可以设置接收人即参与者类型。

- 参与者类型：下拉选择，有用户、角色、用户组、角色组、人员、职责、岗位7个选项。
- 编码/名称：根据参与者类型选择不同的接收者。
- 消息中心：当一个预警条目触发时是否给选择的参与者类型对应的用户送一条预警消息到消息中心。
- 电子邮件：将预警信息发送到用户的电子邮箱中。添加电子邮件地址时，可以通过人员档案参照来选择（需要在人员档案中设置电子邮件地址），也可以手动添加电子邮件地址。电子邮件要发送成功，必须正确配置邮件服务器。
- 手机短信：有属于某个用户的预警信息产生时，通过手机短信提示用户登录系统查看。使用该功能时，需要企业客户向无线通信公司提出短信服务开户申请，并在人员档案中维护手机号，并在用户管理中设置用户和业务员的关联。

## 3.3消息模板

消息模板：可以按照不同的领域模块定制消息模板，在模板中配置发送消息的内容和格式。

添加一个消息模板，按如下步骤进行：

1. 打开消息模板类型注册节点，注册消息模版类型，关联实体是为了在消息模版中使用元数据字段，另外可以在下面的子表中添加自定义变量
	![](https://community.yonyou.com/datadict/bipbook/assets/20._UIsQ-ou.png)
2. 打开消息模板-全局节点，进入消息模板的新增界面，输入模板名称、模板描述、消息标题、消息内容，点击“保存”按钮。
	![](https://community.yonyou.com/datadict/bipbook/assets/21.DsrrXwN2.png)
- 选择模板语种：模板支持的语种跟系统安装的语种类型有关，这里系统只安装了简体中文、繁体中文和英语三个语种，模板也支持简体中文、繁体中文和英语三种；
- 模板编码：当首次新增某一个消息模板时，模板编码需要手工输入；同一个消息模板的不同语种的模板编码是相同的；当已存在一个语种的模板，新增另一个语种的模板时，模板编码不允许编辑。
- 消息内容：消息内容中可以引用变量。例如：“您好，人员编码@&m code@生日即将到来”；其中@......@即是所引用的变量。具体操作是从右边的系统变量页签、元数据页签或者业务函数页签中选择一个变量，双击该变量，则相应的变量@&m code@就作为消息内容的一部分。注意：消息模板所属的模板分类不同，可以应用的变量也可能不同，有些模板可以应用系统变量、元数据设置消息内容，有些可以应用系统变量、业务函数设置消息内容，同样高级设置中应用到的是元数据还是业务函数都与消息模板分类有关

## 3.4预警监控和预警日志

业务预警监控是对预警平台配置的所有预警条目进行监督控制，只有定时任务且被激活的预警需要监督控制；如果打开【业务预警监控】节点列表没有显示预警条目可以光标定位到“任务名称”搜索框并回车或者鼠标点击刷新按钮。这里关于业务预警监控说明几点：

1、一个任务分为等待运行、正在运行、暂停等几个状态，业务预警监控中心可以临时启动一个等待调度的任务。

2、任务列表中的任务，如果处于等待状态，可点击“执行”按钮，开始执行任务。任务被执行后本条任务的运行次数+1，上次耗时记录上次执行该任务使用的时间。

3\*\*、\*\*在任务界面中选择欲删除的条目，点击“删除”按钮，系统弹出确认提示框，如果选择是，那么该任务被删除。 对于处于等待和可执行状态的任务可以删除该任务实体。被删除以后，对应的预警条目不会再被执行，这种情况下，虽然在预警条目配置界面还能看到该条目，但实际上它已经不发挥作用了。

## 4.2预警平台业务插件接口

定义预警类型时必须提供做业务检查的业务插件，由开发人员编写。该业务插件必须实现预警业务插件接口nc.bs.pub.pa.IPreAlertPlugin。预警服务运行时，根据定义的预警条目执行业务插件的适当业务，并将产生的预警信息写入预警文件，进行企业业务预警。

```java
javapublic interface IPreAlertPlugin {

   /**
     * 预警插件执行体
     *
     * @param context 执行环境
     *
     * @return PreAlertObject 该返回值不允许为null
     * 若预警插件成功执行但不打算发送消息，
     * 那么请将PreAlertObject的returnType设为PreAlertReturnType.RETURNNOTHING
     * 或者直接返回一个nc.bs.pub.pa.NullPreAlertObject的实例
     * @throws BusinessException
     */
    PreAlertObject executeTask(PreAlertContext context) throws BusinessException;

}
```

关于返回的PreAlertObject对象设置返回值的步骤：

1. 调用setTitle()方法设置预警/后台任务消息标题，注意：如果采用消息模板方式，可以不用设置消息标题
2. 调用setReturnType()方法设置返回值类型
3. 调用setReturnObj()方法设置返回值
4. 若在消息模板分类中为该预警/后台任务类型关联了业务实体，请调用setBusiObj()方法设置业务实体对象
	若在消息模板分类中为该预警/后台任务类型中设置了自定义变量，请调用setBusiCalculater()方法设置变量对应的值

下面表格是关于returnType取值及说明，其中“取值”列均为枚举nc.bs.pub.pa.PreAlertReturnType的枚举值。

| **取值** | **说明** |
| --- | --- |
| RETURNNOTHING | 无消息返回 |
| RETURNOBJECT | 对象类型 |
| RETURNFORMATMSG | 格式化消息，可转化成打印模板数据源实现接口 IAlertMessage |
| RETURNMESSAGE | 返回固定字符串类型 |
| RETURNDATASOURCE | 打印模板数据源，只展现自定义变量实现IDataSource接口，直接使用元数据实体上的字段或元数据实体字段与自定义变量同时存在时实现接口IMetaDataDataSource；实现 IDataSource.getItemValuesByExpress 方法获取自定义变量的值，实现 IMetaDataDataSource.getMDObjects 得到元数据实体 |
| RETURNMULTILANGTEXT | 返回多语文本 |
| RETURN\_MULTIPLE\_RECEIVER\_OBJECT | 返回一个MultiReceiverPreAlertObject 对象，用于实现多接收者接收不同的消息的场景 |
