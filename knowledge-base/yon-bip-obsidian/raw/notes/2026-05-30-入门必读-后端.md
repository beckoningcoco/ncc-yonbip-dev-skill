## 后端脚本与API脚本

## 后端脚本与API脚本区别

后端脚本应用场景为：规则链、调度任务；

API脚本应用场景为：前端脚本调用api脚本，提供前端调后端能力；发布为对外接口，供三方系统调用

常见问题：

前端脚本需要调用后端函数时，页面操作选择不到指定的后端函数。

原因：可能是创建成了后端脚本，后端脚本不具备前端调用后端的能力，所以系统给过滤了。创建的脚本类型应为API脚本。

## 后端脚本举例

后端脚本中包含param传参，可以通过param取出规则间传递的参数， 取出后可以通过参数的set方法重新赋值。该能力常用于规则链中用来设置默认值， 调整参数值等场景。 如：更改保存数据的值。

```javascript
let AbstractTrigger = require('AbstractTrigger');
  class MyTrigger extends AbstractTrigger {
    execute(context,param){
      var email = param.data[0].email;  //一般从param中取规则链中传递的值
      param.data[0].set('new1','22'+""); //可以回写修改new1的值,这里的值要加个空引号
     return {};
    }
  }
  exports({"entryPoint":MyTrigger});
```

## 前端函数调用API函数

## 应用场景

- 简单的前端调后端，实现增删查改操作

前端脚本调用后端脚本时，在没有通过ObjectStore相关方法进行新增、更新、删除的情况下，都要加{"domainKey":"developplatform"} 参数，将流量转移到标准服务，减少领域服务压力 例如:

```javascript
cb.rest.invokeFunction("AMPUB.backDefaultGroup.ceshi001", {"key":value},
      function(err, res) {
           alert(JSON.stringify(res));
      }
  ,viewModel,{"domainKey":"developplatform"})
```
