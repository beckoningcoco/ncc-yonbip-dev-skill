## 原厂页面跳转

### 原厂跳转自建单据

```
let data = {
        //TODO：填写单据类型及单据号
        billtype: 'Voucher',// 单据类型
        billno: 'text11',// 单据号
        domainKey: 'developplatform', // 领域编码；自建单据为'developplatform'
        params: {
          mode: 'add', // (编辑态edit、新增态add、浏览态browse)
          id:'add'  //(编辑或浏览态，需要添加目标的单据id)
        },
busiObj:""; // YPD框架需要添加此属性，值为目标跳转单据业务对象编码
  };
  //打开一个单据，并在当前页面显示
  cb.loader.runCommandLine('bill', data, viewModel);
```

## 原厂单据更改自建单据数据

原厂单据扩展中，后端函数不能以实体操作的方式直接对自建单据中得数据进行增、删、改，需要在自建应用中创建并发布api函数（提供对自建应用中数据的增、删、改操作），然后在原厂扩展应用中调用自建应用发布的api，以实现此能力。

自建应用中创建并发布api

参考文档： [【API发布】](https://c2.yonyoucloud.com/iuap-hc-client/ucf-wh/client/index.html#/detail/APIFB12?nodeId=c6f56484-8817-4b87-abd2-eb630bd48e95&productline=CommunityDoc&type=enableDoc "【API发布】")

原厂扩展后端函数调用api

```
let AbstractTrigger = require('AbstractTrigger');
  class MyTrigger extends AbstractTrigger {
    execute(context,param){
      let body = {};
      let url = "https://**************";
      //SACT 为当前应用编码
      let apiResponse = openLinker("POST", url, "SACT", JSON.stringify(body));
      //throw new Error(JSON.stringify(apiResponse));
      return {};
    }
  }
  exports({"entryPoint":MyTrigger});
```
