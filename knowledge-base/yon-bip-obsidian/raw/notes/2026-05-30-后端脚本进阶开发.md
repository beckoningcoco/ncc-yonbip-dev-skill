## 后端函数调用BIP开放平台接口

### 概述

后端函数通过openLinker系统预制函数，根据传参应用编码自动匹配查找应用内部的appkey，appsecrect等信息获取token。使用方不需要关注token生成方式，只需要关注接口请求地址、请求方法（get/post）、请求参数。

### openLinker方法

1、POST方法示例；

```
let AbstractTrigger = require('AbstractTrigger');
  class MyTrigger extends AbstractTrigger {
    execute(context,param){
       let url = "https://dbox.yonyoucloud.com/iuap-api-gateway/yonbip/digitalModel/orgunit/querytree";
       let body = {};  //请求参数
       let apiResponse = openLinker("POST", url, "应用编码",JSON.stringify(body));  //TODO：注意填写应用编码(请看注意事项)
     return {apiResponse};
    }
  }
  exports({"entryPoint":MyTrigger});
```

2、GET方法示例；


## 后端函数调用三方系统接口

函数脚本调用外部接口时返回的数据量大小有限制，领域服务最多500条，标准服务最多2000条。

### 概述

本章讲解在应用构建中后端函数如何调用三方系统接口，通过postman或者postmanPure系统预制函数，传入请求方法（get/post）、请求地址、请求头、请求体，即可调用三方系统接口。

### get请求

请求示例如下：

```
let AbstractTrigger = require('AbstractTrigger');
 class MyTrigger extends AbstractTrigger {
     execute(context, param) {
         //信息体
         let body = {
         };
         //信息头
         let header = {
             'Content-Type': "application/json;charset=UTF-8"
         };
         // *****必须是https请求
         let responseObj = postman("get", "https://****.com/test", JSON.stringify(header), JSON.stringify(body));
         return {
             responseObj
         };
     }
 }
 exports({
     "entryPoint": MyTrigger
 });
```

### postman、postmanPure区别

问题：postman或者postmanPure有什么区别？

答案：postman：此方法在发起请求时，默认携带了当前请求的上下文信息（如当前租户的 yht\_access\_token）。这使其在调用本系统（即当前 BIP 环境）内部的 REST 接口时非常方便，无需手动处理 Token。但这也意味着，当调用外部系统或遇到 HTTPS 证书问题时，可能会受到当前上下文或平台安全策略的拦截。

postmanPure：此方法发起的是 “纯净”的 HTTP 请求，不携带任何默认的上下文信息。这意味着您需要手动管理所有必要的请求头，包括身份认证 Token（如 yht\_access\_token）。这种“无状态”的特性使其兼容性更好，尤其适用于跨租户调用、调用外部系统，或在 postman 调用失败（例如因 HTTPS 证书问题或第三方系统拦截）时作为备选方案。

### popenLinker、openLinkerPure区别

问题：openLinker或者openLinkerPure有什么区别？

答案：openLinker调用本系统REST接口可不用加yht\_access\_token，默认使用当前租户token调用，openLinkerPure需要在header中加yht\_access\_token，适用跨租户调用的情况。

### 获取当前数据中心域名

问题：如何获取当前数据中心域名？

答案：通过ObjectStore.env().url获取当前数据中心域名。

### openLinkerPure无法调通

问题：openLinkerPure调用时报错，openLinker正常调用？

答案：需要在header中加入appKey和appSecret，如

let header = { appkey: "fdb4d8f5f1794fd38f20b81a5e36dd25", appsecret: "bdbb6c7c2e246946ccc8a863d796b5c48ebfe78d" }。

## 常见问题

### 问题：sendMessage参数中的channels可以选择哪些？

答案：消息发送参数里的channels包含 mail、sms、uspace、weixin、enterprise\_wechat
一般不传channels参数即表示全部发送，只有只发某些通道才需要指定。

### 问题：sendMessage支持带附件吗？

答案：建议将附件以URL的方式放在消息内容里进行发送，或者在消息模板添加固定邮件。

### 问题：调用方法sendMessage方法发送邮件提示当前设置邮箱服务器端口号不可用？

答案：检查消息通道配置，默认邮件服务器的配置，点击测试发送，能发送成功才能使用。

### 问题：专业版开发支持后端函数发送邮件吗？

答案：专业版开发后端函数支持发送邮件。

### 问题：发送友空间消息时候uspaceReceiver在哪获取？

答案：uspaceReceiver在用户管理菜单中的用户ID。

## 常见问题

### 调度任务失败怎么办

可以手动触发，立即执行进行补偿，或者等待下一次执行。

### 调度任务一直失败是什么原因

可能是该调度任务对应的后端函数业务逻辑发生了变化，需要对函数内容进行排查。

### 调度任务可以设置指定时间触发吗

可以，在定时规则中进行设置。

### 如果调度任务过期了怎么办

可以重新设置定时规则，重新设置执行时间区间和有效时间范围。

### 为什么前端函数不支持调度任务

因为调度任务是在服务器端执行，前端函数作用于web端，非常不安全。

### 无中间服务，是否直接将BIP待办发送至目标系统？

可以，通过应用构建，函数事件订阅的方式，可以接收到待办消息，然后再根据三方的接口规范，调用接口将待办信息发送给三方。

### 添加事件订阅，选择不到自己需要的怎么办？

如果未发现需要的事件，说明原厂并未提供，请提交需求工单；

### 配置事件订阅函数后切换页签回来绑定的函数丢失了？

在添加函数后，请在事件订阅页面点击保存后再切换页签；

### 三方系统如何接收消息？

需要三方系统开放出公网可访问的REST接口，在函数中通过postman函数调用即可；
