## 业务场景

对“客户档案”实体扩展主要列举两个业务场景，

一个是实体字段扩展，可以通过平行表或特征进行扩展，但大文本目前仅支持通过平行表扩展。使用特征时，务必遵循开发者社区开发规范的 [特征使用规范](https://c2.yonyoucloud.com/iuap-hc-client/ucf-wh/client/index.html#/detail/KKGF1?nodeId=9ef2aba0-7dfe-484f-b1c5-13e152ec3df3&productline=CommunityDoc&type=enableDoc "特征使用规范")

注：ISV开发不能使用特征，必须使用平行表。

另一个是新增子实体扩展，新增子实体只能通过平行表进行扩展。

1. 主实体扩展字段：

通过平行表扩展两个字段，分别是：是否特殊客户，特殊客户说明，其中“是否特殊客户”是布尔类型。

通过特征扩展两个字段，分别是：合同期限类型，合作开始日期。

## 代码扩展

对“客户档案”进行原厂扩展后，产生了租户级的页面模板，根据实际业务需求可以对模板进行业务逻辑扩展开发。

本小结主要讲解采用脚本的方式进行业务逻辑的扩展开发，涉及开发点有前端脚本扩展，按钮命令扩展，以及子表业务逻辑扩展开发。

其实原厂扩展开发所采用的开发技术，例如脚本扩展，规则扩展，命令扩展，子表扩展等等，和自建应用所使用开发技术都是一样的。

## 业务场景

本章分为两种业务场景扩展：

1. 前端脚本扩展： a. 字段“是否特殊客户”添加值改变事件，如果选择“是”，则字段“特殊客户说明”可以填写信息，否则置灰，不可填写，用前端脚本扩展实现；
2. 规则链扩展： b. 在“客户档案卡片”新增页面，点击“保存”按钮时进行校验，如果字段“是否特殊客户”选择“是”，则字段“特殊客户说明”不能为空。

## 前端脚本扩展

第一个场景全部用前端脚本扩展实现（前2小节），第二种通过前端脚本扩展调用后端脚手架接口实现（第3小节）。

```
  } else {
    viewModel.get('extendNote').setDisabled(true);
    viewModel.get('extendNote').setValue('');
  }
});
```
1. 同时“特殊客户说明””默认应该是置灰的，所以在页面初始化中，还要添加“页面初始化完成”方法，并添加置灰代码，这需要放到“customInit”脚本内。
```
viewModel.on('customInit', function (data) {
    viewModel.on('afterLoadData', function (data) {
        viewModel.get("extendNote").setDisabled(true);
    });
});
```

## 常见问题

## 要扩展原厂应用必须要有应用构建吗？

是的，没有应用构建是无法扩展原厂应用的

## 应用构建中没有想要扩展的应用怎么办？

需要增购 对应的领域

## 原厂实体可以扩展吗？

可以，和自建应用建立实体是一样的

## 原厂页面可以扩展哪些内容？

可以扩展页面组件，前端函数，页面布局等。

## 原厂扩展应用可以迁移到其他租户吗？

可以，迁移后会覆盖目标租户的应用内容，实现一次开发多次使用

## 实体扩展字段，开放api能操作吗？

答：可以，详情页设计器中增加扩展字段后，可使用开放api新增修改

## 实体扩展字段怎么在页面显示？

答：分别在列表和详情页设计器中通过字段设置增加扩展字段

## 是否可以通过开放平台的保存接口保存扩展的子实体数据

答：支持，可以通过ui模板，或者详情接口查找子实体的字段名称

## 扩展子页签的按钮为什么不显示

答：请检查 页面配置 是否设置了显示

检查是否该按钮受权限管控，如果authid属性不为空，需要给当前用户分配按钮角色，才能看到按钮。

## 扩展是否可以删除原厂不使用的按钮？

答：不可以删除，但是可以隐藏；

## 是否可以通过扩展去调整原厂预置的动作规则？

答：不可以，原厂预置的动作规则限定不可修改、不可删除、不可覆盖、不可拖动调整顺序；

## 新扩展的时候，原厂按钮预置的执行命令限定什么？

答：原厂按钮预置的执行命令限定，不可修改、不可删除；

## 原厂扩展，页面按钮能做哪些操作？

答：与UI模板的配置不同，原厂扩展允许用户增加新的按钮，及增加按钮动作的配置，故原在UI上被隐藏掉的按钮工具栏全部放开，包括卡片页面的右上角、右下角的按钮，表格行上的按钮等；

## 原厂扩展，直接在原厂按钮上面增加事件调用后端函数怎么无法生效？

答：原厂扩展按钮功能，设计完后端函数，按钮绑定事件之后，还需要在原厂按钮例如“保存”按钮规则链上添加后端函数，函数才能生效。
