# 业务流开发

> 业务流是以单据为核心的系列规则流转服务，支持推单/拉单/分单/回写，可自定义转换规则，实现上下游单据的自动生成。

## 两个引擎

| 引擎 | 特点 | 适用 |
|------|------|------|
| **YonSuite 引擎** | 基于交易类型确定唯一业务流，严格按流程推进 | 供应链、营销预置流程 |
| **低代码引擎** | 灵活推送关系，不依赖交易类型，所有启用规则同时生效 | 用户自定义流程 |

现在正在融合两者：一个业务流中支持完整业务场景 + 多单据入口 + 多规则转换。

## 核心概念

- **推单**：上游单据手动/自动触发 → 生成下游单据
- **拉单**：下游单据主动向上游索取数据
- **分单**：一张上游单据拆分为多张下游单据
- **回写**：下游单据完成后更新上游单据状态
- **单据联查**：图形化查询上下游关联单据链

## 推单逻辑

触发来源：用户手动触发 / 审批流回调 / 上游单据动作触发

```
上游单据 → 转换规则 → 下游单据
                ↓
        数据映射 + 字段赋值
```

## 常用 API

```java
// 单条推单
pushBill.upstreamBillId, conversionRuleId

// 批量推单
batchPushBill.upstreamBillIds, conversionRuleId

// 查询适用业务流规则
findApplicableRules.upstreamBillId

// 单据联查
queryBillRelationChain.upstreamBillId
```

## 配置步骤

1. 创建业务流 → 添加节点（单据类型）
2. 配置转换规则（上游→下游的字段映射）
3. 设置触发时机（手动/审批后/保存后）
4. 配置分单规则（按数量/金额等维度拆分）
5. 启用 → 验证推单结果
