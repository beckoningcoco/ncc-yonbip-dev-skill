# 元数据开发

> iuap 平台的核心抽象层，提供设计态元数据管理和运行态 SDK 查询引擎。

## 架构分层

| 层 | 说明 |
|------|------|
| **设计态** | 元数据设计器，维护实体、接口、组件的定义 |
| **运行态** | 以 SDK 提供，含元数据查询接口、持久化引擎、查询引擎 |
| **租户扩展** | 支持租户级元数据扩展，不修改原厂定义 |

## 核心模型

- **组件 (Component)**：元数据的基本组织单元，对应一个业务模块
- **实体 (Entity)**：数据模型定义，映射到数据库表，含主实体/子实体/聚合实体
- **属性 (Attribute)**：实体的字段定义，支持 String、Integer、Decimal、Date、Boolean、枚举、参照类型
- **接口 (Interface)**：实体实现的抽象契约，支持多接口继承
- **组合关系 (Composition)**：主子实体间强关联，子实体随主实体生命周期
- **继承 (Inheritance)**：实体可以继承父实体的属性

## 业务对象 (BizObject)

运行时数据载体，基于元数据实例化。通过 `BizObject` 类操作数据：

```java
BizObject obj = new BizObject("实体URI");
obj.set("fieldName", value);
obj.get("fieldName");
```

## QuerySchema 查询

SDK 提供的声明式查询方案：

```java
QuerySchema schema = QuerySchema.create()
    .addSelect("id", "code", "name")
    .appendQueryCondition(
        QueryCondition.name("code").eq("001"),
        QueryCondition.name("ytenant").eq(tenantId));
List<Map<String, Object>> result = billQueryRepository.queryMapBySchema("entity.uri", schema);
```

**常用查询模式**：常量条件 / CASE 语句 / DISTINCT 去重 / 字段别名 / 函数调用 / 分组过滤 / 动态条件 / 动态类型
