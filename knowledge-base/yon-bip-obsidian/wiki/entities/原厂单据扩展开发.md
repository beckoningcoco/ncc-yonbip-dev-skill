# 原厂单据扩展开发

> [[可视化应用扩展]] 的具体技术实现，支持对原厂单据进行字段、页面、动作、脚本、流程等多维度的扩展开发。

## 官方定义

1. 支持字段扩展、子实体扩展
2. 扩展字段支持 OpenAPI 传参
3. 设计器中容器和控件支持各种组合设计，有很强的灵活性。常用扩展场景可插入各种表单元素控件、多页签、表格、按钮、底部栏、按钮栏等
4. 支持扩展原厂单据动作（save、delete、submit、unsubmit、audit、unaudit）
5. 支持编写前端脚本和后端脚本
6. 审批流流程扩展
7. 业务流推拉单回写
8. 参照扩展
9. 以及其他扩展能力

## 扩展能力清单

| 扩展维度 | 具体能力 |
|---------|---------|
| 数据模型 | 字段扩展、子实体扩展、OpenAPI 传参 |
| UI 布局 | 表单控件、多页签、表格、按钮、底部栏、按钮栏 |
| 单据动作 | save/delete/submit/unsubmit/audit/unaudit 前后拦截 |
| 脚本 | 前端脚本（MDF/YNF）+ 后端脚本（YonQL/远程调用） |
| 流程 | 审批流扩展 + 业务流推拉单回写 |
| 参照 | 参照扩展（过滤/带入/显示字段调整） |

## 与专业开发的关系

原厂扩展开发是低代码能力的一部分。如果低代码扩展能力不够用，可配合 [[专业开发]]（YPD 脚手架）进行深度定制。
