# 后端函数扩展

> YonBIP 标准版环境下，通过写 Groovy 脚本实现服务端业务逻辑，通常挂载在保存/提交等动作节点。

## 常见场景

- **保存前校验**：检查字段必填、业务规则合法性
- **数据转换**：保存前自动计算/补充字段值
- **联动操作**：保存后更新关联数据

## 开发步骤

1. UI设计器 → 选择动作节点（如保存按钮）
2. 添加后端函数 → 打开 Groovy 脚本编辑器
3. 编写校验/业务逻辑
4. 保存 → 页面验证

## 调试方式

- 查看后端日志（YMS 日志或服务端日志）
- 在脚本中打印日志输出

## 开发步骤

1. 打开扩展应用 → **脚本** 页签 → 创建目录"后端脚本"
2. 点击"新增脚本" → 函数名称输入（如 `saveBeforeCheck`）→ 函数类型选"后端函数"
3. 点击"设计"编写 Groovy 代码
4. 在保存按钮的规则链上添加后端函数

## 配置步骤

1. 编辑页面 → 点击"保存"按钮 → 右侧"动作"页签 → "单击"事件 → 点击"配置"
2. 在规则链末尾添加规则：`extBackEndRule-执行函数` → 选择刚创建的函数
3. 将新规则**拖到最上面**（保存前校验需优先执行）

## 完整示例 1：主表字段校验

```groovy
let AbstractTrigger = require('AbstractTrigger');
class MyTrigger extends AbstractTrigger {
    execute(context, param) {
        var pdata = param.data[0];
        if (pdata != null) {
            var extendNote = pdata.extendNote;
            var extendSpecial = pdata.extendSpecial;
            if (extendSpecial == true && (extendNote == null || extendNote == '')) {
                throw new Error('是否特殊客户选择是时，特殊客户说明不能为空！');
            }
        }
        return {};
    }
}
exports({ "entryPoint": MyTrigger });
```

## 完整示例 2：子表行校验

在上面的 `saveBeforeCheck` 中继续添加子表校验逻辑：

```groovy
// 检查客户关系维护经理
if (pdata.merchantrelationList != null) {
    pdata.merchantrelationList.forEach(data => {
        var isDefault = data.extendManagerDefault;
        var mobile = data.extendManagerPhone;
        if (isDefault === true && (mobile == null || mobile === '')) {
            throw new Error('联系人选择默认时，手机不能为空！');
        }
    });
}
```

## 调试方式

1. 在后端函数点击"调试" → 复制调试脚本
2. 在页面按 `SHIFT+ALT+D` 打开对话框 → 粘贴调试脚本
3. 点击"保存"按钮触发调试

## 与规则扩展的区别

| | 后端函数 | 规则扩展 |
|------|---------|---------|
| 语言 | Groovy | Java |
| 环境 | 标准版 | 专业版 |
| 复杂度 | 简单校验 | 复杂业务逻辑+分布式回滚 |
| 部署 | 无需编译 | 需 YDS 制盘部署 |
