# 应用扩展

> 在 YonBIP 标准产品基础上，通过低代码平台对应用进行 UI、逻辑、规则层面的定制化扩展。

## 三种模式

| 模式 | 环境 | 说明 |
|------|------|------|
| **UI模板扩展** | 标准版 | 基于原厂模板创建扩展模板，修改页面布局和字段 |
| **前端/后端函数扩展** | 标准版 | 写 JS（前端）或 Groovy（后端）脚本，挂载到控件事件 |
| **规则扩展** | 专业版 | 用 Java 开发 Rule，支持普通远程和分布式远程模式 |

## UI模板扩展流程

1. 打开原厂应用 → 业务扩展 → 找到目标单据
2. 创建扩展模板 → 进入页面设计器
3. 手工拖拽实体扩展字段到页面
4. 页面验证 → UI模板配置发布
5. 主子实体均可扩展

## 前端函数扩展流程

1. UI设计器 → 选择目标控件
2. 添加事件（如值改变 `afterValueChange`）
3. 写 JavaScript 逻辑
4. 页面验证 → 浏览器 F12 调试

## 后端函数扩展流程

1. UI设计器 → 选择动作节点（如保存按钮）
2. 添加后端函数 → 写 Groovy 脚本
3. 逻辑校验、数据转换等
4. 页面验证 → 后端日志调试

## 规则扩展流程（专业版）

1. 在 Developer 中创建 Rule 工程
2. 继承 `AbstractRule`，实现 `execute()` 和 `cancel()` 
3. 流水线发布 → YDS 制盘 → 部署
4. 在 UI 保存按钮配置远程调用

详见 [[规则扩展]]。
