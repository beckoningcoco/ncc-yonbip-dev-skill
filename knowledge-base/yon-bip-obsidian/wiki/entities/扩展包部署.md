# 扩展包部署

> 将开发环境中完成的低代码扩展成果（实体扩展、UI模板、函数、规则等）迁移到目标环境。

## 流程

```
导出扩展包 → 目标环境导入 → 部署验证
```

## 步骤

### 1. 导出扩展包

- 开发环境 → **业务扩展** → 选择应用
- 点击"导出" → 选择要导出的扩展项
- 生成 `.zip` 扩展包文件

### 2. 扩展包部署

- 目标环境 → **业务扩展** → 导入
- 选择扩展包 → 上传 → 系统自动解包并应用
- 支持跨环境迁移：dev → UAT → 生产

### 3. 部署验证

- 登录目标环境
- 打开对应应用 → 检查新增字段、UI变更是否生效
- 测试函数和规则是否正常执行

## 规则引擎部署

专业版规则扩展需要额外部署：
- 通过 YDS 制盘生成补丁包
- 在安装器中部署 Java 代码
- 目标环境配置远程调用地址
