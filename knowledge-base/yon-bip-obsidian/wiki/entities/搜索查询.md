# 搜索查询

> BIP 旗舰版的前端查询区和后端查询方案。

## 前端：FilterViewModel

```javascript
var searchModel = viewModel.getCache('FilterViewModel');
searchModel.getParams().autoLoad = false;  // 阻止自动查询
viewModel.on('beforeSearch', fn);          // 动态添加过滤条件
```

条件构建：
```javascript
args.params.condition.simpleVOs = [
    { field: 'xxx', op: 'eq', value1: 'value' }
];
```
条件操作符：`eq`/`in`/`elt`/`egt`/`like`/`is_null`/`is_not_null`

## 后端：ObjectStore 查询

| 方法 | 说明 |
|------|------|
| `selectById(URI, {id, compositions})` | 按ID查，可指定返回子表 |
| `selectBatchIds(URI, {ids, compositions})` | 按多ID批量查 |
| `selectByMap(URI, {key: val})` | 条件查询 |
| `queryByYonQL(sql, domainKey)` | YonQL查询，原厂表必填domainKey |

### YonQL 关键语法

- 主元数据**不允许**有别名，子元数据**必须**有别名
- `like '山'` → 自动转 `%山%`（不需要手动加`%`）
- `leftlike '河'` = MySQL `like '河%'`
- `rightlike '东'` = MySQL `like '%东'`
- 支持 `left join` / `inner join`，暂不支持 `right join`
- 分页：`limit page, size`

### 多表查询（基于关联关系）
```javascript
var sql = "select son_id.sonCode, son_id.main_id.mainName
  from GT22161AT175.GT22161AT175.grandson
  left join son_id son on son_id=son.id
  left join son_id.main_id main";
var res = ObjectStore.queryByYonQL(sql);
```
前提：实体之间有关联关系（主子表或引用关系）

### 重要限制
- 自建表和系统表不支持关联查询（domainKey不同，不在同一库）
- 跨领域原厂表不支持关联查询
- YonQL只支持查询原厂数据，增删改用开放平台接口
