# 环境搭建

> BIP 旗舰版开发环境搭建方案。

## 方案对比

| 版本 | 工具/方式 | 说明 |
|------|----------|------|
| V3 R6 以下 | IDEA | 需手动配置 Maven、数据源、启动参数 |
| V3 R6 | Web 平台（YPD） | 数字化工作台 Web 界面创建引擎，自动推送脚手架到 Git |
| V3 R6 及以上 / BIP5 | YDS 工具 | 官方 IDE，一键创建引擎 + 启动脚手架 + 调试 |

> **YPD vs YDS**：R6 的引擎类型是 YPD，通过 Web 平台创建；BIP5 推荐 YDS 本地 IDE 模式。YPD 模式下脚手架自动推送到 Git，开发人员通过 Git 克隆到本地。

## IDEA 搭建要点

1. 更换 Maven 仓库，启用离线模式
2. 修改 pom 文件、属性配置、config 配置
3. 配置启动参数（JVM options）
4. 配置 hosts 映射
5. Rebuild + Maven clean install

## 数据库

默认使用达梦（DM）数据库：
```
jdbc:dm://host:port/DATABASE_NAME
```

## 部署方案

四种部署方案均支持客户化定制和 ISV 生态开发：

| 方案 | 说明 |
|------|------|
| 公有云 | 部署在公有云服务器，SaaS 模式。需购买开发沙箱 + 测试沙箱 |
| 私有云 | 客户自建/指定基础设施上部署，独立运维，满足数据安全合规要求 |
| 专属云 | 介于公/私有云之间，专有化部署 + 资源隔离 |
| 本地部署 | 客户自有数据中心，完全自主掌控硬件和数据 |

### 公有云资源准备

- 购买沙箱租户（ISV 走云市场，客开走商务系统）
- 2 套 PASS 资源包（测试 + 生产各一）
- 提交 DSP 工单，由用友运维创建服务链路（流水线、数据库、YMS、Nginx、域名、Redis）

### 私有云/本地部署网络准备

1. **网络连通**：确保本地能访问应用平台、技术平台、数据库、Redis。通过 VPN 或零信任客户端
2. **Hosts 配置**（域名不在公网 DNS 时）：

```
# 应用平台 & 技术平台
1xx.xxx.xxx.xxx tech-xxxr6.xxx.com
1xx.xxx.xxx.xxx xxxr6.xxx.com

# 本地调试（域名后两段和业务中台一致）
127.0.0.1 local.xxx.com
```

3. **本地开发环境**：JDK + Node.js。YDS 已内置前后端环境，不需要手动搭建

## YDS All-in-One

V3 R6 及以上推荐使用 YDS，内置 JDK、Node.js、Maven，不需要单独配置环境变量和依赖。

## R6 YPD 引擎部署（Web 平台）

> 适用 R6 版本，有租户/无租户均可。通过数字化工作台 Web 界面创建，无需 YDS 工具。

### 部署准备

R6 私有化项目需提前准备：

| 资源 | 说明 |
|------|------|
| 开发环境信息 | 平台地址、账号密码、数据库、网络 → 通过运维文档交付 |
| GitLab | 私有化项目需自建，确保 YonBIP 服务器可访问 |
| Maven 私库 | 私有化项目需自建，平台自带也可用项目现有 |
| 数据库 | 默认使用平台数据库 |
| Redis | 默认使用平台 Redis 缓存 |
| 文件服务 | 默认使用平台文件服务 |

### 创建引擎

1. 数字化工作台 → 云平台 → 开发平台 → 专业开发服务 → 应用引擎
2. 点击【新建引擎】，填写：
   - 引擎名称 + 引擎编码（产品自动添加前缀 `c-`）
   - 选择资源池（对应环境）
   - 引擎类型：**YPD**
   - Git 地址（建议 SSH）+ 上传 Git 私钥
3. 技术平台自动执行：代码生成、YMS 配置、中间件绑定、流水线初始化等
4. 脚手架自动推送到 Git，开发人员克隆到本地即可

### 健康检查

```
https://技术平台域名/引擎编码/extend/healthycheck
```

返回 `status: ok` 表示部署成功。R6 版本后端服务访问前缀为引擎编码值（如 `c-kk-fn-pxlx`）。内网检查地址：`http://127.0.0.1:8080/extend/healthycheck`。

### 数据库与 YMS

- **数据库**：引擎编码中划线替换为下划线 + `_db`，如 `c-kk-fn-pxlx` → `c_kk_fn_pxlx_db`
- **YMS**：自动注册产品并配置逻辑数据源、Redis、文件存储
- 自定义改动需在 YMS 控制台**发布配置 + 重启服务**后生效

### 常见问题（YPD）

| 问题 | 解决 |
|------|------|
| 未执行初始化脚本 | 手动执行：网盘 → 开发工具-引擎数据库初始化脚本 |
| 自定义数据库 | YMS 添加数据库连接池 → 关联逻辑数据源 → 同步库表 → 发布重启 |
| 前后端分不同 Git | 手动提交 fe 目录到独立 Git → 修改流水线前端 Git 地址 |
| 外网健康检查报错 | 先查容器内 `http://127.0.0.1:8080/extend/healthycheck`，正常则排查 Nginx/Ingress |

## YNF 前端环境搭建

> YNF 使用 `ynf-dx-cli` 命令行工具，端口 **9090**，调试需手动配置 Resource Override。

```bash
npm install ynpm-tool -g          # YonBIP 包管理工具
ynpm install ynf-dx-cli -g        # YNF 脚手架工具
ynf -v                            # 验证
ynf init app ynf-${domainKey}     # 初始化（app/lib/sdk）
ynpm install                       # 安装依赖
ynf add                            # 添加扩展页面
npm run start                      # 启动
```

部署目前手动上传 Nginx。FAQ：`ynpm set usein=npm`（pnpm 报错时）、`npm install -g typescript`（ynf init 报错时）。

## 常见问题

| 问题 | 解决 |
|------|------|
| 运维交付手册怎么获取？ | 服务部署后发给项目经理 |
| 访问应用平台提示服务不可用？ | ping 检查域名指向 → hosts 配置未生效 |
| Node.js 能用其他版本吗？ | 不建议，教程版本已通过验证 |

## 脚手架工程搭建（传统 → YDS 升级）

> 仅适用私有云/专属云/本地部署，公有云不适用。

### 准备

1. 确认环境信息（平台地址、账号、网络、hosts、Maven 私库配置）
2. 获取引擎信息（Git 地址、数据库访问信息）
3. 下载脚手架：`enginecode-be`（后端）+ `enginecode-fe`（前端）

### 前端（升级前）

- VS Code 打开 `xxxxx-fe`，修改 `src/common/config.env.js`
- 安装 **ynpm 4.0.5**：`npm install -g ynpm@4.0.5`
- 启动：`npm run debug`（同时启动 server + extend）

### 后端（升级前）

- IDEA 打开 `enginecode-be` → 配置 Maven settings.xml
- 修改 `config-dev.properties` + `application.properties`
- Maven Reload → compile → package
- 启动类 `MDFApplication.java`，VM 参数：**`-Denable_cloud_debug=true`**
- 健康检查：`http://local.yyuap.com:8080/extend/healthycheck` → `Connected` + `ok`

### 升级后（YDS 一键模式）

- 前端：YDS 点击"启动前端脚手架"，自动下载依赖 + 启动
- 后端：settings.xml 放 `.m2`，YDS 点击后端启动按钮
- 健康检查：`http://localhost:62871/extend/healthycheck`

### 关键端口

| 端口 | 用途 |
|------|------|
| 8080 | 升级前/后 后端健康检查 |
| 62871 | 升级后 YDS 后端健康检查 |
| 3003 | 前端本地调试 |

### 常见坑

| 问题 | 原因/解决 |
|------|-----------|
| Maven 依赖拉取失败 | 检查 iNodeVPN + 私库配置 |
| 找不到 yms.jar（FileNotFoundException） | 工程目录不能有中文 |
| 前端脚手架初始化卡住 | VPN/网络，或使用离线包 |
| 离线包启动报错 | nodejs/ynpm 版本需与离线包一致 |
| 前端组件加载失败 | 用 nvm 切换 nodejs 版本 |
| YDS 预览单据报错 | 先查健康检查接口，再查防火墙 62871 端口 |
