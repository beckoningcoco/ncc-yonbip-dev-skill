## 关键业务规则

1. **分区策略**: `uordercorp` (iCorpId) 标记 `isPartition: true`，数据按公司分区存储
2. **编码规则**: `code`(cOrderNo) 标记 `isCode: true` + `REF.CODE`，可被其他单据引用；标记 `isMultiCodeRule` 支持多套编码
3. **必填字段**: `code`（单据编号）、`vouchdate`（单据日期）
4. **数据权限**: `saleAreaId`（销售区域）标记 `data_auth: true`，受数据权限体系控制
5. **业务日志**: `modifierId` 标记 `isBusinessLog: true`，修改操作自动记录变更日志
6. **已废弃属性**: `headFreeItem`、`headItem`（旧版自定义项，`deactivated: true`，新项目不应使用）
7. **核心单据**: `isCoreBill` 业务场景，支持工作流/审批流驱动
8. **文档级**: 标记 `doc` term，参与文档生命周期管理
9. **条件使用**: 标记 `isUseCondition: true`，支持条件规则

## 使用说明

### 查询订单数据

```sql
-- 直接查询 orders 表（注意分区键 iCorpId）
SELECT id, cOrderNo, vouchdate, status, fTotalMoney, fRealMoney,
       iCustomerId, iSalesOrgId, iSubmiterId, dCreated
FROM orders
WHERE iCorpId = ? -- 必须指定分区键
  AND cOrderNo = ?
```

### 查询子表数据

```sql
-- 订单明细（通过 orderId 关联）
SELECT * FROM order_detail WHERE orderId = ?

-- 收款计划
SELECT * FROM payment_schedules WHERE mainid = ?

-- 状态历史
SELECT * FROM order_status WHERE orderId = ? ORDER BY audit_time
```

### 通过 API 获取元数据

```
GET /base/itf/voucher.order.Order
```

返回本页所描述的完整元数据 JSON。
