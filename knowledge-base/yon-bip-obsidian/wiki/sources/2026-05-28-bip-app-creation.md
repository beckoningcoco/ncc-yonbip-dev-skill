# 应用创建

> 专业版应用的快速创建过程：领域注册 → 菜单管理 → 确认引擎 → 新建应用。适用所有部署方案。

## 适用场景

适用所有部署方案（公有云/私有云/专属云/本地部署），客户化定制开发 + ISV 生态开发均支持。

## 前置准备

1. **领域**：如领域不存在，需先注册
2. **应用引擎**：需提前创建，参考"新建应用引擎"
3. **权限**：需有"应用管理"功能权限
4. **ISV 注意**：应用编码需提前申请

## 创建流程

### 1. 领域注册（如已有则跳过）

数字化工作台 → 数字化建模 → 系统管理 → 应用管理 → 领域管理 → 新建领域

填写：领域编码、领域名称、领域图标

> 领域对应一级导航目录菜单。如需使用已有菜单的一级目录，可跳过菜单管理步骤。

### 2. 菜单管理（可选）

在菜单管理下添加一级目录，用于应用功能发布。

### 3. 新建应用

数字化工作台 → 云平台 → 开发平台 → 低代码开发 → 应用构建

1. 使用 **"客开模板"** 新建应用
2. 选择对应的**领域**与**自建应用引擎**
3. 填写应用编码与应用名称
   - 编码格式：`c{1}{2}{3}`，**禁止出现 `-`、`_` 等特殊字符**
   - 示例：`cpurart` → 名称 `cpurart商品采购`
4. 点击【完成】，可进入应用进行开发

### 关键提示

基于 YDS 创建的引擎，**必须通过 YDS 打开的浏览器访问工作台**进行相关开发，否则服务端会报错。

## 常见问题

| 问题 | 解决 |
|------|------|
| 怎么创建领域？ | 数字化建模 → 系统管理 → 应用管理 → 领域管理 |
| 导航菜单没有"应用管理"？ | 功能权限不足，换有权限的账号或申请授权 |
| 编码已重复？ | 修改编码值后重新保存 |
| 只有公共引擎可选？ | 将账号添加到自建引擎的**成员列表**中 |
| 应用构建打不开/白页？ | 确认服务已安装 → 技术平台查看微服务运行状态 → 重启流水线 → 工单求助 |
