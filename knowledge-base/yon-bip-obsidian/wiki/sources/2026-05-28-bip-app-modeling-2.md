## APP 数字化建模（2）：流程建模与业务对象建模

YonBIP V5.0.2507 APP 数字化建模培训课件第 2 部分（约 77 页），覆盖流程建模（工作流/业务流）和业务对象建模（特征/编码/模板/消息/规则）。

### 一、流程建模

#### 1. 工作流设计

**设计流程**：按领域-模块-业务对象（单据类型）创建流程模型，支持创建多个（通过发起条件控制运行时匹配）。图形化拖拽设计器，定义环节的参与人、抄送人、操作、时限及消息提醒。

**新版设计器亮点**：
- 环节属性设置更高效（图例图文结合）
- 选人方式更直观（选人方式、已选、待选一目了然）
- 批量设置环节属性

**流程匹配规则**：依据发起主组织，优先匹配本级组织流程模型 → 逐级向上查找。匹配到唯一 → 启动；匹配到多个 → 提示；未匹配 → 不走流程。

**流程分级管控**（三种模式）：
| 管控方式 | 规则 | 适用场景 |
|---------|------|---------|
| 统一管控 | 下级必须使用上级流程，不允许新建 | 强管控集团 |
| 扩展管控 | 下级遵守上级流程，仅能在指定环节扩展 | 有限自治 |
| 自治管控 | 下级可自建流程，没有则不走审批 | 松散管控 |

**流程仿真**：支持模拟单据测试，自动仿真执行，记录执行日志。精准仿真可减少测试工作量、提前发现设计问题、避免上线后返工。

#### 2. 流程审批

- **参与人设置**：支持按用户、角色、部门、职务、岗位、部门负责人、分管领导、发起人、单据字段等多种方式
- **代理设置**：管理员可按组织-领域维度设置代理人，支持生效/失效日期，任务同时发送代理人和被代理人
- **流程预测**：提交前/审批中可预测后续审批路径，API 保证数据安全隐私
- **流程监控**：全局流程实例监控，支持人工调整（改派、加签、终止、挂起/取消挂起、流程复活）、异常日志和干预日志

**典型场景**：员工报销 → 部门经理审批 → 机器人初审 → 财务经理复审 → 自动支付

#### 3. 业务流与单据转换

**业务流**：多个单据类型串接的业务流程，每个单据有上下游关系。支持预置流程（开箱即用，可调整）和自定义流程。流程类型包括：原厂-原厂、原厂-自定义、自定义-自定义、自定义-原厂。

**关键能力**：
- 手工+自动双轮驱动业务流转
- 根据不同业务属性走不同分支流程
- 支持消息驱动（自动/手工推单场景下的业务待办消息）
- 支持 ISV 开发场景（制品盘发布）
- 全面支持多语（全球化业务推广）

**单据转换规则**：定义上下游单据间的字段映射（VO 对照）、数据过滤条件、分单规则、回写规则。支持系统级/租户级/流程级三层。

**联查配置**：全局级业务流程追溯。低代码/零代码单据自动生成联查设置，无需手工配置。支持单据状态显示、列表模式、以当前单据为中心的上查/下查。

### 二、业务对象建模

#### 1. 业务模型管理

BIP 业务模型分层架构：
```
模型层：元数据(Entity/Attribute) → 扩展实体/属性 → 特征
展现层：UI模板、打印模板、导入导出模板、消息模板、分析报表
规则层：规则库(编码规则/相关性规则/唯一性规则/非空规则/休假/考勤/报销...)
行为层：增删改查、提交/审批/驳回、工作流/业务流引擎驱动
```

**企业业务模型管理一体化**：以业务对象为枢纽，构建端到端数字化链路。一次配置影响全链路服务（UI 字段变更 → 批量影响 UI 模板、流程选人、编码规则等）。

#### 2. 特征管理（核心亮点）

特征是 BIP 新架构提供的"支持租户自定义扩展业务对象属性"的能力。

**两大应用场景**：

**场景 1：扩展单据/档案自定义项**
- 将特征分配到业务对象（单据/档案），实现自定义项扩展
- 同一业务对象的所有实例都具有该特征字段
- 通过 UI 模板按角色/组织设置字段可见性差异

**场景 2：弹性扩展物料属性**
- 物料属性特征（静态属性）：描述物料的固有属性，通过"特征→特征组→物料模板→物料档案"链路
- 自由项特征（动态属性）：物料+自由项特征维度细化管理，可压缩物料数量
- SKU 属性特征：描述 SKU 静态属性，由"物料+SKU 敏感特征"生成 SKU 档案
- 选配特征：按单选配的派生物料模式参数化选配

**特征支撑服务**：前端/后端框架、UI 模板、导入导出、打印、公式、编码规则、相关性规则、报表、工作流、业务流、消息模板、数据权限等全链路支持。

**特征分级管控**：集团制定标准自上而下贯彻（不可变更），各下属企业个性化扩展相互隔离，可在上级标准之上按需扩展。

**商品 SKU**：自由项特征标记 SKU 敏感 → 生成 SKU 档案 → 应用于营销领域（订货、商城、零售、定价、信用、促销等）。SKU 敏感是存量敏感的子集。

**限制**：每种特征组类型下所有特征组可扩展特征数量总和 ≤ 765 个（自定义档案+字符类 ≤ 165、数值类 ≤ 150、基础档案类 ≤ 150、布尔类 ≤ 150、日期类 ≤ 150）。

#### 3. 编码规则

- 为档案和业务单据提供按数字/字母流水编码能力
- 编码元素灵活配置：属性、公式、数字/字母流水、流水依据
- 支持不同条件维度设置不同的编码规则
- **物料结构化编码**：基于物料字段+特征的结构化编码，特征可参与编码流水或公式计算
- **组织级编码规则**：支持组织级管控，满足分级管理需求
- 并发控制和缓存机制，提供大请求下稳定的编码能力

#### 4. 模板管理

**UI 模板**：
- 系统模板（原厂提供，不支持设计）→ 预置模板（复制系统模板）→ 自定义模板（租户级扩展）
- 四个层级：应用构建（偏向开发）→ 高级设计器（偏向实施顾问）→ 简版设计器（偏向实施顾问）→ 用户级列表设置（偏向普通用户）
- 四种扩展能力：显示扩展（字段名/排列/显隐/必填）、管理扩展（按活动/交易类型/组织分配）、业务扩展（保存校验/页面规则/脚本）、赋值扩展（公式/参照/默认值）

**打印模板**：支持单据/小票/套打/合同/标签等多种模板类型，支持多模板切换、批量打印、PDF 打印、输出 Excel、字体管理、分级管控、导入导出复用。

#### 5. 消息平台

消息中间件，提供模板能力、通道集成（邮件/短信/微信/企业微信/消息中心/IM）、消息策略拦截、业务触发能力。

**消息类型**：待办、通知、预警
**消息来源**：业务动作、业务事件、预警、业务流程
**关键能力**：跨租户多模板多通道、统一消息中心、按业务条件设置发送时机/通道/接收人、消息发送策略（频率/时间）、日志追踪与诊断

#### 6. 相关性规则

统一的业务对象级规则定义入口，规范业务对象属性之间的相互约束关系。

**四种规则类型**：
| 类型 | 说明 | 示例 |
|------|------|------|
| 赋值 | 根据条件自动填充字段值 | 销售订单按销售组织+商品分类自动赋值售价比率 |
| 值范围 | 控制字段可选值范围 | 费用项目按费用属性控制不同可选范围 |
| 合法性校验 | 校验字段间关系是否合规 | 物料尺寸1和尺寸2之间的约束关系 |
| 可用性 | 控制字段可见性 | 按客户/地区控制字段显示 |

**AI 辅助**：智能公式（取物料品牌、客户编码等）、智能规则（根据描述语言自动生成规则）。
