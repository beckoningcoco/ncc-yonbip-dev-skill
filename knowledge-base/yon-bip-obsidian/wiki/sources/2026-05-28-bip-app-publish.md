# 应用发布

> 开发完成后，将应用功能发布到数字化工作台，通过权限分配让业务用户使用。

## 适用场景

适用所有部署方案，客户化定制 + ISV 生态开发。

## 发布流程

1. 进入应用 → **发布管理**界面
2. 左侧添加**分组**（即菜单的分组结构）
3. 选择分组，点击右侧 **"+"** 添加功能节点，填写**服务编码**（可直接复制单据编码）
4. 勾选功能节点 → 【添加】→ 右上角【发布应用】
5. 弹框中配置：
   - 发布目标：工作台（默认）/ 移动端
   - 所属领域：应用创建时的领域
   - 一级菜单：选择后展示在导航"企业自定义"下
   - 着陆页：默认按结构生成
6. 点击【确定】发布

> 发布后需协同实施人员进行菜单管理与权限分配。

## 常见问题

| 问题 | 解决 |
|------|------|
| 服务编码重复 | 添加后缀解决 |
| 角色编辑页找不到功能节点 | 客开单据属业务类角色，平台功能属管理类角色 |
| 授权后菜单没显示 | 等缓存刷新（10 分钟内），刷新浏览器或重新登录 |
| 页面建模修改后需再发布？ | 不涉及功能权限的修改不需要重新发布 |
| 数据权限在哪授权？ | 角色管理 → 数据权限子页签 |
| 菜单功能报 502 | 引擎服务异常。YDS 本地正常但他人报错 → 线上未部署，需通过开发成果迁移生成制品盘安装 |
