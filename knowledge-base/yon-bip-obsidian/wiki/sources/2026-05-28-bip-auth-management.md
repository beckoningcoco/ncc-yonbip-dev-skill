# 权限管理

> BIP 核心权限服务（BIP-AUTH）完整指南：功能权限、主组织权限、数据权限三位一体的权限体系（51K 字）。

## 三大权限范畴

| 范畴 | 说明 | 控制粒度 |
|------|------|----------|
| **功能权限** | 控制用户可访问的领域/应用/服务/按钮 | 菜单、按钮、字段、页签 |
| **主组织权限** | 以组织为主隔离维度，控制业务数据行访问 | 用户-角色-组织三维 |
| **数据权限** | 在主组织权限基础上进一步精细化数据过滤 | 权限维度 + 权限范围 |

> 三者三位一体、不可分割。主组织权限是强管控模型，默认无组织权限时无法访问任何业务数据。

## RBAC 权限模型

以**角色为核心**，将所有权限（功能/组织/数据）打包分配给角色，用户通过成为角色成员获得权限。

### 两种功能权限分配方式

1. **RBAC 模式**：角色 → 服务/按钮/字段/页签权限
2. **全员应用模式**：直接给全体成员/管理员/自定义范围（组织/用户/标签/身份）授权

## 核心术语

| 术语 | 定义 |
|------|------|
| 领域/应用/服务 | 系统功能的三个层级划分 |
| 菜单 | 领域-应用-服务体系的可配置用户视图 |
| 用户身份 | 员工/商家/客户/供应商/开发者/外部/三员 |
| 角色 | RBAC 核心，关联功能/数据/敏感权限，必然属于组织 |
| 角色组 | 角色分组，一个角色仅属一个组 |
| 授权 | 用户与角色关联的动作 |
| 管理员 | 全员应用授权中具备特殊赋权能力的用户 |
| 主组织 | 业务数据唯一标记的核心组织，影响数据权限 |
| 权限管理维度 | 系统预置，描述数据权限受控列类型 |
| 权限范围 | 租户级配置，描述关联管理维度的数据取值范围 |
| 受控对象 | 需受控的应用/单据服务/档案（打标元数据） |
| 敏感字段权限 | 字段级：无权/只读/读写 |

## 技术架构

- 基于 RBAC 模型
- 核心数据模型：用户 → 角色 → 功能权限 + 组织权限 + 数据权限
- 与 MDD 框架结合，元数据字段标记声明数据权限过滤
