# 配置迁移

> YonBIP 社区文档中心 — 环境间配置数据的打包、传输和部署。

## 核心内容

### 背景
实施交付中环境间配置复制和验证工作占实施成本 20%+，小项目达 40%。配置迁移工具解决沙箱→生产的配置数据迁移。

### 迁移流程

**构建阶段**：新增传输包 → 加入迁移清单 → 提交传输包 → 构建 → 下载

**部署阶段**：导入传输包 → 环境校验 → 数据校验 → 部署 → 数据检查

### 迁移顺序（必须严格）

```
对象建模 → 应用 → 枚举 → 实体 → 页面
```

自建应用首次必须先迁"业务对象"，依赖关系决定了顺序，漏掉被依赖数据会导致迁移失败。

### 两种部署模式

| 模式 | 说明 |
|------|------|
| 指定目标部署 | 绑定目标环境/租户，部署时校验微服务版本和库表一致性 |
| 任意目标部署 | 可在任意环境部署（不含当前租户），跳过环境/租户校验 |

支持"弱校验"：跳过微服务不一致，库表校验只要目标不缺少字段即可通过。

### 私有云 COPY 数据库

copy 前必须清理 `iuap_apcom_migration.dms_*` 表（R5: 17张表，R6/YonBIP 5.0: 20张表），否则主键对照残留导致后续迁移出错。

### 常见问题速查

| 问题 | 解决 |
|------|------|
| UI模板校验失败 | 迁依赖的特征/安装扩展元数据 |
| 工作流构建失败 | 保存并发布工作流 |
| 参照默认值报错 | 先通过档案迁移同步数据 |
| 开多语翻译丢失 | 单独迁移"翻译工作台" |
| 特征迁移失败 | 检查数据类型、URI、用户匹配 |
