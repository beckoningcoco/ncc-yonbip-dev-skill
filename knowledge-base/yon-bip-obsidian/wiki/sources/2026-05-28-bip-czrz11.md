## 日志开发

BIP 日志开发的代码实践指南，展示如何在单据插件中集成业务日志和操作日志的 SDK 调用。

## 业务日志代码示例

在单据插件（`@BillPlugin`）中注入 `IBusiLogService`，通过 `BusinessLogObjectBuilder` 构建日志对象：

```java
BusinessLogObjectDTO businessLogObject = new BusinessLogObjectBuilder()
    .newObject(billContext.getBillDO())
    .uri(billContext.getBillDO().getMetaFullName())
    .operCodeTypes(OperCodeTypes.open)
    .serviceCode(billContext.getBaseBillContext().getServiceCode())
    .busiObjTypeCode(billContext.getBillDO().getCBillNo())
    .busiObjTypeName(billContext.getBaseBillContext().getBillName())
    .defaultOperationName(billContext.getBaseBillContext().getActionInfo().getName())
    .defaultBusiObjTypeName(billContext.getBaseBillContext().getBusiObj())
    .logMode(BusinessLogMode.BUSINESS_OBJECT)
    .build();
busiLogService.saveBusinessLog(businessLogObject);
```

在 `beforeSave` 扩展点调用 `saveBusiLog()` 即可自动记录。

## 操作日志代码示例

注入 `IOperationLogService`，构建 `OperationLogVO`：

```java
OperationLogVO operationLog = new OperationLogVO();
operationLog.setServiceCode(billContext.getBaseBillContext().getServiceCode());
operationLog.setDevice(DeviceTypes.web.getIndex());
operationLog.setIp(InvocationInfoProxy.getValByKey(ContextConstans.X_FORWARDED_CLIENTIP));
operationLog.setButton(OperCodeTypes.save.getDefaultOperateName());
operationLog.setOperCode(OperCodeTypes.save.getCode());
operationLog.setDefaultButton(OperCodeTypes.save.getMultilingual());
operationLog.setOperUser(InvocationInfoProxy.getUserid());
operationLog.setTenantId(InvocationInfoProxy.getTenantid());
operationLogService.saveOperation(operationLog);
```

## 关键注意事项

- **RPC 通信限制**：`IBusiLogService` 和 `IOperationLogService` 使用 RPC 通信，**ISV 开发不能使用**，需用 OpenAPI 方式接入
- **日志存储**：数据最终持久化到 ES，在平台功能节点中展现
- **OpenAPI 流程**：数据 → MQ → 日志接入层 → 数据处理 → 持久化 → ES
- **脚手架依赖**：日志 SDK 在 YPD 脚手架中已默认依赖
