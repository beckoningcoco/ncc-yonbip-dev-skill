## 调度任务

调度任务是 BIP 平台的分布式定时任务框架，基于 Quartz 实现，支持系统级/租户级/个人级三级调度任务的分级管理。

## 核心架构

- **分布式任务调度系统**：高可用，每个 Scheduler 负责部分任务（分表确保每次分发 < 4 万条）
- **任务分发流程**：启动调度服务 → 任务工厂创建计划实例 → 业务用户增删改查 → 持久化 → 分发线程循环分发到工作线程 → 发送到 Kafka → 客户端调用 Rest 接口
- **超时设置**：Consumer 执行 Rest 调用超时 2 秒，推荐业务接口异步化
- **依赖中间件**：Kafka（消息队列）、Quartz（调度引擎）

## 调度类型配置

- 编码/名称不能与当前租户下其他类型重复
- 接口字段需完整 REST 路径，支持 POST 请求
- 强烈建议业务接口按异步方式实现（SDK 支持异步执行器）
- 启用不勾选则任务配置时无法选取该类型
- 可定义参数：字符串、数字、日期、参照
- 用户可见参数会出现在调度任务界面供动态修改

## 调度任务

三级场景：
- **系统级任务**：在开发态创建，全局性跨租户，如权限同步
- **租户级任务（出厂默认）**：在开发态创建但不启用，租户开通时初始化
- **租户自行创建**：用户在运行态根据类型自行创建

任务配置包含：条件设置（继承类型参数）、定时规则（cron 或时间配置）、消息设置（成功失败都发/只失败发/都不发）

## 开放 API

| API | URL | Method |
|-----|-----|--------|
| 新增 cron 任务 | `/warning-pub/warning/task/save` | POST |
| 批量新增 | `/warning-pub/warning/task/batch/save` | POST |
| 修改 cron 表达式 | `/warning-pub/warning/task/update` | POST |
| 删除任务 | `/warning-pub/warning/task/delete/{taskId}` | GET |
| 批量删除 | `/warning-pub/warning/task/batch/delete` | GET |
| 停用任务 | `/warning-pub/warning/task/shutdown/{taskId}` | GET |
| 启用任务 | `/warning-pub/warning/task/startup/{taskId}` | GET |
| 立即执行 | `/warning-pub/warning/task/run/{taskId}` | GET |

## 业务接口规范

请求 Header 携带：`logId`、`sysId`、`tenantId`、`userId`；Body 为 JSON 格式传类型定义的参数。

- **异步接口**：返回 `{"asynchronized":"true"}`，执行完后回调 `/warning/warning/async/updateTaskLog`
- **同步接口**：直接返回 `status`、`content`、`title`、`msg`

## 异步执行器 SDK

配置 `dispatch.url`，实现 `IDispatchTask` 接口的 `execute()` 方法，返回 `TaskSendMsgDTO`。页面配置 URL 格式：`/项目根路径/dispatchtask/实现类bean名`

## 参数类型枚举

STRING(0)、REF(1)、NUMBER(2)、DATE(3)、ENUM(4)、TIME(5)、DECIMAL(6)、ENTITY(7)、DATETIME(8)、OTHER(9)
