## 调度任务开发

BIP 调度任务的开发实践指南，涵盖异步/同步接口实现、后端函数方式及数据库存储。

## 核心概念

- **调度任务服务**：基于预设固定时间点/间隔/执行次数自动执行任务的框架
- **调度类型**：开发态配置，定义任务接口和参数
- **调度任务**：运行态实例，支持添加/删除/停用/启用，提供 REST 服务
- **系统级任务**：全局性跨租户任务（如权限同步），运营人员在开发态配置
- **租户级任务**：租户内部定时任务，租户管理员在租户态配置
- **Scheduler**：Quartz 定时调度器

## 架构要点

- 分布式高可用，支持三级任务分级管理
- 任务全异步执行，支持限流
- Kafka 分发 → 客户端 Rest 调用 → 超时 2 秒
- 每个 Scheduler 分表确保分发量 < 4 万

## 开发方式

### 异步接口（推荐）

接口先返回 `{"asynchronized":"true"}`，业务逻辑在 `ymsExecutor` 线程池中异步执行，完成后回调 `/warning/warning/async/updateTaskLog`（POST，参数：id、status、content、title、msg）。

### 同步接口（60s 超时限制）

直接返回 JSON：`{"status":"1", "id":"666", "msg":"执行成功", "content":"content执行成功"}`。

### 后端函数（公有云唯一方式）

调度类型选择【后端脚本】，在脚本中通过 `ObjectStore.postman()` 调用 REST 接口（仅支持同步调用）：

```javascript
let AbstractTrigger = require('AbstractTrigger');
class MyTrigger extends AbstractTrigger {
    execute(context, param) {
        let response = ObjectStore.postman('post', url, header, body);
        return { data: response, url: url };
    }
}
exports({ entryPoint: MyTrigger });
```

### 数据库存储

- Schema：`iuap_yms_data`
- 调度类型表：`dispatch_taskway`
- 调度任务表：`dispatch_task`
- 任务用户表：`dispatch_taskuser`

## 常见问题

- 最小调度间隔：**5 分钟**
- 调度任务不执行：检查系统级调度任务是否被停用（系统级停用会导致租户级全部不执行）
- 老版本任务类型注册：`私有化域名/iuap-apcom-scheduler/warning/#/ruleslistdev/`
- REST 接口不适用于公有云（公有云只能用后端脚本）
