## DEV 开发平台课件

YonBIP V5.0.2507 DEV 企业应用开发与扩展培训课件（97 页），面向售前和实施人员，系统阐述开发平台的方案定位、三种开发模式（零代码/低代码/专业开发）、AI 辅助开发能力、以及最佳实践案例。

### 方案定位：多角色融合开发

DEV 开发平台的核心理念是**多角色融合开发**，覆盖三种开发模式的不同角色：

- **零代码** → 业务用户：通过可视化拖拽快速搭建表单、流程、分析页面，低成本实现业务在线化
- **低代码** → 业务专家、实施顾问：通过模型设计和脚本编写实现业务规则、个性化运算，具备更好的逻辑控制
- **专业开发** → 程序员：通过自建应用引擎，提供元数据驱动的前后端、移动端脚手架工程，支持团队协作的 DevOps 一体化

**同源架构**是核心竞争力：三种模式使用同一底层架构（元数据、脚本、API、权限、多租户），从一个模式升级到另一个无需重构，资产可继承。

### 核心优势

1. **高效率**（70%/20%/10%）：消除"业务需求→技术实现"转换损耗
2. **全覆盖**：三层开发工具，按需求开启不同模式
3. **有后台**：完整信息化底座支撑企业数智业务
4. **AI 赋能**：AI 开发助理全流程赋能（需求生成、建模、表单、公式、代码生成及诊断）
5. **闭环**：与数据、集成、DevOps 深度协同，形成"开发-数据-集成-运维"端到端闭环
6. **兼容标准化**：脚手架方式规范代码，杜绝"千人千码"

### 三种开发模式详解

#### 1. 零代码开发

面向业务用户，完整的应用构建闭环：

- **表单构建**：可视化拖拽设计器 → 流程设计（审批流）→ 表单设置（多视图、打印、脚本）→ 表单发布（权限设置）→ 数据管理
- **数据分析**：基于业务对象构建数据模型，通过仪表板可视化展现，PC 和移动需分别设计
- **集成配置**：表单自动发布 API，支持 Webhook 事件订阅推送三方系统
- **应用发布**：支持工作台和外链访问，权限按角色配置

适用场景：数据收集、审批流程、业务执行类表单（多表单业务流转）。

#### 2. 低代码开发

面向业务专家和实施顾问，基于业务模型的开发：

**主要流程**：需求分析 → 定义模型 → 建立模型 → 准备/验证数据 → 流程开发 → 测试验证 → 性能优化

**核心能力**：
- **公式扩展**：系统级函数 + 业务级函数 + 自定义脚本函数（动态取数、复杂计算、分录级循环处理、字段联动）
- **流程扩展**：委员会专家规则（运行时按规则表动态获取执行人）、脚本节点做复杂阶梯计算
- **OpenAPI 开发**：为大模型提供个性化业务数据，为业务应用提供专属智能体配置（关键业务监控、待办提醒、业务处理伴随）
- **原厂功能扩展**：扩展原厂单据字段、新增单据与原厂集成、通过特征扩展原厂业务属性

**典型场景**：某快消品集团 2000+ 经销商的返利计算系统（4 天完成，业务专家完成 80% 模型流程设计，实施顾问完成 20% 逻辑处理）。

#### 3. 专业开发

面向程序员，覆盖高定制化需求：

**开发过程**：环境搭建（VSCode + Maven + JDK + Git）→ 创建工程（引擎/数据库/脚手架/YMS/微服务）→ 开发调试（编译/本地调试/安装盘/QP 补丁）→ 上线部署（安装器/公有云流水线）

**YDS（VSCode 插件）**：集成工程管理、本地开发（前后端建模调试）、支撑服务配置、本地发布（安装脚本抽取/产品盘/补丁盘/远程安装）四大能力。

**核心能力**：
- **脚手架开发**：低代码框架的单据前后端扩展规范，支持第三方页面注册
- **后端服务扩展**：单据动作规则（保存前校验）、命令操作（按钮实现计算逻辑）、API 发布
- **业务对象支撑服务配置**：以业务对象维度配置元数据、UI 模板、工作流、编码规则等
- **扩展插件管理**：原厂/客开微服务动态加载扩展插件，支持调试模式免重启

#### 4. 软件测试

"零代码"自动化测试技术：

- **UI 零代码测试**：基于 RPA 客户端录制业务操作，自动生成测试脚本。原来 1 天/流程 → 30 分钟，通过率 80% → 100%
- **API 零代码测试**：Chrome 开发者工具插件抓取接口，智能匹配+自动参数化+智能断言。原来 1 天/流程 → 30 分钟
- **UI+API 混合模型**：采购订单→入库→发票→凭证的业财一体化自动化测试

### AI 辅助开发（五大能力）

1. **AI + 需求文档**：一句话需求 → 秒级生成完整系统设计（复用全行业原厂业务模型、标准单据、基础数据）
2. **AI + 数据建模**：自动复用原厂档案、关联原厂业务，零手工建模
3. **AI + 页面建模**：智能生成含字段、校验、布局、按钮、流程的全套单据
4. **AI + 公式**：自然语言描述业务需求 → 自动生成计算公式
5. **AI + 代码**：写注释 → 生成脚本代码，大模型检查脚本逻辑

### 关键特性

- **融合开发**：零代码/低代码/专业开发无缝集成，按需升级不需重构
- **模型分级管理**：管理态/实施态/运行时三层，支持 L1 客群租户级模型扩展的组织级能力
- **企业模型管理**：基于业务模型的支撑服务一体化管理，元数据变更批量影响 UI 模板、流程、编码规则等
- **业务规则对象化**：基于相同对象的规则分配到不同业务场景
- **关联查询优化**：流程导览模式、单据折叠、字段权限控制、联查配置优化

### 产品组合

| 层次 | 核心工具 | 涉及平台 |
|------|---------|---------|
| 业务支撑 | 零代码 | 开放平台、智能分析 |
| 业务支撑+管控 | 零代码+低代码 | 应用平台、开放平台、智能分析、智能体、意图流 |
| 业务管控+开发管理 | 零代码+低代码+专业开发 | 应用平台、集成平台、数据平台、智能平台 |

### 最佳实践案例

1. **雅江集团** — 500 个决策事项全部线上化，通过业务对象继承关系快速完成规模化业务开发，2 周完成模型和页面设计
2. **航天科技** — 航天级保密合规（三员分立、密级标识），一套开发引擎撑起 6 大人才业务，平均 2-3 天上线一个新业务模块
3. **中核工程** — 平台可用性 99.95%，核心业务 99.99%，每 2 天发布一个应用；三维融合（PDMS/Revit 模型与业务数据绑定），尾项闭环效率提升 90%
4. **明日控股** — 60+ 张业务单据、5 大能力中心，业务人员 2 周完成原型迭代；需求响应周期缩短 70%

### 适合实施人员的要点

- 零代码是快速启动的最佳路径（数据收集、审批流）
- 低代码解决需要逻辑控制的场景（规则引擎、脚本计算、动态审批人）
- 同源架构意味着零代码搭建的应用后续可以用低代码/专业开发增强，无需推倒重来
- AI 辅助开发可大幅缩短交付周期（需求→建模→表单→公式→代码全链路 AI 支持）
- 四个案例展示了从标准化开发（雅江）到高安全合规（航天）到高可用（中核）到业务中台化（明日）的完整客户谱系
