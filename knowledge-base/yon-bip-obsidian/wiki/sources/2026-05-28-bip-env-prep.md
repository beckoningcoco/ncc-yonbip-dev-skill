# 环境准备说明

> YonBIP 社区文档中心 — 专业开发前的环境准备工作：网络、账号、本地配置。

## 四种部署方案

| 方案 | 说明 |
|------|------|
| 公有云 | 部署在公有云服务器上，SaaS 模式 |
| 私有云 | 部署在客户自建或指定的基础设施上，独立运维 |
| 专属云 | 介于公/私有云之间，专有化部署 + 资源隔离 |
| 本地部署 | 客户自有数据中心，完全自主掌控 |

四种方案均支持客户化定制和 ISV 生态开发。

## 公有云环境准备

购买商用开发环境租户（开发沙箱 + 测试沙箱）：
- ISV 场景 → 云市场购买 ISV 沙箱
- 客开场景 → 商务系统购买开发/测试沙箱，规格复制生产环境

需要 2 套 PASS 资源包（测试 + 生产各一套）。ISV 可申请开发周期内 PASS 资源补贴。

资源准备后提交 DSP 工单，由用友运维创建服务链路（流水线、数据库、YMS、Nginx、域名、Redis 等）。

## 私有云/专属云/本地部署

需要提前从运维交付手册获取环境信息和账号。

### 网络连通
确保本地能访问应用平台、技术平台、数据库、Redis 等。一般通过 VPN 或零信任客户端连通。

### Hosts 配置

域名不在公网 DNS 解析时，需配置 hosts 映射：

```
# 应用平台 & 技术平台
1xx.xxx.xxx.xxx tech-xxxr6.xxx.com
1xx.xxx.xxx.xxx xxxr6.xxx.com

# 本地调试（域名后两段和业务中台一致）
127.0.0.1 local.xxx.com
```

## 本地开发环境

后端 SpringBoot（JDK）+ 前端 React（Node.js）。

> 基于 YDS 的 All-in-One 模式不需要手动搭建，YDS 已内置前后端环境。

## 常见问题

| 问题 | 解决 |
|------|------|
| 运维交付手册怎么获取？ | 服务部署后发给项目经理 |
| 访问应用平台提示服务不可用？ | ping 检查域名指向，hosts 配置未生效 |
| 前端 Node.js 能用其他版本吗？ | 不建议，教程版本已通过验证 |
