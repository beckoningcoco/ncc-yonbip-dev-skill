## 哈勃监控技术

哈勃监控中心是 BIP 平台的统一监控解决方案，利用多种数据采集手段将 CPU、内存、磁盘、网络、日志、服务调用等数据统一收集管理，提供资源、日志、服务调用等多种可视化分析视角。

## 技术架构

### 采集器层
- **Prometheus Exporters**：Node Exporter（主机）、cAdvisor（容器）、MySQL/PostgreSQL/Redis/ZooKeeper/RabbitMQ/Nginx/Elasticsearch Exporters
- **Async Profiler**：生成 CPU、内存火焰图
- **哈勃链路 Agent**：链路录制 + 实时链路两个版本
- **Fluentd**：日志采集器

### 中间件/存储层
- **Prometheus**：性能指标收集存储
- **Grafana**：可视化面板
- **Elasticsearch**：日志和链路数据存储
- **Kibana**：日志检索和可视化
- **MinIO**：业务日志归档存储和下载
- **Kafka**：链路数据接收
- **MySQL**：基础数据 + 链路录制数据 + 性能数据聚合降级
- **Redis**：静态数据缓存

### 服务层
- **全链路监控服务**：旗舰服务，承载监控中心大部分功能
- **监控数据收集服务**：处理哈勃链路录制和微服务链路数据
- **日志管理服务**：基于 Fluentd → Kafka → Elasticsearch / MinIO
- **实时链路服务**：基于 OpenTelemetry + Jaeger（实验阶段）

## 四大功能模块

1. **指标监控**：资源池监控、微服务监控（按产品/产品线/数据中心聚合）、中间件监控
2. **日志中心**：访问日志（Nginx/K8S Ingress）、业务日志、K8S 集群事件、SQL 慢日志、CPU/内存火焰图
3. **报警中心**：基于 Prometheus Alertmanager，支持运行时规则更新、短信/邮件/友空间通知；开放自定义报警推送接口
4. **链路跟踪**：链路录制（场景级）和实时链路（全量，实验阶段）

## 五大工作台

- **IT 管理工作台**：主机/服务 CPU、内存、磁盘、网络等性能指标趋势
- **运维工作台**：系统访问、中间件监控、K8S 运行情况
- **开发工作台**：链路录制、SQL 慢日志、业务日志
- **性能工作台**：火焰图生成、JVM 监控、内存 Dump
- **报警中心**：策略管理、通知规则、告警查询认领

## 日志分析技巧

业务日志索引分词字段：appId、instanceId（默认分词）、logger、filepath、requestUrl（IK 分词）、message（空格分词）。遵循 Kibana KQL/Lucene 语法。

## 故障排查流程（典型案例）

收到微服务超时报警 → 检查微服务 CPU/内存峰值 → 查 SQL 慢日志（某 SQL 执行 25s，返回 7 万行，扫描 1000 万行）→ K8S 集群事件（健康检查失败，多次重启）→ 访问日志（499 状态占 59%）→ 业务日志（数据库连接池耗尽）→ CPU 火焰图（死循环：6 个线程占满 6 核 CPU）

## 开发接入

### 后端链路录制
- 依赖 `com.yonyou.cloud.apm:apm-common`，在应用管理启用哈勃监控
- 专属化需配置 `hubble_agent_url` 和 `hubble_collect_url`

### 前端链路录制
- 引入录制 JS → `uis.start()` → `hubble.startRecord()` / `hubble.stopRecord()`

### 报警推送 API
- POST `/iuap-saas-billcode-service/billcoderest/getPreBillCode`
- 参数：metric、endpoint、timestamp、step、value、level（info/warning/error）、tags
