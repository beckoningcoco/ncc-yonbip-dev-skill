# 多语（国际化）

> YonBIP 多语红皮书：资源多语、内容多语的完整技术架构和接入方案。

## 概述

多语资源分为**界面多语**和**内容多语**两种类型。核心要点：

1. 资源多语的技术架构
2. 内容多语的技术架构
3. 翻译工作台功能介绍
4. 前端/后端代码、UI 元数据、实体元数据的多语资源抽取方法
5. 资源多语的维护和翻译
6. 代码如何接入多语运行时
7. 多语服务对外接口

## 资源多语

### 资源标识规范

资源 ID 由多语服务统一生成：`{产品线}_{领域}_{项目}_{流水号(10位)}`

- 预发环境前缀：`P_`
- 测试环境前缀：`T_`

### 资源生命周期

1. **开发态**：研发通过抽取工具/手工导入 properties 文件/后端接口将资源条目放入资源池
2. **翻译态**：翻译人员导出资源文件 → 翻译 → 重新导入，或直接手工维护
3. **运行时**分组版本机制：分组内资源有更新时自动新增版本，前端自动加载

### 权限规范

- **研发人员**：增加资源条目（工具抽取、导入 properties、后端接口调用）
- **翻译人员**：变更资源条目内容（直接修改、导入导出 Excel）
  - 系统翻译人员 → 0 租户资源
  - 租户翻译人员 → 仅当前租户

### 核心组件

- 多语服务
- 资源抽取工具
- 后端 Java 多语运行时 SDK
- 前端 React 多语运行时 SDK

## 内容多语

### 语种字段映射

支持 6 个语种字段（name ~ name6）：

| 字段 | 默认语种 |
|------|----------|
| name | 中文（zh_CN） |
| name2 | 英文（en_US） |
| name3 | 繁体（zh_TW） |
| name4 | 法文（fr_FR） |
| name5/name6 | 不固定，租户自定义 |

语种与字段关系通过 `LangSequence` 决定：`LangSequence=1` → name，`LangSequence=2` → name2，以此类推。

### 元数据支持

元数据中为需要多语的字段定义为"多语字段"，生成物理模型时自动生成 name~name6 六个字段。

## 关键流程

### 开发态抽取

研发通过工具抽取资源条目 → 放入资源池 → 关联到领域/应用/项目

- UI 元数据和模板资源可关联到业务实体和单据对象
- 前端/后端资源只能关联到领域/应用/项目，需人工打标

### 翻译维护

只有经产品经理确认的资源才提交翻译，减少垃圾资源数量。
