## 低代码开发用户手册

YonBIP V5.0.2507 低代码开发用户手册（57 页），系统介绍应用构建平台的完整开发流程。

### 三种开发模式

| 模式 | 适用场景 | 特征 |
|------|----------|------|
| 快速交付模式 | 模式化记录型业务应用 | 无代码配置为主 |
| 简单开发模式 | 需要个性化的业务应用 | 在线脚本+配置 |
| 原生开发模式 | 复杂创新型业务应用 | 脚手架+本地开发+CI/CD |

### 应用构建全流程

**1. 对象建模**

- **业务对象**：具体业务模型的抽象集合，一个业务对象支持一个主实体+多个子实体，支撑服务以业务对象为配置维度
- **实体**：具体业务存储模型数据描述，含关系引用描述，支持创建业务属性和系统属性
- **枚举**：预设值集合，用于单选/多选引用
- **引用接口**：审批、业务流、自动编码、树型结构、主组织、交易类型、档案
- **实体管理**：支持从空白新建、复制已有实体、批量操作、唯一性校验（联合唯一）、非空校验、场景支持（字段权限/数据权限/转换规则）
- **系统属性**：根据引用接口自动生成（基础可选项、审批、业务流、交易类型等八种）

**2. 页面建模**

- **页面类型**：单据（列表/单卡/一主多子/主子孙/左树右卡/左树右表/树形表/行编辑表）、空白布局、档案、分析、第三方页面
- **设计器三区域**：组件区（布局容器/数据容器/工具栏/按钮/基础控件/业务控件/协作控件）→ 设计区 → 属性区
- **布局容器**：网格、模态框、卡片组、卡片、多页签、子页签、锚点容器
- **数据容器**：表单、表格、树型表格、查询区、搜索树、穿梭框
- **关键控件配置**：单选参照（只选末级/默认展开级次/模糊匹配/联想匹配/展现方式/参照设置/参照带入/参照过滤）
- **页面规则**：交互规则（显示隐藏/必填/可编辑/赋值/数值精度/合计）+ 校验规则（不能保存条件+提示）
- **参照**：支持创建并发布为公共参照，"参照展示带入"和"元数据带入"两种带入方式
- **打印模板**：需先在对象建模中勾选"打印"场景

**3. 流程 & 自动化**

- **工作流**：为单据配置审批流程，业务对象需勾选"审批流"场景
- **业务流**：单据间推拉单业务，业务对象需勾选"业务流"场景
- **脚本**：统一管理前后端脚本资源
  - 前端页面脚本：每页面有且仅一个
  - 后端脚本（函数）：页面组件/命令扩展的后端逻辑
  - API 脚本：后端函数，支持前端调用后端（后端不支持调前端）
  - 计算属性脚本：公式/页面规则自动生成

**4. 集成配置**

- **API 授权**：将开放平台 API 授权给应用，通过 appKey/appSecret 访问
- **API 发布**：将应用数据通过后端函数发布为 API 供外部调用
- **事件订阅**：通过函数方式接收事件，解决原厂单据和自建单据数据一致性问题

**5. 发布管理**

- **菜单结构**：创建分组→添加页面→发布应用，生成对应菜单层级（一级→二级→三级）
- **应用发布**：指定一级菜单挂载，可选发布移动端（指定应用类别/着陆页/终端）
- **移动发布逻辑**：按应用生成 vs 按分组生成，着陆页默认结构 vs 指定页面
- **单页面发布**：发布为独立链接+二维码，支持有效期和登录方式（友互通/手机号）

**6. 配置迁移**

- 支持不同租户间迁移开发过程和成果，应用可按整体或单个支撑服务维度迁移
- 首次迁移为新增，再次迁移为覆盖
- **两步迁移**：① 先迁移对象建模数据 → ② 迁移整个应用（含枚举/页面模板/参照/打印模板/业务流/脚本/API 等）
- 工作流默认不随应用迁移，需单独迁移
- 支持单个支撑服务迁移（业务对象/枚举/参照/打印模板/工作流/业务流/脚本/API 发布/菜单结构）

### 脚本服务

租户级脚本调用，独立脚本资源管理，仅支持后端函数，可被调度任务定时调用。
