## 专业开发用户手册

YonBIP V5.0.2507 专业开发用户手册（17 页），涵盖沙箱管理、应用引擎、开发资源管理三大模块。

### 总体概述

应用构建服务专业版在标准版（无代码声明式配置+在线低代码函数式开发）的基础上，额外提供：自建应用引擎的创建、脚手架下载、本地开发调试、CI/CD、发布上线、生态应用上市的全生命周期管理。

标准版五大模块：数据建模 → 页面建模 → 流程&自动化 → 集成配置 → 发布管理。八大设计器：实体设计器、查询元数据设计器、单据页面设计器、画布式页面设计器、打印模板设计器、工作流设计器、业务流程设计器、函数编程设计器。

### 沙箱管理

沙箱是独立、安全、可销毁回收的环境，用于开发配置/测试调试/集成验证/补丁升级。

- **开发沙箱**：为当前租户开通新沙箱租户，测试更改不会损坏正式数据，变更验证后通过传输包工具打包上载部署
- **测试沙箱**：在开发沙箱能力基础上，提供选择当前租户部分数据同步到沙箱的能力，实现 UAT 验证环境
- **关键操作**：新建沙箱（按购买个数创建）→ 导入用户（选择用户可进入沙箱）→ 进入沙箱（快捷跳转）→ 沙箱历史（关键操作记录）
- **状态管理**：开通中/已开通/失败；到期停用后续费需手动启用

### 应用引擎

应用引擎是 YonBuilder 专业开发服务的核心，用于管理专业开发者基于云原生技术开发的微服务。

- **创建引擎**：输入引擎编码、名称，选择资源池，填写 Git 相关信息
- **自动初始化**：生成代码 → 配置 YMS 和数据 → 注册支撑服务 → 初始化流水线 → 创建产品盘
- **失败重试**：自动执行失败可根据原因调整环境配置后重试
- **开发流程**：引擎创建完成 → 下载脚手架 → 本地开发 → 代码提交 → CI/CD → 部署上线

### 开发资源管理（资源中心）

组件注册中心实现扩展组件的共享，一处定义多处使用，通过设计器进行组件属性/动作的自定义设置。

**四态模型**：
1. **开发态**：CLI 工具创建项目 → 安装依赖（npm/cnpm/ynpm）→ 创建组件 → 本地调试（npm start）
2. **配置态**：资源库（新增组件库）→ 组件库配置（新增组件、上传 index.min.js）→ 配置属性和动作（通过 props 传入，viewModel.execute 触发事件）
3. **设计态**：设计器中拖入扩展组件，配置属性和动作
4. **运行态**：预览直接看到最终效果，与设计态渲染机制一致

**扩展开发流程**：
1. 安装 CLI 工具，创建组件工程
2. 编写组件代码，npm run build 产出 dist/index.min.js
3. 在组件注册中心上传资源，配置属性和动作
4. 打包组件包，同步到设计器
5. 在设计器中使用和配置扩展组件
6. 预览验证
