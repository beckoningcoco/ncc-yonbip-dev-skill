# 专业版引擎部署（YPD）

> V3R6 版本专业版引擎创建与部署指南。与 BIP5 YDS 模式不同，R6 通过数字化工作台 Web 界面创建引擎，自动推送脚手架到 Git。

## 适用场景

| 部署方案 | 客户化定制开发 | ISV 生态开发 |
|----------|:------------:|:----------:|
| 公有云 | 否 | 否 |
| 私有云 | 是 | 是 |
| 专属云 | 是 | 是 |

> R6 的 YPD 模式只支持私有云/专属云；公有云不适用。

## 部署准备

| 资源 | 说明 |
|------|------|
| 开发环境信息 | 平台/系统访问地址、账号密码、数据库、网络等，由项目经理/客开经理通过运维文档交付 |
| 代码管理工具 | 推荐 GitLab，私有化项目需提前安装并确保 YonBIP 部署服务器可访问 |
| Maven 依赖仓库 | 私有化项目需提前安装私有化 Maven 仓库，确保服务器可访问 |
| 数据库 | 默认使用 YonBIP 平台数据库 |
| Redis 缓存 | 默认使用 YonBIP 平台 Redis |
| 文件服务 | 默认使用 YonBIP 平台文件服务 |

## 引擎创建流程

### 功能入口

数字化工作台 → 云平台 → 开发平台 → 专业开发服务 → 应用引擎

### 创建步骤

1. 点击右上角【新建引擎】
2. 输入引擎名称、引擎编码（如 `c-kk-fn-pxlx`，产品已自动添加前缀 `c-`）
3. 选择资源池（对应环境）
4. 引擎类型：**YPD**（默认）
5. 输入 Git 地址（建议 SSH 协议）+ 上传 Git 私钥（可选密码）
6. 点击【确定】

### 自动化过程

创建过程技术平台自动执行：
- 基础配置 → 代码生成 → YMS 配置 → 中间件绑定 → 共享数据源
- 发布配置 → 注册支撑服务 → 注册事件 → 初始化流水线 → 创建产品盘

### 健康检查

```
https://技术平台域名/引擎编码/extend/healthycheck
```

返回 `status: ok` 表示部署成功。R6 后端服务访问前缀为引擎编码值。

## 成员管理

引擎列表页 → 鼠标悬浮引擎条目 → 【成员列表】→ 引用图标 → 选择用户 → 确定

## 开发说明

### 脚手架工程

引擎创建过程自动推送脚手架到 Git，开发人员通过 Git 克隆到本地：

| 目录 | 说明 |
|------|------|
| `{引擎编码}-fe` | 前端工程 |
| `{引擎编码}-be` | 后端工程 |
| 其他 | 数据库脚本、发布目录、产品包制盘文件 |

### 数据库

引擎创建自动在平台默认数据库创建 Schema：`{引擎编码(下划线)}_db`

示例：`c-kk-fn-pxlx` → `c_kk_fn_pxlx_db`

### YMS 配置

引擎创建自动在 YMS 中注册产品并配置中间件：
- 逻辑数据源
- Redis 配置
- 文件存储配置

自定义改动需**发布配置 + 重启服务**后才能生效。

## 常见问题

| 问题 | 答案 |
|------|------|
| 代码管理工具能用其他的吗？ | 推荐 GitLab 及相关 Git 工具 |
| Maven 私库怎么配置？ | 平台自带，也可用项目现有私库，需同步依赖包 |
| 怎么用自定义数据库？ | YMS 控制台添加数据库连接池 → 关联逻辑数据源 → 同步库表和数据 → 发布配置重启 |
| 未执行初始化脚本？ | 手动执行，脚本见网盘：开发工具-引擎数据库初始化脚本 |
| 前后端工程分到不同 Git？ | 手动提交前端 fe 目录到另一个 Git → 修改流水线前端 Git 地址 |
| 健康检查返回错误？ | 确认服务编码（R6 为引擎编码）→ 容器内网检查 `http://127.0.0.1:8080/extend/healthycheck` → 排查 Nginx/Ingress |

## 与 BIP5 YDS 模式对比

| 维度 | R6 YPD | BIP5 YDS |
|------|--------|----------|
| 引擎创建方式 | Web 平台界面 | YDS IDE 命令 |
| 脚手架获取 | 自动推送 Git | YDS 本地生成 |
| 本地调试 | 需手动配置 | YDS 一键启动 |
| JDK 版本 | JDK 8 | JDK 21 |
| 健康检查端口 | 8080 | 62871 |
