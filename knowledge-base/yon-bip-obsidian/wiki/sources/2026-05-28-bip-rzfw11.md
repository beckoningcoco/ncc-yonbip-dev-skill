## 日志服务

BIP 平台的日志服务提供用户操作日志和业务日志的统一记录、存储和查询能力。日志存储在 ES 中，支持数据分析。

## 适用场景

全部部署方案（公有云/私有云/专属云/本地部署）+ 全部开发类型（客户化定制/ISV 生态开发）均适用。

## 日志类型

- **操作日志**：记录用户点击的菜单或按钮操作
- **业务日志**：记录重要业务操作，如订单提交、审批等
- 要求：对业务系统无侵入、不影响性能、统一存储方便后续统计分析

## 应用构建业务日志配置

数据建模时勾选"业务日志"，保存并发布即可自动记录。操作时能在业务日志节点查看。

## 自定义日志开发

### Maven 依赖

```xml
<dependency>
    <groupId>com.yonyou.iuap</groupId>
    <artifactId>iuap-ap-log-sdk</artifactId>
</dependency>
```

### 业务日志 SDK 接入（RPC，ISV 不可用）

使用 `IBusiLogService` 接口，通过 `BusinessLogObjectBuilder` 构建日志对象：

- `newObject`：业务数据（包含 id/code/name 或元数据标签字段）
- `uri`：实体元数据 URI
- `operCodeTypes`：操作类型枚举（新增/更新/删除等）
- `serviceCode`：工作台服务编码
- `busiObjTypeCode`：业务类型编码（可用 BillNum）
- `busiObjTypeName`：业务类型名称（支持多语）
- `logMode`：`BusinessLogMode.BUSINESS_OBJECT`

### 操作日志 SDK 接入（RPC，ISV 不可用）

使用 `IOperationLogService` 接口，构建 `OperationLogVO`：

- `serviceCode`/`serviceName`：服务标识
- `device`：登录设备（web/android/client/iphone）
- `button`/`operCode`：操作按钮和编码（`OperCodeTypes` 枚举）
- `operUser`/`tenantId`：用户和租户
- `ip`：客户端 IP

### 日志 OpenAPI（ISV 可用）

通过日志 API 接入，数据经过 MQ → 日志接入层 → 持久化 → ES：

- **业务日志查询**：`POST 数字化工作台域名/yonbip/digitalModel/log-pub/business/rest/query`
- **业务日志记录**（批量）：`POST 数字化工作台域名/yonbip/digitalModel/log-pub/openapi/rest/business`
- 其他 API 参考环境的 API 文档（云平台 → 连接集成平台 → API 网关 → API 文档）

## 常见问题

- **后端在线函数打印日志**：使用 `logBusinessObject` 方法
- **日志存储**：ES（Elasticsearch）
- **操作日志报表**：R5 及以后提供基本统计报表；复杂报表用操作日志 API 自定义
- **新功能开发单据接入业务日志**：元数据中勾选业务日志标签
- **API 文档不一致**：以项目环境上 API 说明为准，开放平台为公有云环境
