## 事件中心开发

事件中心是 BIP 平台的集中事件管理中心，基于 Kafka 实现业务解耦。事件发出方发布事件，订阅方注册订阅接口，事件触发时事件中心调用订阅接口执行业务。

## 为什么需要事件中心

传统同步调用的问题：
- 上游依赖下游接口（如产品入库单依赖生产订单关闭接口）
- 业务逻辑耦合（本属于下游的逻辑写在上游）
- 同步调用失败影响上游事务
- 新增下游监听需要修改上游代码

事件中心解决：上游发布事件 → 中心节点 → 下游订阅。实现**业务松耦合、易于扩展、异步处理、运行稳定**。

## 三种事件推送模式

| 模式 | 机制 | 记录 |
|------|------|------|
| 同步发送 | 事件中心发到下游，下游立即返回处理结果 | 记录成功/失败 |
| 异步发送 | 事件中心发送后先记录"异步发送中"，下游回调反馈结果 | 记录回调成功/失败 |
| Kafka 推送 | 事件中心发到指定 Topic，下游从 Kafka 拉取后回调 | 记录拉式回调成功/失败 |

## 技术架构

- **事件类型**：定义事件的基本属性，注册在事件中心
- **事件监听**：订阅方注册监听接口，指定监听的事件类型
- **事件发布**：业务方调用事件中心 API 发布事件
- **Kafka**：消息队列，事件传输通道
