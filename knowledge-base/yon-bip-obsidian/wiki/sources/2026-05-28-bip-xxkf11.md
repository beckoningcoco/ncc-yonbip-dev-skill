## 消息开发

消息开发涵盖多种消息通知场景：同租户/跨租户工作通知、消息集成开发（三方系统待办对接）。

## 跨租户工作通知

业务场景：采购商给供应商发送通知（跨租户）。OpenAPI 不支持跨租户接口调用，推荐使用**后端脚手架 SDK**方式（方便代码统一管理）。

### Api 函数方式

通过后端 Api 函数发送跨租户工作通知。

### 后端脚手架 SDK 方式（推荐）

引入 SDK 依赖，编码实现跨租户通知发送。可参考文档中的 SupplierWorkNotifyController 示例。

## 消息集成开发（三方系统待办对接）

场景：三方系统打开 YonBIP/YonSuite 公有云页面进行审批操作。不支持费控移动端和消息待办。

### 对接流程

1. 提交工单给指定租户开通接收待办事件
2. 三方系统注册监听待办事件订阅（创建待办 + 更新待办两个事件）
3. YonBIP 发送待办事件，三方系统根据事件编码走不同逻辑（新增/更新待办）
4. `businessKey` 成对出现：一个新增待办对应一个更新待办
5. 建议三方系统将 businessKey + 动作信息存储入库，处理消息乱序情况：
   - **新增事件**：businessKey 已存在则跳过，不存在则正常走逻辑
   - **更新事件**：businessKey 不存在则先入库标记为更新；存在且是更新动作则跳过；存在且是新增动作则正常处理更新
6. 通过 yhtUserId 调用开放平台接口获取用户编码/手机号/邮箱，与三方系统用户匹配
7. 三方用户在三方系统中点击待办 → 走单点登录

### 待办数据格式

创建待办接收数据包含：`msgTsLong`、`todoType`、`srcMsgId`、`typeName`、`businessKey`、`approveSource`、`title`、`content`、`tenantId`、`yhtUserId` 等字段。
