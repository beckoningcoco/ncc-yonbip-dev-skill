## 预警服务

预警服务是 BIP 平台的预警配置与管理服务，提供开发态的预警类型设置和运行态的预警任务管理。

## 与调度任务的区别

- **定时任务**：强调定时自动操作，只需记录执行结果状态
- **预警**：追求实时化监控，关注触发后给谁发消息、发什么格式、通过哪些通道
- **共同性**：都需要时间规则、可视化监控看板、集中的调度服务

## 核心概念

- **预警类型**：开发态配置，包括插件、参数、阈值等。支持报表订阅模式和接口插件模式
- **预警任务**：运行态管理，包括启用/停用、定时规则、参数设置、接收设置
- **消息接收人**：直接指定 / 规则指定（如某类物料的采购员、单据创建人）

## 产品特性

- 高可用分布式任务调度
- 系统级/租户级/个人级分级管理
- 支持场景内提交异步任务
- 统一监控 + 个性化配置（时间规则、参数、消息）
- 支持多种消息通道（通过消息模板）
- 任务全异步执行 + 限流

## 接入模式

**服务式接入流程**：
1. 在公共服务中注册应用（configKey=prewarning）
2. 添加 Maven 依赖：`iuap-warning-sdk`（3.3.1-RELEASE 或更高版本）

## 预警类型配置

- **基本信息**：编码、名称（不可重复）、应用、级别（全局/租户）、启用开关
- **接口字段**：完整 REST 路径，只支持 POST，强烈建议异步实现
- **时间间隔**：控制最小执行间隔（小时/分钟级）
- **条件**：参数类型支持字符串、数字、日期、枚举、参照；可设用户可见/必填/默认值
- **结果集**：定义接口返回值字段；标记"是否接收人"用于消息接收人设置；标记"是否表头字段"控制消息模板结构
- **关联字段**：当接收人使用规则时，指定过滤字段（如通过 dept 找部门负责人）

## 预警任务配置

三种场景：系统级（开发态创建）、租户级默认任务（开发态预置）、用户自行创建（运行态）

配置项：条件设置、定时规则、消息设置（通道选择 + 模板 + 接收人）

## 开放 API

- **添加 cron 任务**：POST `/prewarning-pub/simpletask/save`
- **异步回调**：POST `/prewarning/log/async/updateTaskLog`

## 加签调用（SDK）

实现 `IPrewarningTask` 接口的 `execute()` 方法，返回 `PrewarningTaskSendMsgDTO`。通过 `IPrewarningTask2AsyncImpl.asyncTask()` 异步调用。
