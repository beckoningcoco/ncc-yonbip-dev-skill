## 预警任务开发

BIP 预警任务的开发实践指南，涵盖预警类型创建、预警任务配置、接口开发（同步/异步）、组织权限过滤和自定义消息接收人。

## 预警类型配置要点

- **编码/名称**不可重复，编码创建后不允许修改
- **预警模式**：报表订阅模式 或 接口模式（需实现领域接口，使用 yms 域名变量如 `${domain.url}/xxx/prewarning/send`）
- **组织过滤**：无过滤 / 主组织权限过滤 / 所属组织过滤
- **时间间隔**：控制任务最小执行间隔（小时/分钟级）
- **条件**：参数 key 为条件编码；用户可见则可在任务界面修改；必填则在任务界面为必输
- **结果集**：定义接口返回值，标记"是否接收人"用于消息接收人设置，"是否表头"控制消息模板变量结构
- **自定义消息接收人**：配置 `$` 占位符开头的接口地址，返回消息接收人列表

## 预警任务配置

选择预警类型后配置：条件设置（继承类型参数）、定时规则、消息设置（通道+模板+接收人选择：用户/角色/组织/规则/自定义接收人/结果集接收人字段）

## 预警接口开发协议

### 请求规范
- **方法**：POST/JSON
- **请求头**：`logId`、`userId`、`tenantId`、`sysId`
- **请求体**：条件字段 + `alertTypeCode`（预警类型编码）+ `constrain`（组织约束，JSON 字符串格式）

### 返回值协议

**异步模式（推荐，适用于耗时较长的场景）**：

立即返回 `{"asynchronized":"true"}`，业务处理完后回调 `/iuap-apcom-prewarning/prewarning/log/async/updateTaskLog`。

**同步模式**：直接返回完整结果。

**_details 字段结构**（每一条 details 独立发送一次消息）：

```json
{
  "status": 1,
  "_details": [{
    "url": "跳转URL",
    "billId": "单据ID",
    "head1": "表头字段值",
    "data": [{"a": "表体字段值", "warnPrimaryOrgId": "主组织ID"}]
  }]
}
```

- `_details` 外的字段为表头（与预警类型结果集表头标记一致）
- `data` 内的字段为表体
- 必须包含 `billId` 才能触发自定义消息接收人回调

### 主组织权限过滤
- 只针对角色，不针对用户/组织/规则
- 返回值表体需带 `warnPrimaryOrgId` 字段

### 所属组织过滤
- 返回值需带 `constrain` 结构：`{"constrainOrgIds": ["orgId1"]}`

## 消息发送规则

1. `_details` 非空 List → 发消息（优先取 `_details` 作为消息体）
2. 报表模式且 url 不为空 → 发消息
3. 非报表模式且 `data` 非空 List → 发消息

## 常见问题

- **自定义消息接收人不触发**：预警接口没有返回 `billId` 字段
- **任务执行失败**：消息接收人规则的接收对象为空
- **没收到消息**：没配置消息接收人 / 接收人规则返回空 / 接口返回不满足发送条件
