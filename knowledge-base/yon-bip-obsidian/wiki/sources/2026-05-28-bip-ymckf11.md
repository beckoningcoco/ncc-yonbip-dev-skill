## YMC开发实践

YMC（YonBIP 监控中心）是新一代监控诊断平台，以实时监控、快速诊断定位问题为主要特点，覆盖主机-微服务-容器-中间件多级资源监控。

## 核心模块

- **YMC-Link**：中控台
- **YMC-Portal**：门户
- **YMC-Proxy**：客户端代理
- 集成哈勃监控的日志分析、报警、监控指标、链路分析、性能分析等能力

## 容器 CPU 问题排查

1. **微服务监控**：首页 → 指标监控 → 微服务监控，录入微服务编码
2. **线程堆栈**：主动点击生成；被动超过 CPU 阈值自动生成
3. **火焰图分析**：y 轴=调用栈深度，x 轴=抽样数（即 CPU 时间占比），宽度大的块=热点函数
4. **Arthas 命令**：`dashboard`（定位高 CPU 线程）→ `thread pid`（查看堆栈）；`profiler start/stop`（生成火焰图）
5. **Linux 命令**：`top` → `top -H -p PID` → `printf "%x\n" tid` → `jstack PID | grep tid`

## 容器内存问题

Pod 内存估算公式：`堆内存最大值 + 非堆(256MB) + OS开销(128MB) + 突发(512MB)`

## JVM GC 频繁问题

- **JVM 监控面板**：性能工具 → JVM 监控面板
- **堆对象**：`jmap -histo PID | head -n20`
- **堆内存 Dump**：主动（需先故障隔离）或被动 OOM 自动生成
- **Arthas**：`memory`、`heapdump`、`profiler start -e alloc`

## 元空间内存溢出

- 趋势图：性能工具 → JVM 监控面板
- `jstat -gc`（MC=容量，MU=使用量）
- 参数：`-XX:MetaspaceSize=512m -XX:MaxMetaspaceSize=512m`

## 微服务接口响应慢

- **哈勃录制** → 查看 BPR 详情和哈勃报告 → 分析 SQL 耗时 TOP
- **trace 跟踪**定位具体耗时点

## 数据库慢查询

**定位**：慢查询警告看板 / `select id from information_schema.processlist where time >5`
**优化**：`explain + SQL`，关注 type（system > const > eq_ref > ref > range > index > ALL）、key（实际使用的索引）、rows（扫描行数越少越好）
