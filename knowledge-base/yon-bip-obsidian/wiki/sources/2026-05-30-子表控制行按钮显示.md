# 详情页子表控制行按钮显示

> 社区文章：通过 beforeSetActionsState 事件控制子表行按钮的可见性。

## 基本信息
- 来源：YonBIP 开发者社区 (article/detail/11077)
- 作者：生态开发
- 类型：前端脚本 / 代码片段

## 核心代码

```javascript
let historyGridModel = viewModel.get("****"); // 获取指定表格
historyGridModel.on('beforeSetActionsState', (actionState) => {
    const rows = historyGridModel.getRows(false);
    const actions = historyGridModel.getCache('actions');
    rows.forEach((data, i) => {
        actions.forEach((action) => {
            // 设置按钮可用不可用
            if (data.contractHistory_verifyState == "0" 
                && (action.cItemName === editBtn || action.cItemName == detailBtn)) {
                actionState[i][action.cItemName] = { visible: true };
            } else {
                actionState[i][action.cItemName] = { visible: false };
            }
        })
    })
})
```

## 关键 API
- `viewModel.get("表格编码")` — 获取指定 GridModel
- `gridModel.on('beforeSetActionsState', callback)` — 行按钮状态设置前回调
- `gridModel.getRows(false)` — 获取所有行数据
- `gridModel.getCache('actions')` — 获取缓存的动作按钮列表
- `actionState[rowIndex][actionName] = { visible: true/false }` — 控制按钮显隐

## 适用场景
- 根据子表行数据状态动态控制行操作按钮（编辑、详情等）
- 不同数据状态显示不同操作按钮
- 审批状态控制按钮可用性
