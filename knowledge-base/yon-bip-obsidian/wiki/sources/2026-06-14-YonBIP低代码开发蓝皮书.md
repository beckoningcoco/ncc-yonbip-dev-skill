# YonBIP旗舰版开发赋能蓝皮书 V3.0

> 用友集团 战略项目架构方案部 | 64 页 | 适用 BIP 202212

## 核心内容

本文档系统讲解如何对 YonBIP 标准产品进行**原厂扩展开发**，以"客户档案"为例贯穿全文。

## 七大开发场景

| 场景 | 类型 | 说明 |
|------|------|------|
| 1 | **实体扩展** | 为"客户档案"扩展两个字段（是否特殊客户、特殊客户说明） |
| 2 | **UI扩展** | 扩展UI卡片模板，手工设置主/子实体扩展字段到页面 |
| 3 | **前端函数扩展** | "是否特殊客户"值改变事件：选"是"则字段可填，否则置灰 |
| 4 | **后端函数扩展** | 保存时校验："是否特殊客户"=是时，"特殊客户说明"不能为空 |
| 5 | **子表业务逻辑** | "客户关系维护经理"页签：默认联系人=默认时，手机号不能为空 |
| 6 | **普通远程规则** | 保存时校验"客户类型"不能为空（专业版） |
| 7 | **分布式远程规则** | 两个分布式规则，execute 报错后执行 cancel 回滚（专业版） |

## 关键概念

- **实体扩展**：在标准产品实体上添加新字段，主实体和子实体均可扩展
- **UI模板扩展**：基于原厂UI模板创建扩展模板，通过业务扩展点挂载
- **前端函数**：标准版写 JavaScript，在 UI 设计器中配置
- **后端函数**：标准版写 Groovy 脚本，在保存/更新等动作节点挂载
- **规则扩展**：专业版用 Java 开发 Rule，通过流水线发布后远程调用
- **扩展包部署**：导出→导入→验证，支持 dev → UAT → 生产
