# 外部申请单驱动合并或拆分开票

> `ContentType	application/json` 请求方式	POST | 分类: Invoice Request (TAX)


## 1. 请求说明

请求域名	动态域名，获取方式详见 获取租户所在数据中心域名

请求地址	/yonbip/tax/output-tax/api/invoice-will/mergeOrSplit/invoicing

请求方式	POST

ContentType	application/json

应用场景	开放API

事务和幂等性	无

用户身份	支持传递普通用户身份，详细说明见开放平台用户认证接入规范

来源	系统级


## 2. 请求参数

| access_token | string | query | 是 | 调用方应用token |
Body参数

| fpxz | string | 否 | 是 | 发票性质 01：蓝票（只支持数电票） 02：红票（只支持数电票） 示例：01 |

| fplx | string | 否 | 是 | 发票类型 31：数电专用发票； 32：数电普通发票； 33：数电纸质发票(增值税专用发票)； 34：数电纸质发票(普通发票)； 36：数电纸质发票(机动车销售统一发票) 示例：32 |

| gmfMc | string | 否 | 否 | 购买方名称 当蓝票校验类型等于“N”时，此栏位必填且会带入蓝票中； 当蓝票校验类型等于“Y”或空时，此栏位无需填写，会根据原始单据带出 示例：个人 |
| gmfNsrsbh | string | 否 | 否 | 购买方税号 当蓝票校验类型等于“N”且【发票类型】为“31”、“33”时，此栏位必填，填写的数值会带入蓝票中； 当蓝票校验类型等于“Y”或空时，此栏位无需填写，会根据原始单据带出 |
| gmfDz | string | 否 | 否 | 购买方地址 如果传递则以传递为准，如不传，则使用申请单中的数据，如申请单数据不一致，会主动获取一个获取到的地址 |
| gmfDh | string | 否 | 否 | 购买方电话 如果传递则以传递为准，如不传，则使用申请单中的数据，如申请单数据不一致，会主动获取一个获取到的电话 |
| gmfYh | string | 否 | 否 | 购买方银行 如果传递则以传递为准，如不传，则使用申请单中的数据，如申请单数据不一致，会主动获取一个获取到的银行 |
| gmfZh | string | 否 | 否 | 购买方账号 如果传递则以传递为准，如不传，则使用申请单中的数据，如申请单数据不一致，会主动获取一个获取到的账号 |
| allElcUserName | string | 否 | 否 | 数电发票用户名 如需要指定开票用户，则需要传递用户名，开票会根据指定人员开票，如无需指定开票人，无需填写，此字段需按照国密4密文加密 |
| allElcPassWord | string | 否 | 否 | 数电发票密码 如为基础通道且为账号密码登录模式，如账号未在税务云配置，则可传递发票密码，便于自动登录开票； 此字段需按照国密4密文加密 |
| kpr | string | 否 | 否 | 开票人 如果为空，获取对应开票点配置的开票人 |
| skr | string | 否 | 否 | 收款人 如果为空，获取对应开票点配置的收款人 |
| fhr | string | 否 | 否 | 复核人 如果为空，获取对应开票点配置的复核人 |
| bz | string | 否 | 否 | 发票备注 1、红字发票无需填写，系统自动生成； 2、蓝票备注信息需自行组装传递 |
| zdybz | string | 否 | 否 | 自定义备注 用户自定义备注，不带入发票信息，用于标注一些辅助备注信息 |
| einvoiceShowGxfDzDh | string | 否 | 否 | 备注是否显示地址、电话 0：不显示 1：仅显示销售方银行、账号 2：仅显示购买方银行、账号 3：购销方银行、账号都显示 示例：0 |
| einvoiceShowGxfYhZh | string | 否 | 否 | 备注是否显示购销方银行、账号 0：不显示 1：仅显示销地址、电话 2：仅显示购买方地址、电话 3：购销方地址、电话都显示 示例：0 |
| hcyy | string | 否 | 否 | 红冲原因 发票性质为红票时，红字确认单编号为空时，此栏位必填 红冲原因说明 ：1 销货退回 2 开票有误 3 开票中止 4 销售折让 |
| yfpDm | string | 否 | 否 | 原蓝字发票代码 发票性质为红票时，红字确认单编号为空时，此栏位必填 |
| yfpHm | string | 否 | 否 | 原蓝字发票号码 发票性质为红票时，红字确认单编号为空时，此栏位必填 |
| hzxxbbh | string | 否 | 否 | 红字确认单编号 目前仅支持购销双方已确认的红字确认单 |
| revphone | string | 否 | 否 | 收票手机 如存在多个，可用英文逗号隔开 |
| revemail | string | 否 | 否 | 收票邮箱 如存在多个，可用英文逗号隔开 |
| revurl | string | 否 | 否 | 回调url 发票开具完成后，回调的地址信息，回调接口需要按照系统标准格式创建 |
| autoMake | string | 否 | 否 | 是否自动开票 true:自动开票，即不需要人工在税务云确认开票，直接进行开票 ； false:不自动开票，即需要人工在税务云确认开票； 示例：false 默认值：true |
