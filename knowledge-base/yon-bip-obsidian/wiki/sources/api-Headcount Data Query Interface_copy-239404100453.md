# 岗位维度同步接口

>  请求方式	POST | Headcount Allocation (CHR)


## 1. 请求说明

请求域名	开放API: 动态域名，获取方式详见 获取租户所在数据中心域名
集成API: 详细域名信息，请见 连接配置
请求地址	/yonbip/HCC/syncPostToStaffingSchema
请求方式	POST
ContentType	application/json
应用场景	开放API/集成API
API类别	批量保存/更新
事务和幂等性	无
来源	系统级

## 2. 请求参数

名称	类型	参数位置	必填	描述
access_token	string	query	是	调用方应用token
Body参数
名称	类型	数组	必填	描述
schemaIds	string	否	否	编制方案Id列表，用逗号隔开，可以为null，此时同步所有启用且有岗位维度的编制方案 示例：1,2
deptIds	string	否	否	岗位所在的部门id列表，用逗号隔开 示例：3,4

## 3. 请求示例

Url: /yonbip/HCC/syncPostToStaffingSchema?access_token=访问令牌
Body: {
	"schemaIds": "1,2",
	"deptIds": "3,4"
}
