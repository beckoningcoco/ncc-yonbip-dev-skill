# MDF问题流转声明

> 来源：https://c2.yonyoucloud.com/iuap-yonbuilder-designer/ucf-wh/docs-mdf/mdf/index.html#/introduce/02-mdf-to-bug-description | 所属：介绍

# [](#mdf问题流转声明)MDF问题流转声明

## [](#1-重点内容)1. 重点内容

领域给框架转问题需先进行以下验证：

- 禁用扩展脚本仍有问题
- 代理MDF代码到生产环境版本没有问题
- 其它领域单据也有同样问题

未经过以上步骤验证的问题一律原路转回，框架不是帮领域排查问题的工具人（问题太多了，请理解）

## [](#2-详细说明)2. 详细说明

### [](#21-禁用扩展脚本)2.1. 禁用扩展脚本

- **方法一：** 使用MDF插件：[MDF谷歌调试插件](#/introduce/02-chrome-debug-plugin)》
- **方法二：** 在工作台或单据url拼接参数?extend=false禁用扩展脚本

如脚本禁用后没有问题，则需由领域架构师定位，如果需框架协助，由架构师在MDF接口人群联系框架，不可将问题直接转至框架，如果禁用后还有问题，进行其它检查项

### [](#22-代码mdf代码版本)2.2. 代码MDF代码版本

- **方法一：** 使用MDF插件：《[MDF谷歌调试插件](#/introduce/02-chrome-debug-plugin)》
- **方法二：** 在工作台或单据url拼接参数?mdf=release-3.5代理生产环境资源

如在某环境有问题但生产环境没有问题，适合用这种方式排查，如果代理后问题仍然存在，则可排除框架影响，需由领域前端架构师进行排查，如需框架协助，由架构师在MDF接口人群联系框架，不可将bug直接转至框架，如果代理后没问题，备注清楚把问题转给框架

### [](#23-确认是否公共问题)2.3. 确认是否公共问题

打开其它领域单据，确认是否有同样问题，如果其它领域单据无问题则说明是领域扩展问题，如果其它领域也有同样问题，备注清楚把问题转给框架

### [](#24-禁用公式和交互规则)2.4. 禁用公式和交互规则

在工作台或单据url拼接参数?extendRule=false，如拼接参数后无问题，则为公式或者交互规则问题，需领域自行排查，如需框架协助，由架构师在MDF接口人群询问，不可将bug直接转至框架

### [](#25-问题信息完整)2.5. 问题信息完整

问题需包含以下信息：

- 账号、密码、租户名称、单据名称
- 详细的复现步骤
- 专属化环境需要提供完整的环境信息

如信息不全，框架开发可直接打回。问题提出人不可直接给框架提Bug（先提给具体的节点负责人）
