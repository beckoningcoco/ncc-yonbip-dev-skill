# 消息服务开发与问题排查

## 消息分类与 configKey

| 分类 | configKey | 说明 |
|------|-----------|------|
| 业务事件 | `businessevent` | 事件触发类消息 |
| 业务动作 | `businessaction` | 单据动作触发 |
| 业务流程 | 无需注册 | 开通流程自动出现 |
| 预警 | 见预警文档 | 条件触发 |

## 开发流程

### 注册应用

在公共服务应用配置表中注册业务应用。

### 预置模板类型

访问 `{消息平台域名}/devops/#/`，业务动作 → 领域 → 应用 → 单据 → 新建模板类型。

### MDD 排查清单

1. 业务调用方无请求拦截/阻断（前置 rule）
2. 消息模板已启用且设为默认
3. 多语发送时对应语种模板已配置
4. 消息通道已配置且验证
5. 接收人手机/邮箱已配置
6. 消息策略未关闭通道

## 常见问题速查

| 问题 | 解决 |
|------|------|
| 消息发送失败 | 接口返回的 `mgslist` 有详细失败原因 |
| 短信发不出 | 公有云：确认余量和租户授权。专属化：检查 YMS 的 `apilink.sms.send.apicode` |
| 163/QQ 邮箱密码 | 需要使用 SMTP 授权码，非明文密码 |
| 短信签名错误 | 境内需申请头/尾签名，与授权一致 |
| 邮件被退信 | 发件→发件服务器→收件服务器→收件客户端，三阶段分离 |
| 模板取不到值 | 检查 MDD 应用注册表 `queryapi` 的 configKey |
| 跨租户模板 | 不允许；系统模板全租户可用 |
| Rule 不生效 | `billruleregister` 新增 rule 后需重启或刷新 Redis |

## plainMessage 发送

可以不创建模板类型，直接组装消息走 plainMessage 接口发送，消息通道共享。
