# YPR 打包出盘与补丁管理

## 制盘流程

```
选择流水线 → 下载 deploy 信息 (alone.xml/standard.xml/mini.xml)
→ 拷贝微服务 → 下载二方包到 MavenRepo → 更新镜像 → 出盘
```

## 补丁类型

| 类型 | 说明 |
|------|------|
| SP | 可包含多个制品，含"更新""已修改""新增"服务 |
| QP | 只包含 1 个制品，仅"更新"状态 |
| KB | 知识库补丁，1 个制品 |

## 补丁制作参数

| 参数 | 说明 |
|------|------|
| 是否专项 | 专项补丁，默认通项 |
| 依赖补丁 | 基于盘或已发布补丁 |
| 是否携带镜像 | 镜像 ID 变化时自动携带 |
| 是否抽取二三方包 | sha1 不同或新增的 jar 自动携带 |
| 是否下载 YMS 配置 | 专属化时 YMS 配置 (5-12 后上线) |

## 权限模型

| 角色 | 权限 |
|------|------|
| 管理员 | 模板管理、删除、制盘、验证、发布、下架 |
| 开发者 | 删除、制盘 |
| 测试者 | 验证通过/不通过 |
| 审核者 | 发布、下架 |

## 发布规则

- 同一产品+版本+形态只有一个 Release
- 版本号只能逐级递增
- RC 版本号 RC1→RC2... 递增
- Release 下架后可再次发布新 Release

## 导出规则

- 产品盘按流水号直接拷贝 YPR 制品包
- 中间件按操作系统+状态筛选
- 镜像+二方包组装为 System 扩展包
- 可选择是否携带安装器
