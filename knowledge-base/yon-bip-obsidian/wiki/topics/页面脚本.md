# 页面脚本

> BIP 平台前端和后端脚本开发的核心知识点汇总。

## 前端脚本 — MDF 模型体系

MDF 框架将页面元素封装为模型，通过 JS 调用 API 控制 UI。事件通信采用**发布订阅模式**。

| 模型类型 | 页面元素 | 关键方法 | 关键事件 |
|----------|----------|----------|----------|
| **BaseModel** | 所有模型基类 | `setVisible()`, `on()` | — |
| **viewModel** | 单据页面容器 | `get(field)`, `getGridModel()`, `hasFeature()` | `beforeSave`, `afterSave`, `beforeSearch`, `afterLoadData` |
| **SimpleModel** | 输入框、按钮 | `getValue()`, `setValue()`, `clear()` | `beforeValueChange`, `afterValueChange` |
| **ListModel** | 下拉选框、复选框 | `setDataSource([...])` | `beforeSetDataSource`, `afterSetDataSource` |
| **GridModel** | 表格 | `getCellValue()`, `setCellValue()`, `getSelectedRows()`, `setColumnState()` | `beforeCellValueChange`, `afterCellValueChange` |
| **ReferModel** | 参照选框 | `setFilter()`, `browse()` | `beforeBrowse`, `beforeValueChange` |

## FilterViewModel 查询区扩展

### 生命周期

| 事件 | 时机 | 用途 |
|------|------|------|
| `beforeInit` | 查询区脚本加载前 | 修改查询数据 |
| `afterInit` | 组件渲染后、首次搜索前 | 设默认值、监听事件 |
| `beforeSearch` | 点击查询按钮后 | 过滤/修改查询条件 |
| `afterResetClick` | 重置按钮后 | 可 `return false` 阻止搜索 |
| `updateAdvanceModel` | 高级查询 | or/and 混合条件 |

### 模型获取

```javascript
const filtervm = viewModel.getCache('FilterViewModel');
const realModel = filtervm.get('cItemName').getFromModel(); // between 区间用 getToModel
```

### 常用操作

- **隐藏方案选择**：`filtervm.setState('bHideFilterScheme', true)`
- **隐藏字段**：`filtervm.execute('updateViewMeta', {code:'xxx', visible:false})`
- **设置行数**：`filtervm.getParams().filterRows = 2`
- **参照默认值**：`cb.utils.triggerReferBrowse(referModel, [{field:'id', op:'eq', value1:'xxx'}])`
- **代码触发搜索**：`filtervm.get('search').fireEvent('click')`
- **必填校验**：`filtervm.get('xxx').setState('mustInput', 1)`
- **自动搜索**：`filtervm.getParams().autoLoad = true`

### op 过滤参数

`eq` / `neq` / `gt` / `lt` / `egt` / `elt` / `like` / `leftlike` / `rightlike` / `between`(value1~value2) / `in` / `nin` / `is_null` / `is_not_null`

## GridModel 样式扩展

**配置方法**：脚手架 `extend.less`（推荐） / YonBuilder 设计器 / 技术中台 MDF 配置。

| API | 说明 | 示例 |
|-----|------|------|
| `setRowState(idx, 'className', 'xxx')` | 行样式 | CSS: `.public_fixedDataTableRow_xxx [role="gridcell"]` |
| `setColumnState('field', 'style', {...})` | 列样式 | 或用 tinper-next 内置类如 `bg-purple` |
| `setCellState(row, 'field', 'style', {...})` | 单元格样式 | 精确到单元格 |

**tinper-next 内置背景**：`bg-red/pink/purple/indigo/blue/light-blue/cyan/green/yellow`

## 事件体系

| 事件类型 | 绑定方式 | 用途 |
|----------|----------|------|
| [[保存事件]] | `viewModel.on('beforeSave'/'afterSave', fn)` | 保存前校验/保存后处理 |
| [[按钮点击事件]] | `viewModel.get('btn').on('click'/'beforeclick', fn)` | 按钮交互 |
| [[字段值改变]] | `viewModel.on('afterValueChange'/'beforeValueChange', fn)` | 字段联动 |
| 搜索查询 | `viewModel.on('beforeSearch', fn)` | 动态构建查询条件 |

## 前端 API 调用

| 方式 | 层级 | 说明 |
|------|------|------|
| `cb.rest.DynamicProxy.create` | 前端调后端Controller | 支持 `async: false` |
| `cb.rest.invokeFunction` | 前端调后端函数 | 回调式，非ObjectStore需加 `domainKey` |

## 页面路由 & 模态框

- **跳转**：`cb.loader.runCommandLine('bill', {billtype, billno, domainKey, params}, viewModel)`
- **模态框**：`templateType: 'modal'` + 回调 `(subModel) => { return subModel }`
- **接收参数**：`viewModel.getParams()`
- 弹窗获取父模型：`viewModel.getCache('parentViewModel')`

## 后端脚本

### AbstractTrigger（后端脚本）
用于规则链、调度任务，通过 `param.data[0]` 取参

### AbstractAPIHandler（API脚本）
用于前端调后端、发布接口，通过 `request` 取参

### 后端远程调用

| 方法 | 场景 | Token |
|------|------|-------|
| `openLinker(method, url, appCode, body)` | BIP内部接口 | 自动 |
| `openLinkerPure(...)` | 跨租户BIP接口 | 手动 |
| `postman(method, url, header, body)` | 本系统REST | 携带上下文 |
| `postmanPure(...)` | 外部/三方系统 | 纯净请求 |

### 后端函数互调
`extrequire("函数路径").execute(context, param)`

### 消息触达 — sendMessage
支持：邮件、友空间通知/待办/预警，渠道：`mail`/`sms`/`uspace`/`weixin`/`enterprise_wechat`

### 调度任务
后端脚本 → 获取地址 → 调度任务节点 → 配置定时规则

### 事件订阅
API脚本注册到事件 → 通过 `postman()` 转发给三方系统

## MDF 前端扩展脚本

### 文件命名与位置

- **命名**：`{应用编码}_{billNo}_VM.Extend.js`
- **位置**：`src/business/{应用编码}/`
- **公共脚本**：`business/common/common_VM.Extend.js`

### 脚本结构

```javascript
cb.define(process.env.__DOMAINKEY__, ['common'], function (common) {
  const Extend_VM = {
      doAction: function (name, viewModel) {
          if (this[name]) this[name](viewModel);
      },
      init: function (viewModel) {
          // 业务逻辑：事件绑定、赋值等
      }
  }
  return Extend_VM;
});
```

### 模态枚举（modeChange 事件）

| mode | 说明 |
|------|------|
| `browse` | 预览态 |
| `edit` | 编辑态 |
| `add` | 新增态 |

`isArray`、`extend`、`getNewId`、`getCookie`、`isEmpty`、`alert`、`queryString`(get/set/has/toStr)

## 前端脚本 — YNF 扩展开发

> YNF（新一代框架）使用 MobX + hooks，通过 `index.extend.js` 编写扩展逻辑。与 MDF 的 viewModel 模式不同，YNF 使用 rootStore/stores/actions/reactions 架构。

### YNF vs MDF 核心差异

| 维度 | MDF | YNF |
|------|-----|-----|
| 架构模式 | viewModel + Model | rootStore + stores + actions + reactions |
| 扩展文件 | `{appCode}_{billNo}_VM.Extend.js` | `src/pages/{billNo}/index.extend.js` |
| 注册方式 | 约定命名自动加载 | `yds.config.js` → `tnsConfig.providerEntry` 显式注册 |
| 本地端口 | 3003 | 9090 |
| 调试代理 | YDS 内置自动 | 手动 Resource Override 插件 |
| 脚手架工具 | YDS/VS Code 内置 | `ynf-dx-cli` 命令行 |

### 扩展脚本结构

导出四个函数，每个函数接收 `rootStore` 参数：

| 函数 | 定义 | 使用场景 |
|------|------|----------|
| `ExtendStore` | Domain Store — 扩展领域/行业业务 stores | 自定义数据模型 |
| `ExtendAction` | 明确的、静态的 action 动作 | 自定义事件行为 |
| `ExtendReaction` | 副作用 — track 状态值并按逻辑更新 | 生命周期钩子（`afterHook`） |
| `ExtendRootStore` | 全局 api/utils/pageStore 扩展 | 子表字段校验、自定义工具方法 |

### reaction afterHook 模式

给内置 reaction（如 `init`）增加执行后钩子：

```javascript
export function ExtendReaction(rootStore) {
  return {
    init: {
      afterHook(rootStore, res) {
        if (rootStore.cardStore.isAdd) {
          // 新增态逻辑：设置默认值、禁用字段等
        }
      },
    },
  }
}
```

### 子表自定义校验（UI 协议绑定）

```javascript
export function ExtendRootStore(rootStore) {
  const tableStore = rootStore.{子表Store名称}
  tableStore.onFieldChange = function (newVal, row) {
    // 校验不通过时返回 false 阻止继续
    rootStore.utils.message.danger('错误提示')
    return false
  }
}
```

> 自定义方法需与字段的 UI 协议绑定才能生效。

### YNF 关键 API

| API | 说明 |
|-----|------|
| `rootStore.cardStore.isAdd` | 判断是否新增态 |
| `rootStore.formStore.get('field')` | 获取主表字段 |
| `rootStore.formStore.setValue('val')` | 设置字段值 |
| `rootStore.formStore.setDisabled(true)` | 禁用字段 |
| `rootStore.{tableStore}.getValue()` | 获取子表全量数据 |
| `rootStore.utils.message.danger('msg')` | 错误消息 |
| `rootStore.pageStore.push({billNo, params})` | 页面跳转 |
| `rootStore.pageStore.openModal({...})` | 弹框/抽屉 |
| `rootStore.actions.{name}.doAction` | 调用 action |

### FAQ

- **页面跳转**：`rootStore.pageStore.push({ billNo, params })`
- **自定义弹框**：`rootStore.pageStore.openModal({ uiType:'component', component, containerType:'drawer', onOk, onCancel })`
- **覆盖默认 action**：通过 `cardReaction` 配置覆盖新增/浏览/编辑/复制态的默认数据加载
