# 下推单据时，传递参数给下游单据脚本

```
需求描述
1)【通用收款合同】单据，没有勾选<合同收款条款>数据时，点击"收入确认"按钮，给出提示"当前没有勾选<合同收款条款>数据，请勾选数据"；
注意：此处标品中，不管有没有勾选<合同收款条款>数据，点击此按钮时，都是所有数据都到转单界面了，因此做此改造；
2)【通用收款合同】单据，勾选<合同收款条款>数据时，点击"收入确认"按钮：只有勾选的<合同收款条款>数据转换到转单界面；【通用收款合同】中表头的"项目"字段的值为多选，可能会存在多个，在转单时，需要将第一个值赋值给转单界面子表的"项目"字段上；
注意：此处标品中，不管有没有勾选<合同收款条款>数据，点击此按钮时，都是所有数据都到转单界面了，因此做此改造；
```

## 解决方案

编写前端页面脚本，从 viewModel 中获取选中行的数据，若未获到数据数量为0，则报错。若获取到了选中行数据，则将这条明细行数据的 id 设置到下推参数中去，再从通用收入确认单的下推后脚本处，获取上游传来的参数，根据 id 筛选出需要保留的行，重新设置 viewModel 的行数据。

编辑通用收款合同的脚本，在下推前脚本中，获取表头项目字段的信息，并取出第一个项目的 id 和名称，放入下推参数中去，并在通用收入确认单处，取出项目 id 和名称，分别设置表体的项目字段的值。

![image-20250722165330358](下推单据时，传递参数给下游单据脚本/image-20250722165330358.png)

```js
//在上游单据，触发下推时，校验是否勾选表体行，以及将必要参数传给下游单据
/* 点击收入确认，校验是否勾选表体 20250708 -----------start */
viewModel.on('beforePush', (data) => {
    //获取下推规则id，根据id去校验，不要影响到其他下推逻辑
  const value = data.args.carryParams.custMap.ruleSysId;
  if ('2298242420597850113' == value) {
    let gridModel = viewModel.get('ContractCollectionList');
    var executiosRows = gridModel.getSelectedRows();
    if (!executiosRows || executiosRows.length == 0) {
      cb.utils.alert('当前没有勾选<合同收款条款>数据，请勾选数据', 'error');
      return false;
    }

    //将选中的行的id放入query参数
    let ids = [];
    executiosRows.forEach((item, index) => {
      ids.push(item.id);
    });

    //通用收款单表头的项目是多选的，而下游收入确认单表体项目是单选的，此处获取项目id传入下游确认单。
    let headproject = getValue('PaymentContract_projectMutipleList');
    var projectId = '';
    var projectName = '';
    if (headproject && headproject.length > 0) {
      var project = headproject[0];
      projectId = project.id;
      projectName = project.PaymentContract_projectMutipleList;
    }
    let contractPushParam = {
      ids: ids,
      project_id: projectId,
      project_name: projectName,
    };
    //设置参数，可以在下游单据，通过脚本取出参数
    viewModel.getParams().query.contractPushParam = contractPushParam;
  }
});
/* 点击收入确认，校验是否勾选表体 20250708 -----------end */
```

```js
//下游 根据上游传过来的参数，进行逻辑处理
/* 合同下推单据走的 下推规则，默认将所有的明细行都带过来生下游单据，现在改造成只带勾选的表体数据  ---- start     */
viewModel.on('afterLoadData', function (args) {
  ContractDataAfterPush(viewModel);
});
//合同下推后处理
const ContractDataAfterPush = (viewModel) => {
  debugger;
  var viewParams = viewModel.getParams();
  if (viewParams.action !== 'push') {
    return;
  }
   //根据下推规则id判断是否执行逻辑，以免影响到其他下推逻辑
  var ruleid = viewParams.carryParams.custMap.ruleSysId;
  if ('2298242420597850113' == ruleid || '2298807621114134531' == ruleid) {
  // 判断单据来源
  let receivableDetailsNewBodyList = viewModel.get('bodyItem');
  let rows = receivableDetailsNewBodyList.getRows();
  if (null == rows || rows.length == 0) {
    return;
  }
  var source = rows[0].sourceCode;
  if (source !== 'PaymentContract' && source !== 'RentalContract') {
    return;
  }
  // contractPushParam 是合同下推时传的query参数
  var contractPushParam = viewParams.query.contractPushParam;
  //contractPushParam 是个 json对象，取出 ids，是选中的行的id
  var ids = contractPushParam.ids;
  var projectid = contractPushParam.project_id;
  var projectname = contractPushParam.project_name;
  let addRows = [];
  // 循环当前表体，如果来源id 是选中的行，则保留这一行
  rows.forEach((item, index) => {
    if (ids.includes(item.sourceChildId)) {
      let row = Object.assign({}, item);
      row.project = projectid;
      row.projectName = projectname;
      addRows.push(row);
    }
  });
  //删除所有行，然后把选中行添加进去。
  receivableDetailsNewBodyList.deleteAllRows();
  receivableDetailsNewBodyList.insertRows(addRows.length, addRows);
}
};
/* 合同下推单据走的 下推规则，默认将所有的明细行都带过来生下游单据，现在改造成只带勾选的表体数据  ---- end     */
```
