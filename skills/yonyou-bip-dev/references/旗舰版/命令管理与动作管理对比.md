# 命令管理与动作管理对比

| 特性 | **动作管理** | **命令管理** |
|------|-------------|-------------|
| **框架版本** | 新版 YPD 框架 | 旧版/传统框架 |
| **扩展方式** | `afterActionSave` / `beforeActionSave` 扩展点 | 规则链（Rule Chain） |
| **技术架构** | 动作调度器 + 插件切面 | 命令模式 + 规则链 |
| **灵活性** | 高，可直接扩展前后事件 | 低，只能通过规则链插入逻辑 |
| **单据类型** | 新平台单据、已升级单据 | 存量单据、未升级单据 |

## 只有命令管理的单据如何扩展保存后事件？

### 方案一：前端脚本扩展（推荐短期，最简单）

`viewModel.on('afterSave')` 是框架级事件，与动作管理/命令管理无关，所有单据都可用：

```javascript
viewModel.on('afterSave', function(data) {
    if(!data.err) {
        let saveData = data.res;
        let proxy = viewModel.setProxy({
            customOperation: { url: '/your/custom/api', method: 'POST' }
        });
        proxy.customOperation({ billId: saveData.id, billCode: saveData.code }, function(err, result) {
            if(!err) { cb.utils.alert('扩展操作成功', 'success'); }
        });
    }
});
```

### 方案二：后端规则链插入自定义规则

```java
public class CustomAfterSaveRule implements IRule {
    @Override
    public void execute(RuleContext context) {
        BillData billData = context.getBillData();
        callExternalApi(billData.getId(), billData.getCode());
    }
}
```

### 方案三：事件中心 + 业务流（多系统解耦）

```java
// 在规则链最后发布事件
EventCenter.publish("bill.saved", Map.of(
    "billId", billData.getId(),
    "billType", "otherOutStock"
));
// 在业务流中订阅处理
```

### 方案四：升级单据到 YPD 新框架（长期方案）

在新扩展的应用下重新扩展该单据，获得完整的动作管理和 `afterActionSave` 扩展能力。

## 方案选择建议

| 场景 | 推荐方案 |
|------|----------|
| 短期需求、快速实现 | 方案一：前端 `afterSave` |
| 复杂业务、数据一致性要求高 | 方案二：后端规则链 |
| 多系统解耦、异步处理 | 方案三：事件中心 + 业务流 |
| 长期维护、大量定制 | 方案四：升级到新应用 |
