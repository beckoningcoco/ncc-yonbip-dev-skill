## 什么叫做翻译

```
在用友的开发体系（低代码/零代码平台）中，“翻译档案” 是一种非常核心的数据关联与映射机制。
简单来说，它的作用就是“通过一个代码（ID），去获取对应的详细信息（Name/Value）”，就像查字典一样。
以下是结合用友开发场景的详细解读：
1. 核心定义：什么是翻译档案？
在用友的元数据或业务单据配置中，当你看到“翻译档案”这个选项或概念时，它通常指：
将当前字段的值（如：部门ID），映射到另一个档案（如：部门档案）中，自动获取并填充该档案的其他属性（如：部门名称、负责人、编码）。
技术本质：它是一种预设的 SQL 关联查询（Join）。
目的：为了保证数据的一致性，避免手动录入错误，并减少开发人员写关联查询代码的工作量。
2. 举个通俗的例子 🌰
假设你在开发一个“员工档案”单据：
场景：你需要录入“所属部门”。
如果没有翻译档案：你只能手动输入部门名称，或者写代码去查数据库。
使用了翻译档案：
你设置“部门ID”字段关联到“部门档案”。
当你输入 001 时，系统会自动去“部门档案”里查找 ID 为 001 的记录。
系统自动帮你把“部门名称”填为“财务部”，“负责人”填为“张三”。
3. 在代码和接口中的体现
结合你之前上传的 旗舰版自动提交功能.md 文档，翻译档案的概念在以下场景中会体现出来：
接口报文格式：
在提交 JSON 报文时，你通常需要同时传入 ID 和 Name（或者 _name 后缀的字段）。
例子：
json

编辑



{
  "settlementOrg": "1786953725369122858", // 这是ID（代码）
  "settlementOrg_name": "成都城投建筑工程有限公司" // 这是翻译出来的名称
}
如果不传 Name：有时候系统会自动根据 ID 去“翻译”出 Name；但为了保险起见（防止翻译档案配置错误或查不到），在接口自动化中通常建议ID 和 Name 都传。
参照字段：
文档中提到的“参照”（Reference），其实就是翻译档案的前端表现形式。你在页面上看到的那个弹出选择框（比如选择供应商、选择物料），背后就是由“翻译档案”驱动的。
4. 总结
在用友客开中，你可以把翻译档案理解为：
对于配置人员：一种“自动填表”的规则配置（配置 A 字段关联 B 档案，自动带出 B 的信息）。
对于开发人员：一种数据字典关联，用于在保存单据时，将用户选择的 ID 自动转换为系统所需的完整对象信息。

当然 有一点很重要，根据 id去查询出  名称 编码 等字段 叫做翻译，同时根据 名称，编码，或者其他字段，查询出id，这也叫翻译！！
```

## 翻译自定义档案

```java
@Autowired
private DefDocListService defDocListService;
@Autowired
private DefDocService defDocService;

public DefDoc queryDefDocByCode(String listCode, String code) {
    BdRestParam bdRestParam = new BdRestParam();
    bdRestParam.setSysId("diwork");
    bdRestParam.setTenantId(InvocationInfoProxy.getTenantid());
    bdRestParam.setUserId(InvocationInfoProxy.getUserid());
    defDocListService.setBdRestParam(bdRestParam);

    ConditionVO conditionVo = new ConditionVO("code", listCode);
    Condition condition = new Condition();
    condition.addCondition(conditionVo);
    Sorter sorter = new Sorter();
    sorter.addOrder("code", "ASC");
    List<DefDocList> list = defDocListService.list(condition, sorter);
    if (CollectionUtils.isNotEmpty(list)) {
        defDocService.setBdRestParam(bdRestParam);
        conditionVo = new ConditionVO("custdocdefid", list.get(0).getId());
        condition = new Condition();
        condition.addCondition(conditionVo);
        conditionVo = new ConditionVO("code", code);
        condition.addCondition(conditionVo);
        List<DefDoc> doclist = defDocService.list(condition, sorter);
        if (CollectionUtils.isNotEmpty(doclist)) {
            return doclist.get(0);
        }
    }
    return null;
}
```
