# 旗舰版配置事件订阅

```
1.什么是事件订阅？系统中的每一个业务操作都可以被称为是一个事件，例如客户档案新增，修改，删除，单据审批，提交......
2.事件订阅可以监听到这些事件，当事件发生时，自动调用开发人员指定的回调地址，将事件的相关信息当做参数去调用接口。
3.事件订阅是异步的，所以不支持事务。
```

# 事件监听

- 事件监听，也是检测到某个事件源发生之后，去回调某个接口，异步处理业务。

## 第一步，使用管理员账号登录事件中心

- 这个管理员账号很重要，如果随便用一个账号登录，可能没有权限去查看事件中心的日志信息。

```
https://bipdev.cnmc.com.cn/iuap-apcom-event/ucf-wh/eventCenter/index.html#/
```

## 第二步，在事件类型中找到需要配置监听的事件源

## 第三步，配置监听

事件类型: 虽然是灰色的，但点击后可以选择。
监听名称：随便取
节点编码：输入客开的微服务编码
微服务编码: 输入客开的微服务编码
所属应用编码：点开之后可以选择。
领域应用编码：和所属应用编码保持一致即可。
请求方式：Rest（还有一个RPC方式，暂时没接触）
事件接收地址：`${domain.url}/微服务编码(注意，没有-be哈)/controller地址`

## 回调接口的Controller的写法

- 注意：事件监听或者事件订阅，接口返回就是字符串的 true/success，其他的不太行！

```java
@ResponseBody
@RequestMapping(value = "/listener", method = RequestMethod.POST)
public String handle(HttpServletRequest request) {
    String sourceID = request.getParameter("sourceID");
    String eventType = request.getParameter("eventType");
    // userObject是单据id
    String userObject = request.getParameter("userObject");
    String queueName = request.getParameter("queueName");
    String tenantCode = request.getParameter("tenantCode");
    
    LOGGER.info("监听到新的事件发生，sourceID: {} eventType: {} userObject: {} queueName: {} tenantCode: {}",
            sourceID, eventType, userObject, queueName, tenantCode);
    try {
        switch (eventType) {
            case EventListenerType.WAPAY_APPROVEPASS_TYPE:
                if ("waPayPass2Expense".equals(queueName)) {
                    waPayService.pushExpenseBill(userObject);
                }
                break;
            case EventListenerType.PAYFILE_CANCELAPPROVE_YON:
                if ("cancelApproveDelExpenseBill".equals(queueName)) {
                    List list = mapper.readValue(userObject, List.class);
                    if (CollectionUtils.isNotEmpty(list)) {
                        Map<String, Object> dataMap = (Map) list.get(0);
                        waPayService.delExpenseBill((String) dataMap.get("payFileId"));
                    }
                }
                break;
            default:
                break;
        }
    } catch (Exception e) {
        throw new BusinessException("999", e.getMessage());
    }
    return "success";
}
```
