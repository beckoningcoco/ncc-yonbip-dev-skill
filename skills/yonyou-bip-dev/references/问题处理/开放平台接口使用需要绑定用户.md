- 这个问题发生在 机械院项目。
- 需求是通过 调度任务，获取智慧芽系统的附件，在调度任务中调用文件上传API将附件上传到服务器。

    /**
     * 文件上传

     * @param body

    public Map<String, Object> uploadFile(String businessType, String businessId, String fileName,
            String base64File) {
        String requestUrl = UrlConstant.USPACE_UPLOAD_FILE + "?access_token=" + getToken()
                + "&businessType=" + businessType + "&businessId=" + businessId;
        String respBody = doPostFile(requestUrl, fileName, base64File);
        log.error("uploadFile 文件上传---> " + "响应信息:" + respBody + " 请求路径:" + requestUrl);
        Map<String, Object> resp = JacksonUtils.parseObject(respBody, Map.class);
        String code = (String) resp.get("code");
        if (("200").equals(code)) {
            return (Map) resp.get("data");
        }
        return null;
    }

 private String doPostFile(String url, String fileName, String base64File) {
        InputStream inputStream = null;
        File tempFile = null;

        try {
            // 1 从base64字符串解码为字节数组
            byte[] fileBytes = Base64.getDecoder().decode(base64File);

            // 2 创建临时文件
            tempFile = File.createTempFile("upload_", "_" + fileName);
            tempFile.deleteOnExit(); // 确保JVM退出时删除

            // 3 将流写入临时文件
            try (FileOutputStream fos = new FileOutputStream(tempFile)) {
                fos.write(fileBytes);
            }

            // 4. 使用临时文件进行上传
            YmsHttpMultipartEntity entity = YmsHttpMultipartEntityBuilder.create()
                    .setContentType(YmsHttpContentType.create("multipart/form-data", StandardCharsets.UTF_8))
                    .addFileBody("files", tempFile)
                    .build();
            YmsHttpHeader header = new YmsHttpHeader();
            YmsHttpRequest request = new YmsHttpRequestBuilder().url(baseUrl + url).method(YmsHttpMethod.POST)
                    .multipartEntity(entity).addHeader(header).build();

            // request.setHeader("yht_access_token",
            //         yonBipTokenUtil.getYhtAccessToken("ec4326d0-d490-4d5a-9105-260b51fd16d0"));

            log.error("文件上传 ， YmsHttpRequest 完整信息: \n{}", JSON.toJSONString(request, true));
            YmsHttpResponse response = client.execute(request);
            log.error("文件上传 ， YmsHttpResponse 完整信息: \n{}", JSON.toJSONString(response, true));
            int statusCode = response.getStatusCode();
            if (200 != statusCode) {
                throw new BusinessException("999", "文件处理失败:" + response.getBodyString());
            }
            return response.getBodyString();
        } catch (IOException e) {
            throw new BusinessException("999", "文件处理失败: " + e.getMessage());
        } finally {
            // 5. 清理资源
            if (inputStream != null) {
                try {
                    inputStream.close();
                } catch (IOException e) {
                    // 记录日志但不中断流程
                }
            }
            if (tempFile != null && tempFile.exists()) {
                tempFile.delete();
            }
        }
    }

    private String getToken() {
        String url = baseUrl + "/iuap-api-auth/open-auth/selfAppAuth/getAccessToken";
        long timestamp = System.currentTimeMillis(); // 使用currentTimeMillis更明确
        Map<String, String> rsMap = new HashMap<>(4); // 指定初始容量避免扩容
        rsMap.put("appKey", appKey);
        rsMap.put("timestamp", String.valueOf(timestamp));
        String signature = sign(rsMap, appSecret);
        String requestUrl = String.format("%s?appKey=%s&timestamp=%d&signature=%s", url, appKey, timestamp, signature);
        YmsHttpRequest request = new YmsHttpRequestBuilder().url(requestUrl).method(YmsHttpMethod.GET).build();
        try {
            YmsHttpResponse ymsHttpResponse = client.execute(request);
            String respBody = ymsHttpResponse.getBodyString();
            Map<String, Object> resp = JacksonUtils.parseObject(respBody, Map.class);
            Map<String, Object> data = (Map) resp.get("data");
            return (String) data.get("access_token");
        } catch (YmsHttpException e) {
            throw new BusinessException("999", e.getMessage());
        }
    }

## 解决方案

- 首先，集团老师阐明这个问题发生的原因，在调度任务中，框架本身会对线程中的 token赋值为一个机器人账号的 yht_access_token,这个用户身份在调用那些对身份要求强的 API时，是无法成功的。
- 解决办法有两种，第一种，在代码中，根据 user_id 获取一下 yht_access_token， 在调用API时，赋值到 请求头中，覆盖原有的机器人账号的token。

​      key   yht_access_token   value  获取到的yht_access_token的值

- 关于 如何使用  后端代码生成  根据用户id获取 yht_access_token 的代码，请参考 `references/根据用户id获取yht_access_token.md`



- 第二种解决办法就是新建一个线程，异步执行逻辑。既然初始线程中，对yht_access_token赋值了机器人身份的token，那么只需要新起一个不带机器人身份的线程即可

```
        new Thread(() -> {
         //这里执行业务逻辑~
        }).start();
```

**注意，如果你使用的时 用友框架封装好的      private ExecutorService ymsExecutor = YmsExecutors.getYmsExecutor();  这是不可以的，用 ymsExecutor 构建的线程，会默认继承主线程的身份信息！！！！**
