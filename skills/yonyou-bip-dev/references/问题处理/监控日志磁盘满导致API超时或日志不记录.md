# 监控日志磁盘满导致 API 超时或日志不记录

## 现象

1. **监控日志节点不再记录日志**：开发时突然发现监控日志节点无法记录日志
2. **调用原厂 API 耗时极长**：销售订单接口耗时很长，批量 100 条总有几条耗时 60+ 秒，其他正常

## 根因

监控日志服务的磁盘满了，导致：
- API 记录日志时写入超时，影响接口响应时间
- 监控日志节点无法写入新的日志数据

## 排查步骤

1. 进入**技术中台**，查看监控日志服务的日志信息
2. 检查日志中是否有磁盘空间相关的告警
3. 确认磁盘使用率是否达到 100%

## 解决方案

- **临时方案**：清理监控日志磁盘上的旧数据
- **长期方案**：扩容磁盘空间

清理磁盘后，API 调用恢复正常，监控日志节点也能正常记录。
