## SQL相关

当用户提出关于SQL语句相关的命令，例如帮助用户生成一条sql，或者优化SQL，或者调整SQL，首先遵循下列规则。

- 第一，上下文中，是否可以判断用户当前在做什么项目？ 如果无法判断，则需要提醒用户输入项目名称。
- 若用户不输入项目名称，则视为通用SQL，自行检索回答即可。
- 若用户输入了项目名称，则需要根据`project-config.md`配置文件，检索环境信息，确认 数据库类型，是达梦？MYSQL？oracle？ ; 还要确认项目版本，是BIP版本 还是 NCC版本，因为二者的数据库查询规则有些不同。具体的不同请参阅 `数据库查询约束.md`
- 当用户希望帮他快速生成SQL时，可以参考SQL语句模板，参考历史查询记录。
- 当为用户生成了SQL后，必须先主动检索 `SQL语句模板.md` 中是否存在相同或类似的SQL记录：
  - 若已存在类似记录，则告知用户模板中已有，无需重复维护。
  - 若不存在，则主动询问用户是否需要将本次SQL作为模板记录到 `SQL语句模板.md` 中。
  - 用户同意后，按照已有样式新增一条记录，**注意将项目具体值替换为通用的 `{占位符}` 形式**（如 `{schema}`、`{table_name}`、`{column_name}` 等），避免暴露项目环境信息。

## Maven 仓库清理

当用户提供 Maven 本地仓库路径时，帮助清理所有 `_remote.repositories` 文件。

- 脚本位置: `scripts/clean_maven_remote.bat`
- Windows 命令:
  ```cmd
  cmd /c "for /r "<仓库路径>" %i in (_remote.repositories) do @if exist "%i" del /q "%i""
  ```
- Bash 命令:
  ```bash
  find "<仓库路径>" -name "_remote.repositories" -type f -delete
  ```
