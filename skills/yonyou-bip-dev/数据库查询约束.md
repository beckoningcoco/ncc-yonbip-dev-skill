# 数据库查询安全约束

## 连接前：必须确认项目和环境

连接数据库前，必须让用户**逐项确认**：

1. **项目**: 列出可用项目，确认当前操作的是哪个项目
2. **环境**: 明确是 test（测试）还是 prod（生产）
3. **用户**: 确认使用的数据库用户

**禁止**直接使用默认配置连接数据库，尤其是当配置中包含生产环境地址时。

## 生产环境：双重确认

如果用户选择的生产环境，必须执行以下流程：

```
第一次确认: "你正在连接 [项目名] 的 生产环境 数据库，确认? (输入 'yes')"
第二次确认: 如果执行的是写操作，要求输入项目名称拼音 + 输入 'YES-PROD'
```

**生产环境默认只允许执行 SELECT 语句**，禁止一切写入操作，除非经过明确的二次确认。

## 写操作（UPDATE/DELETE/INSERT/DROP/ALTER/TRUNCATE）

无论测试环境还是生产环境，执行写操作前必须：

1. **展示 SQL**: 完整显示将要执行的语句
2. **影响评估**: 如可能，先执行等价的 SELECT COUNT(*) 展示影响行数
3. **确认**: 用户必须明确确认执行
4. **生产环境**: 在以上基础上，要求输入 `YES-PROD` 再次确认

## 配置文件

数据库连接配置存储在: `本技能目录下的 db_config.json`

新增项目时，按以下格式添加到 `projects` 对象中：

```json
"项目名称": {
  "test": {
    "host": "测试库地址",
    "port": 1521,
    "service_name": "服务名",
    "users": { "用户名": "密码" }
  },
  "prod": {
    "host": "生产库地址",
    "port": 1521,
    "service_name": "服务名",
    "users": { "用户名": "密码" }
  }
}
```

## 自动执行规则

| 操作 | 测试环境 | 生产环境 |
|------|---------|---------|
| SELECT | 可直接执行 | 需首次确认连接 |
| DESC/表结构查询 | 可直接执行 | 可直接执行 |
| SHOW/INFO | 可直接执行 | 可直接执行 |
| UPDATE/DELETE | 需确认 | 需双重确认 |
| INSERT | 需确认 | 需双重确认 |
| DROP/ALTER/TRUNCATE | 需确认 | 阻止（需特批） |
