# BIP 组织模型与会计主体

## 核心概念

"会计主体"不是独立创建的实体，而是对业务单元（组织）勾选"会计主体"职能后的角色。主体仍是同一业务单元，**ID 不变**。

```
业务单元 A (org_id = 123)
  ├── 勾选职能：库存组织
  ├── 勾选职能：采购组织
  └── 勾选职能：会计主体  ← 同一个 org_id = 123
```

## 数据层面

| 表/实体 | 字段 | 说明 |
|---------|------|------|
| 业务单元主表 | id / org_id | 组织唯一标识 |
| 组织职能关系表 | org_id + func_code | 标记该组织有哪些职能 |
| 会计主体参照 | bd_financeorgtreeref | 过滤出有会计主体职能的组织，ID = 原组织 ID |

## 编码影响

当单据上同时有"组织"和"会计主体"时，**直接使用 orgId 作为 accentity_id**，无需额外查询转换。

```java
// ❌ 错误：多此一举查 accentity_id
String accentityId = dao.getAccentityId(orgId);

// ✅ 正确：直接使用 orgId
boolean settled = dao.isSettled(orgId, periodCode);
```

## 特殊情况

| 场景 | accentity_id == org_id |
|------|----------------------|
| 组织勾选了会计主体职能 | ✅ 相同 |
| 组织没勾选会计主体职能 | ❌ 不会出现在会计主体参照中 |
| 租户根节点 | 一般不直接作为会计主体使用 |
