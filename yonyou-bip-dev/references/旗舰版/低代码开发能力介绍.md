## 一、适用范围

低代码开发**仅适用于公有云客户化定制开发**，不适用私有云/专属云/本地部署，也不适用 ISV 生态开发。

| 部署方案 | 开发类型 | 是否适用 |
|---------|---------|---------|
| 公有云 | 客户化定制开发 | ✅ |
| 私有云 | 客户化定制开发 | ❌ |
| 专属云 | 客户化定制开发 | ❌ |
| 本地部署 | 客户化定制开发 | ❌ |
| 全部 | ISV 生态开发 | ❌ |

---

## 二、业务场景

低代码开发 = **可视化应用构建** + **可视化应用扩展**

1. **可视化应用构建**：以低代码方式新建应用（应用建模），支持新建应用的设计能力
2. **可视化应用扩展**：打开原厂应用进行数据扩展、页面扩展、业务规则扩展，适配个性化需求
3. **数据分析类页面**：设计态和运行时统一受"平台-数字化建模-报表平台"规格许可控制

## 三、核心名词速查

| 术语 | 一句话解释 |
|------|-----------|
| **YonBIP** | 用友新一代商业创新平台，涵盖平台/应用/业务/数据服务 |
| **YonSuite** | BIP 架构下专为成长型企业设计的一体化云服务包 |
| **业务对象** | 现实世界对象在系统中的抽象，封装数据模型+业务规则+访问方式 |
| **实体 (Entity)** | 对现实事物的抽象数字化表示，核心数据载体与功能单元 |
| **枚举 (Enum)** | 一组离散值的命名常量集合，限制变量取值范围 |
| **特征** | BIP 新架构提供的实施态扩展属性定义能力，统一了业务对象扩展属性的语义定义 |
| **元数据** | "关于实体的数据"，描述实体名称、属性构成、关系规则、存储方式等元信息 |
| **档案** | 企业长期维护的基础核心数据（客户档案、物料档案等），稳定性高、复用性强 |
| **单据** | 记录企业业务活动的凭证式数据（采购订单、销售订单等） |
| **节点** | 业务流程中的具体处理环节或操作步骤，最小执行单元 |
| **参照** | 通过关联已有档案/标准数据完成当前业务记录的机制 |
| **原厂单据** | 系统出厂自带的标准业务单据，结构/关联/流程由厂商预定义 |
| **自建单据** | 企业通过应用构建自定义的专属业务单据 |
| **功能权限** | 谁能用哪些功能（菜单、按钮级别） |
| **数据权限** | 能用某功能时能看到/操作哪些数据（数据范围级别） |
| **工作流** | 多参与者间按预定义规则自动传递文档/信息/任务的过程（审批流） |
| **业务流** | 单据按交易类型的业务流程自定义，支持跨领域单据流转（推单/拉单/回写） |
| **YonQL** | 基于元数据的查询语言，风格偏向 SQL，用于对元数据实体进行数据查询 |
| **函数** | JavaScript 中封装特定逻辑的可调用代码块，一等公民 |

## 四、低代码开发架构

分层架构（从上到下）：
1. **应用层** — 可视化应用构建 / 可视化应用扩展
2. **设计器层** — 页面设计器、参照设计器、报表设计器、流程设计器
3. **引擎层** — 页面渲染引擎、规则引擎、公式引擎、流程引擎、脚本引擎
4. **平台层** — 元数据驱动框架、YonQL 查询引擎、开放平台、权限框架

## 五、开发能力矩阵（11大模块）

### 5.1 单据开发
- 页面规则：交互规则、校验规则
- 页面设计器操作
- 公式
- → 详见：[页面设计器](DJYMJM12)、[公式](CYGSSY12)

### 5.2 参照开发
- 自定义参照（自建单据参照、原厂元数据参照）
- 参照带入、原厂参照调整显示字段
- 参照过滤、联想匹配
- 参照数据权限控制
- 自定义参照数据（前端覆盖返回结果）
- → 详见：[参照页面建模](CZYMJM12)

### 5.3 报表开发
- 智能分析
- 语义模型制作报表（支持业务对象、SQL、数据库表）
- 集成三方数据源（数据中台能力）
- → 详见：[报表开发](BBKF12)

### 5.4 前端脚本
- MDF、YNF 预置的所有函数
- 常见场景：列表/卡片页面生命周期、列表过滤、参照过滤、表格操作、模态框、赋值、调用 API 脚本
- → 详见：[前端脚本](QDJB12)

### 5.5 后端脚本 / API 脚本
1. YonQL 操作数据库（增删改查）
2. 远程调用：开放平台接口、三方 REST 接口、其他后端函数
3. 发送邮件、友空间待办、通知、短信
4. 规则链：获取上下文信息，调整前端业务请求参数
5. 发起/撤回工作流、下推业务流
6. 调度任务（定时任务可调用后端脚本执行）
7. 能力同后端脚本，使用场景不同（API 脚本不支持规则链）
8. 发布开放平台接口
9. 接收开放平台事件订阅数据
10. 支持预制按钮、自定义按钮扩展开发
- → 详见：[后端脚本](HDJB12)

### 5.6 工作流
- 支持扩展提交、撤回、各级审批（同意/退回）、最后一级审批
- 支持设置每级审批人对字段的查看/编辑权限
- 支持脚本活动（编写后端脚本实现复杂审批人设置）
- → 详见：[工作流](LCZDH12)

### 5.7 业务流
- 原厂单据（部分不支持）与自建单据建立推单/拉单/回写规则
- 自建单据间建立推单/拉单/回写规则
- → 详见：[业务流](LCZDH12)

### 5.8 移动端
- 支持构建：单卡、一主多子、主子孙、列表
- 设计器控件和属性调整
- 页面规则和公式与 PC 一致
- 前端脚本基本与 PC 一致
- 后端脚本与 PC 一致
- 审批流/业务流与 PC 共用
- 数据权限与 PC 共用（移动端设计器开启即可）
- → 详见：[移动端](YDDKFRM11)

### 5.9 文件上传下载
- 三方调用开放 API 为指定单据上传附件
- 前端 JS API 操作附件：上传、下载、复制等
- → 详见：[文件上传](YYGJWJSC1)

### 5.10 原厂单据扩展开发
- 字段扩展、子实体扩展
- 扩展字段支持 OpenAPI 传参
- 设计器灵活组合：表单元素控件、多页签、表格、按钮、底部栏、按钮栏
- 扩展原厂单据动作：save、delete、submit、unsubmit、audit、unaudit
- 前端脚本和后端脚本
- 审批流流程扩展
- 业务流推拉单回写
- 参照扩展
- → 详见：[原厂扩展开发](YCKZKF1)

### 5.11 其他
- **数据权限**：功能权限 + 数据权限配置 → [数据权限](SJQXNLHZ1)
- **配置迁移**：开发成果迁移至其他租户 → [配置迁移](CZZN-PZQY12)
