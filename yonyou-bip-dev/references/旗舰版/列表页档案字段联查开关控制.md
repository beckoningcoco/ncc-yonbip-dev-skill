# 列表页档案字段联查开关控制（bJointQuery）

## 问题场景

列表页的档案字段（如物料、客户、供应商等）默认自动带超链接（`bJointQuery` 联查），点击可跳转到档案详情。用户希望去掉某个字段的超链接行为。

## 解决方案

### 方案1：全局关闭（影响所有单据）

菜单路径：**"企业风格设置" → "业务偏好设置"** → 取消勾选 **"档案信息支持查看细节"**

> ⚠️ **注意事项**：
> - **仅对三大档案**（物料、客户、供应商等基础档案）生效
> - 影响范围是**全局所有单据**，打开/关闭后所有单据列表的档案字段联查都会受影响
> - 适合统一管控场景

### 方案2：单据级别控制

| 操作 | 入口 | 配置项 | 说明 |
|------|------|--------|------|
| **关闭联查** | UI 模板设计器 → 选中字段 → 右侧属性 → 扩展属性 | `disableJointQuery: true` | 只对当前 UI 模板的当前字段生效 |
| **开启联查** | 设计器 → 打开字段编辑器 | `bJointQuery: true` | 显式启用联查（默认档案字段已自动启用） |

### 两种方案对比

| 维度 | 全局开关 | 单据级开关 |
|------|----------|------------|
| 生效范围 | 所有单据 | 当前 UI 模板 |
| 控制粒度 | 粗（一刀切） | 细（字段级） |
| 配置位置 | 企业风格设置 | UI 模板设计器 |
| 适用场景 | 整个租户不需要档案联查 | 某单据的某字段不需要联查 |

## 补充：程序化控制 bJointQuery

如果需要在**代码中动态控制**联查行为（而非设计器静态配置），参考 [[文本字段超链接跳转页面功能]]，使用：

```js
// 动态启用联查
gridModel.setColumnState('字段名', 'bJointQuery', true);
gridModel.setColumnState('字段名', 'enableJointQuery', true);

// 监听联查点击事件，可拦截并自定义跳转逻辑
gridModel.on('beforeCellJointQuery', (args) => {
  // 自定义跳转逻辑...
  return false; // 阻止默认联查行为
});
```
