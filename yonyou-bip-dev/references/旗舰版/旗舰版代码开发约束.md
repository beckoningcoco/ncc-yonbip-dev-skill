BIP 旗舰版客开代码中必须遵守的约束和最佳实践，基于实际踩坑经验总结。

## 1. 租户处理

### 禁止硬编码租户 ID

```java
// ❌ 错误：硬编码 ytenant_id
String sql = "SELECT * FROM org_orgs WHERE ytenant_id = 'bgq1ldsj'";

// ✅ 正确：动态获取
import com.yonyou.iuap.context.InvocationInfoProxy;
String tenantId = InvocationInfoProxy.getTenantid();
if (tenantId == null || tenantId.isEmpty()) {
    tenantId = "bgq1ldsj";  // YMS 定时任务无上下文时的兜底
}
```

### IBillQueryRepository 查询必须显式加租户条件

```java
// ❌ 错误：不主动加租户，依赖平台"自动"处理（不可靠）
QuerySchema schema = QuerySchema.create().addSelect("id,code,name");
billQueryRepository.queryMapBySchema("org.func.BaseOrg", schema);

// ✅ 正确：显式添加租户条件
QuerySchema schema = QuerySchema.create()
    .addSelect("id,code,name")
    .appendQueryCondition(QueryCondition.name("ytenant").eq(tenantId));
```

### IYmsJdbcApi 直接 SQL 中的租户

租户用 `?` 占位符，由 DAO 方法动态传入，不拼在 SQL 字符串里：

```java
// ❌ 错误："... AND ytenant_id = 'bgq1ldsj'"
// ✅ 正确："... AND ytenant_id = ?"
// 参数通过 SQLParameter.addParam(tenantId) 传入

## 2. 数据库查询：用哪种方式

```

# 2. 数据库查询：用哪种方式

| 场景 | 使用 | 原因 |
|------|------|------|
| 单实体/字典查询（组织、仓库、物料、供应商等） | `IBillQueryRepository` | 平台 ORM，按 ID/条件查询，无需关心表名 |
| 复杂跨表 JOIN（如 ia_ledger_b + ia_ledger_b_x_account） | `IYmsJdbcApi` 直接 SQL | 平台 ORM 不支持复杂 JOIN |
| 非标准中间表 Upsert | `IYmsJdbcApi.batchUpdate` | 目标表不在平台建模中 |

### IBillQueryRepository 常用方法

```java
@Autowired
private IBillQueryRepository billQueryRepository;

// 按 ID 查单个（实体全名参考项目已有调用）
IBillDO org = billQueryRepository.findById("org.func.BaseOrg", Long.valueOf(id));

// 按条件查询，返回 List<Map>
QuerySchema schema = QuerySchema.create()
    .addSelect("id,code,name")
    .appendQueryCondition(QueryCondition.name("code").eq(orgCode))
    .appendQueryCondition(QueryCondition.name("ytenant").eq(tenantId));
List<Map<String, Object>> list = billQueryRepository.queryMapBySchema(entityFullName, schema);

// 批量 IN 查询
QueryCondition.name("id").in(idArray);  // idArray 是 String[]
```

### 常用实体全名（已确认）

| 字典 | 实体全名 |
|------|----------|
| 组织 | `org.func.BaseOrg` |
| 部门 | `org.func.AdminOrg` |
| 供应商 | `aa.vendor.Vendor` |
| 员工 | `bd.staff.Staff`（注释代码中确认） |

> 不确定的实体全名先在项目中搜索已有调用（grep `billQueryRepository.findById`），找不到再查 BIP 建模器。

### IYmsJdbcApi 使用规范

```java
@Autowired
protected IYmsJdbcApi ymsJdbcApi;

// 参数化查询（禁止字符串拼接SQL）
SQLParameter param = new SQLParameter();
param.addParam(value1);
param.addParam(value2);
List<Map> result = ymsJdbcApi.queryForList(sql, param, new MapListProcessor());

// 批量操作
List<SQLParameter> batchParams = new ArrayList<>();
for (...) {
    SQLParameter p = new SQLParameter();
    p.addParam(val1); p.addParam(val2); ...
    batchParams.add(p);
}
int[] results = ymsJdbcApi.batchUpdate(sql, batchParams);
```

## 3. 字典查询性能：按需懒加载，禁止全量预加载

数据库可能有数万条字典数据（组织、仓库、物料等），不能一次性全部加载到内存。

```java
// ❌ 错误：全量预加载
public void preloadCaches() {
    loadAllCache("org.func.BaseOrg", orgCache);  // 可能几万条！
    loadAllCache("aa.warehouse.Warehouse", whCache);
    // ...
}

// ✅ 正确：按需懒加载（每批数据只查涉及的 ID）
public void ensureIdsLoaded(List<Map<String, Object>> rows) {
    // 1. 收集本批数据中所有唯一的字典 ID
    Set<String> orgIds = new HashSet<>();
    for (Map row : rows) {
        collect(orgIds, row.get("stock_org_id"));
        // ...
    }
    // 2. 过滤掉缓存中已有的
    List<String> missing = orgIds.stream()
        .filter(id -> !cache.containsKey(id)).collect(toList());
    if (missing.isEmpty()) return;  // 都在缓存中，不查DB
    // 3. 用 IN 条件批量查询缺失的
    QuerySchema schema = QuerySchema.create()
        .addSelect("id,code,name")
        .appendQueryCondition(QueryCondition.name("id").in(missing.toArray(new String[0])))
        .appendQueryCondition(QueryCondition.name("ytenant").eq(tenantId));
    List<Map> list = billQueryRepository.queryMapBySchema(entityName, schema);
    // 4. 写入缓存（跨页复用）
    for (Map row : list) cache.put(...);
}
```

**效果**：第 1 页查询量最大（查所有涉及的 ID），后续页面大部分命中缓存，查询量趋近于零。

## 4. Controller 层规范

遵循项目现有 `DsswSCMController` 风格：

```java
@RestController
@Slf4j
@RequestMapping("/yonbip/{module}/xxx")
public class XxxController {
    private Logger logger = LoggerFactory.getLogger(XxxController.class);
    @Autowired
    private IXxxService xxxService;

    @PostMapping("/someEndpoint")
    public ResponseUtil someEndpoint(@RequestBody JSONObject params) {
        logger.error("someEndpoint_params====>" + params.toString());
        ResponseUtil responseUtil = new ResponseUtil();
        try {
            JSONObject result = xxxService.doSomething(params);
            responseUtil.setCode("200");
            responseUtil.setMessage("操作成功");
            responseUtil.setData(result);
        } catch (Exception e) {
            logger.error("操作异常：" + e.getMessage(), e);
            responseUtil.setCode("500");
            responseUtil.setMessage("操作异常：" + e.getMessage());
        }
        return responseUtil;
    }
}
```

**关键点**：
- 日志用 `logger.error`（项目惯例，标记为 error 级别便于运维查看）
- 使用 `ResponseUtil`（非 `ResponseEntity`）
- 参数用 `@RequestBody JSONObject params`
- Controller 只做参数接收和异常包装，不写业务逻辑

## 5. Service 层规范

```java
@Service
@Slf4j
public class XxxServiceImpl implements IXxxService {

    @Override
    public JSONObject xxxMethod(JSONObject params) throws Exception {
        // 1. 参数校验（校验失败直接抛 RuntimeException）
        String org = params.getString("org");
        if (StringUtils.isBlank(org)) {
            throw new RuntimeException("操作失败：组织参数不能为空");
        }

        // 2. 默认值处理（日期默认前一天等）
        LocalDate startDate = resolveStartDate(params);

        // 3. 核心逻辑 + 分页
        for (int page = 1; page <= totalPages; page++) {
            // ... 查询、映射、写入
        }

        // 4. 返回汇总
        JSONObject summary = new JSONObject();
        summary.put("totalRows", total);
        return summary;
    }
}
```

**关键点**：
- 校验失败用 `throw new RuntimeException`（由 Controller 捕获）
- 不使用 Spring 的 `@Valid` 校验（项目惯例）
- 返回 `JSONObject`（非 `ResponseUtil`，`ResponseUtil` 是 Controller 层的职责）

## 6. 分页与批量写入

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 源表分页大小 | 1000 | 平衡内存和 DB 查询次数 |
| Upsert 批大小 | 50~100 | MySQL 批量写入最佳范围 |
| 总数 COUNT | 先于分页查询 | 提前判断空数据，避免无意义循环 |

```java
// 先 COUNT，避免空循环
int totalRows = dao.countData(orgId, startTs, endTs);
if (totalRows == 0) return emptyResult();

// 分页循环
int totalPages = (int) Math.ceil((double) totalRows / pageSize);
for (int page = 1; page <= totalPages; page++) {
    List<Map> rows = dao.queryData(orgId, startTs, endTs, offset, limit);
    // ... 映射 + 分批 Upsert
    for (int i = 0; i < batch.size(); i += BATCH_UPSERT_SIZE) {
        int[] results = dao.batchUpsert(subBatch);
        for (int r : results) {
            if (r == 1) inserted++;      // MySQL ON DUPLICATE KEY：1=INSERT
            else if (r == 2) updated++;  // 2=UPDATE
        }
    }
}
```

## 7. MySQL Upsert（ON DUPLICATE KEY UPDATE）

目标表必须建唯一索引（如 `rkmx_id`），然后：

```sql
INSERT INTO target_table (col1, col2, col3, ...)
VALUES (?, ?, ?, ...)
ON DUPLICATE KEY UPDATE
    col1 = VALUES(col1),
    col2 = VALUES(col2),
    ...;  -- 排除主键/唯一键列
```

- `affected_rows = 1` → INSERT
- `affected_rows = 2` → UPDATE
- `affected_rows = 0` → 数据未变化

## 8. 异常隔离：单行失败不中断整批

```java
// 映射阶段的逐行包裹
for (Map row : rows) {
    try {
        Map target = fieldMapper.mapRow(row);
        batch.add(fieldMapper.mapToParams(target));
    } catch (Exception e) {
        log.error("字段映射失败, rkmx_id={}: {}", row.get("rkmx_id"), e.getMessage());
        errorCount++;
    }
}

// 批量写入阶段的逐批包裹
try {
    int[] results = dao.batchUpsert(subBatch);
    // 统计 insert/update
} catch (Exception e) {
    log.error("批量Upsert失败: {}", e.getMessage());
    errorCount += subBatch.size();
    // 继续处理下一批，不抛异常
}
```

## 9. 包结构规范

新增代码统一放在项目已分配的客开包下，不修改原厂代码：

```
com.yonyou.ucf.mdf.{项目包名}/
├── controller/     # REST 端点
├── service/        # 接口
│   └── impl/       # 实现
├── dao/            # 数据访问
├── converter/      # 字段映射/转换
├── constant/       # 常量
└── domain/         # 实体（如有）
```

本次开发用的是 `com.yonyou.ucf.mdf.jiankang`。
