# 不走标准API，根据前台网络请求封装 提交功能代码

## 第一步 **捕获基准请求（F12 大法）**

要实现自动化，首先需要获取系统原生的请求格式。

1. **操作路径**：在前台网页进行一次正常的单据提交操作。

2. **开启监控**：按下 `F12` 打开开发者工具，切换到 **“网络（Network）”** 选项卡。

3. 录制请求

   ：

   - 点击提交按钮，捕获浏览器发出的 POST 请求。
   - 重点关注 **请求路径（URL）**、**请求头（Headers）** 以及 **请求体（Payload）**。
   - *注意*：如果是保存接口，报文中通常需要包含 `_status` 字段，且值应设为 `2`。





## 第二步 **提取关键参数与 Token**

直接复制 F12 捕获的请求往往无法在 Postman 或代码中直接调通，需要进行清洗和补充。

1. 获取 Token

   ：

   - 进入前台登录状态，F12 打开控制台，进入 **“应用程序（Application）”** 选项卡。
   - 在存储（Storage）中查找 `yht_access_token` 和 `Domain-Key`。

2. 请求头（Headers）优化

   ：

   - 将捕获的请求头复制到 Postman 进行测试。
   - **常见坑点**：如果调用成功但数据未新增，尝试**精简请求头**。通常只需保留 `Domain-Key` 和 `yht_access_token` 即可，移除其他浏览器自带的头信息。



## **第三步：代码模拟实现（Java 示例）**

```
import com.yonyou.ucf.mdf.mingyuanconnect.pojo.AutoSumbitEntity;
import okhttp3.OkHttpClient;
import org.springframework.http.HttpEntity;
import org.springframework.http.HttpHeaders;
import org.springframework.http.MediaType;
import org.springframework.http.client.OkHttp3ClientHttpRequestFactory;
import org.springframework.http.converter.StringHttpMessageConverter;
import org.springframework.stereotype.Component;
import org.springframework.web.client.RestTemplate;

import java.nio.charset.StandardCharsets;
import java.util.Arrays;
import java.util.HashMap;
import java.util.Map;
import java.util.concurrent.TimeUnit;
   


public Map<String, Object> autoSubmit(AutoSumbitEntity autoSumbitEntity, String domainKey, String data) {
        //请求地址    注意，如果是客开的节点，这里 注意拼 /domainKey  不然可能部署测试后，调用报 401权限的错误。
    // 有的时候，不换成 /domainKey也可以~.....
        String requestUrl = System.getProperty("domain.url") + "/" + domainKey
                + "/bill/submit?cmdname=cmdSubmit&terminalType=1&busiObj=" + autoSumbitEntity.getBusiObj()
                + "&serviceCode=" + autoSumbitEntity.getServiceCode() + "&sbillno=" + autoSumbitEntity.getSbillno() + "&orgId="
                + autoSumbitEntity.getOrgId();


        //  1. 创建 OkHttpClient 并设置超时（关键步骤）
        OkHttpClient okHttpClient = new OkHttpClient.Builder()
                .connectTimeout(30, TimeUnit.SECONDS)  // 连接超时
                .readTimeout(60, TimeUnit.SECONDS)     // 读取超时（SocketTimeout） ← 最关键
                .writeTimeout(30, TimeUnit.SECONDS)    // 写入超时
                .build();
        OkHttp3ClientHttpRequestFactory factory = new OkHttp3ClientHttpRequestFactory(okHttpClient);

        //  3. 设置 RestTemplate
        RestTemplate restTemplate = new RestTemplate(factory);

        //设置编码格式
        restTemplate.getMessageConverters().set(1, new StringHttpMessageConverter(StandardCharsets.UTF_8));
        HttpHeaders saveHeader = new HttpHeaders();
        saveHeader.setContentType(MediaType.parseMediaType("application/json; charset=UTF-8"));
        saveHeader.put("Domain-Key", Arrays.asList(domainKey));
        Map saveParams = new HashMap<>(20);
        saveParams.put("billnum", autoSumbitEntity.getBillnum());
        saveParams.put("data", data);
        restTemplate.setRequestFactory(factory);
        HttpEntity<String> saveEntity = new HttpEntity<String>(com.alibaba.fastjson.JSONObject.toJSON(saveParams).toString(), saveHeader);
        Map<String, Object> saveResult = restTemplate.postForObject(requestUrl, saveEntity, Map.class);
        return saveResult;
    }


```

## 遇到的一些问题（仅供参考）

- **封装的不是提交动作，而是保存动作，保存后 只有表头，但是没有表体？？？**

  - 报文；里有个字段传的不对！！

  - 是一个参照字段，费用项目

  - 传了一个前台选不到的 费用项目进去，就把表体弄没了。

- **提交报错 ：稽核的VO不能为空？**

  - 提交时，这个 id 字段必须是字符串，检查代码是是不是传成 long类型了。

  - ```
    {
        "billnum": "znbzbx_expensebill",
        "data": {
            "code": "TY2511200014",
            "id": "2408664449237385219",
            "isWfControlled": true
        }
    }
    ```

- **提交报错，未找到流程定义**

  - 泸州老窖这边发现的这个问题，请求报文需要传   "isWfControlled": true ，原因暂时不明。

  - ```
    {
        "billnum": "znbzbx_expensebill",
        "data": {
            "code": "TY2511200014",
            "id": "2408664449237385219",
            "isWfControlled": true
        }
    }
    ```

  - isWfControlled 是主表里的一个字段，在表头，叫是否受审批流控制。
