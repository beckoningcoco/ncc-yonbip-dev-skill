## 1. 概述：这套技能能做什么

这是一套为**用友 BIP 旗舰版 / NCC 后端开发工程师**量身定制的 Claude Code 技能集合。安装后，Claude Code 将变成一个：

- **精通用友平台的后端开发助手**：自动识别旗舰版 vs NCC，生成符合框架规范的 Java 代码
- **内置源码分析工具**：本地反编译 + 类名索引，秒级定位 BIP/NCC 平台源码
- **环境配置速查器**：记录多项目（泸州老窖、天九、毛戈平、机械总院、国投中鲁等）的 VPN、数据库、中间件、账号密码
- **问题处理知识库**：已累积大量踩坑记录和解决方案，遇到报错直接查
- **个人知识库系统**：支持持续消化网页、文档、PDF，自动构建结构化 wiki

一句话总结：**把你的终端变成用友开发专家**。

## 2. 安装指南

### 2.1 前提条件

| 条件 | 说明 |
|------|------|
| **Windows 系统** | 技能脚本针对 Windows 设计 |
| **CMD 或 PowerShell** | 不要在 Git Bash 中运行安装脚本 |
| **Claude Code CLI** | 已安装并可用 |
| **Node.js** | `baoyu-url-to-markdown` 需要（npm 依赖） |
| **Python 3** | NCC 源码索引构建需要 |
| **NCC home 目录** | 至少一个版本的 NCC home 路径（可选，有则建索引） |

### 2.2 克隆并安装

```bash
# 1. 克隆仓库
git clone git@github.com:beckoningcoco/ncc-yonbip-dev-skill.git
cd ncc-yonbip-dev-skill

# 2. 先编辑 path_config.json！把路径改成你自己电脑上的实际路径
#    - NCC_Home_2111 / NCC_Home_2312：NCC home 目录
#    - BIP_Home_V5：BIP 旗舰版 V5 的 home 目录
#    - Chrome调试用户目录：Chrome 调试目录
#    - 知识库目录：Obsidian 知识库目录

# 3. 运行安装脚本
setup.bat                  # 交互式安装（推荐，会提示输入 NCC 路径）
setup.bat --quiet          # 跳过所有交互提示，不建索引
setup.bat --skip-ncc       # 跳过 NCC 索引构建步骤
```

### 2.3 安装后目录结构

安装完成后，你的 `~/.claude/skills/` 目录下会出现：

```
~/.claude/skills/
├── yonyou-bip-dev/          # 旗舰版 BIP 开发技能
├── yon-ncc-dev/             # NCC 定制开发技能（含子技能）
├── baoyu-url-to-markdown/   # 网页转 Markdown
├── llm-wiki/                # 知识库构建系统
└── knowledge-base/          # 预置知识库内容
```

### 2.4 权限配置

安装脚本会自动配置 `~/.claude/settings.json`，添加以下权限：
- `Read` 权限（读取文件不弹窗）
- 技能目录加入受信范围（执行脚本不弹窗）

如果自动配置失败，手动在 `~/.claude/settings.json` 中添加：

```json
{
  "permissions": {
    "allow": ["Read"],
    "additionalDirectories": ["C:\\Users\\你的用户名\\.claude\\skills"]
  }
}
```

### 2.5 更新

```bash
cd ncc-yonbip-dev-skill
git pull
# 如果使用 junction 方式安装，技能会自动同步最新版本
```

## 3. 核心技能详解

### 3.1 yonyou-bip-dev — 旗舰版 BIP 开发

#### 触发条件（什么时候会自动激活）

当你在与 Claude Code 对话时提到以下关键词，该技能会自动被调用：

- `用友`、`YonBIP`、`BIP`、`客开`、`旗舰版`
- BIP 特有的技术名词：`SuperDO`、`BillPlugin`、`YMS`、`YonBipUtil`、`QuerySchema`
- 后端开发通用问题：Java 代码生成、数据库操作、SQL 优化、Linux 服务器操作
- 涉及已登记项目的环境配置（泸州老窖、机械总院、国投中鲁等）

#### 核心能力

| 能力 | 说明 |
|------|------|
| **BIP V5 源码分析** | 自动查类名索引 → 定位 jar → CFR 反编译 → 输出源码分析 |
| **环境配置速查** | 按照"项目 → 环境 → 凭据"结构，快速定位 VPN、数据库、中间件地址和账号密码 |
| **问题处理** | `references/问题处理/` 下积累了大量实战问题的排查思路和解决方案 |
| **便捷帮助** | 生成建表 SQL（支持达梦/MySQL/Oracle）、阿尔萨斯命令、模板代码 |
| **旗舰版参考** | 80+ 篇旗舰版开发参考：API 调用、单据转换、事件订阅、文件上传、页面脚本等 |

#### 问题处理流程

当用户提出报错类问题时，技能会按以下流程处理：

1. **识别问题类型**：问题处理类 / 便捷帮助类 / 其他
2. **查找已有资料**：先在本地 `references/` 中搜索相关文档
3. **标注来源**：如果找到相关资料，标注是哪个项目、哪个版本、什么环境
4. **梳理排查思路**：给出排查步骤
5. **给出解决方案**
6. **知识沉淀**：如果本地没有相关资料，主动询问是否要记录下来

#### 参考资料总览

`references/旗舰版/` 目录下的核心文档（部分）：

- **开发入门**：`环境搭建.md`、`YDS环境使用VSCode搭建.md`、`YDS配置远程调试.md`
- **API 调用**：`旗舰版调用OpenAPI.md`、`后端函数调用外系统接口推送数据.md`、`旗舰版发布API.md`
- **单据操作**：`下推单据时传递参数给下游单据脚本.md`、`旗舰版单据转换规则获取下游单据对象.md`
- **页面脚本**：`旗舰版页面脚本模拟触发点击按钮.md`、`旗舰版页面脚本关闭弹窗.md`、`旗舰版页面脚本转单功能.md`
- **查询方法**：`旗舰版查询方法汇总.md`、`QuerySchema根据自定义条件查询主子表数据.md`
- **文件/影像**：`旗舰版文件上传.md`、`旗舰版影像上传.md`、`旗舰版附件上传的接口方法.md`
- **后端工具**：`YonBipUtil.md`、`后端函数工具类cb-utils.md`、`异步执行代码.md`
- **配置相关**：`旗舰版配置事件订阅.md`、`编码规则配置入库自动生成批号.md`

> 完整文档树可查看 `yonyou-bip-dev/references/` 目录。

#### project-config.md — 多项目环境配置

该文件集中管理多个客开项目的环境信息，包括：

| 项目 | 版本 | 数据库 | 特点 |
|------|------|--------|------|
| **泸州老窖** | BIP 旗舰版 | 达梦(DM) | ATRUST VPN、Nginx 入口 |
| **天九** | NCC 2312 高级版 | Oracle (PolarDB) | 堡垒机登录 |
| **毛戈平** | BIP | 待补充 | dev/uat 双环境 |
| **机械总院** | BIP 旗舰版 本地部署 | 达梦(DM) | 麒麟 Linux、AI 一体机 |
| **国投中鲁** | BIP R6 | MySQL | 数字国投 VPN、ESB 接口 |

每个项目记录了：平台地址、账号密码、数据库连接、VPN 信息、中间件地址、Git 仓库、服务器列表等。

#### 使用效果

- **查环境配置**：说"泸州老窖的 YMS 控制台地址是什么"或"机械总院的达梦数据库密码"，直接得到答案
- **解决报错**：说"启动报 Request YMS 控制台失败"，技能会自动定位到 `references/问题处理/启动报错-请求YMS控制台失败.md` 给出解决方案
- **生成代码**：说"写一个 BIP 审核后插件调外部接口"，生成符合 BIP 框架规范的完整 Java 代码
- **源码分析**：说"看一下 SuperDO 的实现"，自动反编译并分析

```
yon-ncc-dev/
├── ncc-dev/                        # NCC 开发总入口
│   └── references/
│       ├── common/                 # 通用素材（所有子技能共享）
│       │   ├── event-codes.md      # 事件码表（审批前/后、保存前/后等）
│       │   ├── bill-types.md       # 单据类型代码表
│       │   ├── coding-standard.md  # 编码规范
│       │   ├── common-api.md       # 通用 API 参考
│       │   ├── frontend-dev.md     # 前端开发参考
│       │   ├── master-detail-dev.md # 主子表开发参考
│       │   ├── app-menu-registration.md # 应用菜单注册
│       │   └── *-vo-reference.md   # 各模块 VO 与表名对照表
│       └── scenarios/              # 场景化开发指南
│           └── plugin-dev.md       # 业务插件开发完整指南
├── ncc-plugin-dev/                 # 业务插件（事件监听器）开发
│   ├── references/
│   │   ├── examples.md            # 三种典型模式完整代码
│   │   ├── event-codes.md         # 事件码表
```

#### 触发条件

当提到以下关键词时，`ncc-dev` 技能会自动激活：

| 场景 | 触发关键词 |
|------|-----------|
| **业务插件** | 事件监听、审批后/新增后/修改后、IBusinessListener、doAction、单据转换、自动生成、回写字段 |
| **后台任务** | 后台任务、计划任务、定时任务、IBackgroundWorkPlugin、executeTask |
| **参照** | 参照、下拉参照、自定义参照、参照过滤、F7 |
| **业务扩展** | 业务扩展、扩展点、处理器 |
| **编码规则** | 编码规则、单据编号、自动编号、编码生成 |
| **源码分析** | 任何 `nc.xxx.XxxClass` 格式的 Java 类名 |

#### 核心能力

| 能力 | 说明 |
|------|------|
| **场景自动路由** | 根据用户描述自动识别是业务插件、后台任务还是其他场景，读取对应参考文件 |
| **版本自动区分** | 默认版本 NCC 2111，支持切换 2312 等多版本，自动使用对应索引 |
| **源码反编译分析** | 查类名索引 → 定位到具体 jar → CFR 反编译 → 输出带分析的源码 |
| **事件码智能提示** | 自动查找事件码表，生成带常量定义（非魔法值）的规范代码 |
| **数据字典查询** | 覆盖 NCC 全模块的数据库表结构，支持按模块/表名快速定位 |
| **VO 参照速查** | 30+ 模块的 VO 类名与数据库表名对照表（so/pu/ic/arap/gl/bd/uap/ct...） |

#### 使用示例

**示例 1：写一个销售订单审批后自动生成采购入库单的业务插件**

```
用户：写一个销售订单审批后自动生成采购入库单的插件
```

Claude Code 会自动：
1. 识别为 NCC 业务插件场景
2. 查找 `event-codes.md` 获取审批后事件码（1020）
3. 查找 `bill-types.md` 获取销售订单和采购入库单的单据类型代码
4. 参考 `so-vo-reference.md` 和 `ic-vo-reference.md` 获取 VO 类名
5. 参考 `examples.md` 获取单据转换模式代码
6. 生成完整、规范的 Java 代码（含常量定义、异常处理、事务控制）

**示例 2：写一个超期应收单预警的后台任务**

```
用户：写一个定时任务，每天检查超期应收单并发送预警消息
```

Claude Code 会自动：
1. 识别为 NCC 后台任务场景
2. 切换到 `ncc-background-task` 子技能
3. 参考 `IBackgroundWorkPlugin` 接口源码
4. 参考 `examples.md` 获取后台任务模板
5. 查找 `arap-vo-reference.md` 获取应收单相关 VO
6. 生成完整的后台任务代码（含注册、部署说明）

**示例 3：查看 NCC 某个类的源码**

```
用户：看一下 nc.bs.pub.taskcenter.IBackgroundWorkPlugin 的源码
```

Claude Code 会自动：
1. 查找 `class_index_2111.json` 中该类所在的 jar 位置
2. 用 `cfr-0.152.jar` 反编译
3. 输出带注释的完整源码分析

#### 使用效果

- **业务插件开发**：从"提需求"到"拿到可直接粘贴的 Java 代码"，减少查找事件码、VO 类名、参考示例的时间
- **后台任务开发**：自动包含接口约束（如返回值不允许为 null 等注意事项），避免踩坑
- **数据字典查询**：不用打开数据库客户端，直接问"应收单主表有哪些字段"
- **源码分析**：不用 IDE 索引，秒级定位 NCC 平台源码

### 3.3 llm-wiki — 个人知识库构建

#### 这是什么

llm-wiki 是一个基于 Karpathy llm-wiki 方法论的**个人知识库构建系统**。它不是笔记软件，而是让 AI 帮你持续维护的 wiki 系统：

- 你给素材（链接、文件、文本），AI 提取核心知识
- AI 自动整理成互相链接的 wiki 页面
- 所有内容都是本地 Markdown 文件，可用 Obsidian 打开
- 知识库随每次使用变得越来越丰富

#### 触发条件

需要在对话中**明确提到**知识库操作意图：

- `知识库`、`wiki`、`llm-wiki`
- 要求对已初始化的知识库执行消化、查询、健康检查等操作

> ⚠️ 注意：只是说"总结这篇文章"不会触发知识库——必须是明确的知识库相关意图。

#### 核心操作

| 操作 | 命令示例 | 说明 |
|------|---------|------|
| **初始化** | "帮我初始化一个知识库" | 创建知识库目录结构和配置 |
| **消化素材** | "帮我消化这篇 [URL]" | 从网页/文档/PDF 提取知识 |
| **查询知识** | "知识库里有没有关于 X 的内容" | 搜索已有知识 |
| **健康检查** | "检查知识库的健康状态" | 发现孤立页面、断链等 |

#### 预置知识库内容

本工具包预置了一个已构建好的**用友 BIP 知识库**，内容非常丰富（300+ 篇素材）：

| 主题 | 内容规模 | 涵盖范围 |
|------|---------|---------|
| **旗舰版开发基础** | 6 大主题 | 环境搭建、日志、Token 认证、数据库、YMS 配置 |
| **页面脚本** | 9 类代码片段合集 | 保存/按钮/字段/搜索/API/状态/生命周期/参照/跳转 |
| **BIP 后端开发模板** | 6 类模板 | OpenAPI/查询/实体/插件/规则/推单 |
| **低代码开发体系** | 完整能力体系 | 概念→架构→模块→流程→学习路径 |
| **问题处理** | 持续累积 | 启动报错、API 报错、影像上传、权限、界面问题 |
| **BIP 核心概念术语** | 19 个术语 | 平台核心术语官方定义速查 |

知识库索引文件位于：`knowledge-base/yon-bip-obsidian/index.md`

#### 使用效果

- **新人入职**：直接问"BIP 的环境怎么搭建"，知识库返回结构化教程
- **快速查阅**：问"BIP 前端脚本 beforeSave 怎么写"，返回代码片段和注意事项
- **持续积累**：解决了一个问题后，一句话"把这个记录到知识库"，下次自动检索

### 3.4 baoyu-url-to-markdown — 网页转 Markdown

#### 功能

通过 Chrome CDP（远程调试协议）抓取网页，渲染完成后转换为干净的 Markdown 文件。

#### 使用场景

- 抓取用友官方文档、论坛帖子保存到本地
- 把网页教程转成 Markdown 方便阅读
- 配合 llm-wiki 知识库：抓取 → 消化 → 归档

#### 使用方式

```
用户：把这个网页转成 markdown：https://xxx.com/doc
```

#### 特点

- 使用真实 Chrome 渲染（支持 SPA 页面、需要 JS 执行的页面）
- 支持需要登录的页面（wait-for-user 模式）
- 自动提取 YouTube 字幕
- 失败时可回退到云端 API

## 4. 推荐额外安装的公网技能

以下技能来自社区，与本技能包互补，建议额外安装（通过 npm 一键安装）：

### 4.1 check — 代码审查

```bash
npx skills add tw93/waza@check -g -y
```

**功能**：代码审查助手，自动审查代码变更，发现潜在 bug、安全问题和优化建议。

**适用场景**：
- 写完代码后自查：`/check` 审查当前改动
- 合并 PR 前做最后检查
- 新人写的代码需要 review 时

**效果**：在提交前自动发现常见问题，减少 Code Review 往返次数。

---

### 4.2 hunt — 故障排查

```bash
npx skills add tw93/waza@hunt -g -y
```

**功能**：报错/崩溃时先找根因再修，不是直接改症状。

**适用场景**：
- 启动报错、运行时异常
- 日志里的错误堆栈无从下手
- 环境问题（连不上数据库、端口冲突等）

**效果**：系统化排查思路，先理解"为什么出错"再修，避免反复试错。

---

### 4.3 think — 方案设计

```bash
npx skills add tw93/waza@think -g -y
```

**功能**：架构/技术选型先出方案再写代码。

**适用场景**：
- 接到新需求，不确定怎么设计
- 有多种实现方案，需要对比选型
- 需要输出设计文档给团队 review

**效果**：先设计再编码，减少返工。特别适合"做推单功能还是做拉单功能"这种需要分析业务场景的选择。

---

### 4.4 self-improving-agent — 自我进化

```bash
npx skills add charon-fan/agent-playbook@self-improving-agent -g -y
```

**功能**：从对话反馈中学习优化，记录你的偏好和习惯。

**适用场景**：
- 持续使用后，AI 会越来越了解你的编码风格
- 记录你的项目特殊约定（如"我们项目用达梦，建表语句默认生成达梦的"）
- 从错误中学习，下次同类问题直接给正确方案

**效果**：用得越多越精准，减少每次都重新交代背景信息。

---

### 4.5 agent-browser — 浏览器操控

```bash
npx skills add vercel-labs/agent-browser@agent-browser -g -y
```

**功能**：AI 直接打开网页、点击、填表、截图。

**适用场景**：
- 让 AI 帮你登录 BIP 平台截图
- 自动浏览用友官方文档并提取信息
- 日常网页操作自动化

**效果**：把浏览器操作流程自动化，特别是反复操作 BIP 平台页面时节省时间。

## 5. 典型使用场景

### 场景 1：新人接手客开项目

**需求**：了解项目环境配置、搭建开发环境

**使用方式**：
```
你：泸州老窖项目的开发环境怎么搭建？
```
→ `yonyou-bip-dev` 自动激活 → 从 `project-config.md` 返回 VPN、平台地址、数据库连接、Git 仓库等全部信息

```
你：BIP 旗舰版用 YDS 怎么搭建开发环境？
```
→ 返回 `references/旗舰版/环境搭建.md` 的完整步骤

### 场景 2：开发业务插件

**需求**：采购订单审核通过后自动生成付款单

**使用方式**：
```
你：写一个 NCC 审批后插件，采购订单审核通过后自动生成付款单
```
→ `ncc-plugin-dev` 自动激活 → 查找事件码、单据类型、VO 对照表 → 生成完整 Java 代码

### 场景 3：排查线上问题

**需求**：应用启动报错"请求 YMS 控制台失败"

**使用方式**：
```
你：应用启动报错：请求 YMS 控制台失败，怎么解决？
```
→ 自动定位到 `references/问题处理/启动报错-请求YMS控制台失败.md` → 给出排查思路和解决方案

### 场景 4：查阅平台源码

**需求**：理解 SuperDO 的实现机制

**使用方式**：
```
你：BIP 的 SuperDO 是怎么实现的？帮我看看源码
```
→ 查类名索引 → CFR 反编译 → 输出带分析的源码

### 场景 5：积累团队知识

**需求**：解决了一个罕见问题，想记录下来

**使用方式**：
```
你：帮我记录一下这个问题到知识库：[描述问题和解决方案]
```
或
```
你：把这个网页文档消化到我的知识库：https://xxx
```

### 场景 6：代码审查与方案设计

**需求**：写完代码后自查，或设计新功能架构

**使用方式**：
```
你：/check           # 审查当前代码改动
你：/think           # 帮我设计这个功能的实现方案
你：/hunt            # 这个报错帮我排查根因
```

## 7. 最佳实践与注意事项

### 7.1 版本识别

> ⚠️ **最重要的一条**：用友有两个主要产品线——旗舰版（BIP）和 NCC（NC Cloud）。两个技能会自动根据关键词区分，但如果不确定，主动说清楚是哪个版本。

- 旗舰版 → 触发 `yonyou-bip-dev`
- NCC → 触发 `yon-ncc-dev`

### 7.2 主动记录问题

当遇到一个技能中没有记录的新问题时，Claude Code 会主动询问是否要记录。**建议同意记录**——这是知识库持续增长的关键。

### 7.3 定期更新技能

```bash
cd ncc-yonbip-dev-skill
git pull
# 技能使用 junction 安装，会自动同步
```

### 7.4 维护路径配置

更换电脑、移动 NCC home 目录后，记得更新 `path_config.json`。

### 7.5 安全提醒

- `project-config.md` 中记录了真实的账号密码和服务器信息
- **不要**将此仓库设为 public
- **不要**将配置信息分享给无关人员
- 建议在 `.gitignore` 中确认敏感文件不会推送到远端

### 7.6 知识库使用建议

- 初次使用先打开 Obsidian 浏览一遍预置知识库索引（`index.md`）
- 解决完一个问题后顺手"记录到知识库"
- 定期做健康检查，清理冗余内容
- 知识库内容可以在团队成员间共享
